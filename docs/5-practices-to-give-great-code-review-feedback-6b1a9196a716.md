## 优秀的代码评审提高了代码质量，创造了一个学习环境

![](img/3e392aba3919b527c61dd563f38baecb.png)

在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的 [ThisisEngineering RAEng](https://unsplash.com/@thisisengineering?utm_source=medium&utm_medium=referral)

每个故事都有两面性。[上周](/5-tips-to-create-better-pull-requests-a75c42a32ff?source=your_stories_page-------------------------------------)，我们谈论了如何准备一个好的公关。今天，我想谈谈如何执行一个好的代码审查。

代码审查是每个开发人员一天中非常重要的一部分。代码评审的两个主要目标是:

*   提高代码库的质量。
*   指导其他开发人员，让他们提高技能。

此外，评审者在执行代码评审的过程中经常会学到一些新的东西。记住要保持开放的心态，这个过程是双向的:贡献者和评论者之间的对话。

让我们看看我们能做些什么来改进我们执行代码审查的方式。

# 1.阅读标题、描述和所有参考资料

首先要做的是对公关有一个非常清晰的**背景**。有时候我们想走得快；我们认为我们已经知道我们的同事想要做什么，最好的方法，我们只是浏览一下描述。

然而，最好花些时间仔细阅读 PR 的标题和描述，尤其是后者，因为我们可以找到指导我们同事的所有假设。我们可以找到一个更详细的任务描述，也许还能很好地描述他们在开发时所面临的主要问题。

这可以为我们提供进行建设性审查所需的所有信息，同时考虑到审查的所有相关方面。

# 2.关闭公关。自己为这个任务草拟一个解决方案

这是最有争议的建议。我们应该自己想出解决这个问题的适当方法。在仔细阅读了描述之后，我们应该和我们的同事有相同的上下文。因此，我们拥有解决问题所需的一切。

我们不必真的写出解决方案。这将是做同样的任务两次，这是浪费时间。然而，我们应该对解决问题的最佳方法有一个概念，也许还应该对所需步骤有一个大致的轮廓，并对所涉及的组成部分有一个明确的设想。

这给了我们一个基线来比较实现的解决方案和我们的想法。最后，我们不想过多地关注变量名或类似的吹毛求疵；我们应该讨论公关的逻辑及其背后的想法。拥有一个预期的解决方案和一个实现的解决方案应该将讨论引向那些主要的方面，而不是次要的细节。

# 3.有礼貌，总是要求澄清，从不假设。

代码评审的目的之一就是让其他开发人员成长。没人想让同事难受，也没人想炫耀。

创建 PR 的人通常会花大量的时间考虑它，尝试不同的方法，并测试他们的代码。很有可能他们遇到了一些实际问题，使得理想的解决方案难以实现。

每当有事情与我们的想法不一致或看起来不清楚时，总是问为什么有人做出这样的选择，并试图理解他们的理由。这是一个很好的方法来了解是否有一些概念需要解释，或者是否有一些 *gotcha* 在文档中没有明确。

不管怎样，通过问一个简单而礼貌的问题，我们创造了一个机会:

*   当我们解释获得相同结果的其他方法时，教给我们的同事一些东西。
*   当他们用他们的思路回答我们的问题时，向他们学习一些东西。
*   当答案很好，可以解释为什么要做出决策时，改进代码库/文档。

这种方法也让对方觉得对自己的工作负责。他们必须能够解释他们的选择，这比仅仅要求他们以另一种方式改变代码库更有力量。当一个人对自己的工作负责时，他们会想做得更好，尽可能做出最好的东西。他们希望为自己的工作感到自豪，这是创造伟大产品和解决方案的最强大的情感之一。

# 4.专注于重要的事情

当审查一段代码时，关注最重要的部分:逻辑、数据结构和算法的选择、测试中是否覆盖了所有的边缘情况，等等。

许多其他的语法/格式元素应该由一个工具来处理，比如 linter、formatter、拼写检查器。等等。在评论中强调它们是没有意义的。

同样的想法也适用于文档的编写。应该有一些约定，如果他们没有遵守，告诉贡献者是可以的。每当文档不清楚或不可读时，留下注释也是有意义的。

然而，没有什么比打开一份公关，看到大量关于措辞微小变化的评论，要求改变，却没有对公关逻辑或概念的评论更令人沮丧的了。

# 5.赞美美好的事物

有时候我们在一篇 PR 里读到一些非常巧妙的东西，或者我们看到投稿人通过考虑其他我们没有想到的方面解决了一个问题。

在这段代码上留下一个好的评论可能会以下列方式提高创建 PR 的人的士气:

*   这让他们感到有价值和自豪。
*   这促使他们总是做得更好。
*   这有助于他们给团队带来积极性。

重要的是不要表扬每一位公关，尤其是如果没有什么特别的事情需要表扬的话。如果我们强迫自己总是寻找一些值得称赞的东西，我们可能会毁了这种做法，使它失去任何价值。

然而，适时的表扬可以让同事精神振奋，帮助他们找到更多自尊，让他们更加热爱自己的工作。
