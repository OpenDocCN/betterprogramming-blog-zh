代码评审是软件开发中最重要的部分之一，它需要你和评审者的共同努力。代码审查通常包括三个主要步骤:

*   您在单独的分支上对项目进行更改。
*   你和评论者一起改进它们。
*   您的工作被合并到主代码库中。

这个过程对于保持项目的良好维护和正常运行是必要的，如果你没有做好准备，这可能会有点痛苦。这篇文章将提供几个简单的步骤，让你为复习做好准备。

# 首先检查你自己的代码

如果你不接受你作为评审者提交的代码，就不要提交它。你应该假装你是评审者，检查你的代码，在让别人评审你的工作之前*修复你遇到的任何错误。*

如果你想改变什么，积极主动地去改变它。你应该总是第一个彻底检查你的代码。

自我审查应该包括修复你能找到的每一个可能的问题，你应该继续工作，直到你觉得你的代码马上就能通过审查。当然，你不会什么都抓到，但你应该先靠自己尽可能接近。

## 测试您的代码

你的自我审查的一部分应该是测试你的代码。无论您是使用自动测试框架还是简单地手动运行相关代码，您都应该首先确保您的代码功能正常。

# 向代码中添加注释

所有主要的编程语言都允许你在你的代码中添加注释，解释为什么它会这样工作。

Mohammad Faisal 为[良好的评论实践](/15-rules-for-writing-clean-comments-18390394941f)写了 15 条有用的规则。总而言之，您通常应该避免对显而易见的事情进行评论，尽可能让代码自己说话。

向代码中添加注释可以确保审阅者和任何接触到您的代码的人能够理解正在发生的事情。没有它们，代码有时会令人困惑，难以理解。但是也不要加太多。

# 接受建设性的反馈

代码评审的整个目标是改进一组变更，直到它们变得完美。所以，当批评来临时，你应该准备好接受批评——这是有原因的。

你还应该确保你能及时回复别人的评论。如果有人花时间来看你的代码，你应该尊重他们，尽快回复他们。

当然，这并不意味着你应该总是盲目地跟随每一个评论。运用你的最佳判断来决定需要采取什么行动(如果有的话)。有些话题需要比其他话题更多的讨论，以找出处理它们的最佳方式。
