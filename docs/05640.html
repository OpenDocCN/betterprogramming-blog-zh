如何开始一个新的开源项目

原文：https://betterprogramming.pub/how-to-start-a-new-open-source-project-9dcc2be49e08?source=collection_archive---------10-----------------------#2020-07-23

这需要一些努力，但完全值得

开始一个全新的开源项目需要做大量的工作，但是一旦你了解了基础知识，事情就会变得简单得多，这是100%值得的。

本文旨在向您传授关于启动和维护一个开源项目的所有知识，但它并不关注编程方面。相反，随着时间的推移，你会学到从为你的项目找到贡献者到维护它的一切。

为什么要开源？

首先，你为什么要分享你的项目的源代码呢？开源你的项目有很多好处！

一起工作

英雄所见略同，吸引这些伟人的唯一方法是分享你的作品，而不是把它锁起来。

当您邀请其他人参与您的项目时，您的项目可以在其他人更好的想法和代码的帮助下得到改进。

透明度

当潜在用户看到你分享你的源代码时，他们会有一定的安全感。毕竟，如果你不怕分享源代码，那么很明显你没有隐藏任何恶意。

透明性还意味着任何人都可以看到你是如何完成项目目标的，他们甚至可以在自己的代码中使用相同的概念。

添加到您的简历中

通过使你的项目开源，你允许潜在的雇主(和其他任何人)看到你是如何工作的。

展示你有一个伟大的开源项目可以帮助你在未来找到一份好工作。

你提供什么？

在你开始你的项目之前，有两个问题你应该问自己。第一个是，“这个项目将完成什么？”从最终目标开始，对你的工作实际上会做什么有一个清晰的愿景。

你需要问自己的第二个问题是，“为什么人们需要这个？”你的项目应该为他人提供某种价值。

不要只是复制

提供有价值的资源的一个重要部分是，它不应该是另一个项目的副本。

如果已经存在另一个与你的想法具有相同一般特征的项目，但是没有很好地维护或更新，你仍然可以为你的用户提供价值！如果您想要添加或更改功能以改进另一个类似项目的概念，这同样适用。

奠定基础

没有一个合适的起点，你不能指望每个人都理解你试图完成的事情。如果你希望其他人也能有所贡献，你必须先打下一些基础。

选择一个名字

你为你的项目选择的名字将代表你的工作给所有人看。它应该是独特的，容易记忆的，并且与项目的目标相关。

例如， Sentry 监控应用程序的崩溃和错误。哨兵这个名字很容易记住，并且清楚地代表了这个应用程序的功能。

这一切都从自述文件开始

首先制作一个详细简洁的自述文件，告诉你项目中最重要的事情。大多数看到你的项目的人会从自述文件开始，所以分享你需要分享的所有内容是很重要的。

当有人能够不用看代码或太多文档就能使用您项目的基本功能时，您就知道您的自述文件是完整的。

自述文件应该包括你的项目是什么，它实际上是什么样子，如何使用它，以及任何其他相关的细节。请记住，自述文件应该尽可能简短，不能再短了。您可能还想添加关于贡献和许可证信息的详细信息。

制作自述文件网站有一个模板，你可以用它来制作自述文件。

开始编写代码

开始编写自己的代码是一个好主意，这样其他人会对你的项目有更多的了解，知道会发生什么，知道他们是否愿意帮忙。

这也意味着你可以发布你的项目的测试版来增加可见性。

从格式良好的代码开始，甚至加入一些注释。这将帮助其他人理解代码中的思想以及项目的目的。

挑选许可证

没有许可证，没有人能够真正为你的项目做出贡献。GitHub的网站Choose a open source license是选择许可的一个很好的资源，它为您提供了每个许可选项的详细信息。

开源软件中最常用的许可证是 MIT许可证、 Apache许可证2.0 和 GNU通用公共许可证v3.0 。每个许可证都有独特的权限、条件和限制，因此最好仔细寻找适合您项目的许可证。

投稿指南

建立清晰的贡献指南很重要，这样其他人就知道他们可以如何帮助你的项目。这通常放在根目录下一个名为CONTRIBUTING或CONTRIBUTING.md的文件中，以便容易找到指南。

您可能希望包括以下信息:

  • 如何提交错误报告
  • 如何请求功能
  • 如何设置开发环境和运行测试

你的投稿指南应该简洁明了，同时仍然要提到需要包含的所有内容。

记录你的工作

即使你一开始没有很多代码，从一开始就记录你所做的一切也是很重要的。

您甚至可以将您的文档作为项目源代码的一部分，以便其他参与者在添加代码时可以添加到其中。

像 GitHub Pages 和Read Docs这样的工具可以让你在一个漂亮的网站上免费托管文档。

主持项目

一旦你有一个项目准备分享，你需要一个地方来分享它。最常用的网站是 GitHub ，尽管还有许多其他选项，如 GitLab 或 Bitbucket 。

列出的所有网站都很容易使用，但是如果你想开始一个开源项目，你可能需要学习一点关于Git的知识。你可以看看Anne Bonner撰写的深入文章“Getting started with Git and GitHub:the complete beginner ' s guide”来获得关于如何开始使用Git和GitHub的精彩教程。

吸引注意力

一个项目只有在其他人希望的情况下才能成长。你必须与世界分享你的想法，这样你才能得到一些贡献者和用户来启动你的项目。

一旦你的项目开始，你可能想在 Reddit 、黑客新闻、 Quora 或其他网站上分享新闻，以吸引一些对你的产品感兴趣的用户。

GitHub问题

GitHub的一个简洁的功能是问题部分。来自GitHub的问题指南页面:

“问题是跟踪项目的任务、增强和缺陷的好方法。它们有点像电子邮件——除了它们可以与你团队的其他成员分享和讨论。大多数软件项目都有某种类型的bug跟踪器。GitHub的追踪器名为 Issues ，在每个存储库中都有自己的部分。

使用GitHub问题的一个好处是，你可以添加“好的第一个问题”和“需要帮助”标签，使问题更容易被发现。

像这样的网站待价而沽(这甚至是开源本身！)甚至用这些标签聚集问题，鼓励其他人为新项目做贡献。这是让你的项目中的一些合作者愿意帮忙的一个好方法。

持续维护和开发

一旦你的项目获得了更多的动力，它就不仅仅是写代码了。如果一切顺利，你将围绕你的项目发展出一个相当大的社区。

在一个大社区里，你必须时刻保持专业和礼貌。毕竟，越多的人对你的项目保持热情，你就能得到越多的贡献者，你的工作也将增长得越多。一段时间后，你的项目将发展成为整个社区的工作，而不仅仅是你一个人的工作。

这里有一个关于维护人员最佳实践的好指南。

结论

启动和维护一个开源项目可能需要大量的工作，但这是值得的。通过与世界分享一个新项目，你可以帮助很多人，而不仅仅是你自己。这些项目可以发展成更大更好的东西，你可以从知道你开始了人们可以依赖和帮助的事情中获得一些快乐。
