# 如何从自我管理开始

> 原文：<https://betterprogramming.pub/how-to-start-with-self-management-in-a-great-way-8f1bd4aab215>

## 你的组织在灌输自我管理方面有困难吗？那么这就是给你的文章！

![](img/7421ea857385c613363507c7114a3414.png)

[Icons8 团队](https://unsplash.com/@icons8?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

*“从第一次冲刺开始，这个 Scrum 团队就是自我管理的！你将决定工作的原因、内容和方式。不会有任何人告诉你该做什么，不会再有微观管理！”一家高科技机构的首席执行官彼得说。就这样，自主权被给予了，没有人告诉老鹰队的人将从事什么工作。*

老鹰们自己也松了一口气。过去的微观管理终于结束了。尽管如此，约翰还是定期过来，询问他们在做什么。这位前项目经理随时都会来看看团队在做什么。他很乐意看到团队意识到他在产品 Backlog 中的投入。而不是今天。

他在 backlog 中输入的问题是与 Eagles 的产品目标无关。然而，他们把它放在那里是为了取悦约翰，让他不要再开始咆哮说这样做有多重要。

下一个 Sprint 计划，老鹰队的产品负责人 Shruti 提出，与 John 相关的产品待办事项在这个 Sprint 中非常重要。*“我知道这不是我们想要做的，但是让我们继续吧。他会对完成的工作感到满意的。有他在我们这边真好。”仍然犹豫不决的开发人员将项目前期准备工作纳入到产品计划中。*

随着 Sprint 的进展，Shruti 增加了更多的 Sprint Backlog。*“我们甚至没有时间完成摆在首位的项目。你希望我们如何完成这一切？”其中一名开发人员指出。“嗯，”她回答道，“你在自我管理。你们决定如何完成。我只是在 Sprint 结束时需要它，因为我们真的需要它来实现 Sprint 目标。”*

他们不知道如何把事情做好。这是在每日的争论中公开讨论的。由于 John 的项目以及新项目，Sprint 目标的进展受到了阻碍。Shruti 了解到这一点后，请另一个团队的一位友好同事来解决他们面临的问题。

老鹰内部出现了巨大的分歧。这是在 Sprint 回顾的时候出来的。看起来这些项目是在外面完成的，但当被问及可维护性和引擎盖下的状态时，似乎是用了偷工减料的方式来获得这些功能。技术债务是全方位的。

不用说，这使 Shruti 和 John 都感到意外。在一次扔泥巴比赛之后，团队将这种情况带入了 Sprint 回顾会。我会告诉你接下来发生了什么。

这似乎是一个非常夸张的故事。不会是真的吧？嗯，确实是。我进入这个团队(当然名字不同)。几乎在冲刺的最后。很容易指出这里缺乏信任，但事实并非如此。在对情况进行了适当的分析之后(在团队的要求下，也让 John 参与进来)，我发现这里经常发生两件事:

*   没有人知道自我管理意味着什么
*   自主权不是一天给另一天的

我见过一些组织在墙上贴海报，总结敏捷转型带来的所有新角色和责任的期望。成百上千的人被派去参加课程，这样每个人都能理解这个框架。高度聪明的人，在另一个层面上，当谈到技术专长，但未能讨论和调整自我管理意味着什么的概念。当团队觉得 John 的行为非常可怕，Shruti 将新工作放入 Sprint Backlog 是错误的时候，John 和 Shruti 都觉得他们在这种情况下做得最好。

如果他们一开始就讨论自我管理对他们意味着什么，以及对彼此的期望是什么，他们可能不会陷入如此严重的分裂。这使我想到第二点。

自主权不是一朝一夕赋予的。这和教我的孩子骑自行车是一样的。当我给我三岁的孩子一辆崭新的自行车，期望他们当场自己骑走，会发生什么？他们会立刻变得呆若木鸡，惊恐地哭泣，而我会对昂贵的全新自行车上的划痕感到不安。

我几乎肯定你觉得这种假设的情况是相当愚蠢的。你说得对，这很蠢。但这与许多组织在过渡到自我管理团队时所做的完全相同。*“从现在开始享受自我管理吧，好运！”他们出发了。不是这样的。变得能自我管理，就像学骑自行车一样，需要训练轮子。*

# 自我管理就像一个凳子

职业 Scrum 培训师 Simon Reindl 和 Stephanie Ockerman 写了一本很棒的书，叫做[掌握职业 Scrum](https://www.informit.com/store/mastering-professional-scrum-a-practitioner-s-guide-9780134841526?utm_source=Referral&utm_medium=ScrumOrg&utm_campaign=mastering) 。除此之外，总的来说这是一本很棒的书，我真的很喜欢他们对自我管理的清晰描述。

他们将其描述为一个三条腿的凳子，有效的自我管理(或者 Scrum 指南中所说的自我组织)需要三条腿保持平衡。这三条腿是:

*   共同的目标
*   边界
*   明确的责任

从本文开头的场景中可以看出，这种平衡完全丧失了。没有共同的目标，因为老鹰队不理解约翰的愿望与产品和冲刺目标之间的联系，但无论如何还是做到了。Shruti 推进工作并让其他人来解决开发人员的问题，这表明边界没有被理解。每个人都承担着不同的责任。真是一团糟。

# 从达成共识开始

我的双胞胎目前正在学习骑自行车(实际上，当我写这篇文章时，我可以在后院听到他们的声音)。他们知道他们需要训练轮，这将逐渐提高难度。但是这种渐进的方法也是经验主义的一部分，帮助他们找到平衡。他们知道只有这样他们才能继续拆除他们的训练轮。他们明白，因为我们讨论过多次。当他们跳上哥哥的自行车，害怕摔倒时，他们也注意到了这一点。

决不要认为我觉得 Scrum 团队就像带着辅助轮的孩子。但是 Scrum 团队和围绕它的组织也需要一个渐进的、有指导的方法。从达成共识开始，理解那些共同的目标、界限和责任。让自己走向成功，而不是失败。确保参与的每个人都理解一开始对他们的期望，并不断检查和适应以改进它。

我很想知道在你的组织中自我管理是什么样子的。你们讨论过吗？
