说到成为一名优秀的程序员，有一些习惯会立即浮现在你的脑海中。有一些习惯是大多数程序员都会同意的，但实际上，他们中的大多数人自己并没有这些习惯。

众所周知，我们被习惯所定义。要成为一名更好的程序员，我们应该努力培养良好的编程习惯。

这里有六个很好的编程习惯，你应该努力养成，以便脱颖而出。

# 1.清理您的代码

一个很好的编程习惯是，每当你修改一段代码时，你都应该尝试改进这段代码——不管你是仅仅修改一行代码还是给现有代码添加一个全新的特性。

罗伯特·c·马丁对这种基于童子军规则的习惯有一句名言:

> "总是让你正在编辑的代码比你发现的更好."

当你阅读一些代码时，你会惊奇地发现你可以改进多少小事情。您所做的更改可以小到重命名一个变量或函数。只要确保一次至少做一个改进。您将使代码保持更好的状态，并大大减少代码味道的数量。

这是一个好习惯的原因是我们不断地学习。昨天被认为是好的事情今天可以用其他方式做得更好。随着时间的推移，代码往往会腐烂，所以照顾好您的代码对于保持代码的可维护性至关重要。

# 2.想想谁是下一个

每当你写一段代码的时候，你都应该考虑下一个开发者在看这段代码的时候会发现什么。

编写这段代码时，要尽可能让它易于理解和阅读。阅读代码和编写代码的时间比远远超过 10 比 1。这意味着，从长远来看，您可以通过多花一点精力使代码可读来节省大量时间。

不要试图在编码方面花里胡哨——写每个人都能理解的简单代码。真的没有必要去做一些你从 Stack Overflow 抄袭来的连你自己都没有真正理解的神奇的一行程序。

# 3.做需要做的事情——除此之外别无他法

开发人员往往有过度开发的奇怪习惯。当某个特性被请求时，大多数开发人员会构建一些额外的东西“以防万一”

这就是我们所说的 *overengineering* ，这是做过头的艺术。代码变得比必要的更健壮或更复杂。过度工程的问题是没有好的理由去做它。

大多数情况下，比实际需要建造的更多的原因是基于投机。开发人员已经根据自己的想法添加了将来可能有用的代码。这段额外的代码被添加到代码库中，但是它可能永远不会被使用。

过度工程化会导致代码被设计得过于一般化，以至于忽略了它最初被设计来执行的主要任务。因此，它变得不仅难以使用，而且从根本上难以理解。

# 4.规划您的代码

真正的程序员爱做一件事，而且只做一件事，那就是编程。这不应该真的让你震惊。因此，很明显，大多数程序员在被分配某项任务时，会直接跳到他们最喜欢的部分。

他们立即开始编码。

一开始，匆忙编写代码似乎很令人兴奋。然而，这种兴奋可能会耗费你大量的时间。当直接进入编码部分时，你最终会忽略更大的画面。

在你开始编码之前，你需要计划和组织。思考你可能会发现的问题，以及如何解决它们。你将如何构建你的代码？你打算实现这个特性的原因是什么？

在你开始编码之前，这些都是很好的问题。这些问题可以让你更加意识到这样一个事实:在编写代码之前，有很多事情需要考虑。

# 5.文件

是的，我知道……肯定不是这个清单上最有趣的习惯，但却是一个很好的习惯。记录你的工作是非常重要的。

您是否曾经不得不检查一个没有任何关于如何正确设置一切或应用程序应该如何工作的文档的存储库？

在通读代码几个小时后，您可能仍然没有任何线索。那时你想要一些文件。

最起码，我会为你的项目推荐一个合适的`readme.md`，解释应用程序应该做什么，以及你如何让它运行起来。

关于文档的令人兴奋的部分是你可以在编码的时候做它。花些时间考虑一下描述性的变量名——这是值得的。为你的变量、方法和类选择一个好的名字需要时间，但是节省的时间比花费的多。

如果你想了解更多关于如何在代码中创建有意义的名字，请阅读我写的这篇文章。

# 6.永远不要停止学习

正如艾萨克·阿西莫夫所说，“你停止学习的那一天，就是你开始堕落的那一天。”在科技领域尤其如此，因为这里的事物瞬息万变。

不可能跟上所有变化的东西，即使你能，我也不建议你这么做。不要走极端，但每隔一段时间学习一种新的框架或编程语言是有好处的。

这并不意味着你必须成为所有这些编程语言或框架的专家。尝试其中的一些，看看你是否喜欢，这很好。

您将获得许多新的见解，了解某些问题在其他编程语言或框架中是如何解决的，这将帮助您成长为一名程序员。

# 包装它

当谈到作为一名程序员能够脱颖而出时，你可以培养许多习惯。我们已经讨论了大多数开发人员没有的六个习惯。

这些习惯会让你脱颖而出:

*   清理您的代码
*   想想下一个要看你作品的人
*   不要做多余的事情
*   规划您的代码
*   记录你的工作
*   永远不要停止学习

试着为自己培养这些习惯，一次一个。记住，建立它们需要时间和坚持——这不会在一夜之间发生。
