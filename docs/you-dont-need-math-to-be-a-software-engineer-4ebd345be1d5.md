当人们开始把软件工程作为职业道路的时候，典型的第一个问题是“我不太擅长数学。软件工程需要我了解吗？”简短的回答是“可能不会”

我将在这里区分软件工程和计算机科学。计算机科学——这个学位——需要大量的数学知识。它对需要大量数学知识的领域进行了全面的理论概述(像离散结构这样的课程需要大量的证明)。

软件工程——这个职业——通常需要的数学知识要少得多。它关注的不是理论结构，而是用代码解决实际问题。现在，如果你在定量或算法领域工作，这个真理就不成立。定量或算法软件工程师将在日常工作中使用大量的数学。标准应用程序或 web 开发人员可能根本不会使用太多。

# 数学会让我在软件工程生涯中受益吗？

数学背后的逻辑思维过程与软件工程中的思维过程非常相似，因此在这方面，擅长数学应该可以很好地转化为擅长软件工程。很多软件工程都是关于解决问题和在方法上有创造性。这与数学非常相似，要解决一个问题，你必须选择一种特定的技术，并创造性地使用它。

这并不是说微积分等特定主题的知识可以直接应用，而是说，如果你可以理解更高级的数学概念，你应该可以轻松理解软件工程中更复杂的主题。

此外，软件工程的一些领域实际上需要大量的数学，在这些情况下，擅长数学将直接帮助你。正如我之前提到的，量化(想想投资银行或交易)和算法软件工程领域是两个关键领域。

# 3 项技能比数学更重要

如果你打算从事软件工程，但你不擅长数学，我不认为让自己成为更有吸引力的候选人的最佳途径是学习更多的数学。有许多领域你可以做得更好或者学习，这对你的职业生涯会有更切实的好处。

## 人际关系技巧

这是一个似乎被忽视的领域，但是提高你在这方面的能力将会帮助你从面试阶段一直进入你的职业生涯。从面试阶段开始，你可能会有一个行为面试。你拥有的任何软技能肯定会增加你被聘用的机会。即使在技术面试中，软技能也会帮助你更好地阐述你的解决方案以及你可能需要的任何帮助。

一旦你进入职业生涯，软技能将帮助你与同事发展更好的关系，更好地参与会议，更好地表达你的工作。

## 学习一个**额外的专业领域**

大多数人进入这个领域时都有一个关键的专业领域。这太棒了。在一件事情上非常胜任是确保你的技能在未来很长一段时间内保持价值的好方法。

在一个领域变得非常有能力是非常困难的，并且需要很长时间，所以在另一个领域变得熟练是不太可能的，但是有可能的。一个人可以希望做的是把自己介绍给新的领域，并获得一个基础。这个新的基础可能会成为你职业生涯的垫脚石，让你过渡到一个新的领域，或者在你当前的领域进一步发展你的洞察力。你可能会获得一种适用于你当前工作的新模式。

## 学习设计技巧

虽然可能不是一个受欢迎的选择，但这可能会被证明是相当有用的。你不必出去成为皮克斯动画制作人或 Photoshop 专家，但对工具和概念的基本理解将有助于你的软件工程工作。在苹果和谷歌实习时，我被要求整理演示文稿，有时我会努力在视觉上表达出我想要的东西。在这种情况下，理解布局会很有用。

如果你在移动或网络领域工作，了解基本的 UI/UX 设计肯定会有帮助。您将能够为 UI/UX 决策提供更多有用的输入，并在编写未来的代码时牢记这些原则。

最后，如果你只是想通过副业打入这个领域，你会发现设计技能非常有用。我做过很多 iOS 开发，每当我需要设计一个截图、一个图标或者一个资产的时候，它从来都不是我想要的样子。我一直有这个想法，但由于我缺乏 Photoshop 和 Pixelmator 等设计工具的知识以及设计理论的基础知识，我在 iOS 应用程序中的很多资产都很平庸。

# 结论

无论你选择学习什么样的新技能，学习本身不仅对你的软件工程生涯有益，而且对生活的方方面面都有益。
