## 一个学生出身的开发者对软件行业和大学之间鲜明对比的思考

![](img/cd197ea85e060545c733ccb4b4d1a42f.png)

安妮·斯普拉特在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

在本文中，我将与您分享我在软件行业工作时的一些经历，以及这些经历与我在大学时的经历有何不同。

在现有的众多学习编程的方法中，大学学位通常在社会上受到很高的评价和尊重，这是许多人选择的一条道路。有些人认为大学是成就好事业的唯一途径。我不同意这一点。

不要误解我。我不讨厌大学。事实上，我现在正在攻读大学学位，我喜欢这份工作。对我来说，大学最大的好处是能够和其他那些真正想成为伟大的开发人员的积极而专注的学生一起学习。我的学院奥尔堡大学的环境和文化邀请你分享知识并帮助他人提高。此外，教授们也很棒，能对困难的话题提供见解。最终，我知道这个软件工程学位将使我成为一名优秀的开发人员，具备所需的技能，如批判性思维、解决问题、团队合作、对高级算法的理解——还有很多。然而，通过我在现实世界中的兼职工作，我逐渐意识到，尽管大学有很多好处，但有很多情况是它不会让你做好准备的！

这里有一些我经历过的事情，是大学无法让你准备好的。

# 学校是安全的，行业不是

当我开始在这个行业工作时，我有一个可怕的发现，突然之间我的行为会产生真正的后果。如果你在公司销售或依赖的代码中引入错误或 bug，很可能会耗费公司的金钱或时间。

在大学期间，行动和效果之间的这种关联并没有那么强烈。当然，你也必须在大学里表现出色，如果你写了糟糕的代码或者没有花足够的时间学习课程，这可能会对你的成绩产生影响，但那又怎么样呢？这到底意味着什么？以我的经验来看，这真的没什么区别，未来的雇主不会看你的成绩单，他们会更关注你实际能做什么。当我开始开发销售给客户的软件时，我突然感觉到一种全新的责任感。

# **怪异的限制**

我在大学完成的所有项目和编码任务的共同点是，它们在你选择如何解决它们方面有很高的自由度。我有一定的限制，比如使用特定的语言，但有时规则也是完全开放的。这种选择的自由，绝对是你离开大学这个安全空间就不能指望拥有的奢侈品。你将被雇佣的公司将会和他们自己的技术团队一起工作，你必须接受你很可能不得不和他们一起工作。

除了技术方面的限制，其他的限制也会出现在工作中。你工作的公司可能会在他们的产品线中加入某个硬件，你将要开发的软件必须在这个硬件上运行。或者他们的服务只支持十年前的技术。例如，这可能是运行特定固件版本的特定电视型号。

您可能已经编写了一段很棒的 JavaScript 逻辑，当您在自己的计算机上的 Chrome 环境中试用时，它完全按照预期的方式工作！接下来，当代码突然在电视上不起作用时，你会感到困惑。这怎么可能？它在你的电脑上运行得非常好，而且电视和电脑都使用 Chrome，对吗？嗯，很可能这台电视运行的是旧版本的 Chrome，不支持新的 ECMAScript 6 `import` 语句。你会怎么做？你可能无法改变电视，所以你必须改变你的代码，并尝试以另一种方式解决问题。

在大学，你可以自由选择不同的技术，你不会以同样的方式经历这种困境。您通常可以选择最适合您的用例的技术。否则，您也可以通过改变项目的范围或方向来规避有害的限制。这在工业上是行不通的。最有可能的是，你不能仅仅改变所用的硬件，因为它可能是公司产品线的重要组成部分。

你只需找到解决问题的方法。这也是他们雇用你的原因，对吗？

# **截止日期……那动了？**

在大学，你习惯了有截止日期。考试在特定的日期举行，有些项目和作业必须在某个时间提交。你几乎总是可以确定这些日期是固定的，不会改变。那些是工作的好环境。

然而，正如你可能已经猜到的，当与客户和顾客一起工作时，情况并不总是这样。期限突然变了，有时候变化很大！你可能会在早上来到办公室，期望有两个多月的时间来完成你正在开发的 React 应用程序，却发现新客户已经到来，他们期待的是与公司完全不同的东西。为了优先考虑这个新客户，你舒适的两个月时间表现在可能已经缩短了一半。这只是游戏的名字。环境在变化，你必须适应它们。当你在学校时，情况就不一样了。

# 你没有经验

即使大学学位需要很多年，你也不可能学会所有关于软件的知识。没有一个大学毕业生具备高级开发人员的知识和经验。即使上完大学，你还是很青涩。此外，技术发展迅速，每天都有一些前沿的框架或库发布。

当你开始为真正的公司工作时，经理和老板会带着要求你编写你从未尝试过的东西的任务来找你。这是非常好的；然后你只需坐下来学习如何去做。开始学习一些你从未想过的利基技术可能会很有趣。例如，我曾经被要求使用 Nullsoft Scriptable Install System(NSIS)为 Windows installer 编写脚本，在我开始从事这项具体工作之前，我从未听说过这种系统。

编写 Windows installer 脚本并不是互联网上搜索最多的话题，也不是大多数开发人员都在做的事情。因此，我很难在网上找到关于如何制作静默安装程序或在成功安装后自动执行程序的安装程序的帮助。

关键是你会在大学里学到很多有用的东西(你也会学到一些不太有用的东西)，但它永远不会让你为你将要做的所有怪异和小众的编码项目做好准备。

这些只是我在攻读大学学位和行业工作期间观察到的一些真实情况。如果你渴望开始学习，或者已经是一名学生，那么我希望这不会让你害怕获得大学学位，也不会让你担心毕业后的工作生活会是什么样子。我的目标只是试着让你对即将到来的事情和你可以期待的事情有所准备。当然，每个人的情况会有所不同，这些仅仅是一个人的经历。你的旅程很可能与我的有一些相同之处，但肯定也会有不同之处。

如果你对大学这个话题以及它如何让你为这个行业做好准备有一些想法或经历，请留下评论或联系我。我很想听听你的意见。
