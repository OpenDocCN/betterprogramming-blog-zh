# 创建您自己的笔记/个人库

自从一年多前开始接触 ReactJS，我就一直在做这件事。

在我们必须记住要做的这么多事情中，当你是一名开发人员时，尝试将事情自动化一直是主要的想法之一——尝试尽可能地节省时间，缩短我们的处理时间，并消除重复的任务。

作为开发人员，我们需要适应并练习消除重复性任务。

例如，如果你需要使用 React 路由器，像其他开发者一样，我们点击 Google 搜索按钮，找到文档并学习它。然后一切都结束了，我们继续前进。然而，几天过去了，甚至几周过去了，当我们再次遇到 React 路由器时，我们再次点击谷歌搜索，并重复。

如果不是 React Router，那么 CSS 属性呢——我们一直在使用它们，一次又一次。

与其试图去记起你在哪里找到了那个最喜欢的动画，那个 CSS 属性，为什么不创建你自己的库，你的定位笔记呢？

也可能是你已经解决了的，或者想出的解决方案，你不想再处理同样的错误，我们容易忘记事情，所以你必须开始记录它。

无论是在编程时，还是在个人生活中，使用这个不重复任务的原则将会帮助你变得更有效率和生产力。

时间是黄金，明智地使用它！

# 不要只是复制粘贴——明白吗

不到万不得已不要停下来。

一开始，复制粘贴代码是可以理解的。然而，如果你想成为一名优秀的开发人员，你需要理解事情发生的原因。如果你不知道你的项目发生了什么，你就不能称自己为开发者。

你必须了解它们是如何工作的，为什么工作，以及它们是如何联系和共存的。这就是代码的美妙之处，它是令人着迷和令人兴奋的结构和架构，就像一件艺术品。

在构建项目的过程中，您会遇到一些错误和 bug。如果你知道你在做什么，调试就不会那么难了。然后，每当你遇到某件事而上网寻找解决方案时，你也可以把它们添加到你的个人文档中。你开始了解那个解决方案，所以当你再次处理它或类似事情的时候，你知道该怎么做。

这就是为什么你必须花一些时间去理解它——这是学习编程的唯一途径，也是成为一名优秀程序员的最佳途径。

练习阅读代码，因为作为一名开发人员不仅仅是创建和构建应用程序。在真实的企业开发人员世界中，您将被要求继续他人的代码，重用现有的代码，并增强和维护他人的工作——所以开始阅读他人的工作吧。您将学习良好的实践和编码风格。

# 最好的学习方法是把自己放在那里

我的开发者之旅走在快车道上。

经过六个月紧张艰苦的工作，我找到了工作。我所做的就是工作、学习和找工作。

在我的第一份工作中工作了三个月后，我和另外两个同事一起接受了一个移动应用项目。如果你读过我的大部分文章，你会知道在我的第一份工作中，我与其他八名计算机科学毕业生一起工作，所以你可以想象这对我的压力和不安全感有多大帮助。

我会说这不是最好的旅程，但它是一个地狱般的旅程。我学到了很多，不仅仅是技术方面，还有如何对待自己。

最好的学习方法是把自己放在外面，让自己接触真实的世界。别人怎么想不重要。就算你感受到了世界上所有的疑惑，也没关系。有时候你需要做的就是出现——因为没有什么比克服我们最困难的时候更值得的了。

## 如果我们不经历我们的最低点，我们永远不知道我们能上升多远

你最好的老师是经验，所以不要害怕犯错误。创造很多，最重要的是你如何克服那些错误，如何解决，如何从中吸取教训。

我的第一份工作不是最愉快的，它像过山车一样，但却是最有回报的。我长得比我想象的还要高，还要好。

如果你没有经历过不好的事情，你就不会知道好的。

# 调试的艺术

调试就是一遍又一遍地做同样的事情，却期待不同的结果。

这是编程中不那么有趣的部分，在这一部分，你开始怀疑自己在生活中做过的所有错误的事情，质疑自己做过的所有选择，一遍又一遍地问自己 web 开发是否适合你。

但世界上最美好的感觉之一是当你最终解决了那个 bug。今天早些时候，我终于能够修复一些网络故障连接，我们已经努力了一个多星期。最后，它是如此的令人满足，以至于我在那之后立刻打了个盹。

根据经验，一次解决一个问题会更容易，如果您可以在控制台上看到许多红线，不要紧张，一旦您可以修复少数实际错误，大多数明显的问题都会自行解决。

所以最好的方法是打破它，一次只犯一个错误。这些错误中的大部分可能在您之前的开发人员都遇到过。如果你一次只解决一个问题，相信我——一旦你编译了应用程序，其他的警告和错误就会自己解决。跟着错误走，当你读台词的时候，当你获得经验的时候，你就会明白了。

如果在长时间的尝试调试之后，你仍然不能想出一个解决方案，休息一下，出去走走，呼吸一些新鲜空气——大多数时候，*我们*是我们的问题。

为什么我们会有这种把简单的事情复杂化的习惯？

# 足智多谋将是你最大的优势之一

尤其是当你是一个自学成才的开发人员时。

这是你能获得的最好的技能之一，不仅仅是对于网络开发，而是在日常生活中。

你不能坐等事情发生在你身上。如果你想要什么，那就起来为之努力吧。

作为一个自学成才的开发者，你不需要花费太多就能成为一名开发者。许多人会不同意，但我没有花太多就让自己成为了一个成功的人。除了一些额外的时间和远大的梦想，我一无所有。

我今天拥有的大部分东西只是我昨天的奢侈品，所以不要再找借口了。如果你想学习，想成为一名移动开发者或者软件开发者，你的意志应该比你的借口更强大。没有“正确的时间”，我们所拥有的只是时间和我们选择用它做什么。

努力每次都胜过天赋。我正在努力成为一名高级全栈开发人员。我不打算花上几千就为了成为一个，我已经离我的目标很近了。

我们今天选择做什么或不做什么将是我们明天的基础，所以无论你选择继续睡觉还是起床完成事情，这都是你的选择。

最重要的是，不时地休息一下。如果你生病了，你所有的努力都将付之东流，所以休息一下吧！

> “是时候成为最好版本的自己了。”
