# 我听过的关于技术面试的最好建议

我很幸运，我能够在职业生涯的早期获得一个[高级开发人员](https://javascript.plainenglish.io/top-10-advanced-vs-code-settings-for-senior-developers-46e348351bd6)的职位，至少从我开始[我的中型博客](https://doctorderek.medium.com/)算起。

在过去的 3 年里，我从咨询和自由职业者变成了全职，所以我参加了几十次技术面试。

有一个关键的提示似乎总能让我进入更深层次的面试。很简单，但是很有效。

> *分享你最近的工作以及它对你的挑战。分享你犯的最后一个错误或者你必须解决的产品问题(当然是在合理的范围内)。*
> 
> — [约翰·J·马斯](https://medium.com/u/8f7d594fe6af?source=post_page-----c38d5a4edd38--------------------------------) [在他的博客上](https://medium.com/@jjmasse/tech-screen-like-a-rock-star-b62fd1e435a9)

对我来说，这个建议在技术电话筛选中最有用，当我第一次和在公司工作的工程师交谈时，或者更好的是，在我将加入的团队中工作。

大多数面试官都非常有礼貌和专业，这意味着作为一名候选人，你有足够的时间来交谈和引导对话。

对我来说最成功的面试，包括让我找到现在这份工作的面试，都是我以一句“哇”开始的。

当你通过提及最近的成就来介绍自己时，你可能会立刻给面试官留下深刻印象。

当然，你可以表现得很自吹自擂，或者你可以最终提到公司技术堆栈中没有使用的技术，但是十有八九你会脱颖而出。

# 一个简单的 3 步公式 Ace 技术筛选

即使你没有直截了当地提到你最近的工作，以及它如何在你的“开场白”(你的[介绍](https://www.urbandictionary.com/define.php?term=intro))中挑战你，你仍然可以在面试的后面有一个[麦克风掉落的时刻](https://www.urbandictionary.com/define.php?term=Mic%20Drop)。

当我讨论我作为一名[专业网站开发人员](/how-to-set-up-vs-code-like-a-pro-in-just-5-minutes-65aaa5788c0d)或[软件工程师](https://blog.devgenius.io/why-you-cant-just-be-a-software-engineer-689e56eacef3)的技术工作时，我喜欢使用这个简单的 3 步模板:

> 1 —以下是我所做的
> 
> 2——这是我学到的
> 
> 3 —这是我下次要做的

我从职业教练那里得到的最好的建议之一是，面试很像约会。

我是如何理解这句话的意思的——它可能对你不起作用——我应该在面试中带头展示我的适应能力。

当我谈论我的技术工作时，我总是试图包括一些细节，比如哪里出错了，以及我为下次学到了什么。

我发现绝大多数雇主希望有人愿意改进，尤其是不需要别人要求的人。

# 沿着记忆的小路散步

我从 2019 年开始的成就之一，当我在 [React Hooks](https://javascript.plainenglish.io/how-to-add-to-an-array-in-react-state-3d08ddb2e1dc) 发布后不久第一次学习 React 时，与[beautiful](https://prettier.io/)有关。

那时候，我正在制作许多简单的演示项目，并发表在我当时的新媒体博客上

在[我简单、干净的代码](/why-you-should-make-your-code-as-simple-as-possible-3b35e89f137)和[生物信息学的学士和硕士](https://medium.com/about-me-stories/how-i-went-from-physical-therapist-to-seo-website-performance-expert-8dbfd557150)(计算机科学专业)的结合中，我能够在一个由 3 名开发人员组成的分布式远程团队中找到一份兼职工作。

我已经有一段时间没有认真看过别人的产品代码了，所以我很兴奋地启动 GitHub 并检查一些[拉请求](https://www.urbandictionary.com/define.php?term=pull%20request)(更好的说法是 [PRs](https://www.urbandictionary.com/define.php?term=PR) )。

很快，我们遇到了一个问题，每个人的拉请求都是无意义的，因为一些开发人员使用分号，而其他人没有，这个问题通常通过[用更漂亮的](https://levelup.gitconnected.com/how-to-format-all-files-in-a-directory-with-prettier-5f0ff5f4ffb2)格式化所有文件来解决。

这个计划只有一个小小的障碍——团队没有使用更漂亮的。这还是在我在博客上解决 [VS 代码设置](/how-to-set-up-vs-code-like-a-pro-in-just-5-minutes-65aaa5788c0d)之前。

在我工作的第二周，我为整个团队布置得更漂亮了，这立即提高了我们的工作效率和协作能力。

在我的技术面试中，我仍然能够用这个简单的例子作为我学到的东西和我成长的经验。

底线是，我了解到在现代编程中，像 Prettier 这样的代码格式化程序并不是一个“好东西”,而是一个必需品。

# 美丽寓言的士气:关键外卖

当然，这是一个有点琐碎的例子，但这种“轻松取胜”在技术面试和电话面试中获得回报的原因很容易理解。

这是一个易于沟通的故事，有清晰的开头、中间和结尾，我分享了我是如何受到挑战并从中成长的。

我下次学到的是，作为远程团队的一部分，要对建立最佳实践有信心，因为这样做的好处是值得的。也许我不应该等到第二周才着手这个项目。

这个故事的寓意是，每当我讨论我的技术工作和专业背景时，招聘经理都会产生共鸣。

*你会带来什么？*让他们知道你在将代码投入生产并从中学习方面有着坚实的背景。

这就是你赢得下一次技术面试的方法。
