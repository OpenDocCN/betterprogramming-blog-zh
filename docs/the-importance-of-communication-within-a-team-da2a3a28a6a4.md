## 有效沟通在工作中非常重要

![](img/4d087405bade5e185331f63568e30262.png)

普里西拉·杜·普里兹在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

虽然听起来不像婚姻顾问，但沟通在生活的各个方面都非常重要，对吗？

它是两方或多方之间的信息传递。这听起来很琐碎，但实际上并非如此。

为了有效地交流，你不能仅仅发送你的信息。你需要以这样一种方式发送它，它将被预期的人正确地解释。仅仅因为它对你有意义，并不意味着它对所有的接收者都有意义。

在技术领域(如软件团队),这可能需要一些时间，需要有意识地努力去了解你的团队，他们对某些技术堆栈的理解水平，他们的领域知识，甚至你实际使用的口语和书面语，毕竟，英语不是每个人的第一语言。

那么，我们如何让这变得更容易呢？好吧，我真的很想再次呼吁婚姻顾问的建议，所以我会的。

通过有意识地努力去了解一个人，通过倾听和提问，我们可以更好地了解他。毕竟，有效的沟通是双向的。

当同时与多人交流时，显然更难做到“个人化”。但是你可以利用你的个人关系和对团队成员的了解，至少和他们中的一些人建立联系，让讨论进行下去。自然地，随着对话的发展和话题的偏离，你可能会和越来越多的团队成员产生联系。

在我目前的职位上，我负责主持大多数团队会议和敏捷仪式。因此，我需要能够向来自不同国家、文化和背景的团队成员传达信息。

我们的沟通并不完美，但我们在不断改进，所以这里有一些我发现行之有效的方法(不管你在团队中的位置如何)。

# **重复自己**

我经常用稍微不同的词语重复自己的话，或者用类比的方法来表达理解。

我还经常跟进书面解释对相关方的期望——这样他们就有书面记录可以参考(如果还没有的话)(比如在吉拉问题上)。

# **不要做假设**

现在这个超级难。你不可能说什么都 100%明确。

你会很快失去你的观众。但事实是，假设是错误、返工和延迟的主要原因。问自己一个问题:给定这个信息，我做了什么假设(并认真思考)？

然后，当你回顾假设时，问问你自己，如果被误解，哪些假设会显著影响工作项目的范围，这些是你应该非常明确的事情。

![](img/5e15a6ac1a228638022eb05c5d0a34a3.png)

# **过度沟通**

事情几乎每天都在变化(至少对我来说是这样)。因此，确保人们尽可能获得最新的信息。

根据我的经验，公司和个人通常遵循“需要知道”的文化，我完全不同意这一点。

我相信给你的团队尽可能多的信息，然后相信他们会做出最明智的决定。

# **DoDs(完成的定义)**

这是我引以为豪的，因为它对我们的团队产生了非常积极的影响。

让我们面对它，在开发中“完成”通常有 t 和 c 的附加(它完成了，但是…)，这些 DOD 是在我们的 Sprint 规划会议之前创建的，然后在 Sprint 规划期间进行讨论和调整(或者完全修改，如果需要的话)。

它们向我们提供了一个共同的理解，即对于要完成的项目的开发意味着什么，这最小化了 QA、支持和开发之间的来回。

# **语气和用词**

随着向远程工作的转变以及通过微软团队、Slack 等以书面形式进行的交流越来越多，在发送邮件之前对其进行审查是非常必要的。当你不在最佳状态或者你在对比对方所说的话时更是如此。

我有过一些经历，我被告知有人对别人对他们说话的方式不满意，经过调查。

通常归结为语气和用词的选择。所以，附上一句请、谢谢或其他友好的话，让对方知道你并不想表现得粗鲁或居高临下。

毕竟，我们也是人，我们有时都会在交流中有所欠缺，这没关系，不要为此自责，但我们可以努力做得更好，为什么不呢？
