随着对软件快速高效交付的需求增加，对 DevOps 工程师的需求也在持续增长，这使其成为当今市场上最受欢迎的技术职位之一。

为了成为一名成功的 DevOps 工程师，并在竞争异常激烈的世界中取得成功，你需要一些特殊的技能。

以下是该领域职业生涯中必不可少的五大技能。

# 1.自动化

自动化是 DevOps 的核心。

“等等，所以你每天都手动完成这项任务，每次都要花一个小时？”作为一名 DevOps 工程师，你会经常问别人这种问题。

几乎所有手动执行的开发和发布任务都应该由 DevOps 工程师自动完成。非常重要的是*软件开发生命周期中尽可能少的手工步骤。如果你想让发布软件高效且一致(谁不想呢？)，那么自动化就是答案。*

## **那么，DevOps 工程师自动化的是什么？**

一名工程师将自动化开发和发布过程的所有方面。为了称自己为 DevOps 工程师，您需要能够创建一个完全自动化的端到端管道，从从存储库构建代码开始，到将软件发布到生产环境结束。这个管道通常还包括基础设施创建、配置管理和安全自动化。

## **这怎么可能？**

利用各种可用的 DevOps 工具，并且能够编写一两个奇怪的脚本。重要的是要知道你不需要能够使用每一种 DevOps 技术(有很多！)而且无论如何你都不需要成为一个编码大师。

因此，您不需要了解所有的 DevOps 工具，但是您通常需要能够使用以下每一组中的至少一种工具:

*   [持续部署](https://www.ibm.com/cloud/learn/continuous-deployment)工具，如 Jenkins、Azure DevOps、Spinnaker 和 Travis。
*   [配置管理](https://www.redhat.com/en/topics/automation/what-is-configuration-management)Puppet、Chef、Ansible 等工具。
*   [基础设施为代码](https://www.redhat.com/en/topics/automation/what-is-infrastructure-as-code-iac) 工具如 Terraform 和 Pulumi。

就脚本而言，很好地掌握像 Ruby、Python 和 Shell 这样的语言对于自动化来说肯定是很方便的。

# 2.沟通和协作

为了理解为什么沟通和协作对这个角色很重要，你需要理解为什么 DevOps 的实践会形成。

DevOps 背后的一个主要驱动因素是开发和运营团队对版本的不同看法。

一方面，开发人员希望发布尽可能多的特性。“客户需要这种功能。我们需要释放”是他们的观点。

另一方面，有些运营部门希望减少软件发布的频率。他们的观点是“每一个版本都是一个变化，而变化就是一个风险。”

DevOps 对这个问题的解决方案是，一组人负责确保以安全、可靠和有效的方式创建、测试和部署软件。那群人是 DevOps 工程师。

## **那么，这与沟通或协作有什么关系呢？**

DevOps 就是要打破前面提到的团队之间的孤岛，这意味着 DevOps 工程师需要能够与人们建立联系，并将他们从整个组织的团队中聚集到一起。有了这种团结，所有的团队都朝着向客户交付高质量软件的共同目标一起工作。

此外，为了在一个组织中实现 DevOps 文化，您需要教育其他人关于 DevOps 原则和工具。没有强大的沟通和协作技能，这是不可能的。

# 3.云

谈到基础设施，许多公司选择云计算来存放他们的 it 服务和应用程序，而不是传统的现场选项。

[90%的公司使用某种类型的云服务](https://leftronic.com/blog/cloud-computing-statistics/)。

借助云提供商提供的服务，DevOps 工程师能够无缝管理基础设施，因为云使组织能够创建可扩展、灵活且安全的应用。

有许多不同的云提供商都提供自己独特的服务——其中最常见的是 AWS、Azure 和 Google。

您需要深入了解的云中的基本服务类型有:

*   建立工作关系网
*   虚拟化(虚拟机/容器/无服务器)
*   安全性
*   储存；储备
*   监视

如果你足够幸运，在一家将基础设施应用程序托管在云中的公司工作，那么你需要熟悉云架构的设计和开发。例如，知道何时使用 IaaS 产品或 PaaS 产品更好。

# 4.安全性

如果安全性是事后才想到的，那么 DevOps 支持的高软件发布率可能是危险的。DevOps 工程师必须确保在软件开发生命周期的所有部分都牢记安全性。

添加新的应用程序功能？让我们确保在它进入生产之前对它进行漏洞扫描。

添加新的云服务？让我们确保只有我们想访问的资源才能访问它。

DevOps 工程师通过在任何代码和基础设施变更发生之前实施安全性，改变了安全性是一个部署后过程的观念。这通常被称为 *DevSecOps* 。

# 5.操作支持

优秀的 DevOps 工程师不仅仅从事开发工作。DevOps 包罗万象，因此您应该对 IT 操作有扎实的技术理解。

DevOps 工程师应确保所有系统都具备监控和警报功能，以协助生产服务的可用性和正常运行时间。有许多工具可供你使用——Prometheus 和 Grafana 就是其中的两个例子。

在出现问题时，具备在相应操作系统上排除技术故障的技能非常重要。了解 Linux 和 Windows 服务器非常重要。

在完全失去服务的情况下，开发运维工程师通常会启动灾难恢复流程来恢复服务。理想情况下，灾难恢复过程将是自动化的！

# 结论

发展这些技能对于成为一名成功的 DevOps 工程师至关重要。DevOps 工程师是一名出色的团队成员，他优先考虑客户满意度。您需要能够对变化做出良好的反应，在技术上很强，并且对 DevOps 工具有很好的理解。
