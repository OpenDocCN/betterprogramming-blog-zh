## **通过对象初始化器进行初始化**

与构造函数相比，对象初始化器显式声明字段名和传递的参数值。这使得正在发生的事情更加清楚。如果您的所有字段都有公共 setters，这意味着您的对象不是不可变的，那么这种方法完全没问题。C# 9.0 引入了`init`关键字，这是一个专门的设置器，允许用对象初始化器设置字段的值。

## 分段对象创建

另一种方法是分段创建对象。这意味着您首先创建一个空对象，然后逐个设置字段的值。这可能非常重复，并且需要一个不带任何参数的构造函数方法。

# 建设者

构建器是一个单独负责对象创建的组件。引入构建器组件清楚地将您的逻辑和模型类与创建和初始化所需的实例分开。

这种创造性模式的主要好处是它使代码库更具可读性和可维护性。创建对象变得更加明确，因为它在创建时清楚地陈述了这个特定实例的内容和数据。

此外，它允许您重载构建器方法，这使得创建具有不同配置的实例变得非常容易。最后，它支持在某些情况下可能需要的不可变对象。

# 要点和限制

构建器模式是实例化类的好方法。就我个人而言，我主要使用这种模式进行测试。这是一种创建模拟对象的简单方法，可以清楚地说明特定场景的测试设置。另一个实际的例子是将它用于定制的 HttpClients，可以用不同的(预)配置来创建。

尽管它带来了优势，但 fluent builder 使继承变得非常困难。你需要使用递归泛型。这种方法是可行的，但是会使构建器代码更加复杂。
