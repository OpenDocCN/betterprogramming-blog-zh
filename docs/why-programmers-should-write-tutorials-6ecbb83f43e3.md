想象一下这种情况:你接到一个任务，要在你公司的软件中实现一个新特性。你以为会很容易，因为几个月前你也做过类似的事情。复制粘贴就好，开心就好。你甚至开始做一些与问题无关的事情，因为你有时间。

截止日期前几个小时，你打开旧代码，惊慌失措。你不明白你写了什么，也不明白你为什么选择那些数据结构。如果你足够幸运，已经编写了一个代码，你就可以开始检查测试代码。

在你运行测试，做一些改变看看会发生什么，并在[堆栈溢出](https://stackoverflow.com/)上寻找替代方法之后，你最终能够理解代码并实现最初的任务。

最后，您浪费了几个小时(并遇到了许多自我怀疑)才想起您必须安装一个不在原始文档中的库。一个简单的命令。不感到沮丧是必然的。

作为一名程序员，你可能经历过这种情况。你获得的经验越多，你再次面对这种情况的机会就越大——仅仅因为你没有把它写下来。

# 写下来

作为一名程序员，养成记录你发现的任何新知识的习惯是很重要的。如果只是简单的一个[插件配置](https://medium.com/vacatronics/how-to-configure-remote-access-in-vs-code-9067af6010af)或者类似于[参考指南](https://levelup.gitconnected.com/a-guide-to-python-decorators-f3b03dd05514)之类的也没关系。就像你三年级老师常说的，“用你自己的话写下来就行了。”

它不必是畅销书的质量。你只需要描述完成任务所需的所有步骤，如果需要的话，还要描述每个步骤背后的原因。

它不需要在你完成后就对了，但是不要等太久才开始做。令人惊讶的是，你的大脑可以如此迅速和容易地从一个重要的话题切换到一些你甚至不喜欢的新流行歌曲。

# 它节省时间

写教程和文章的主要好处之一是节省时间。

这就像写一份食谱。你可以随时检查，重复这个过程，得到同样的结果，而不必记住每一步。

相信我，你未来的自己会感激的。我支持我过去写的许多教程。那个文档节省了我很多时间。但是要保持一切井井有条，因为如果你不去寻找你是如何做到的，而是去寻找合适的文本，这并没有什么不同。

也可以节省别人的时间。如果你是一名高级开发人员，你的一些初级同事(如果不是其他高级同事)可能会经常问你如何做某事。如果是简单的问题，你可以在几分钟内回答，无论如何你都会被打断。

如果你像我一样，被打断是可能发生的最讨厌的事情之一。想象一下，在你还没有意识到的情况下，你的手指正在打字，有人停下来问你为什么最后一次提交没有编译。

文档将节省他们和你的时间。他们不必去寻找解决方案，你也不必停下你正在做的事情。

# 证明文件

写作也是确保你理解问题和解决方案背后的所有步骤和概念的一种方式。它帮助你将知识储存到你的大脑中。

它可以作为你未来的参考指南。每当你有疑问的时候，你都可以查阅的图书馆。

这是记录代码的一种方式。每个程序员都至少有一次想知道当他们写那行代码时他们在想什么，或者为什么他们必须安装一些程序或库。当你写教程的时候，你记录了你所做的所有决定。相信我，你会为此感谢自己的。

它可以作为一个如何使用你的代码的例子，甚至可以作为将来你的包的文档。

当您使用第三方库时，如果文档和您需要的一样好，您将会很幸运。很多时候是不完整的，或者是过时的。

# 结对编程

在某种程度上，这是结对编程的一种形式。你必须解释你做了什么，为了解决你遇到的问题，你需要做出什么样的选择。当你向你的*伙伴*解释这一点时，你可以确保所有的代码都在正确的位置，改进一个功能，或者甚至创建一个更好的测试用例。

即使是对一些实际上不是问题的东西(例如一个概念或一些基本的安装步骤)，这也是有用的。

有时候，你不能和其他人谈论你的代码。他们在自己的区域或其他地方忙碌着。

想想其他程序员，他们将不得不使用你正在编写的代码，他们需要如何理解每一步。这会迫使你把事情说得尽可能清楚和详细。

# 你可以挣钱

你甚至可以从你的教程中赚钱。它可以通过你自己的博客，通过一些出版物，甚至是媒体。你将帮助你自己，帮助别人，并在此过程中赚取一些额外的钱。

这不是很好吗？

想象一下，请你的家人在一家高级餐厅享受一个美好的夜晚，因为你只是将你的知识用语言表达出来。

这也是一种创建内容和向世界展示你对某个主题或工具的了解的方式。你可以通过写作在某些圈子里出名。这可以让你的老板知道你值得加薪。你甚至可能在另一家公司找到一份更好的工作。

此外，随着时间的推移，你可以将你所有的文章分组，并创建一本电子书甚至一门课程。有很多培训平台可供你上传课程。而且在亚马逊上自助出版也没那么难。

# 结论

如果你还没有开始写，今天就开始吧！从简单的安装说明或代码片段开始。没关系。

无论你是学生还是专家，写下你所学到的东西。它可以是编程技巧、教程或一些见解。

相信以后会让你受益匪浅。
