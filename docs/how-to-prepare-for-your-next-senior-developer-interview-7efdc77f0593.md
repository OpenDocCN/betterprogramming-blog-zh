我申请的高级开发人员职位都没有回复我的简历。

对于几乎所有的职位，我都写了个性化的求职信和适合这些职位的定制简历。我甚至自学了很多新技能，只是为了把它们写进我的简历。

# 高级开发人员的招聘是不同的

当雇佣初级开发人员时，公司会出去寻找开发人员。当雇佣高级开发人员时，开发人员必须推销自己。

每一位高级开发人员都要经过面试，测试他们的:

*   最低资格(人才招聘面试)。
*   编码能力(现场采访/异地/coder pad-代码共享)。
*   架构能力(又名系统面试)。
*   团队/管理能力(采访队友/CTO /CEO)。
*   适合公司文化(额外的人力资源面试)。

这与初级开发人员形成了鲜明的对比，他们经常从黑客马拉松、校园面试或在线编码竞赛活动中大量加入。

鉴于每个阶段投入的时间量，高级开发人员的招聘成本因此呈指数级增长。和金钱，如果有外部招聘人员的话。

你最好用你的简历、求职信和第一个电话来证明这一点。而做到这一点的关键就是推销自己。

以下是高级开发者应用的四个小技巧。

# 研究，研究，更多的研究

营销人员在接近顾客之前通常会研究他们的顾客。作者对他们的出版商也是如此。只有程序员会给从旧金山到悉尼的 90 个雇主写同样的样板申请信。

程序员可以发现样板代码，但不需要程序员去感知样板求职信。避免样板文件和使你的求职信个性化的唯一方法是做深入的研究。

## 1.研究他们做什么

*“我对高性能的 UX 非常感兴趣，尤其是动画 gif 和视频内容。”*

一家托管 yoga 视频市场的软件公司可能会喜欢你求职信*中的这句话，因为你需要一个前端开发职位*。如果是同一家公司的视频压缩工程师职位，没有*极度热情的*部分，那条线可能会做得很好。如果你申请的是股票交易算法应用，完全没有必要。

审查你在求职信中写的每一行。**关于你，它说了些什么，也和你申请的产品部门有交集？**

只保留最重要的，去掉其余的。

## 2.关于它们为什么以及如何存在的研究

申请创业公司更是如此。

创业公司经常讨论他们自己的创业故事。是什么打动了他们，他们看到了哪些痛点，他们是如何提出解决方案的，以及他们对改变现状有多大的热情。

当他们从渴望成为自己创业公司一部分的人那里听到同样的话时，他们会感到被认可。

想象一下他们的喜悦——他们不仅得到了一名开发人员，还得到了一名行走的福音传道者。能够向潜在利益相关者推销他们的故事的人。有人会写一些关于创建下一代 VR 游戏/人工智能支持的数据库/前端框架有多棒的病毒式推文。触及到财大气粗的投资者的推文。

永远记住:人是用心买的，用头脑来证明。

迎合他们内心想要的，但只要确保你自己也能感受到。不要听起来像假的。

## 3.研究一下你要向谁申请

这是相当微不足道的一点，但不是不重要。

在把你的申请发给招聘经理的时候，对他们来自哪里，他们曾经从事过什么工作等等进行初步调查。查看他们的 LinkedIn 也无妨。如果他们会来找你，他们也会这么做的。

它可以给你带来明显的优势，也可以让你避免有害的求职信错误。

例如，如果你是英国人，电子邮件的收件人也是英国人，讽刺一下伦敦的天气可能会打破僵局。但是如果你不是英国人，这可能会导致一个有偏见的人立即拒绝你。

## 4.领会言外之意(说出“最好的拥有者”)

去年，我申请了一家斯堪的纳维亚国家领先银行的高级开发职位。

他们很高兴地欢迎我，但我最终被拒绝了，可能是因为我对一个样板问题的愚蠢回答:**你编写了多少测试用例？**我说，**有些，不是很多。**'

我知道这是工作要求，但只是一个额外的要求。我不认为它有多重要。

一年后我带着很多新技能申请了。我又被拒绝了，甚至在面试之前。

然后，我在一次开发者大会上碰到了他们的前开发者。他吹嘘自己是如何因为被迫编写愚蠢的测试用例而离开那家银行的。

那发生在十八个月前。就在我采访他们之前。

当他们再次发布那份工作时，我放了一个 Github repo，专门演示我的自动化测试用例编写，我在求职信中简单而明确地提到了它。

仅仅两周和一轮超级简单的面试后，我得到了一个团队领导的职位。

这个故事的寓意？好的技能通常是必备的要求，你永远不要忽视它们。

在银行的例子中，他们认为测试用例是开发人员的责任，不管它有多小。

违背这一信念伤害了他们。对这种信念的确认导致了对所有需要的框的检查。

这是重新审视的推销员原则:**人们用他们的心去购买，他们用他们的头脑去证明它的合理性。**

# 结论

软件公司雇佣高级开发人员不仅仅是为了编程。他们经常雇佣他们向其他开发人员销售他们的愿景/产品，并最终向外界销售。

如果你对推销自己有足够的了解，你的求职信就能脱颖而出，让你获得面试邀请。
