# Backlog Limbo 是什么鬼东西？

是啊，不是那个[游戏/舞蹈](https://en.wikipedia.org/wiki/Limbo_(dance))。我的意思是被一个既不会被删除也不会被处理的任务所困扰。

积压任务的问题在于，它们似乎一直占据着你大脑深处的一个空间。

不停地唠叨，提醒你有些事情需要做，而你却什么都不做。

我很高兴地告诉你，有一个解决办法，而且非常简单，虽然不容易。

继续 limbo 的比喻，它将使我们能够把这些该死的任务送上天堂(完成)或地狱(删除)。

# GTD 解决(几乎)你所有问题的方法

## **将任务改写为清晰的物理动作**

简单，但不容易。

它还有一个可选的第二步:如果一项任务不能被重写为一个明确的物理动作，它就不是一项任务。你在处理一个项目。

项目只是包含多个物理操作任务的高层任务的抽象。所以你的第二步是将任务定义为一个项目，并将其分解为实际行动的子任务。

在我们开始举例之前，让我们解释一下为什么这个解决方案如此有效。

## 为什么这种解决方案如此有效？

很简单，真的。我们的猴子思维不能很好地处理抽象概念。那是因为他们将抽象与时间距离联系在一起，时间距离是“时间上遥远”的代名词

也就是说，如果你抽象地定义事物，你的大脑会假设它们会在遥远的未来发生，所以没有必要现在就解决它们。这一事实似乎与拖延症密切相关。

例如，观察以下两个任务:

1.
逃避藏在灌木丛后面的老虎。“发现生命的意义”

这两项任务都很重要，但是你会拖延哪一项呢？

一句话:如果你的任务不具体，你会认为那是遥远的未来，所以你更有可能拖延。

# 例子——从抽象到具体

让我们从一个简单的例子开始。让我们将下面的任务从抽象转换为物理:*“检查我们是否能做 x”.*

“支票”绝对是一个抽象名词。
要将其转化为具体的物质，让我们问问自己:
我们需要采取的实际行动是什么？

在这种情况下，我们实际上需要和 Tim 谈谈 X.
所以任务就变成了:*“给 Tim 打电话谈谈 X 的可行性。”* 调用“是具体的，所以我们的大脑知道需要做什么。

我们倾向于拖延的其他类型的任务是“思考”任务。比如我们把*“想想用 X”*转换成有用的东西。

“想”绝对是一个抽象。这项任务短期内不会发生。让我们换一种说法:*“与我的团队安排一次会议，讨论我在 x 上看过的文章。”*

有点长，但是很清楚。我需要安排一次会议——这是一个物理动作，包括进入我的日历应用程序并点击一个时间段。

我也有参与者的具体名单——我的团队，以及我们将要谈论的内容——我在 x 上读过的文章。

这项任务发生的可能性可能要高 10 倍。

现在让我们看一个更复杂的例子——“处理服务中的错误”

乍看之下，除了有点笼统——什么服务，哪些错误？看起来不算太糟，对吧？

但是你的大脑如何将这项任务转化为身体动作呢？
手柄是什么意思？“句柄”还是一个抽象概念。

你能把它重新表述为一个明确的身体动作吗？大概不会。

那是因为这不是单个任务，而是一个项目。让我们来分解一下:

*   创建一个“项目”类型的新任务，这样我们就可以组织具体的子任务
*   开放吉拉/阿萨纳/星期一等。，过滤“生产错误”——在你最喜欢的笔记应用程序上把它们写下来。
*   安排与团队成员的会议——议程是“确定错误列表的优先级”
*   阅读会议记录，并为每个错误添加一个子任务——“处理服务 Y 中的错误 X”——这稍微好一点，但仍然是抽象的——这意味着每个子任务都是我们需要分解的另一个“项目”:
    ——运行流 Z 以重现错误 X
    ——观察堆栈跟踪和日志
    ——应用修复(根据用例，该任务可以是抽象的/具体的)

这可能看起来有点过了，但是如果你看看上面的步骤，大多数只需要几分钟。完成具体的小任务也有助于通过少量的多巴胺激发你的动力。

所以，是的，你花了几分钟分解一项任务。但作为回报，你终于在一项被搁置数月的任务上取得了进展。
你终于把事情做完了。

从待办事项中删除任务的加分，尤其是如果你不能将它们重写为清晰的物理动作。

# 底线

我们的猴子思维不能很好地处理抽象概念。
[那是因为他们把抽象和时间距离](https://journals.sagepub.com/doi/10.1111/j.1467-9280.2008.02240.x)联系在一起。这意味着抽象的任务将会被搁置一段时间。
除非你把它们改写成清晰的物理动作。

对于复杂的任务，也就是项目，也需要将任务分解成具体的子任务。

一旦任务是有形的、具体的，你的思维最终可以对它们采取行动。这就是你如何最终摆脱积压-冷宫。
