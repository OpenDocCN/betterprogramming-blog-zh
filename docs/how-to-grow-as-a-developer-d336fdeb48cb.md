## 作为一名开发人员，你不必仅仅依靠你的开发职业来成长

如果你有时觉得自己在职业生涯中没有成长，或者没有机会学习新东西，因为你忙于满足客户的要求，我完全可以理解。

今天，我们将讨论一些关于你如何独自成长为一名开发人员的要点。

# 作为一名开发者，你如何成长？

1.  成长可能意味着你已经练习了足够多，现在可以更快地解决曾经遇到的问题。当有人要求您这样做时，您知道如何在您的应用程序中实现该功能。
2.  您质疑现有的最佳实践和解决方案。你努力寻找更好的解决问题的方法。可能有现成的解决方案，但是您有能力为社区改进它们。
3.  作为社区的一员，你参与讨论并做出贡献。你从社区中其他开发者创造的内容中消费和学习，并且总是紧跟潮流。
4.  你也用你的知识和贡献为社区的发展做出了贡献。

# 1.练习——尝试建立自己的挑战

![](img/d60a07ec44223abb959716e9a686cd26.png)

照片由[詹姆斯·庞德](https://unsplash.com/@jamesponddotco?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄。

*   有没有你喜欢或者好奇的功能(比如视频流、文件上传、认证流程)？
*   你在 Medium/Dribble 上看到过什么很棒的设计吗？你想知道现实世界的应用程序是如何实现它的吗？
*   你可以建立一个像网飞这样的流行网站的简单版本。
*   你可以创建像博客、电子商务应用，甚至是一个愚蠢的待办事项列表应用这样的常青树应用。这将帮助你掌握最简单的 CRUD 操作。

重点是在遵循最佳实践的同时着手实施它们。在实施时，请随意提出您自己的最佳实践。

一旦开始，你就已经决定了你要构建的一系列特性，并且会模仿你的应用程序的其他特性。这会把你引向一些新的问题，你会开始学习寻找合适的解决方法。

你将使用 Google、Stack Overflow、GitHub、官方文档、YouTube 视频和媒体文章来自己搜索解决方案——这是你开始成长的地方。

# 2.挑战解决方案

![](img/ee1be3d4b68abe8b1884fafda615d39a.png)

[张家瑜](https://unsplash.com/@danielkcheung?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照。

当我们试图解决特定问题时，我们会提出最佳实践。但是你要解决的问题不一定是一样的，那么为什么解决方案要一样呢？

因此，你应该质疑现有的解决方案，尝试优化事情，尝试做得更好。你就是这样想出自己的解决方案的。只是因为有开发人员挑战解决方案，并总是试图找到更好的方法来做事情，所以我们在社区中有这些最佳实践和指南。

尝试提出自己的模式和解决方案，并根据最佳实践验证它们。有时候，你会找到更好的方法。其他时候，你会了解你当前方法中需要改进的地方。你在学习和成长。

# 3.合作

![](img/4ef72c59da6fc03967e4a3f32da808ba.png)

由[拍摄的照片将在](https://unsplash.com/@will0629?utm_source=medium&utm_medium=referral) [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上 Porada 。

你可以在 Stack Overflow、GitHub、Reddit、Linkedin、Telegram、Slack、YouTube comments、Quora、Medium、脸书群组等网站上找到很多开发伙伴。向他们寻求帮助，或者讨论某个话题。

不要只是通过所有这些来源消费来自社区的内容，也要开始回馈一些东西。至少从投票表决堆栈溢出的有用答案或更新一些过时的答案开始。你可以写中型文章来分享你的解决方案。

一旦你看到其他开发者同意你的内容或者觉得它有帮助，你会获得一种自豪感和认同感。会上瘾的。然后你会找到带来更多价值的方法，成长为一个开发者。

不要成为通过贬损性评论攻击他人的有毒开发者——尤其是那些可能会问一些基本问题的新手。刻薄不会给任何人带来任何价值，所以不要刻薄。

没有人从中学到什么，如果你看到有人这样，提醒他们，他们没有给任何人增加任何价值。

# 4.贡献和支持

![](img/3f0155c47e959ae61eed77b9f07ac475.png)

[张家瑜](https://unsplash.com/@danielkcheung?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照。

为别人的项目做贡献。您将从其他开发人员那里获得大量反馈，以改进您的编码方式和解决问题的方式。

不要害怕失败、被纠正或被别人挑战。所有这些只会帮助你学习和成长。

你可以帮助其他人学习。引导他们找到正确的资源并教他们。你可以通过任何可能的媒介来教学，比如 YouTube，媒体文章，课程。天空是这里的极限。

你也可以向那些帮助你找到工作或教会你一些有价值的东西的流行节目内容创作者提供经济上的帮助。今天你可能没有钱可以捐赠，但是当你有了钱的时候，给他们一些回报，感谢他们为了让你成为一个更好的程序员而付出的努力。

至少，你可以投票支持他们的内容，订阅/关注他们，为他们的故事鼓掌，评论，感谢他们，等等。一切都有帮助。
