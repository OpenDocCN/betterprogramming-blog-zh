# 如果 Python 是这种语言，请注意你写的内容

几年前，C 是我的主要语言。有一天我用 300 行代码写了一个`filter`程序。一旦我完成了，程序编译并成功运行，我就高兴了。

我很高兴，因为这似乎很公平。投入到代码生成中的工作量与程序的预期成正比。我的意思是，编写一个复制 Photoshop 行为的程序是一项严格的任务。

Python 有另一种说法。

几个月后，我偶然发现了这个程序的 Python 版本。我惊讶地发现，三行代码就足以让 Python 产生与 C 语言相同的输出。

我知道 Python 版本更加紧凑和简洁，这要归功于该语言丰富的库。但是与 Python 相反，找到适合您需求的外部 C 库比大海捞针要困难一些。

我感觉糟透了。

我之所以有这种感觉，是因为当人们争论用 Python 创建高级程序的时候，我仍然被 c 语言的低级混乱所困扰。

我想我赶上蟒蛇列车有点晚了。但是一旦我抓住了它，我就用双手紧紧抓住它。

但事后看来，C 是一次有收获的旅程。

虽然用 JAVA 和 C 等顽固的语言编码比用 Python 编码要困难得多，但我发现用 C 思维编写 Python 代码是一种很好的编程方法。

我过去用 C 语言编写代码时总是想着“让它工作”。但是对于 Python，我编码时会考虑“让它尽善尽美”。

我变得对 Python [反模式](https://towardsdatascience.com/four-unusual-python-code-smells-7bf3d93f5e6c)更加过敏。不要责备我，要责备 c。

> 注意:我想强调的是，我说的是“用 C 思维方式编码”，而不是“用 C 风格编码”用 C 风格写 Python 代码就像从右向左写英语一样。

# 你的代码就是你的影子

编程是一门艺术？

当然不是。

假设你在博物馆。一件艺术品引起了你的注意。显然，发生这种情况是因为你被艺术家的最终结果所吸引。

现在假设你正在研究软件。产量不错。然而，源代码可能没有那么好。

好的艺术=好的结果

好代码=好(结果+分析)

这里主要的一点是，程序员在编码的时候，应该是为了别人而编码，而不是为了自己。你需要牢记这一点来编码。

[马丁·福勒](https://www.goodreads.com/quotes/6341736-any-fool-can-write-code-that-a-computer-can-understand)说道:

> “任何傻瓜都能写出计算机能理解的代码。优秀的程序员会写出人类能理解的代码。”

上面的引用适用于一般的编程。但我觉得与其他编程语言相比，它与 Python 联系紧密。

记住，写 Python 代码就像写英语一样。如果你的代码读起来不像英语，你就失败了。同样，就像英语一样，你的热情应该闪耀在整个代码中。

Python 的目标是让生活变得更简单。如果我迷失在你的 Python 代码中，如果你用 C 编码会怎么样？坦白地说，一想到会发生什么，我就害怕。

用 C 或其他静态类型语言编码是很耗时的。Python 节省了大量时间。这就是 Python 被广泛用于原型制作的原因。因此，Python 开发人员应该致力于将用 C 等语言编写代码节省下来的时间投入到使他们的代码对其他人更具可读性的工作中。

如果我看不懂你的代码，我不会责怪我的愚蠢。我只会为你感到遗憾，因为 Python 很容易，而你却让它变得很难。Python 的创造者说:

> “Python 可以教到中学的孩子。”—吉多·范·罗苏姆

最终，编程不是在真空中完成的。这是团队合作。因此，你应该注意，总有人要收拾你的烂摊子。如果你用 Python 编程，确保你的代码不那么混乱。

程序是程序员的一面镜子。如果你的程序没救了，你也没救了。

值得注意的是，程序就像人类一样:它们会衰老。它们会随着时间退化。Python 在这方面是臭名昭著的。新版本一个接一个地出现。

与劣质代码相比，优质代码是抗老化的。因此，努力编写高质量的代码。

# 外卖食品

编程曾经是一匹黑马。只有少数人敢于涉足这一领域。

现在它是开放的，每个人都可以访问。Python 使得编程变得更加容易。多亏了 Python，每个人都可以和他们的机器交流——专业人士、非专业人士、孩子，甚至你的祖父母。

从这个意义上说，我恳请你在与机器交流时，明智地选择你的语言。
