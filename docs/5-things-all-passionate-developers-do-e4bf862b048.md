## 编码不仅仅是代码

![](img/b761dcb73aa591a9b35cc1cf538d62bb.png)

摄于 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 的[佘山 R](https://unsplash.com/@thelankankid?utm_source=medium&utm_medium=referral)

每个开发人员都想享受编码，充满激情地创造东西。帮助我改变对编码的看法的方法将会为你所用。有时它会影响你，让你觉得被工作榨干了。[冒名顶替综合症](https://en.wikipedia.org/wiki/Impostor_syndrome)也可能发生，你可能会失去兴趣。

我会给你一些获得激情的小技巧，这些小技巧会帮助你获得编程的激情。

# 让你的大脑休息一下

![](img/54bcb6ddee3917027f56a0c2006ebb3c.png)

照片由[凯尔·米尔斯](https://unsplash.com/@thehammockhermit?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

离开你的办公桌听起来可能违反直觉，但是散步有帮助。散步让你保持头脑清醒的状态，让你休息。不要仅仅认为它有助于编码——它对健康也有好处。

徒步旅行很棒，因为你会有一种胜利的感觉。到目前为止我亲自爬了三座山，感觉很棒。

当你处于笨拙的释放或最后期限时，散步能让你平静下来。你一伸腿，新的想法就源源而来。

心理健康对开发人员来说至关重要。说到底，我们每天都处于压力之下。编码之旅包括遗留代码、寻找讨厌的 bug 和有毒的测试人员。

# 实施最佳实践

学长可能会错意，那就另想办法吧。一个问题有多种解决方案:你的目标是实现最好的。你会为这部分代码感到骄傲；这会增强你的信心。花点时间构思一个完美的解决方案，或者留有改进的空间。

每个项目都有技术债务，所以一定要发现它。无论你在哪里看，测试覆盖率都可以提高。寻找可以减少代码行的地方。这将提升你对团队的价值。

# 结对编程

帮助一个团队成员为获得帮助打开了一个机会。这会激发你的激情，因为在这段旅程中你并不孤单。

公司喜欢团队合作。毕竟，这是项目成败的关键。一起工作会鼓励你们一起支持一个功能。如果某件事没有按预期的那样发挥作用，身边知道这件事的人就多了。

即使是最好的开发人员也会遇到没有出路的问题，所以一定要谈论你的问题。毕竟[橡皮鸭调试](https://en.wikipedia.org/wiki/Rubber_duck_debugging)是个东西。如果你不分享你的问题，你会很快对解决方案失去兴趣。

# 走进未知的世界

![](img/2ee13e63cc3943aa62fa99ce00b0f3c7.png)

[Gabriel](https://unsplash.com/@natural?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

了解一些未知的东西会帮助你扩大知识面。更多的知识给你更多的力量，你可以适当地传播或使用它。

你将绘制通向未知的路径。成为对任何话题都了解最多的人。这会让你变得重要和有价值。

# 避免难闻的代码

每天尽你所能。你写的代码应该是一流的。从长远来看，现在的小调整会有所帮助。正确地测试你所做的一切，从单元测试到集成测试。

bug 是会发生的，如果不是你造成的，你会觉得很强大。写伟大的代码会给你更多的激情。编码是一门艺术，所以你应该三思而后行。

一定要考虑向团队展示你的解决方案。与团队一起传播知识，找到更好的解决方案。

# 额外收获:做代码审查

评审是每个项目不可或缺的一部分。不要只是浏览代码而不知道它是做什么的。联系团队成员，一起审查代码。在浏览代码时，可能会出现误解或错误的解决方案。我就是这样！

开发人员应该花更多的时间阅读，而不是写作。编程语言不仅是为了写作，也是为了人类消费。一定要检查代码中的每一个不一致之处。这将有助于你提升自己的价值，并让你的项目知识更加丰富。
