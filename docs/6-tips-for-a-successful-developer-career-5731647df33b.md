最近，许多初级开发人员向我咨询如何适应技术。基本上，他们问的是如何在科技领域拥有一个长久而繁荣的职业生涯。现在，我在科技行业离完美还很远——我仍然认为自己是个初学者——但我想我可以从经验中给出一些建议。所以，我可以提供一些建议，但在我们开始之前，我想确保你明白，即使你还没有进入工作环境，这些对你来说仍然非常有价值。

# 1.适应性

是的，我知道这是老生常谈，但这就是你如何在技术领域获得长期职业生涯的方法。你看，这个行业的变化太大了，你必须适应并跟上发展的步伐。这一点从你在科技领域的社交媒体上看到的内容中可以明显看出。你如何保持适应性？第一件事就是给自己打造一个信息管道。订阅几个 RSS 订阅源，获取不同的时事通讯，养成每天去有科技内容的内容网站的习惯，并坚持下去。

在那之后，想出一个在笔记本或类似的东西里储存信息的方法。这允许你根据一个特定的主题组织所有的链接、提示和技巧、文章和视频。随着时间的推移，你会对该领域发生的一切有一种广度和深度的感觉，并成为其他人的资源。

另一件事是要么做你的项目，要么做开源工作。这可以让你获得大量的经验，你可以把这些经验放到今天的简历中，这也是大多数招聘者正在寻找的。我的意思是，所有的招聘人员都在寻找他们可以利用的实用技能。当你在寻找开源工作，特别是你想创建的项目时，找到你认为是你的弱点的领域。我认为做所有这些事情的一个额外好处是，人们——在职业生涯的早期和晚期——确实患有冒名顶替综合症。

治疗冒名顶替综合症的一个好方法是，知道得越多，感觉越舒服。冒名顶替综合症的一部分是你觉得自己好像什么都知道，对吗？那么，走出去，让消费新信息、尝试新事物成为一种日常习惯，然后你会觉得自己更像是科技世界中的一名公民。

适应能力的另一个要素是你想学习模式而不是学习特定的技术。特定的技术总是来来去去。然而，你如何构建你的代码、算法和数据结构并不会随着时间而改变很多。人们做的另一件事是找到对他们有用的东西，并把它作为他们生活的中心。这并不坏，但技术变化太多，太频繁，你不能把自己锁在任何东西里。作为替代，如果你想在特定技术的观点上做出更明智的决定，看看利弊，然后从那里开始。

# 2.找到合适的位置

我知道我建议不要进入任何特定的技术或类似的领域，但有些专业领域你可以用来建立自己的技能组合，让你对公司更有价值。这些领域包括可访问性、数据可视化，甚至机器学习，专攻这些领域会让你成为一家公司特别优秀的候选人。例如，如果你有全栈背景，并在 React Native 中有一些子专业，那么你就成为一家公司更有价值的资产。

# 3.沟通和移情

这两件事通常是密切相关的。如果你是自由职业者，你需要关心你的同事和客户。其中一部分是理解人们想从你这里听到什么，设身处地地为他们着想，理解他们如何看待一个特定的项目或项目中的某个功能。这将有助于你精心制作你的信息，为他们工作，并有效地沟通。

# 4.像经理一样思考

正如我之前所说，移情的一种形式是学习如何像别人一样思考。嗯，像雇佣你的人一样思考。学习业务。你想做什么？企业想要完成什么？这将使你在关于如何推动你用来建立业务的软件选择的对话中更有价值。

管理领域的一个重大决策是购买还是建造。如果您了解业务，您可以帮助做出更明智的决策，了解公司将软件资金花在了什么地方——是从货架上拿走一些东西，还是投资构建一些东西，以便根据它是否是任务关键型的来成为业务的一部分。

另一个重要的是公司文化。这很常见，因为在你职业生涯的某个阶段，你可能会被要求加入进来，成为文化变革的推动者。当这种情况发生时，或者有人要求你这样做时，退后一步，评估一下你正在进入的文化。这主要是因为公司文化的存在是有原因的。如果一家公司通过销售产品赚钱，那么这种文化就是为他们服务的。

找出哪些文化元素值得改变，以及它们是否能被你即将进入的文化所接受。从外面看起来不正常的东西在里面可能是完全正常的，尽管看起来有点奇怪。

这些系统中的一切都是有原因的，在试图改变它之前，你必须首先理解这个原因。要明白你的团队或经理要求你做的每件事都是一个测试。如果有人让你做，比如说，一个模糊的项目，把它当作一个测试。分配给您的不同任务用于评估您的职业发展。例如，他们可能会要求你担任某项特定任务的团队领导。在这种任务中，他们会评估你成为团队领导的能力。你必须知道他们想看到的测试结果。

# 5.友好点

首先，我知道这听起来很荒谬。但事实是:社交因素对建立长期关系很重要。当和你一起工作的人让你参与一个项目，甚至是未来的合作时，你就会知道你是否成功地发展了这些关系。

如果你从逻辑上思考这个问题，当你在整个职业生涯中前进的时候，你会构建出那个社会图表。不要误解我:我不是说你总是去酒吧。仔细想想，友好就是好生意。人们更喜欢和一个很好的人一起工作——他能在一天内完成一个功能——而不是一个能在一小时内完成同样工作的混蛋；他们知道好人会更灵活。

# 6.工作就是工作

压力是一种东西，尤其是在科技领域。我知道技术似乎是一个我们一直坐着的职业，你知道，在一个凉爽的空间里，关键任务的错误可能不会以死亡告终，但仍然有很多压力。考虑一下这个问题。当你建立的网站关闭或者你直接关联的应用崩溃时，通常会发生什么？参与上述项目的每个人都加入进来，尽自己的一份力量，努力让项目重新运转起来。或者找出它不工作的原因并修复它。那会非常有压力。因此，你需要考虑你的压力水平，培养你的压力管理技能，使它们与你将要从事的职业相匹配。

例如，作为一名数据科学家，你不需要站在生产、管理或开发的前沿。我是说，你很少会在周末被叫去修理东西。这与从事前端、后端或开发工作的人不同。对他们来说，一个平静的周末可能会变成一个工作日，因为他们可能会被叫去做一些工作，紧急部署一个新功能，甚至修复一个以前没有见过的 bug。给自己留点时间很重要。要明白每个人都会经历压力，而且有时压力很难控制。休息一段时间将确保你总是有动力去工作，并让你在那个职位上的经历变得更好。

我希望这篇文章能让你明白为什么你在这个领域可能不像你应该的那样舒适，并帮助你获得迄今为止最好的开发体验。
