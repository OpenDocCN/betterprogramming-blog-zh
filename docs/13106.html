使用Hashlips art generator、IPFS、Remix和Dapp的全包式分步指南

现在网上有很多关于如何创造和打造你的NFT收藏的指南。但是当你不得不自己创造一切的时候，尤其是当你是一个艺术家的时候，这就变得很棘手了。

我为那些团队中没有开发人员，但仍然希望使用自己的智能合同启动NFT项目的人准备了本指南，与直接通过市场或使用“即用型”服务相比，智能合同更加可靠和灵活。

1.艺术

首先，你需要艺术。如果你是一名艺术家——我不需要告诉你这种艺术是如何被创造出来的。如果你不是——就用Photoshop、Figma或Procreate。我个人使用生育，这是相当可怕的。所有你需要的是画一些层。层数越多，你的收藏就越独特。

2.NFT一代

目前在这个领域中，NFT一代最好的工具之一是由hash lips开发的名为“Hashlips Art Engine”的工具。它是开源的，所以你可以从 Hashlips Github仓库下载并安装它。

只需克隆它或者下载一个ZIP文件，用任何代码编辑器从根文件夹中打开项目，例如— VS Code 。

另外，安装 Node.js 推荐包，并在项目终端运行npm install。

抓取您的图层，按类型分类放入文件夹。比如——背景、脸、眼镜等。并将项目的图层文件夹— C:\Users\yaros\Documents\hashlips_art_engine\layers中的所有内容替换到您的图层中。

然后转到C:\hashlips_art_engine\src\config.js文件，把这个文件里面的所有数据都换成你真正的采集数据。

更改以下几行:

const namePrefix = “Your Collection”;
const description = "Remember to replace this description";

暂时跳过这一行:

const baseUri = “ipfs://NewUriToReplace”;

注意你的层在layerConfigurations数组中的正确顺序。它应该按照你的NFT在画布上显示的顺序排列。例如，从最远的图层(背景)开始，到最近的图层(眼镜)结束。GrowEditionSizeTo为收款金额。如果层数足够的话，可以设置为1000甚至10000。

const layerConfigurations = [ { growEditionSizeTo: 5, layersOrder: [ { name: "Background" }, { name: "Eyeball" }, { name: "Eye color" }, { name: "Iris" }, { name: "Shine" }, { name: "Bottom lid" }, { name: "Top lid" }, ], }, ];

此外，您可以通过将#50添加到您的收藏中来增加稀有性。png文件名。例如，背景黑#50将是背景红#100的两倍。

现在，打开VS代码终端，运行npm run generate。这将生成您的收藏并将其放在build文件夹中。在这里你会找到images和json文件夹。在下一节中，我们需要将它们部署到IPFS。

3.部署到IPFS

IPFS是一个分布式系统，用于存储和访问文件、网站、应用程序和数据。与集中式数据存储相比，IPFS的主要优势是可以从不受一个组织管理的多个位置下载文件。

去https://ipfs.io/下载IPFS桌面应用。

转到文件选项卡，然后按导入按钮。这将上传你的图像文件夹到IPFS。我们需要复制CID并进入我们的艺术生成应用程序。

将config.json中的这一行替换为您新获得的图像文件夹的CID。

const baseUri = "ipfs://QmRXh4mEZ1uKaPLGzjhrk1zTMCWJJb9UjXcE7qDrmGFxGx";

然后到VS代码终端运行npm run update_info 脚本。 这将重新生成元数据文件，并更新每个JSON文件的IMG URL。这将是未来像https://opensea.io/这样的市场正确展示你的NFT所需要的。

在我们更新了我们的元数据之后——让我们回到IPFS应用程序并将json 文件夹上传到IPFS。

现在，我们准备创建我们的第一个智能合同！

4.智能合同ERC-721

转到 Hashlips git repository 并复制智能合约的代码。

转到 Remix IDE 并在contracts文件夹中创建一个新文件。

复制智能合同代码并将其粘贴到您的.sol文件中。

更改这些变量以满足您的收集要求。你不能再改变maxSupply 了，但是你可以改变NFT成本。

uint256 public cost = 0.05 ether; // Cost of your NFTuint256 public maxSupply = 10000; // Collection Amountuint256 public maxMintAmount = 20; // Max mint amount for 1 wallet

进入实体编译器选项卡，按下编译按钮。这将为我们的部署合同做准备。

在Metamask wallet上启用测试网络。我们将使用Rinkbey网络。

在将智能合约部署到主网络之前，必须将其部署到测试网络，如Rinkeby，以确保一切正常运行——你可以在市场上看到NFTs，你可以铸造NFT，你可以从智能合约中提取资金。

转到Remix编辑器的部署和运行事务选项卡，并执行以下操作:

  • 选择环境-注入提供元掩码
  • 默认情况下，应选择元掩码帐户。这个沃勒将是合同的所有者。为了能够将它部署到区块链的林克比，你需要一些钱。你可以在这里买到ETH的测试赛车——https://faucets.chain.link/rinkeby。
  • 在合同下拉列表中选择您的合同
  • 转到部署部分并打开它。您将看到3个字段:_NAME、_SYMBOL和_INTBASEURI —您应该像上面的截图一样用您的收集数据填充它们。Name是你的收藏名称，Symbol是你在区块链上的令牌符号，BaseUri —是你在IPFS上的JSON文件夹的链接。

非常重要的是，您要以这样的格式添加它——ipfs://qmesv 16 rcskwcizau 67y 5 fpwpjchfbnaarc 8 KD 25 ci 8 eik/

按下交易按钮——这将打开你的Metamask钱包，你应该确认交易。

瞧啊。你的第一份智能合同在区块链上。

要将其部署到主区块链网络，您只需将metmask钱包网络更改为以太坊或多边形。

5.创建DAPP

分散式应用程序(DApp、Dapp、dapp或dApp)是一个可以自主运行的应用程序，通常通过使用智能契约，它运行在分散式计算、区块链或其他分布式总账系统上。

我们将使用由 Hashlips 提供的牛逼解决方案。

只需克隆或者下载一个ZIP文件，用 VS代码从根文件夹打开项目。然后在项目的终端中运行npm install。

运行npm运行开始，您将看到这个神奇的应用程序:

您可以更改背景和图像，也可以调整样式以匹配您的收藏主题。

转到hashlips_nft_minting_dapp\public\config\config.json文件，更新您的收藏和网络设置。确保设置正确的合同地址、网络名称、符号和id。集合名称、符号和最大供应量应与我们最近部署的智能合同中的数据相匹配。重要的是从魏的Dapp中设定正确的NFT价格。你可以在这里把ETH转换成魏——。

我们准备从我们的Dapp铸造我们的第一个NFT！在此之前，我们必须修复项目中的一行代码。

转到hashlips_nft_minting_dapp\src\App.js文件，通过简单地添加blockchain.account作为mint函数的第一个参数，修复第133行的代码。并按CTRL-S保存您的更改。

.mint(blockchain.account, mintAmount)

现在你可以用你的Dapp铸造一个NFT了！

6.部署DAPP

首先，您需要将本地代码添加到Github存储库中。转到https://github.com/new并创建您的远程存储库。然后将您的本地代码推送给它。如果你不熟悉Github，请跟随这些文档。

我们将使用Netlify将我们的Dapp部署到生产中。进入Netlify“从Git资源库导入现有项目”页面—【https://app.netlify.com/start T4】，将您的帐户连接到您的Github资源库。

选择您的Github存储库:

接下来，选择分支名称(默认为main)、构建命令(可能是“npm run build”或“yarn build”)和发布目录(默认为build)。并按下“部署地点”。
