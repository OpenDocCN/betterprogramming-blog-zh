如何为你的新项目选择最好的编程语言

在你开始新的工作之前，考虑一下适合这项工作的语言

大约一年前，我得到了职业生涯中的一个机会。领导层找到我，问我是否想领导一个新的云原生团队。

“当然，我愿意。”

我花了一些时间从我公司的开发组织中召集了一个具有前瞻性思维的团队，我认为他们会使团队取得成功。我建立了一个体面的平衡，这些人的经验范围从只有几年的经验一直到20多岁。

从历史上看，我们不是云开发者。我们是一家大型企业软件公司，在微软技术体系中有着深厚的根基。我们知道一切都将发生翻天覆地的变化，但我们从未考虑过的一个变化是编程语言。为什么？

因为我们一直都是这样做的。

如果有效，为什么要改变？它是我们所知道的，所以我们不必花时间去学习一门新的编程语言。我们将花所有的时间来学习CI/CD、无服务器开发和NoSQL。

不要害怕扰乱

罗伯特·克里格尔在他的书 中讲述了旧的商业流程是如何扼杀创新的。很久以前就应该重新评估的过程，但从未受到质疑，因为它们一直都在进行。

我和我的团队很快意识到我们都习惯的编程语言不适合我们的新应用程序。我们构建了一些Lambda函数，由于冷启动时间的原因，这些函数非常慢。我们考虑过解决冷启动问题的方法，但是我们觉得这是在与我们所掌握的工具作斗争。

所以我们决定成为颠覆者。我们加入这个团队是为了做出改变，我们不会让旧的流程和神圣的牛挡住我们的路。

逆潮流而动的决定是一个艰难的决定，但我们想确保我们的成功。中断通常意味着独自面对，因为没有合适的流程。但是如果你像我一样有一个稳固的团队，一切皆有可能。

使用合适的工具完成工作

你不会用锤子在一块木头上钉一颗螺丝钉，也不会用钢锯在2x4的边缘锯出一条路线。不，你应该使用适合手头任务的工具。

你的编程语言就是你的工具。这实际上就是你用来构建产品的东西。为了在最短的时间内做出最好的应用，使用最好的工具。

以下是一些主要编程语言的普遍接受的用例:

  • python——数据科学、机器学习、后端web开发
  • C# —基于Windows的胖客户端应用程序、游戏
  • Java —胖客户端应用、安卓应用、后端web开发
  • JavaScript —快速web开发
  • Swift — iOS应用程序

显然，有更多的语言可以考虑，但是请记住，有些语言比其他语言更容易解决问题。

如果需要，可以改变

在开发过程的早期，您有一个窗口，可以在对生产率影响最小的情况下更改编程语言。您已经完成了概念验证，发现了一些陷阱，看起来风险大于回报。

改变它。

已经做了最少的工作。作为一个团队聚在一起讨论利弊。如果团队一致认为这种编程语言不是正确的方法，那么尽早打电话并做出改变。等得越久，重构就越难。

这种情况发生在我的团队身上不止一次，而是两次。在这个过程的早期，我们很快决定使用Python。几个星期后，我们意识到对于我们的用例来说，这不是一种可维护的语言。所以我们聚在一起，讨论我们的选择，并转向不同的语言。

你能做的最糟糕的事情就是把自己锁在一种不能解决你的问题的编程语言里。在项目的早期阶段保持高度警惕，以便尽早发现并做出积极的改变。

想想未来

你的余生都不会在这个产品上了。总有一天，你要么被调到一个新项目，要么离开公司，去别处追求一些东西。

这意味着你需要考虑你的继任者。在你之后来维护应用程序的人会比你花更多的时间在上面。所以想想什么会让他们的生活更轻松。问自己以下问题:

  • 这是语言行业标准吗？
  • 我能从街上雇人在最短的加速时间内开发这个应用程序吗？
  • 有社区支持吗？
  • 我能在栈溢出上找到问题和答案吗？

作为一名领导者，你的工作是对你的公司产生影响。听起来很傻，为一个项目选择正确的编程语言可以决定你留下的是积极的还是消极的。

开始建造

你能做的最糟糕的事情就是忽略选项，因为它们看起来很难。追求一个方向，因为这是正确的路，而不是容易的路。不要假设一个方向，因为你总是这样做。

“如果你想要新的东西，你必须停止做旧的东西。”— 彼得·德鲁克

尝试一种新的编程语言，它能以最好的方式解决你的问题。如果不行，吸取教训，再找一个。做出敏捷的决定。不要即兴发挥，而是和你的团队一起做出明智的决定。给它一个老的大学尝试。

你会发现，你最早做出的关于应用程序技术方面的决定——它是用什么写的——会产生最大的影响。每个人都有学习新事物的能力。不要害怕走上不败之路。

建造。学习。创新。一天比一天好。
