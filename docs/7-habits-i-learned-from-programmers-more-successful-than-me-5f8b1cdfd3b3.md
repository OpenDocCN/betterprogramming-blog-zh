## 协作、设定编码标准等等

![](img/a4a34e3efb4c05671dffd29c890d7449.png)

在 [Unsplash](https://unsplash.com/?utm_source=medium&utm_medium=referral) 上由 [Max Duzij](https://unsplash.com/@max_duz?utm_source=medium&utm_medium=referral) 拍照

作为一名程序员，你需要学习的东西——而不是在训练营或大学里教的——是一个帮助你成为更好的程序员的好习惯。好习惯帮助你更接近你的目标，减少大量的时间浪费，并在你所做的事情上变得有效。

在本文中，我们将看到我从高效的程序员那里学到的七个习惯，我相信任何人都应该遵守，尤其是像我这样自学成才的程序员。这是你在编码教程或指南中找不到的，但你需要学习成为一名更好的程序员。

确保你只是不去读这些习惯，而是将它们应用到你的日常生活中。

# 1.合作

你是那种试图自己解决所有问题的程序员吗？你可能会花上几个小时或几天的时间在某件事情上，即使你能够解决它，也没有人会把浪费的时间还给你。由此可见求助有多重要。

试着与他人分享你的代码。不管是你创造的简单的石头、剪子、布游戏，还是你建立的复杂的机器学习模型，你都应该与朋友、同事和任何能给你一些建议的人分享。万一你不知道谁可以帮忙，就在 GitHub 或 Stack Overflow 上在线分享你的代码，这样其他人就可以找到你的代码，修复 bug，添加特性。

以后，GitHub 可以成为向潜在雇主展示你的项目的好地方。如果你想更进一步，考虑写一篇内容丰富的博文，解释你在项目中取得的成就。这将有助于你从网上的人那里获得更多的反馈，也能联系到对你的工作感兴趣的人。除此之外，你可以在简历中加入你的职位，这样即使招聘人员不阅读代码，他们也会很清楚你的代码是关于什么的。我通常在 GitHub 上分享我的代码，必要时我会写一篇文章讨论它所解决的问题。例如，我写了一篇文章解释[我如何编写一个预测足球比赛结果的脚本](https://medium.datadriveninvestor.com/can-we-beat-the-bookies-predicting-football-games-with-a-simple-model-43fe020fb384)。

**如何实现这个:**开始在 GitHub 上发布你的代码，在 StackOverflow 上征求反馈。

# 2.正确记录您的代码

我曾经打开旧剧本，然后问自己一个问题，“我到底在想什么？”

试图理解没有正确记录的代码是很费时间的。如果你很难理解你写的代码行，想象一下当其他人试图使用你的代码或为你的代码做贡献时，他们会经历什么。这就是为什么记录一个小脚本或一个大项目是重要的。

> "读代码比写代码更常见."
> 
> — *吉多·范·罗苏姆*

我遇到的所有优秀程序员都有评论和记录代码的好习惯。第一部分指导读者更好地理解代码及其用途和设计，而第二部分向用户描述其用途和功能。

**如何实现:**阅读关于如何用你喜欢的编程语言编写文档的指南是一个好的开始。例如，[这本](https://realpython.com/documenting-python-code/)是关于如何记录 Python 代码的完整指南。在学习了它的基础知识之后，在你自己的脚本中实现它，并且阅读好的代码文档来适应它。

# 3.保持简单

说到写代码，少并不总是多。将一些代码行缩减为一个短而复杂的代码不会帮助其他人容易地理解你的脚本。在这种情况下，代码行越少越好。

优秀的程序员让复杂的代码变得简单。代码越简单，就越容易理解你要完成的任务。这是我自己做编程课程的时候学到的。我过去常常把代码减少到一行，即使是在不必要的时候。然后我问自己“我的学生会很容易理解吗？”简短的回答是“不”

与团队合作时，问自己一个类似的问题。你的同事以后会感谢你的。

# 4.不要拘泥于一种语言

每个人都有自己喜欢的编程语言——我的是 Python——但是，你不应该和那种语言结婚。

您的项目更重要！这就是为什么您应该考虑项目需要高效完成的原因。然后考虑哪种语言适合项目的需要。它有没有可以帮助你轻松构建你想要的库？这种语言提供了最大化项目潜力的工具吗？所有这些问题都会帮助你找到合适的语言来解决你想要解决的问题。

不要试图用你喜欢的语言来适应这个项目，或者用“最好的”编程语言来解决你遇到的所有问题。相反，走出你的舒适区，学习你需要什么来完成一项任务。

也就是说，你不需要学习无数的编程语言，只需要学习那些你能充分利用的语言。

如何做到这一点:掌握一门语言，然后分析用这门语言你不能完成什么事情。然后，想一想有哪些潜在的编程语言可以轻松解决你的问题。涵盖你大部分需求的语言将是值得学习的。

# 5.设定编码标准

如今，组织希望他们的程序员保持一种定义良好的标准编码风格，称为编码标准。即使你不在大公司工作，你也应该有一个编码标准:

*   为不同人编写的代码提供统一的外观。
*   提高代码的可读性和可维护性，同时降低复杂性。
*   有助于轻松检测错误。
*   促进合理的编程实践，提高其他程序员的效率。

为了实现这一点，您应该坚持定义良好的语法和命名约定。您设置的约定越多，读写代码就越容易。这将让你在加入一个已经有一定标准、惯例和规则的团队之前做好准备。

如何实现:阅读向您展示编码标准范例的指南。这个[指南](https://www.geeksforgeeks.org/coding-standards-and-guidelines/)包含了一些众所周知的编码标准。仅举几个例子:

*   命名约定变量:局部变量应该以小写字母开头(例如 localData)，而全局变量应该以大写字母开头(例如 GlobalData)。
*   错误返回值和异常处理约定:遇到错误的函数应该返回 0 或 1，以简化调试过程。
*   代码应该被很好地记录:正确地记录你的代码，这样任何人都可以很容易地理解它。

# 6.避免分心

如果你时不时的看一眼手机，你就会失去焦点，没有焦点是不可能解决一些问题的。

我知道很难抗拒我们每天收到的电子邮件或短信通知，但是相信我，专注于一件事情的好处是惊人的。在我的 iPhone 掉到地上并停止工作后，我才知道这一点。已经[六个多月没有电话了](https://medium.com/curious/six-months-without-an-iphone-this-is-how-my-life-changed-86a8b8437723)，现在我可以轻松地专注于重要的事情。

当然，你不必为了专注于编码而丢掉手机——关掉通知或手机就足够了。

# 7.休息一下

作为一名程序员，你可能有时会花几个小时写代码。尽管这显示了你对一个项目或工作有多投入，但从长远来看，这可能会击垮你。

好好照顾自己。不时地休息一下。这不仅有助于你的健康，还能减少编码时犯愚蠢错误的机会，并获得如何解决特定问题的新想法。
