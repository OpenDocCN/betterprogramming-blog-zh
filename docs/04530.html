使用纽曼，邮差的CLI工具

REST APIs现在是日常软件工程实践的一部分。不久前，我写了一篇文章讨论如何设计REST APIs。在本文中，我将讨论如何对API进行自动化测试，并将其集成到CI管道中。

本文假设您对REST APIs有基本的了解。

邮递员

Postman是一个API开发的协作平台。Postman为API的使用和测试提供了一个基于GUI、易于使用且安全的应用程序。它只是作为一个HTTP客户端。

首先，你需要下载并安装邮递员应用程序。

纽曼

Newman是邮递员集合的基于CLI的运行程序。这意味着您可以使用Newman CLI运行一组API(集合)。你需要按照本页上给出的说明安装纽曼。

既然先决条件已经安装，让我们开始测试API。

测试用REST APIs

对于本教程，我使用来自http://dummy.restapiexample.com/的虚拟REST APIs

这些API可以公开使用。

创建邮递员集合

让我们首先创建一个Postman集合来使用两个API:

  • "获取所有员工"
  • "创建员工"

Postman的UI足够直观，你可以轻松地创建一个Postman集合。如果发现问题，可以参考文档。

一旦创建了集合，就可以添加上述API并运行它们。

为API响应编写测试

既然我们已经尝试了API，我们可以开始编写测试了。一般来说，在API测试中，你可以考虑这样的测试:

  • 验证返回的HTTP状态代码
  • 验证API响应时间
  • 验证内容类型标头
  • 验证响应正文
  • 等等。

Postman提供了一个名为Tests 的选项卡，您可以在其中编写上述测试。

集成到CI渠道中

现在，您可以从命令行执行集合了，让我们尝试与Azure DevOps管道集成。

为此，您必须将Postman集合JSON提交到Git repo中。

接下来，您需要设置管道:

  1. 使用npm任务安装Newman。
  2. 运行测试套件
  3. 发布测试结果。

在这里，我创建了一个名为npm的文件夹，Newman npm包将安装在本地的构建代理上。我还添加了一个样本npm/package.json文件。

在运行测试套件任务中，您需要执行一个简单的命令，如下所示:

/home/vsts/work/1/s/npm/node_modules/newman/bin/newman.js run ./Example_Employees.postman_collection.json --reporters cli,junit --reporter-junit-export .

在这里，如果您注意到了，我们添加了几个额外的参数，这将有助于在CLI上以JUnit格式生成测试执行报告。

Newman还提供了配置 SSL 证书、 HTTP代理、等选项。
