# 在我开始编程生涯的时候，我应该更加关注的 5 件事

## 学习编码是很棘手的

2015 年 1 月，我抑郁了。我已经全职学习软件开发超过半年了，感觉我离找到工作一步也没有。

几个月后，我有了第一份工作。又过了几年，我成了一家软件公司的联合创始人，并开始雇佣开发人员。

以下是我从失败和成功中学到的关于软件开发的东西。

# 1.专注是一种超能力

决定我是否能学到东西的最大因素是专注。当我集中注意力时，我的工作效率飙升，我的心情也变好了。当我注意力不集中时，我的进步消失了，抑郁开始袭来。

专注在两个方面至关重要:尽可能缩小学习范围，专注于手头的任务。在我的职业生涯中，我两次都失败了。

我软件开发之路的起点到处都是。在我决定专注于基础——计算机科学之前，我简要地学习了 C++。在学习了三个月的算法和数据结构之后，我觉得我已经受够了。是时候开始做些真正的事情了。我在 CS 课上学过一点 Python，所以用 Django 和 Flask 做后台是很自然的选择。我无法决定选哪一个，于是两个都选了。在短暂的崩溃之后，当我想到我不能做编码，而应该学习测试时，我筋疲力尽了。

幸运的是，这也是我发现前端的时候。在从一件事跳到另一件事的七个月后，我决定要么做前台，要么回到我的老工作。我只关注 HTML、CSS 和 jQuery。四个月后，我收到了两份工作邀请。

专注于一件事拯救了我的编程生涯。

注意力不集中也拖慢了我的日常学习。我全职自学，所以我有一整天的时间来学习。我花了很多时间懒洋洋地看教程，同时在手机上浏览网页。很快，我意识到这样度过的每一天都让我感觉筋疲力尽，好像我已经工作了很多。但不知何故，我的进度仍然落后。在发现两个小时的紧张学习胜过六个小时的半心半意地看课程后，我的学习速度和生活质量都得到了提高。

# 2.实践每次都胜过理论

我提到学习计算机科学几个月，不是吗？进展不太顺利。在我找到工作之前，我几乎忘记了一切。即使是现在，每当我需要实现一些更高级的数据结构时，我都需要修改我的 CS 笔记。

这是为什么呢？

因为实践战胜理论。我主要参与 web 前端开发，不需要太多计算机科学知识。所以我很少有机会在实践中使用，知识也很快消逝。

看课程或者看教程，并不能教会你很多东西。为什么？因为很被动。我们边做边学，理论知识如果不付诸实践，蒸发的很快。

所以跳过下一个你想看的教程，把手弄脏。自己造点东西。不一定要复杂。从简单的应用程序开始。但是要习惯依靠自己，寻求答案，探索什么是可能的，什么是不可能的。

除了学得更快，直接练习还有其他一些好处。

## 短反馈回路

当你开始学习一个新东西的时候，你不知道你会不会喜欢它。所以在你投入数百小时观看课程之前，试着做些小东西。如果你喜欢，这是一个值得追求的技能。如果你没有，那很可能不适合你。

## 建立投资组合

工作证明对每个开发人员都很重要。你可以看所有的教程，看所有你想看的博客，但不会打动任何人。优秀的作品集会给每个人留下深刻印象。

## 战胜拖延症

看视频教程很难集中注意力。你的大脑一直在捉弄你。它会让你在看视频、做饭、发短信的同时，觉得自己在学习。实际上，这只是拖延的一种形式。

你通过采取行动战胜拖延症。你不能构建一个应用程序而分心。创造迫使你专注。

## 满意

有所建树的感觉很棒。让别人看看你的作品就更好了。建造你引以为豪的东西。

# 3.倦怠是真实的

学习编码是一个迷人的旅程。就像每一件有价值的事情一样，它有好的一面，也有不好的一面。一些不利因素会让你焦虑几个月。

每个有抱负的软件开发人员都很快意识到编程很难。当你开始的时候，它可能看起来很简单，但是随着时间的推移，它变得越来越复杂。简单的条件或循环与真实的应用程序之间的转换尤其困难。我经常感到停滞不前，不知道如何实现一个特性或修复一个 bug。与此同时，我发现自己一次又一次地重新学习这些相同的基础知识。

焦虑的另一个来源是冒名顶替综合症。这对你打击很大。学习编码的时候，经常会觉得力不从心。太蠢了。你会在网上到处看到伟大的程序员，并认为你永远不可能成为他们中的一员。

但对我来说，最大的挑战是一件事:编程需要很多时间——对我来说，是独自度过的时间。我没有参加任何新兵训练营，所以对我来说，这是几个月独自坐在家里，阅读和编写代码。

孤独变得令人沮丧，特别是当我的一些工作申请被拒绝时，我开始质疑我是否浪费了几乎一年的时间。幸运的是，奋斗过后，我找到了一份很棒的工作，遇到了很棒的人。

不要重复我的错误。注意你的心理健康。见见朋友，找个编码伙伴，甚至去个新兵训练营。学习编码并不是一切，如果你能照顾好自己，你会过得更好。

# 4.在公共场合学习有帮助

当我开始编码时，我并不存在于社交媒体或网络上的任何地方。我认为学习是我的私事，其他一切——尤其是推特或脸书——都会分散我的注意力。我错了。

活跃在软件开发社区中既重要又被低估。你不仅可以从阅读他人创作的精美内容中受益(你在 Twitter 上与任何人都只有一条消息之隔)，还可以向他人展示你的作品。展示你的作品意味着很多事情:获得帮助、反馈、赞赏，甚至帮助他人。

我说的公开学习是什么意思？很多事情。您可以:

*   博客。
*   在社交媒体上分享你的学习进度(Twitter 尤其支持)。
*   分享你的学习笔记。
*   制作 YouTube 视频。
*   回答关于堆栈溢出的问题。

你可以选择任何你喜欢的，但是从我的经验来看，对初学者来说最简单的是写博客和在 Twitter 上分享你的学习起伏。

## 博客

写博客能增强你的学习。要写作，你必须理解主题并能简单地解释它。

你的博客也是你进步的证明。定期写博客向你未来的雇主或客户表明你对你的软件开发职业是认真的，所以你被雇用的机会会增加。

博客让你建立一个网络。如果你的博客变得受欢迎，你就拥有了结识他人甚至帮助他人的独特能力。这不仅有趣，还能帮助你被注意到，并通过推荐找到工作。

## 推特

甚至只有在 Twitter 上关注伟大的程序员才能教会你很多东西。但是学习编码时使用 Twitter 的最好方法是分享你的进步。

每当你达到某个里程碑，即使是一个小的里程碑，分享一两个观点。也要说说自己的失败和疑惑。你很可能会找到一群志同道合的人，他们会在你的学习之旅中帮助和支持你。他们可能会给你一句鼓励和建议的话，或者甚至帮你联系一个有工作给你的人。

如果你不确定如何开始，寻找带有挑战的标签(如#100DaysOfCode)或寻找你的社区(如# WomenInTech)。我相信你会找到你的位置。通过分享你的进步，你会建立许多联系，其中一些可能对你的职业生涯有价值。

# 5.在你准备好之前申请

没有人准备申请这份工作。软件开发的世界是广阔的，总会有一些东西需要学习。此外，招聘启事通常只是一个愿望清单——它们包含了比工作所需更多的东西。

所以，当你知道招聘启事中一半以上的要求时，就去申请吧。最糟糕的事情是他们会拒绝你。他们总有机会想采访你。

如果你和我一样，害怕面试。有时候你会被拒绝。有一次，在和一个潜在雇主呆了一天后，我感到非常羞愧，以至于我想永远退出编程。但是时间带走了刺痛，每一次失败的面试都变成了一次学习的经历。

所以试着把你所有的失败都当作实验，并从中吸取教训。申请这份工作。你能做到的。
