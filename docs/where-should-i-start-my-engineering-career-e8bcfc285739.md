# 创业或公司:你应该从哪里开始软件工程生涯？

> 原文：<https://betterprogramming.pub/where-should-i-start-my-engineering-career-e8bcfc285739>

## 试图决定你应该为一家小型初创企业还是大型企业集团工作？以下是我在这两个地方的经历

![](img/7fe51990accb1801387e6f88cc95749b.png)

克里斯蒂娜@ wocintechchat.com 在 [Unsplash](https://unsplash.com/s/photos/software-engineer?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

我有机会在一家小型初创公司(拥有 5 名员工)和一家财富 500 强公司(拥有超过 1 万名员工)担任了一年多的软件工程师，对这两家公司的优缺点有了很多思考。虽然一年多的时间算不上什么经验，

我至少对这两种公司的生活有所了解，并发现了公司文化和员工日常职责的相似之处和不同之处。虽然两者提供不同的东西，你可能会发现一个更适合你正在寻找的。

快速注意——这些结论是我自己的，来自我的个人经历和我的同龄人的经历。如果你经历了不同的事情，请告诉我！

# 指导和成长

根据你所处的环境和你的同事，导师关系看起来会非常不同。这在很大程度上取决于你的同龄人有什么样的外部经历，他们的教学风格，以及你如何学得最好。

我发现，在小公司，很难抽出时间专门指导和监控增长，因为小团队通常专注于获得新功能和尽快修复漏洞。

另一方面，在这种环境下，你会很快获得全面的教育，因为你可以看到公司的每个动态部分。如果你能找到一个小团队，习惯于雇佣初级员工并指导他们，或者有从内部提拔的良好记录(因此说明他们知道如何培养开发人员)，你很有可能会在公司内部找到导师。

相反，大公司拥有庞大的团队，专注于应用程序或服务的较小部分。这允许团队成员花更多的时间探索新的选项和技术，评估他们的需求和方向，并且通常给员工提供共享知识的带宽。

以我在一家拥有超过 1 万名员工的公司的经历来看，我已经能够与我的同事建立专门的关系，进行指导和学习。很容易看出我已经走了多远，因为我在应用程序的很小一部分上工作，我的同事有时间和愿望来帮助我通过 PR 评审和回顾以及站立更新来成长(参见:[敏捷软件开发](https://en.wikipedia.org/wiki/Agile_software_development))。我的老板非常重视个人和职业发展，这反过来又渗透到团队文化中。

TL；大卫:以我的经验来看，在小公司，当每个人都在做所有事情的时候，很难找到好的导师，也很难学会好的习惯，而在大公司，这似乎更容易，因为节奏更慢，有更多的团队成员可以依靠。

# 福利和补偿

![](img/b404bb54fd10c34be16692de0adcfd49.png)

[Jp 瓦列里](https://unsplash.com/@jpvalery?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText)在 [Unsplash](https://unsplash.com/s/photos/salary?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

这是我在与其他公司的同事交谈时经常提到的一个问题——不同规模公司之间的薪酬差异。大公司有实力和权力提供更高的薪水、更多的奖金和更好的办公室。以我自己的经验来看，从小公司跳槽到大公司，我的工资翻了一倍，此外还获得了签约奖金和年度奖金(小公司通常不会提供这些东西)。

无论规模大小，对于那些寿命较长的公司来说，这一点基本上都是正确的:如果你正在从事的产品或服务在财务上有利可图，并且公司稳定，那么从长远来看，你最终会获得更多的钱。

也就是说，有很多故事讲的是，人们在小公司拿着很低的薪水，却被卖给了一家大公司，并获得了巨额报酬。不过，这在业内是一场豪赌，如果你寻求稳定，可能不值得。

薪酬也不全是薪水！我见过我工作过的大公司和小公司之间提供的福利有很大的差异，而且似乎不太一致。

我工作的小公司提供无限制的 PTO、更长的产假和“夏季星期五”(从 5 月到 9 月，每周五半天)，所有这些都是我在大公司享受不到的福利。不过，我最终选择了后者，因为正如我之前所说，我的工资翻了一倍，我的家人认为这更有价值。

TL；dr:不同公司的薪酬差别很大，你应该在找工作之前做一些研究(并根据你自己的情况决定什么是最重要的)。

# 企业文化

![](img/f18e907b4a0d2abc1995bbc736cd5870.png)

詹姆斯·哈里逊在 [Unsplash](https://unsplash.com/s/photos/developer?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

小公司在薪酬方面可能缺乏的东西，他们完全可以在公司文化中弥补(通常是这样)。当一个团队很精简时(比如一个小公司或初创公司)，它几乎可以在一夜之间改变。小团队可以更加敏捷，因为需要取悦的人更少，也没有多余的繁文缛节。

在我之前的公司，有一个 35 人的团队，我们能够非常迅速地专注于我们的文化努力。当有人决定我们应该追溯到一天的最后，这样我们就可以边喝边做，我们做到了。当我们希望每周有一天在家工作时，这种转变很容易。

我们的团队每个季度都有多次外出活动，比如去虚拟现实密室或者划独木舟顺流而下。小公司往往意识到，如果他们无法与大公司的薪酬方案竞争，他们必须在其他地方弥补，通常是在文化中。

尽管如此，大公司在某一点上开始变小，有时他们能够保持创业时的相似感觉。我现在工作的公司有一个有趣的办公室，有许多不同的协作区域，我的团队整天都专注于配对。这使我们更加紧密地团结在一起，并允许团队直接相互合作。

我们每年还有一个黑客周，在那里我们可以专注于我们常规门票之外的令人兴奋的项目，并最终在整个公司结交了大量新朋友。工作期间和工作之外都有很多机会和同事出去玩，这也是我每周一都会回来的原因。

TL；dr:小公司通过创造一个既有趣又丰富的工作环境来弥补他们在财务上的竞争力，但是不要忽视大公司，除非你已经看到他们缺乏团队文化。

# 产品焦点

项目的规模和范围可能是这个列表中最大的不同。在一家小公司，你可能被期望处理代码库中的任何事情，甚至可能包括客户支持、营销视觉工作和参加产品会议。

有一种“所有人都在甲板上”的感觉，当他们做对时，感觉太小了，这可能是令人兴奋的！然而，当它出错时，它会感觉所有的东西都 27/4 在燃烧(这总是可以被看作是学习更多东西的机会)。

在我在 Octerra 的两年时间里，我学到了比我想象中更多的软件开发知识。我有时会成为一名客服人员、一名视频编辑、一名社交媒体营销人员，但我获得了关于整个商业和行业的大量知识。

在一家大公司，更有可能的情况是，你最终会在产品的一个部门工作，甚至是这个部门的一个小功能。像这样的团队有能力深入软件开发的一个领域，并在一个特定的框架或特性中成为[SME](https://en.wikipedia.org/wiki/Subject-matter_expert)。

在日常工作中花了更多的时间在代码上(而且是代码的一个领域)，我对 React 的精通程度超过了我在一年半时间里的想象。因此，除了技术能力之外，我在工作的其他方面变得更加自信，也因此成为一名更好的队友。

TL；dr:小公司提供知识广度的机会，而大公司通常允许对某一主题有更深入的了解。

# 最后的想法

有各种各样的公司，这不是一个放之四海而皆准的方案。虽然大公司和小公司的基础设施有很大的不同，但是当他们面试你的时候，关于面试一家公司还有很多要说的。

确保你问的是关于导师的问题，以及他们过去雇佣过多少初级或中级工程师。在任何行业找到一个好的契合点都很重要。邀请潜在雇主的现任和前任员工出去喝咖啡，就他们的经历向他们请教。

我很幸运地找到了一家公司和团队，我不仅可以从中学习，还可以为构建我们未来的产品提供见解(这恰好是一家非常大的公司，我发现对我来说这是一个更好的地方)。

这都是根据我的亲身经历。我知道工作与生活的平衡和条件在不同的公司和行业会有很大的不同，到目前为止我只在几个不同的地方工作过。
