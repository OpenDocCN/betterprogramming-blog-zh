这5条规则将帮助你的下一个编码项目取得成功

通过批判性地分析你过去的失败，你可以找到未来成功的关键

最近，我意识到我的副业转换率不可否认是可怕的。算了吧，是零。如果我必须给出一个我成功完成的兼职项目的数字，那是一个巨大的零。

当然，这取决于你如何定义成功。在这里，我说的是建立一个正在运行的东西，有人正在从我的努力中获得价值。

可悲的是，当我的项目被批评为这种成功的定义时，我两手空空。在我开始的所有兼职项目中——有相当多的——在我完成我想要的之前，它们都失败了，被遗忘了。好吧，副业还有其他方面。毫无疑问，我获得了经验，拓宽了我的编程视野，并学到了一些可以应用到我的职业生涯中的新技能。

然而，我真正想从我的下一个副业项目中得到的是构建一个“成功”的东西。为了实现这一点，我决定我需要弄清楚为什么我从来没有坚持过我以前的任何尝试。因此，我深吸一口气，决定批判性地分析我最近的五个副业项目“失败”

第0条规则

虽然我不能声称有一个快速成功的神奇公式，但我知道的是，遵循这五条规则帮助我在最新的兼职项目中比我想象的更快更容易地达到里程碑。即使你不同意这些规则本身，我相信从过去的失败中获得关键教训的过程可以被每个人应用到他们生活的几乎任何部分！

第0条，也可能是最重要的一条，是从过去的失败中吸取教训。

这就是为什么我不会以“一刀切”的方式宣扬我的规则，而是分享我的工作方式。也许你也可以回头看看，并推导出你自己的一套更适用于你的规则。

继续读下去，看看我是如何批判性地评估我失败的副业项目，并能够定义未来成功的五个关键规则。

项目1:密码跟踪器

CryptoTracker是我毕业后寻找第一份编码工作时开始的一个项目。前提是在社交媒体上进行情绪分析，以预测加密货币价格的趋势。这是一种有趣的消遣，它让我在采访中有了不同的话题。但该项目本身从未真正启动。

回过头来看，这有几个原因。首先，我在构建和部署大数据应用程序所需的技术方面没有任何经验。我还找到了第一份工作，这份工作耗费了我大量的时间和精力。最重要的是，最终我犯的最大错误是没有在计划上花费任何努力。我刚刚跳进了深水区，没有执行计划。前一分钟，我还在努力构建一个twitter scraper，下一分钟，我就在琢磨如何在HTML和CSS中居中放置一张图片。我经常陷入困境，因为我没有在设计选择上付出任何努力。

所以，我的第一条规则是先设计，后编码。

规则1:先设计，后编码

作为一个粗略的经验法则，在提交任何代码之前，我现在花大约一个月的时间来设计一个编码项目。也就是说，画图表，研究技术，并制定执行计划。

项目2:网页开发

大约在我开发CryptoTracker失败的同时，我也在尝试网站设计和开发。CryptoTracker需要一个网站，所以尝试一下是有意义的。现在说这是一个彻底的失败可能有点苛刻。我确实成功地为朋友和家人建立了几个网站，并且学到了一大堆关于web框架、部署和编程语言的知识，比如PHP、HTML和CSS。

然而，我把网站开发变成一个有利可图的副业的雄心从未实现。我认为这是因为这是一个巨大的饱和市场。有成千上万的超级天才自由职业者在这个领域工作——更不用说大机构和企业解决方案了。为什么会有人花钱给一个没有经验的毕业生建网站？吸引客户的唯一方法是大幅低估我的时间，以削弱市场。

简而言之，我需要一个利基市场。我确实相信，面对这种程度的竞争，成功是可能的，但你需要发展一套技能，让你获得竞争优势。自然，每个程序员都有不同的经验，并根据他们的个人兴趣和独特的职业道路开发各种不同的编程技能。为什么不利用这一事实，在你的下一个副业项目中利用你自己独特的优势呢？你可以利用你天生的竞争优势！

规则2:你是独一无二的！追随你的定位

我喜欢这条规则，并努力把它应用到我做的每一件事上。我拥有什么样的技能、经验和知识能让我以不同于他人的方式看待问题？

项目3:列车追踪系统

作为伦敦的通勤者，我不幸地亲身体验了英国铁路的混乱。更不和谐的是，我不得不为这种特权支付高得离谱的票价！TrainTracker是我尝试开发的一款应用程序，可以自动申请火车晚点补偿。我想自动去除铁路公司强迫你填写的不必要的臃肿和复杂的表格。我构建了一个Python web scraper来收集旅程延迟时间，并开始构建另一个应用程序来填充延迟偿还表单。

然而，这被证明有几层复杂性，最终导致了我的失败。我需要对每个用户进行身份验证以进行索赔，还需要将用户信用卡的详细信息存储在文件中。此外，我正在开发的应用程序会不会被诈骗者利用，他们声称自己进行了实际上从未进行过的旅行？商业可行性和数据保护问题是我在一个月的计划中应该考虑的事情！

因此，我的第三条规则也是围绕着计划项目的非技术方面！具体来说，我现在问自己，“这在商业上可行吗？”

规则3:商业上可行吗？

当规划一个项目时，我发现商业和业务方面不如研究技术方面自然。这条规则提醒我们，在实施之前，要根据商业可行性对所有事情进行审查。

项目4:邻居

Neighbourlee是一个项目，诞生于我对消费主义和浪费的不断不满。我想开发一个应用程序，让人们能够轻松地与邻居分享他们的财产和技能。不用再从亚马逊买新东西了！我知道为了这个项目的成功，我不能仅仅构建一个web应用程序。为了达到最好的影响和采用，Neighbourlee将必须主要是一个移动应用程序(我不知道如何建立)。

通过遵循苹果开发者文档，我开始学习Swift并构建我的第一个iOS移动应用。然而，从零移动应用程序经验开始，开发一个复杂的应用程序对我的业余项目来说是一个巨大的挑战。我很清楚为什么《邻居》没能成功:我贪多嚼不烂。

因此，第四条规则是建立在你已经拥有的技能上。在你的基础技能上更容易，而不是试图在业余时间学习一些全新的东西。

规则4:建立在你的基本技能上

我喜欢学习新东西！然而，如果我过于雄心勃勃，这条规则会约束我。如果我需要从头开始学习一切，我能看到一个项目完成吗？

项目5: LearnScalaSpark

我职业生涯的第一部分是作为大数据工程师度过的。我在Scala中构建了ETL管道，并使用Apache Spark在数十亿行数据上运行它们。由于我坚信“学习任何东西的最好方法是把它教给别人”这句格言，我创建了一个名为 LearnScalaSpark 的网站。目的是在我掌握了这个领域的不同主题后，收集一些文章和学习材料。

然而，我的职业转向了云计算，我发现自己使用这些技术的时间少了很多。因此，我学习它们的欲望也逐渐减弱。因此，我只在网站上发布了几篇文章。回过头来看，明智的做法是让博客更加通用，不要把自己局限于Scala和Spark。

你将随着时间的推移而扩展和进化。你的副业能跟得上吗，还是你把自己束之高阁？第五条规则是提醒你要保持新鲜感！

规则5:不要把自己归类

第五条规则是一个简单的提醒，让你的副业足够普通，六个月内不会过时或失去兴趣！

摘要

总而言之，这是我遵循的五条规则，帮助我最近的副业项目取得成功。

  1. 先设计，后编码！
  2. 你独一无二！追随你的定位。
  3. 商业上可行吗？
  4. 建立在你的基本技能上。
  5. 不要把自己束之高阁。

它们足够通用，我确信它们可以适用于大多数开始一个新的副业项目的人。尽管如此，我认为这里最重要的教训是第0条规则:不要抹去你过去的经历或失败。深呼吸，从你以前的错误中吸取教训，用它们来制定你自己独特的规则，这些规则将会在你未来的努力中推动你前进。

感谢阅读！
