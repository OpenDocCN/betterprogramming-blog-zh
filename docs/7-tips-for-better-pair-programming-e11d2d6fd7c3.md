# 更好的结对编程的 7 个技巧

## 有时候两个脑袋比一个好

![](img/0e566a1244082dbd49060f5f637e7e7f.png)

阿尔瓦罗·雷耶斯在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

有效协作是任何高效工程团队的核心特征之一。每个人都有不同的想法，所以对一个问题有第二双眼睛可以提供多种观点和方法，否则你可能会忽略它们。对于软件工程师来说，最常见的形式是结对编程。

有时结对编程会受到不好的指责。批评者认为，让两个开发人员从事同一项任务会降低团队的整体速度，但从整体来看，这可以节省时间。结对编程不仅能产生更高质量的代码，还能帮助开发人员快速解决问题，清除障碍。最近我经常和同事一起结对编程，所以我想分享一些关于如何最大限度地利用结对编程的技巧。

# 首先关注有挑战性的部分

每个结对编程会话都是不同的，但目标通常是一致的。大多数时候，你都在试图开发一个复杂问题的解决方案，这需要两只眼睛。虽然有些开发人员可能更喜欢先解决简单的问题，但我喜欢在做任何事情之前先解决问题的实质。

容易的部分通常是我自己能很快完成的任务，或者是比其他任何事情都更累人的琐事。当与某人一起工作时，我不想浪费他们的时间，让他们看我为我的界面编写文档或添加错误日志。我更喜欢跳进来，攻击需要两个人来解决的挑战。

留下信号，让你自己稍后清理，比如评论或待办事项。定义可以在以后实现的空方法头和类，或者用伪代码填充部分。如果您打算在会话期间运行您的代码，请添加一个框架实现，该框架实现可能只覆盖快乐路径，而排除边缘情况。

# 不要担心完美主义

基于最后一点，我通常会在结对编程时尽量避免吹毛求疵。虱子有它存在的时间和地点。对我来说，通常是在接近尾声的时候，我会检查写好的代码并清理干净，这样我就可以把它送去审查了。坚持近乎完美的代码在清理阶段是合理的，但是如果你还处于解决问题的阶段，那么吹毛求疵只是让事情慢下来。

# 打开它

尽管我很喜欢坐在驾驶座上，但偶尔改变一下，充当后座角色也是不错的。观察别人如何工作可以像自己做一样有洞察力。我喜欢观察他们如何处理问题，以及他们采取什么步骤来克服这些问题。有时这帮助我学习新的技术，我可以应用到其他情况。

另一方面，如果你通常是观察者，试着控制局面。有时候你只需要用自己的双手*感受*问题，就能了解你正在工作的环境和条件。实践经验通常是学习和培养技能的最有效方式。有一个一起编程的伙伴就像一个安全网，帮助你写出高质量的代码，并快速解决问题和障碍。

结对编程是双向的；如果你只朝一个方向走，你看不到另一个方向，你的伴侣也看不到。确保你们两个人都有机会实践这种经历的两个方面，这样你最终会过得更好。

# 保持随意

有时候我觉得我可能会患上表演焦虑症。当我被监视的时候，我的大脑好像完全关闭了。我忘记了如何打字，犯了很多愚蠢的错误，而且似乎不能正常思考。甚至结对编程的想法也会让我紧张得冒汗。我花了不少时间练习才熟练掌握。

考虑到你的伴侣在你的谈话中可能会有这种感觉。或者，你可能会感到焦虑。结对编程应该是高效的，而不是有压力的。确保你和你的伴侣感到舒适，如果有什么事情困扰着你，不要害怕说出来。

不用说，你应该尊重你的同事。不要因为他们犯错或打字慢而骚扰他们。这样的行为没有任何好处。不是每个人都能在别人看着的时候做到最好。

# 接受新思想

这适用于软件工程的所有方面，尤其是结对编程。不要因为你的伴侣的想法不符合你自己的哲学，就否定他们所有的想法。这就违背了合作的初衷。

如果一个想法实际上是不可行的，首先讨论它，并解释为什么你认为它不可行，而不是完全忽视它。通过讨论，你也许能找到一种方法把它变成可行的东西。在最坏的情况下，记录下考虑的选项总是值得的，即使它们最终没有被应用。

# 设定时间限制

我对结对编程的一个抱怨是，有时很难找到一个好的停止点。在这种情况下，感觉会议拖得太长了，我花的时间比预期的要多。在开始之前，考虑设定一个硬性或软性的时间限制，以防万一。

理想的会议长度是个人喜好，但我喜欢把我的会议保持在 30 分钟左右。通常，我会提前计划好它们，并为它们创建一个日历事件，这样我就可以将它们适当地纳入我的日程安排中。然而，我还是会时不时地参加一个临时会议。

# 关于角色的注释

敏捷教科书谈到了各种不同风格的角色明确的结对编程。常见的例子包括驾驶导航和乒乓风格。如果你想使用其中的一种，那完全没问题，但是我通常不会把我的策略限制在这些风格的细节上。当我结对编程时，我不会明确地制定角色或任何东西的规则。我只是做对我和我的伴侣来说很自然的事情。当然，我通常最喜欢经典的驾驶员导航风格，其中一个人编写代码，而另一个人展示想法。

在大多数情况下，缺乏经验的开发人员应该是驱动者。在结对编程会话中实现高级工程师的设计是一种奇妙的学习体验，因为它有助于您吸收想法并理解建议方案的优点。我曾经遇到过这样的情况，我开车，而一个初级同事导航，但我无意中同时扮演了两个角色，我的搭档什么也学不到。

有经验的开发人员也可以从结对编程中受益，但是根据我的经验，当会议更短、更集中时，这些会议会更好。高级工程师通常更擅长提出一个想法并付诸实施，因此在整个实施过程中并不总是需要合作。

# 外卖食品

结对编程在任何技术工作中都是一种常见的做法，所以熟悉结对编程并学会做好结对编程是获得额外生产力优势的一个好方法。这种做法有很多种，所以你可以随意尝试不同的风格，找到最适合你的。我在这里提出的建议适用于任何形式的结对编程。
