每个行业都有神话。这些一般都是因为外人无法完全理解而产生的。虽然其中一些只是有趣，但也有一些是绝对危险的。很多人对构建软件的样子有夸大的期望，对软件开发人员有刻板的看法。

这种情况也发生在行业内部。比如“Scala 比 Java *更有生产力。”嗯，这取决于你想解决的问题。Scala 既是一种函数式编程语言，也是一种面向对象的编程语言，这使得它在本质上比 Java 更高效、更灵活。但是 Scala 中有一些复杂的特性很难学。Java 很容易学习，并且有丰富的框架和库生态系统。*

让我们来看一些关于软件开发的神话。

# 1.这不是一个创造性的领域

这是我最讨厌的神话。很多人认为编程只是把问题空间翻译成计算机可以理解的语言。这都是错的。

是的，它是技术性和逻辑性的，但它也是一种艺术形式。许多非程序员忽视了优雅的代码之美。即使任何人都可以编写代码，但编写优雅的代码需要优雅。

还需要大量的创造力来找出如何构建一个数据库来模拟一个复杂的现实世界的场景。把不同的技术放在一起，让它们相互交流，开发一个大型的复杂系统，这需要创新。

创造力的第二个方面来自于代码的优化。当你第一次学习编码时，你主要关注的是简单地让代码工作。但是在一些经验和大量实践之后，您开始找出一些技巧和技术来使您的代码运行得更快并消耗更少的资源。

# 2.所有开发人员的软技能都很差

典型开发人员的形象就是喜欢一个人坐在电脑前，一直写代码的独行侠。虽然这与事实相差甚远，但许多开发人员必须通过与项目经理、设计师和用户合作来理解问题，以便能够为此编写代码并记录下来。这就需要良好的沟通能力和写作能力。开发人员在他们的代码中不断地做出决策，并且以各种形式和规模出现。

随着这么多博客平台的出现，越来越多的开发人员开始写博客，讲述他们的经历、他们使用某种编程语言解决的问题、职业建议等等。有些人经营 YouTube 频道，创造了许多好内容。

是时候抹去这个不准确的形象了！

# 3.所有的开发人员都希望有一天成为经理

如果一个人是一个好的程序员，这并不自动意味着他们会是一个好的经理。开发和管理是两种不同类型的工作，都需要特定的技能和思维方式。类似于足球运动员和足球教练。如果一个人是一名优秀的足球运动员，那并不能说明他有执教能力。

管理就是和人们一起工作，并帮助他们走向成功。你通过让人们做他们的工作来获得结果，而不是为他们工作。另一方面，开发人员更多的是个人贡献者。

这种情况正在迅速改变，许多公司现在对这两种职业都有不同的发展方向。如果你想继续做一名个人贡献者，并在同一个方向上进步，你可以成为技术负责人、首席工程师等。对于经理轨道，您可以成为工程经理、工程主管等。

# 4.更大的团队会带来更快的发展

4-8 个的[开发团队被认为表现最好是有原因的。这包括一些开发人员，一个产品经理，一个工程经理，可能还有一个设计师。越来越多的公司遵循这种模式。这就得出一个简单的结论:增加更多的人并不一定会带来更好更快的软件开发。](https://hbr.org/2019/02/research-when-small-teams-are-better-than-big-ones)

这可能看起来违反直觉，但是软件开发不像一个机械的过程(例如数据输入)。在数据输入的情况下，您可以添加更多的人员并产生更多的输出，因为新成员的入职成本非常低，并且个人工作通常不会被其他团队成员阻止。

当您向团队中添加新人时，您需要让他们加入项目。这是团队没有花在产品开发上的时间。

# 5.当你发布产品的时候，就结束了！

软件就像一个活的有机体。它有自己的生命周期，并且不断成长、快速适应和变化。除此之外，现实也在不断变化，从市场和业务趋势到新技术。用户也可能需要新的功能和改进。这是一场持续的斗争。苹果公司刚刚结束了 45 年的历史。

这就是它如此有趣的原因。在产品发布后，还有另一个旅程即将开始。产品仍然需要维护，最终用户报告的任何错误都需要解决。总会有一些优化、安全更新、UX 变更、补丁和其他事情在后台发生。

# 6.AI 和 ML 可以互换

AI 是超集，而 ML 是它的子集。后者更主要用于包含大数据“3v ”(容量、速度和多样性)的大型数据集领域。另一方面，AI 不仅涵盖 ML，还涵盖其他分支，包括自然语言处理、深度学习、计算机视觉和语音识别。

尽管如此，人工智能和人工智能都有一个共同的目标:达到打败人类自然智能的程度。
