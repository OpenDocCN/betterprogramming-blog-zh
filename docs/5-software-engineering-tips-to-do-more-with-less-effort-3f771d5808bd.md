# 事半功倍的 5 个软件工程技巧

> 原文：<https://betterprogramming.pub/5-software-engineering-tips-to-do-more-with-less-effort-3f771d5808bd>

## 通过意识变得更有生产力

![](img/5f79abaa4b14a4e66e45752abb1e4570.png)

普里西拉·杜·普里兹在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片。

在当今的经济形势下，做一名软件开发人员是一份压力很大的工作。在大学(博士和博士后)花了几年时间做编码，在一家初创公司做开发和技术主管，又在一家中型公司做 dev，成为两个孩子的父亲，在一个小办公室、一个小空地、一个大空地等等地方工作……我很乐意分享一点我的经历。

这里有一些帮助你提高工作效率的建议，让你事半功倍。

# 1.有效

有效来自于决定什么需要你努力，什么不需要。称之为帕累托原则，80/20 法则，或 YAGNI。不管你喜欢什么，如果你把所有的精力都集中在需要完成的任务上，那么你就能提高你的工作效率。

此外，如果你生产有用的东西，你的心情也会改善。对于一个从未使用过的特性，有大量的代码行和测试，还有什么比这更糟糕的呢？

放弃不需要的任务对设计也有积极的副作用:如果你不需要某些东西，你就不能有一个清晰的定位。决定做什么和如何实施需要额外的努力，以假设和糟糕的决策的形式，所有的选择看起来都不合适。

# 2.真实的测试代码

写出好的测试。覆盖工具是必须的，但是要聪明，不要简单地假设“绿线”是好的。正确地介绍开关、功能和调用参数。糟糕的测试甚至比糟糕的代码更糟糕:一切看起来都很好，但随后却毫无征兆地崩溃了！

这样做的好处是可以防止错误，减少调试、集成和重构。你之前花了一些时间，之后就没有更多的时间了。

*注意:如果你正在编写的测试在被覆盖的组件发生重大变化时没有失败，那么你就做错了。这看起来很天真，但是如果您想到一个针对您所模仿的依赖项的集成测试，不要期望这涵盖了依赖项的变化。*

# 3.睡个好觉

睡眠模式和习惯非常重要。当你低估了他们，对如何使用你的身体能量没有明确的策略时，那么(意想不到的)夜间工作时间可能会是一个干扰事件。你要么没有充分利用你的能量，错过了截止日期，要么在截止日期前做得过头了(你还是错过了)。

有规律的睡眠习惯、控制咖啡因摄入和健康饮食会有很大的不同。此外，如果你能利用咖啡因的峰值、你的血糖水平和特殊的睡眠模式，你就有了更多的锦囊妙计，以备不时之需。

# 4.改善您的沟通

伟大的故事讲述者比普通的演讲者更有可能受到观众的欢迎。在一个程序员的一生中，这并不意味着成为比尔·盖茨或史蒂夫·乔布斯的区别，但这仍然至关重要。

的确，每个人在相互解释时都可能会遇到一些困难，但对于一名技术工作者来说，在没有行话的情况下向非技术人员解释技术相关概念是一个真正的挑战。大多数时候，那些绘制规格、评估你的工作或转移资金(迟早会成为你的工资)的人都是非技术人员。

所以，实现目标和完成使命是很重要的，但是如果你只能交流你所做的一点点，外界永远不会知道你倾注了多少精力和精力。提高你的沟通技巧、魅力和词汇量。

# 5.耐心调试

调试只是工作，没有娱乐。大多数开发人员根本找不到满足感。耐心和分析是可以利用的武器。试错法或散弹枪调试没有任何帮助，甚至是有害的。慢慢来，阅读整个堆栈跟踪，保持开放，不要因为任何原因忽略代码片段。

而且，了解自己，学习快速调试的窍门。人类是软件开发链中的薄弱环节，因此当你在寻找错误时，最好不要责怪机器，而是寻找好的提示。有几个问题使人脑容易出错。认识他们，知道去哪里找。

# 包扎

几十年前，有一些大型软件公司构建分布在物理介质上的整体程序，这些程序可以在商店里买到。但是创业公司和全球化的经济节奏很难与工作生活平衡和人为错误相融合。

要成为一名更好的开发人员，有很多方法可以采用，但是最好的结果往往来自意想不到的来源。成为一名多才多艺的全面的程序员可以带来巨大的变化——尤其是对非技术人员而言。
