## 写技术文章可以帮助你成为更好的开发者

虽然我的软件工程师生涯才刚刚开始，但我找不到不写技术文章的好理由。相反，我坚信每个开发人员都应该这样做。

撰写与技术相关的文章并不复杂，你也不需要成为一名天才作家。你需要的是写作的好理由——而这正是本文所要讨论的。

让我们来看看我认为开发人员应该写技术文章的五大理由。

# 1.分享您的解决方案

我们都觉得自己很特别，但作为开发人员，我们面临的问题或多或少都是一样的。当你[正在努力寻找让你头疼的问题的解决方案](https://medium.com/quick-code/what-a-recurring-outofmemory-error-taught-me-11f2061063a1)时，在世界的另一边，可能有人处于完全相同的情况。

事实是，你处理的问题不太可能是真正独特或前所未有的。如果开发人员总是共享他们的解决方案，您就可以避免每次出现新问题时浪费如此多的时间和精力。

同样，如果你花时间构思[一个创新的架构](https://medium.com/swlh/designing-a-multi-layered-architecture-for-building-restful-web-services-with-spring-boot-and-a12ef85b77c9)，编写一个库，或者测试新技术，你应该让世界知道。这样，整个社区都会从中受益。

# 2.出色的面试

请记住，你的文章是你的遗产。

无论何时你面临一次工作面试，你都可以迅速证明你的技能——从特定编程语言或框架的知识到解决问题。如果你的文章来自公共存储库，这将变得更加容易。

从某种意义上说，你通过这些文章提高了你的声誉，向整个社区展示了你的能力。这样肯定会在面试的时候加分，这是一个很大的副作用。

# 3.教中学

这听起来可能违反直觉，但我从自己的文章中受益最大。

在撰写新文章的同时，作为一名开发人员，您也在培养新的技能和提高自己。例如，通过养成写作的习惯，你可以学习如何有效地传达信息。这将使你成为一个更好的沟通者，使你的评论和文件更加清晰和详尽。

此外，如果你的目标是为你的读者精心制作高质量的内容，你可能需要[更深入地研究你所涉及的主题](/how-to-avoid-bottlenecks-in-node-js-applications-8085d86b6b2e)。这不应该使你气馁。相反，这是一个学习新事物、扩展专业知识、习惯与官方文档打交道的绝佳机会。

# 4.发现新的机会

如果你想成为一名作家，你需要一个地方来发表你的作品。通过给著名的博客发草稿，[向媒体发表文章](https://medium.com/illumination/a-complete-guide-on-how-to-choose-the-right-publications-a1a9a93507fb)，或者建立你自己的博客，你会给自己带来新的机会。

探索全球 IT 社区能提供什么。不仅你的职业生涯会受益，而且你甚至会开始你从未想过的合作。

另外，解释你所写主题的最好方法之一是引用或参考他人的作品。这将让你发现有用的在线资源，并与来自世界各地的新朋友联系。

# 5.赚外快

你不应该指望以此为生，但你绝对可以货币化你的文章赚外快。

这一点很容易做到，因为关于它的文章有一个共同的特点。事实上，与其他话题不同，它们可能不需要被宣传。

因此，你可能会被动地建立一个读者群。看到其他开发人员只是偶然遇到它们，不要感到惊讶。教程更是如此。

最后，如果你不想建立博客，你可以在[媒体](https://medium.com/)上免费开始。创建帐户，加入[合作伙伴计划](https://medium.com/earn)，瞧！你准备靠写作挣钱了！
