自从我在谷歌 Play 商店发布我的第一个应用程序已经过去四年了。在我的旅程中，我面临了许多问题，但我也学到了许多新东西。这篇文章是关于我们大多数人在开发个人项目时面临的两个主要问题。

当我几年前开始的时候，我不知道该做什么——比如它是如何工作的，等等。但是现在我有一些事情要分享。初学者的主要问题是我们不知道从哪里开始和做什么。让我们开始吧。

# 从哪里开始

作为应用程序开发人员，我们在智能手机上开发许多应用程序，并亲自探索更多的应用程序。有时候我们觉得一个应用程序缺少一些功能，我们觉得如果他们有这些功能就好了。我们中的一些人联系应用支持团队，告诉他们我们的想法，而我们中的一些人没有采取下一步行动就放弃了。

每个应用程序都有自己的一套功能，可能无法满足所有移动用户的需求，因此总有空间去做一些事情。所以试着去探索。这将是第一步。当我们不确定要做什么时，探索给了我们更多的想法。探索之后，做一些该做的功课。

你的想法可能是你觉得有趣的东西，或者你可能想做一个对其他人有用的应用程序。或者它可能是一个可以为你带来某种收入的应用程序。想法可以是任何东西——只要在开始之前相信它。

# 做什么

一旦我们有了一个想法，我们不只是点击 IDE 并要求我们需要的输出。这是一个循序渐进的过程。

我们中的一些人会在第一阶段退出，一些人会在稍后阶段退出，一小部分人会在最后阶段成功或失败。我们不会在最后阶段都成功，因为如果我们坚持努力，做一些事情，就不会产生影响。这完全取决于市场。那些成功的人口袋里有成功，而其他人有经验，很难说它在市场上有多少价值。

对于任何项目，我们都需要遵循以下步骤:

*   收集要求
*   用户界面设计
*   发展阶段
*   测试
*   部署
*   分析学

# **采集要求**

任何应用程序的第一步都是收集需求。把更多的精力放在需求分析上。我们中的许多人没有花费必要的时间，因为我们很匆忙，或者我们认为它没有发挥主要作用。

然而，这是非常重要的一步。这里我们需要分析很多东西，比如:

*   它将包含哪些功能？
*   将会有多少个屏幕？
*   我们将支持哪些平台？
*   我们在每个平台上支持的最低版本是什么？
*   我们的应用程序的受众群体应该有多大？
*   我们如何使用分析来跟踪我们的用户？
*   什么是货币化或商业模式(我们将如何产生一些收入)？

在开始应用程序开发之前，我们需要考虑许多指标。我们需要搞清楚每一件事情。如果你想做大事，你需要做大事。我的建议是最好先用小的 pop 做实验，这样你会熟悉一些事情和要遵循的程序。这样，以后我们就可以毫无障碍地轻松钓到大鱼了。

# **UI 设计**

相信我:UI 将扮演一个关键角色。UX 设计指的是术语*用户体验设计，*，而 UI 代表的是*用户界面设计。*你应该明白 UI/UX 的重要性。UI 设计是第一个为应用程序带来第一批用户的东西。以后人气会决定 app 的命运。

在初始阶段，当市场上没有人认识你的时候，你准备怎么介绍自己？只有一个办法:app 图标和预览截图。设计应该简单而有吸引力。看看 WhatsApp——他们知道应该有多简单。不要把彩虹的所有颜色都放在你的应用程序里。让应用程序看起来专业而精致。

学习设计——不一定要作为一种职业，但足够满足你的需要。相信我，你迟早会需要它的。

# **开发阶段**

当我们完成了基本步骤，下一步也是最关键的一步是编码部分。现在是时候打我们的 IDEs，并与他们斗争一段时间，以产生我们的想法。伟大的事情需要时间，所以要有耐心，多花点时间。添加一些更好的 UX 动画。生孩子不是一件容易的事——如果你有孩子，你就能理解这一点。

我们不会一蹴而就，所以继续尝试。我们正在研究我们的想法，所以最好在开发中使用所有最新的框架。这有助于应用的未来——也有助于我们的学习。保持自我更新。

我的建议是，如果你正在开发 Android 应用程序，那么在 [Kotlin](https://medium.com/@pavan.careers5208/detailed-guide-on-android-clean-architecture-9eab262a9011) 中构建应用程序，并遵循 MVVM 或 clean architecture。如果需要，使用协程和匕首。请查看我在 [clean architecture，](https://medium.com/@pavan.careers5208/detailed-guide-on-android-clean-architecture-9eab262a9011)上的帖子，这样你的应用程序将易于维护和测试。在发展中遵循[坚实](https://en.wikipedia.org/wiki/SOLID)的原则。检查[优化](https://medium.com/better-programming/improve-apps-performance-with-android-profilers-edb240deeb71)。我建议的所有事情都不是强制性的，但拥有它们是件好事。使用一些崩溃分析框架来分析崩溃。使用版本控制系统(VCS)，比如 GIT，来维护源代码

学习是一门持续的艺术，永远不要停止。

# **测试**

在我们完成开发阶段后，我们需要进行手工测试。

检查错误，并修复它们。做多轮测试。写一个 JUnit 测试用例是很好的。

但这一切都取决于时间。我总是进行手工测试，但是在生产中会出现一些罕见的情况。每次发布前进行适当的测试。

# 部署

要部署应用程序，你需要有一个谷歌 Play 商店账户，在 IOS 上只需花 25 美元购买一次。以苹果应用商店为例，是 99 美元/年。

最初发布 alpha 版本，然后发布 beta 版本，然后将其投入生产。当创建一个新的版本时，预先发布给一定比例的用户，监控崩溃，并在出现重大影响时停止发布。修复错误，然后再次创建发布。然后，慢慢地从 5%增加到 100%。如果你遵循这种渐进的展开，你就可以避免巨大的负面影响。

# 分析学

我们还没有完成——分析发挥了重要作用。如果配置得当，它会为您提供有关用户指标的所有信息。

在某个阶段，我每天安装 1000 个应用程序，卸载 900 个。这是一件非常糟糕的事情。这是分析发挥作用的时候了。理解用户指标，并对其做出反应。谷歌或 Firebase analytics 是更好的开始。但也有其他分析网络，如 CleverTap、Segment 等。在您即将推出的应用中尝试分析平台。

# 结论

处理个人项目将帮助我们探索新事物，提高我们的技能，并可以成为第二收入来源。你会比你简单的工作职责有更多的意识。当您从头开始监控一个完整的产品时，您将理解软件开发生命周期(SDLC)的每个阶段，这将在许多方面受益。

不要太担心失败——如果你停下来担心，你就不能开始。把失败当作一个从中学习的机会。
