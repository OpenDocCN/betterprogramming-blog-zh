# 开始—第一次编程课程

如果你像这个星球上几乎所有的互联网用户或者我一样，你通过在谷歌中键入“如何学习编码”来开始学习编码。就我而言，前两个结果是 Codecademy 和 Udemy tech 课程。

Udemy 吹嘘拥有超过 13 万门课程。哇哦。怎么挑一个？也许是几十门名为“完整的 Web 开发训练营”或“Python —从零到英雄”的课程中的一门？这看起来是一个很好的开始，价格低于 20 美元。那似乎是合理的。

你注册了这门课程。持续 20 个小时。你还不知道，但要完成任何视频课程或教程，你需要大约三倍于课程的时间。

需要一两个月，但是你完成了课程！呜哇！你学到了很多。不久前，您还一无所知，现在您已经对一种或几种编程语言有了扎实的了解。如果这个课程还不错，你也已经开发了一两个应用程序。

下一步是什么？

我很确定你还没有自信。自己造东西还为时过早。也许另一门课？只要 20 美元。你会更新你的知识，学到更多的东西。我们开始吧。

第二道菜做好了。您仍然不确定如何自己构建应用程序。下一步是什么？也许另一门课？这个承诺能让你得到一份工作。

# “我被困住了”阶段

当你完成下一门课程时，你可能还没有准备好自己动手做一些东西。我明白。

事情是这样的:没有人准备好了。

你很想开始另一门课程，但那只是浪费时间。

没有任何课程或教程会让你成为程序员。事情不是这样的。

课程是一种被动的学习方式。你半心半意地看着他们，有点无聊和分心。我不怪你。很难在几个小时内保持专注，初学者的课程经常重复相同的信息。

当然，您可以找到一些很棒的课程，它们有清晰的大纲、许多编码练习以及老师和社区的支持。我参加过许多这样的课程，现在仍然如此。

但是即使是很棒的课程也只能让你到此为止。你在这个课程中工作的环境是人造的。老师掌控一切，掌握所有答案。

更糟糕的是，他们有所有的问题。

每门课程都会给你一些任务去完成。通常情况下，任务是具体的和受约束的。它们是需要解决的小而精心设计的谜题。解决这些难题与自己编码并不十分相似。

当你自己编码的时候，你必须把一个像“我想建立一个待办应用”这样的大问题分解成小的、可管理的问题，比如“我如何存储任务？”或者“我如何删除任务？”将一个问题分解成更小的部分是编程的本质。在这些课程中，有人会替你做。

要学习编码，你需要的不仅仅是最好的课程。你需要奋斗。

# 是时候继续前进了——建造一些东西

如何走出课程循环？

从改变心态开始:没有人能教你任何东西。你得自己学。你是对你的成长负责的人，而不是老师。

通过对自己的学习负责，你一定会做出更好的决定。

在你离开这些课程之前，你应该尽你所能从中吸取所有的知识。如果你感兴趣，我有一些关于从课程中学习的文章，但基本内容很简单:

*   记笔记。
*   试着凭记忆编码——不要和老师同时进行。
*   不时刷新你所学到的东西。最好准备一份重复时间表。
*   找在线测试来检查你的知识。
*   在博客或视频上记录你的学习。
*   向某人解释你学到的东西。

有了基础知识，就该造东西了。但是你应该建造什么呢？

从简单开始。自己一个人搭建一个待办 app 已经足够好了。或者一个天气应用或者 Pokédex。

如果你想成为一名前端开发人员，尽快开始使用一些外部 API。你可以找一个[免费 API 列表](https://rapidapi.com/collection/best-free-apis?)获取灵感。

还不确定要创作什么？看看[这份不同难度等级的好主意列表](https://github.com/florinpop17/app-ideas)。

当你独自工作时，你会很快碰壁。比你想象的要快得多。那很好。我们的工作 90%的时间都在碰壁。允许自己失败并重新开始。

拥抱失败，因为它将伴随你几个月。这是成长的唯一途径。

最终，你可能会碰壁。这个问题你几个星期都解决不了。这意味着你选择的应用程序对你的技能水平来说太复杂了。这是常有的事，没什么可担心的。就这样吧，创建一个新的应用程序。

知道何时放手是拥抱失败的重要一环。只要你在建设和学习，你就没有真正失败过。

每个新问题教给你的东西比几个小时的教程还多，所以不要担心，继续走在成为程序员的道路上。你能做到的。
