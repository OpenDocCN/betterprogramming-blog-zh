如何提出关于编程的问题

你至少要付出n%的努力

编程不是写代码。编程就是解决问题。实际代码，任何人都可以学习。如果你擅长解决问题，编码部分基本上是学习语法和如何搜索函数、库和技巧来解决任务。

不值得回答的问题

这篇文章是基于在不同的脸书小组中问的一系列真实问题。这些问题是合理的，我想研究的只是你如何接近你的观众。

如果你的问题是这样的，为什么有人要花自己的时间来帮助你呢？

“我该怎么解决这个问题？”(一个明显的学校作业的随机截屏。)

“我该怎么做？我什么都试过了。”(明明没有尝试过全世界的一切。)

如果你只是抛出一个问题，而你希望别人为你做所有的工作，你是不尊重别人的。你基本上是在要求随机的陌生人为你做所有的工作，而你却坐着从他们的工作中收获。

示例1:三角形的面积

"如何用Python计算两个三角形组成一个正方形的面积？"

对所有问题的自动回复应该是，“到目前为止你尝试了什么？”(或者，如果这是一个哲学问题，“到目前为止，你对这个主题有什么想法？”提问时请贴上你现在的代码。

如果你的回答是“我还什么都没尝试过。我只是有问题解决它，“请回来时，你已经尝试了一些东西。

上面的例子是一个糟糕的提问方式。以下是我认为提问的好方法:

“如何用Python计算两个三角形组成一个正方形的面积？到目前为止，我已经尝试了以下方法:

我认为这个领域背后的数学原理非常简单。正方形有相等的边，所以我们知道底边和高，因为它们也是一样的。我们甚至不需要邀请毕达哥拉斯——耶！我谷歌了一个三角形的面积，这是公式:
A = 1/2 b*h，这是我现在的代码:

def find_square_area(length):
    return int(length)*int(length)def find_triangles_area(length):
    return (1.2*int(length)*int(length))side_length = input('length of square sides: ')
print (f'The area of the square is {find_square_area(side_length)}')
print (f'The area of each triangle making up the square is {find_triangles_area(side_length)}')

代码看起来可以运行，但是我期望有一半的正方形区域，因为我认为它应该是这样的。我是说，我只是把它切成两半。是我想错了还是编码错了？感谢帮助！

我希望能看到更多这样的问题。显然，假想的用户已经认真考虑过了。他们已经研究了他们想要解决的问题的理论，并且已经完成了代码。

代码似乎工作正常，用户也在反思这里的逻辑错误。这个很容易滑落，但是如果你停下来想一想，每个三角形的面积不会比正方形本身大。显然出了问题。

我确信有几个人会帮忙，我们能够指出find_triangles_area()函数中的数学是不正确的。也许讨论还会引发对像*=这样的操作符的讨论——并不是说你需要使用它，但是它可以激发更多的创造力。

在本次会议中，我们取得了几项胜利。第一，原帖通过差不多搞清楚问题，学到了很多东西。其次，这个问题很吸引人。这是我想要回复的答案。

例子2:毕达哥拉斯

无论如何，让我们邀请毕达哥拉斯参加聚会，并提出一个与理论相关的问题。

"我不明白勾股定理，给我解释一下."

这是一个非常模糊的问题。我该从何说起呢？考虑到网上有多少网站对此进行了精彩的解释，我不相信这是真正的问题。

如果我来回答，也许我会问一些类似的问题:

你对勾股定理的什么地方有疑问？概念，数学，还是别的什么？

这应该会引发一些思考，发布问题的用户可能会说:

“嗯，设置好之后，我们两边都有数字，对吧……24 = 16+b。然后，突然我们在计算320和b的平方根。所有的数字都去哪里了？为什么我们突然开始研究320了？”

结果是，用户实际上只是对表达式有问题。也许他们看到的例子跳过了几个步骤，因此，使它更加混乱。

既然我们已经获得了更多的信息，我们可以一步一步地浏览这一部分，用户可能会得到一个很好的回复。

不过，这个问题还不错，对吧？如果信息从一开始就在那里，我们就不会浪费一点时间。一点点信息就能帮助这里的每个人。

最后

每当你想问一个关于编程或解决问题的问题时，首先要确保你自己已经做了一些事情。如果你想真正从中学到东西，你必须做某事。总的来说，这也是最好的学习方法。你可以看所有你想看的教程，但是如果你从来没有尝试过任何东西，你怎么能找到感觉呢？

如果你想要体面的回答，确保你张贴体面的问题。你会惊讶有多少人真正喜欢帮助别人，包括我自己。我喜欢教书，也喜欢学习。让我们再次让教学变得伟大。感谢阅读！
