# 从事专业编码 3 年后，我放弃了 3 个信念

> 原文：<https://betterprogramming.pub/3-beliefs-i-abandoned-after-3-years-of-professional-coding-d4a71b588100>

## 对错误原则的信仰是如何阻碍你成为专业人士的

![](img/88d9d195ee13f3cd758fc5e34dc2864a.png)

布莱克·康纳利在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片。

我在收拾我所有的东西，回去聪明地工作。当我关掉电脑时，我瞥见了日历。今天是 11 月 5 日。今天标志着我作为专业开发人员工作的第三年。

我看着我的隔间附近所有的空椅子，我无法避免地想起迄今为止我学到的所有东西。但是在我增长的知识的重压下，我并没有感到更重。相反，我感觉更轻松了，因为我知道我已经放弃了过去三年对这个令人惊叹的职业的许多错误信念。

# 1.程序员应该什么都知道

当我十几岁学习如何编码时，我的老师会给我家庭作业练习来练习我的技能。我记得每次我都会尽最大努力不去上网查看其他程序员是如何面对这样的问题的。我对编程一无所知，甚至是细微的差别，这让我感到很尴尬。

随着时间的推移，我不仅允许自己每天访问 Stack Overflow 而不感到羞耻，而且我已经意识到[在这份工作中了解一切是多么不可能。](https://medium.com/javascript-in-plain-english/how-to-survive-your-first-year-as-a-professional-junior-developer-9db4bdd3d8e8)

我已经把自己从不真实的自我期望的负担中解放出来。现在，我知道我不知道一些事情也没关系。觉得有必要检查文档，甚至是最基本的东西。

这份工作不仅仅是你已经能做的。这也是关于你如何管理外面的一切。

# 2.程序员不应该对自己的工作没有安全感

现在，我有一些非常熟练的高级开发人员和我一起工作。他们是有能力的人，对我的拉动式要求有很高的标准。否则，他们会毫不后悔地轻易拒绝他们。

我曾经认为这些人是不会犯任何错误的开发人员。我会敬畏地看着他们，想知道是否有一天我会达到他们的技能水平，并且永远不会写错任何代码。

但是这种想法是错误的。

我没花多少时间就获得了更多的经验和与同事的对话，却意识到他们也有[不安全感](https://medium.com/better-programming/how-i-deal-with-negative-thoughts-when-i-have-to-code-b8664ed5d824)。他们并不总是知道所有的事情，也不总是确定某个问题的最佳解决方案。他们必须像我一样不断更新知识。他们知道，有时他们只是不得不随波逐流，凭直觉判断什么对项目最有利。

我已经把自己从这种信念中解放出来，现在我不总是觉得困在自己的世界里，认为我永远不会成为一名高级开发人员，因为我知道高级开发人员也有我的不安全感。

# **3。编程不允许即兴**

编程是科学。我们称之为[最佳实践](https://www.aversan.com/coding-standards-and-best-practices-2/)的原理科学。但是，我们的应用程序组装在一起的方式是人类思维创作的艺术。

我曾经相信编码是某人的知识和可视化编辑器上出现的代码之间的线性行为，但这是完全不同的东西。

开发人员每天都在努力遵循编写代码背后的最佳原则，但是很多时候，我们必须承认我们必须在工作中随机应变。

我们即兴发挥是因为我们不了解一项新技术，所以我们必须快速掌握它，然后以良好的判断力来使用它。我们即兴发挥，因为我们从来没有在第一次尝试中正确地完成某件事，所以我们必须总是重新发明如何达到目标*这次*。

在开发人员的知识和如何将这些知识应用于项目之间有很多即兴创作。但是我相信最好的开发人员知道这一点，并且知道如何随机应变，倾听代码，及时做出反应，创造和支持他人的想法。

# 结论

我相信，人们成长的一部分就是摆脱阻碍他们飞得更高的负担，这对每个人的职业生涯来说都是如此。

希望这篇文章能启发你这样做。看看你的职业生涯，想想是否有任何信念实际上正在伤害你，阻碍你的成长，或者仅仅是不符合编码的真实情况。
