## 每个开发人员都应该知道的 React 库

![](img/4bc694bcccc289563cacba0cf99e13d7.png)

无论您是一名专业的 React 开发人员还是刚刚开始使用 React，您都希望尽可能地简化开发人员的工作。跟上 React 生态系统的所有最新趋势很难。项目正在快速发展。所以很容易错过一些很棒的 React 库。

这就是为什么我们将仔细阅读每个开发人员都应该知道的前七个 React 库。

# 1.洛达什

[Lodash](https://lodash.com/) 通过消除处理数组、数字、对象和字符串的麻烦，使 JavaScript 变得更简单。在 GitHub 上有超过 40k 个星星，Lodash 是一个受欢迎的库。根据其网站，Lodash 是一个现代 JavaScript 实用程序库，提供模块化、高性能和额外功能。

尽管 Lodash 不是一个特定的 React 库，但它是我个人最喜欢使用的库之一，因为它非常优雅。我强烈建议你去 Lodash 看看。

# 2.反应引导

[React 引导库](https://react-bootstrap.github.io/)顾名思义:用 React 构建的引导组件。Bootstrap 是一个用于 HTML、CSS 和 JS 开发的开源工具包。Bootstrap 是世界上最流行的前端组件库。

React Bootstrap 的伟大之处在于它取代了旧的 Bootstrap JavaScript。该库中的每个组件都是作为真正的 React 组件从头开始构建的，没有像 jQuery 这样不必要的依赖。

作为最古老的 React 库之一，React Bootstrap 已经随着 React 一起发展和成长，使它成为您的 UI 基础的一个很好的选择。

# 3.react-i18 接下来

如果您需要一个多语言应用程序，那么您肯定需要 [React-18next](https://react.i18next.com/) 库。React-i18next 是 React 基于 i18next 的强大国际化框架。它为您提供了一个完整的解决方案，将您的产品从 web 本地化到移动和桌面。

i18next 库具有翻译功能，如复数、格式化和插值。最重要的是，这个库适合服务器端渲染。

# 4.反应 DnD

[反应 DnD](https://react-dnd.github.io/react-dnd/about) (拖放)帮助你构建复杂的拖放界面，同时保持组件的解耦。与大多数其他伟大的拖放库不同，React DnD 是建立在现代 HTML5 拖放 API 之上的。

这个库专注于正确的拖放交互，把它的视觉方面留给你。

过去，这个库有一个主要缺点——不支持触摸屏。如果你想要触摸支持，你可以使用 React DnD 与触摸后端，而不是 HTML5 后端。

# 5.材料用户界面

谁没听说过谷歌的材料设计？[材质 UI](https://github.com/mui-org/material-ui) 是一个包含一组实现著名材质设计的组件的库。

GitHub 上超过 50k 颗星，素材 UI 库绝对要在这个列表上。它是目前最流行的 React UI 包。它已经存在好几年了。这个库易于实现，并且组件引人注目且用户友好。

这个库有太多的组件，无法一一列出，所以我建议你查看 Material UI 本身。

# 6.反应旋转器

在你的应用程序中已经没有空间容纳无聊的旋转器了。 [React spinners](https://www.react-spinners.com/) 是一个提供各种各样花式微调器的库。可以调整这些微调器的属性，以便您可以按照自己的方式完全自定义微调器。

这是一个小库，在 GitHub 上有大约 1k 颗星星，但它可以在 UX 方面给你的应用程序带来真正的提升。

# 7.酶

[Enzyme](https://airbnb.io/enzyme/) 是一个 JavaScript 测试工具，用于测试 React 组件的输出。它由 Airbnb 工程团队创建，并于 2015 年作为开源软件发布。酶的 API 意味着直观和灵活。这个库应该使断言、操作和遍历 React 组件的输出变得更加容易。

Enzyme 可以与各种各样的测试跑步者一起跑步，如 Mocha、Jest、Karma。

如果你是测试 React 组件的新手，我强烈建议你去检查酶。
