# 作为初级开发人员，我犯的五大错误

> 原文：<https://betterprogramming.pub/top-5-mistakes-i-made-as-a-junior-developer-8bb907463754>

## 如果你刚刚开始，看看这些建议

![](img/5736bd7ec02761e7ecfa1f9a93bf78ac.png)

[杰米街](https://unsplash.com/@jamie452?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照。

在这篇文章中，我想谈谈我作为初级开发人员犯的五大错误。它们并不针对我的情况，许多其他开发人员也犯了同样的错误。

因此，我想把它们写下来，以便其他人可以看到并避免它们。事不宜迟，我们来看看这些错误是什么，如何避免。

# 1.我没有问问题

许多初级开发人员的一个误解是他们的问题很愚蠢。有多少次你避免问一个认为很傻的问题？

我不认识你，但是这个误解阻止了我问很多问题。猜猜结果是什么:我很困惑，不知道如何完成任务。

因此，我鼓励你问问题，不管你认为这些问题有多愚蠢。很多时候，其他开发者也有或者曾经有过同样的疑问。如果有人因为问问题而让你感觉不好，那是他们的问题，而不是你的。

# 2.我陷入了教程地狱

这个错误很经典:被困在教程地狱。很多开发者，包括我自己，都是看教程而不是写代码。这是为什么呢？看教程和抄袭老师的代码会给你一种错误的成就感和进步感。然而，从长远来看，情况恰恰相反。

不要误解我:教程是有用的。做一到两次来获得你想要学习的概念，但是不要停滞不前。真正的学习发生在你尝试和奋斗的时候。自己构建一个应用比十个教程教会你更多。原因是*你*必须思考和工作。那里没有教师向你灌输代码和指令。

如果你想学习，你能做的最好的事情之一就是写大量的代码。教程是好的，但只是在一定程度上。做一到两个来了解你想学什么，然后开始构建东西。

# 3.没有风险，只有舒适

我犯的另一个错误是呆在我的舒适区。我这么说是什么意思？我过去常常挑选容易和舒适的任务去做。完成那些任务给了我一种虚假的成就感，从长远来看，这伤害了我的进步。

我没有挑战自己或尝试做更困难的任务。结果我就卡在了同一个水平。呆在自己的舒适区是无法进步的。

有时候出拳超过你的体重是必要的。只做有挑战性的任务也不是办法。那我们该怎么办？混合它们。不要只做简单的任务。这样，你就不会永远停留在同一水平。同时，不要只尝试挑战性的任务，因为迟早你会变得意志消沉。

# 4.社交媒体大师

你能做的最糟糕的事情之一就是听社交媒体专家的话。我说的是那些宣扬不停工作，为“成功”牺牲一切的人，等等。这是有害的积极态度。

你不必为了成为一个好的开发人员而生活和呼吸代码。你不必 24/7 编码。下班后你不必编码。对下班后编码有帮助还是一般多？绝对的。这是成为开发人员或优秀开发人员的要求吗？绝对不行。

在这些大师和我两次筋疲力尽之后，我曾经模拟过我的生活。因此，要意识到这些错误的主张和有毒的阳性。生活不仅仅是编码和工作。

# 5.太安静了

最后，我在会议中太沉默，几乎不参与。我害怕参与讨论，认为我没有什么有价值的东西可以补充。但那不是真的！

即使您可能是初级开发人员，但这并不意味着您没有任何有价值的东西可以添加到讨论中。此外，如果你说了一些技术上的错误，那是可以纠正的。除此之外，它可能会帮助你澄清你对你正在做的项目的误解。

当然，这并不意味着只是为了说话而说话。不要让任何恐惧阻止你表达自己的观点。你是初级开发者，你是有价值的，你有有用的东西可以加入讨论！

# 结论

以上是我作为初级开发人员所犯的错误。你可能会发现自己犯了同样的错误。如果你有，试着从这篇文章中学习并改正它们。

概括一下:

*   问问题，不管你认为他们有多蠢。
*   不要陷入教程地狱。看一两个教程，然后写代码。很多代码。
*   确保你完成了简单和挑战性的任务。
*   不要听信社交媒体专家和有害的积极态度。生活不仅仅是工作和编码。
*   尽你所能做出贡献。不要安静。你有价值！
