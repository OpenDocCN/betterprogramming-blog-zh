## 人非圣贤孰能无过

![](img/99128de496f3897217e9fe95e241fd49.png)

Maruxa Lomoljo Koren 在 [Pexels](https://www.pexels.com/es-es/foto/senalizacion-de-luz-de-neon-roja-3887574/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels) 上拍摄的照片。

> "让无罪的人扔第一块石头."

这句著名的宗教短语几乎可以应用于生活的任何领域。当然，这包括软件开发领域。

在这篇文章中，我列出了我认为我们在开发软件时可能犯的最大错误。无论你是否信教，我相信你对这些大罪都很熟悉。

# 1.渴望

这是我见过很多次的事情，当我开始开发软件时也是这样。想要尝试所有新的东西是非常诱人的，比如使用最新的框架或刚出来的语言。但这几乎总是直接导致灾难。

如果您使用一个刚刚发布的新框架，仅仅是因为它似乎可以解决所有问题，或者因为您喜欢尝试新框架——即使它不成熟或者背后没有大量的用户基础——当您在项目的开发过程中取得进展时，您可能会意识到它的缺点、错误或安全漏洞。这将迫使你修补所有的代码。

更糟糕的是，框架背后的创建者或社区可能会停止支持它并抛弃它。如果发生这种情况，如果你不想为未来造成巨大的技术债务，你将被迫更换它。你将不得不在这上面花很多时间。

*总之:仅仅为了测试没有必要的东西而过度使用库、工具或框架。*

# 2.贪食

这个问题比我们想象的更经常发生。这是我在做代码审查时多次看到的事情。例如:

*   函数的使用没有优化。
*   使用库来做一些你可以用几行代码有效解决的事情，并且不假思索地更新它们。
*   将巨大的文件加载到内存中并处理它们，而不是逐块读取并处理它们。
*   通过为每个操作打开和关闭连接，将数百条记录插入数据库。
*   从数据库中检索不必要的信息。
*   向客户端发送比所需属性更多的对象。
*   数据类型和数据结构选择不当。
*   在客户机上处理高成本的任务，这些任务可以在你的台式计算机上流畅地运行，但不能在你的手机上运行。例如，试图通过下载整个列表(即使列表很大)来对客户端上的对象列表进行分页，而不是在服务器上进行分页，因为这样更容易。
*   并行化任务，使机器的其余部分没有资源用于其他任务。

*简而言之:使用可用资源，不考虑其成本或编码，不关注执行任务所需的硬件要求。*

# 3.贪婪

你听过多少次“我创造了那个代码。不要看着它，听其自然。它工作得很好”或者“我从一开始就在做这个项目，我不想让其他人加入”？我们必须意识到，当我们为他人或团队工作时，代码不是我们的，我们必须与我们的同事分享，尽可能地帮助他们。

我们想了解和掌握所有的时髦语言和框架，这是不可能的。结果，我们对任何事物都没有深入的了解，最终我们错误地使用了它们。

在其他情况下，我们可能会被诱惑，想要成为不可或缺的，成为所有发展的中心，成为做所有决定的人。我们可能认为，如果我们是不可或缺的，那么如果出了问题，我们将是最后一个被解雇的人。

*概括来说:当我们相信一切都属于我们或者试图掩盖我们力所不及的时候。*

# 4.懒惰

让某物工作而不用担心它是如何工作的。

*   没有增加好的测试覆盖率，甚至根本没有增加测试。
*   不关心使用好的实践和遵循诸如固体、干燥或其他的原则。
*   没有使用版本控制系统，或者使用了一个版本控制系统却做错了，挑选或者一次提交许多变更。
*   找到重复的或死的代码行，却什么都不做。
*   写代码的时候不考虑谁会跟着来。
*   从堆栈中复制粘贴代码会溢出而不理解它。
*   忽略错误或警告。
*   没有停下来思考就直接开始编码。
*   构建高度耦合的系统，因为这似乎更容易。

*概括来说:当我们花尽可能少的时间做某件事，却没有停下来想一想这是否是做这件事的最佳方式，因为我们对质量或学习毫无兴趣。*

# 5.愤怒

不合理地对某人或某事发脾气。

*   当有人想使用一种语言或框架，并试图以各种可能的方式证明它是正当的，只是因为你不知道它时，变得恼怒。
*   对某人大喊大叫，他犯了一个`git push`错误，上传了一些破坏构建的东西，让你浪费了一个小时的时间试图修复它，却没有意识到你可能会在其他时间这样做。

总而言之:我从不犯错。为什么我们其他人不小心点？

# 6.羡慕

阻碍别人的工作，这样他们就不会比你做得更好。

*   拒绝使用或批评一项新技术，只是因为你不知道它。
*   知道一个库或实用程序解决了一个其他开发人员正在试图解决的问题，并对此保持沉默，这样他们就可以花更长的时间去做，而不会“得分”

*综上:如果不是我，也不会是别人。*

# 7.骄傲

相信你是最好的，你永远不会错。

*   仅仅因为你认为你会做得更好而修改代码。
*   继续开发某样东西，同时意识到这不是最好的方法，仅仅是因为你没有意识到你犯了一个错误。
*   认为你的代码可以工作，代码中的错误是别人造成的。
*   在没有充分测试的情况下将提交上传到生产分支，认为您的代码不会失败。
*   认为自己是对任何事情都最了解的人，不接受其他开发者的建议或意见。或者更糟糕的是，认为别人做的事情是错误的，而不花五分钟试图理解它，仅仅因为它与你的想法不同。
*   即使一个决定是错误的，也要为它辩护。例如，当像 Postgres 这样的关系数据库已经绰绰有余时，决定使用大数据解决方案。
*   当你试图证明自己是最棒的时候，过度设计，而你所做的只是把事情复杂化。
*   不倾听客户，认为他们不了解自己的领域。

*总之:我做任何事都是最好的，我不接受任何人的批评。*

# 结论

如果你开发软件已经有一段时间了，你可能犯了这七宗罪中的一些。在我的情况下，我可以说我犯了罪，但就像在任何事情上一样，我们可以从错误中吸取教训，不再犯错。
