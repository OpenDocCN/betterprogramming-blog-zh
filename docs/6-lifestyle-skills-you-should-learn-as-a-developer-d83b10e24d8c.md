# 作为一名开发人员，你应该学习的 6 项生活技能

> 原文：<https://betterprogramming.pub/6-lifestyle-skills-you-should-learn-as-a-developer-d83b10e24d8c>

## 因为技术技能只能带你走这么远

![](img/530caacca7c28135dfafc19433e0d564.png)

扎克·贝滕在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

我在 IT 行业已经快七年了。大部分时间我都是做开发的。当我开始我的职业生涯时，就像其他人一样，我认为学习一门编程语言就足以成为一名超级程序员。但是，随着我在这个行业中度过的时间越来越长，我了解到仅仅拥有关于编程语言的知识是不够的。我们还必须发展一些个性技能，尤其是如果我们天生不具备这些技能的话。

当然，学习一门编程语言是做开发人员的必须。但这只是第一步——不是一切。

在这篇文章中，我将分享我认为一个程序员如果想成为一个高效的开发者应该具备的东西。

# 1.一致性

![](img/30d30629456b7a2037d7012c23211cd0.png)

照片由[布雷特·乔丹](https://unsplash.com/@brett_jordan?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄

一致性是成功的必要组成部分，不仅是作为开发人员，在生活中也是如此。不幸的是，大多数时候人们没有意识到一致性是多么重要。

通过持续交付高质量的工作，你建立了声誉。你将被认为是交付可理解和可伸缩代码的人。当其他人认识你时，你会被认为是一个权威和一个可信的来源。

在软件开发团队中，几乎总是有一个人是首选的。他们是首选，因为他们与他们的态度和行动一致。当你承诺某事并坚持到底时，你会获得内心的自信，也会获得团队的信任。记住这一点很重要

> “一致性不是完美。”—未知

毫无疑问，一致性并不意味着你必须产生高质量的代码，这些代码能够完美地工作并且没有错误。不幸的是，大多数时候，这是不可能的。然而，如果你一直只写有小错误的代码，你知道你作为一个开发人员正在进步。这正是你要做的。通过自我教育和从失败中学习来不断提高自己。

# 2.创造力

![](img/474d6b97020bea18d57383393936363c.png)

照片由[创意交流](https://unsplash.com/@thecreative_exchange?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

每当我告诉别人我是一名软件开发人员时，他们总是说，“那你一定非常聪明。”老实说，我对此深表怀疑。作为一个开发者，我有很多次觉得自己很蠢，他们甚至无法想象。

通常，人们有一种思维定势，认为软件开发人员擅长数学。有些开发者可能是，但不是我们所有人。就此而言，数学甚至不总是必需的。毫无疑问，像数据科学、机器学习、人工智能等软件领域。需要大量的数学编程，但是开发一个 web 或移动应用程序，这不是必需的。

与其成为数学天才，你更需要一个有创造力的头脑。因为现在的软件需要你从无到有的创造一个东西:一个空白的屏幕。一种带有编程语言文件扩展名的打开的文本文档。

我当然相信软件开发在某种程度上是有创造性的。我不是说所有的软件都是创造性的努力。我甚至不是说商业软件开发一点也不技术性。我的建议是，作为软件开发人员，有创造力的人能够而且确实表现出色，因为他们拥有正确的思维工具。商业、人、抽象思维大多是创造技能。所以，下一次有人告诉你他们不会成为一名优秀的程序员，因为他们不擅长数学，你可以告诉他们，“你不必这样！”

# 3.问题解决

![](img/1eebd475969d9eca91f5e324e376cdb2.png)

卡拉·埃尔南德斯在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

正如我之前所说，软件正在从无到有地创造一些东西。软件是为人民服务的。每天，每当我的产品负责人与我分享需求时，我都必须思考可能的解决方案。作为开发人员，我们收到的需求是我们必须解决的问题。有时候，你可以从过去的经验中找到解决方案，但其他时候你真的需要集思广益才能找到正确的道路。你要知道你要往哪个方向想；否则，你只会陷在你的想法里。

软件工程首先是解决问题，其次是编码。

需要确切地告诉计算机做什么；他们不能像人类一样在得到模糊指令时做出假设。其次，软件工程师的任务是设计甚至可能还不存在的功能和应用程序，因此开发人员的工作是在前端提供用户界面，在后端提供数据基础设施，以便从头开始。

# 4.责任

![](img/d9b13807bc09c0d2d433d63979a4123c.png)

克里斯蒂安·布纳在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

你还记得你父母告诉你要负责任吗？对你家庭的某些事情负责，对你自己的生活负责？我希望你听了他们的话。因为他们给你上的人格课也会帮助你的职业成长。

当你作为一名开发人员，你要对你的工作负全责；或者这么说吧，对分配给你的任务负全部责任。对你的工作项目负责不仅仅是按时交付，如果你在截止日期前无法完成任何任务，不要说是因为其他人。

我知道，当你开发一个应用程序时，不仅仅是你一个人，在一个团队中工作的具有不同技能的多个人，可能不止一个团队。但是，让我们说，你对其他团队的工作有一些依赖，那么你有责任从他们那边完成它。他们可能会遵循自己的时间表。是你在他们预定的时间之前就需要他们的帮助。所以你必须非常专心。你必须说服他们交付货物，这样你的任务才不会被延误。

当然，这不是争论的问题，而是以一种适当和顺利的方式完成事情。如果你仍然看到有风险的东西，就让你的上级提前知道。我们应该总是避免在最后一刻出现意外。

# 5.协同工作

![](img/bd80e74778cf031eff4636ff29ace6b4.png)

安妮·斯普拉特在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

团队合作可以指许多不同的事情:与他人一起工作以实现一个目标，多个个人的协作行动，或者甚至只是与其他人的有效和高效的协作。

作为一名开发人员，我们应该抓住这个机会获得团队工作的好处。你不必是一个喜欢独自工作的书呆子。你必须明白，当你陷入困境时，你的团队会帮你走出困境。当你的团队很容易接近时，不要让自己太累。

> “如果你想走得快，就一个人走。但如果你想走远，就一起走。”
> 
> —非洲谚语

团队合作允许具有不同互补技能和专业领域的人走到一起，分享他们的想法、思想、知识和解决方案。团队成员的不同视角可以推动**创意**和**创新**。

知识共享是当今组织的一大优势。通过在团队中工作，你可以互相学习，提高你的技能，并增加你在某些发展领域的专业知识。

当团队成员克服障碍时，他们会觉得自己在帮助团队实现目标。拥有他们的工作并对他们的成功或失败承担全部责任是很重要的。一个健康的团队会自我管理并让其他团队成员负责，从而灌输所有权的价值。

# 6.奉献

![](img/eb0c4b04f4c2aeae384849858663b8d9.png)

杰西·奥里科在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

奉献意味着对某人或某事忠诚。奉献是对某件事关心到极致。

> 当你致力于某件事时，你会付出你的一切——你所有的努力、所有的时间、所有的爱。

这是一个重要的特质，尤其是在工作场所。

开发人员的职业生涯充满了障碍。几乎每天，你都会面临新的挑战和新的问题。有时你可以很容易地修复它们，有时你会卡住。如果你没有奉献精神，你不会进步，反而会沮丧和失望。

在工作场所的奉献教会了纪律、激情和时间管理，这些都是成功的关键因素。当你表现出对工作的奉献精神时，你会赢得组织的信任。有你在身边，他们会感到安全。每当他们遇到困难或挑战时，他们会知道谁能处理好。通过做更有挑战性的事情，你会不断学习新的东西，也会提高你的地位。

毫无疑问，做更具挑战性的工作，你有时会感到沮丧。但是如果你对你的目标有奉献精神或热情，在工作完成之前你不会放弃。在一天结束的时候，你会感到满足。

遵循这些生活方式技能将帮助你产生更多高质量的工作。你在工作场所会更有效率。你会受到上司和雇主的重视。这些技能会让你在工作中更快乐，并且知道自己被重视，对自己的职位感到安全。
