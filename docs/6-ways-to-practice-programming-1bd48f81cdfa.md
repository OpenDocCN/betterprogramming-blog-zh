# 向大师们学习

对于高水平的运动员来说，这是很重要的。他们明白练习他们运动中最基本的基础的重要性——尤其是那些顶尖选手。

斯蒂芬库里，NBA 历史上最好的射手，每天练习 500 次投篮。历史上最好的拳击手之一穆罕默德·阿里专门花了 20 分钟来跳绳。有史以来最好的网球运动员罗杰·费德勒在蹦床上练习平衡截击。

这样的例子不胜枚举。如果你喜欢运动，并且在任何一天见过运动员，你可能会惊讶于他们做运动是如此的容易。好像只是走路或者呼吸，一些琐碎的事情。

# 编程；编排

我们作为程序员也应该这样做。不管你是昨天才开始学 Python 的，还是有十几年经验的全栈开发者:天天练。

我不是说要整天练习。你有工作要做，有生活要过。休息和平衡的生活和练习一样重要。但是在你的日常生活中保留一段时间来练习。

可以是两个小时，也可以是 30 分钟。你花在做这件事上的时间将取决于你的经验和你还有多少要学。

数据结构、基本算法、安装过程、环境设置、设计——作为一名开发人员，你在一生中会看到很多执行工作所依赖的概念。

我看到许多新开发人员抱怨编程有多难。但是你猜怎么着？说话也是。然而，你每天都在说话。你可能不记得了，但是学习说话对你来说也很困难。你花了几年时间来掌握你的语言，学习它的语法和规则。你每天通过和朋友交谈、阅读、写作和听别人说话来练习。

编程语言也不例外。你必须坚持读、写、说。练习最基本的语法规则和用法。听别人怎么说话。

# 练习！练习！

在软件方面，我们经常在工作和生产中进行实践。我学了一些语言，因为一个项目需要它们，没有时间去上课或练习。

正因为如此，我犯了很多错误。我相信你们很多人也是如此。

练习课不应该是这样的。我们必须把我们的练习和我们的职业职责分开，把训练和真正的比赛分开。

练习课应该是一段时间，在这段时间里，你可以尝试任何你想尝试的次数，没有项目截止日期的压力，也没有失败的恐惧。事实上，大多数时候你会失败。这是你正在学习的标志。

您不能在项目环境中练习。压力很大，还有截止日期、要做的功能、要修复的 bug。

即使有了多年的经验，你还能在练习中做些什么呢？经过十年的经验，我想和你分享一些我自己做的事情。

## 代码挑战

正如我在本文中[所说，代码挑战是练习和提高编程技能的好方法。](https://medium.com/better-programming/6-things-you-can-do-to-improve-your-programming-skills-7fb0773119)

它们做起来很有趣，并且提供了从基础到高级的广泛主题。

不断重复，即使你已经解决了。即使你认为你掌握了一个主题，稍后再重温它，只是为了检查你是否还记得如何做，或者你是否能提出另一个解决方案。你会惊讶于一个问题可以有多少种不同的解决方法。

有一些提供各种挑战的好网站，比如[黑客排名](https://www.hackerrank.com/dashboard)和 [CodinGame](https://www.codingame.com/home) 。

## 学习一门新语言

如果你已经知道一门编程语言，试着学习一门新的。你会从这个过程中学到很多。

每种编程语言都试图以其特定的方式解决问题。语言有自己的语法和存在的理由。他们实现了不同的编程范式。他们解决问题的方法不同。

这就像说外语一样。你学习他们的文化，母语人士如何思考，他们如何表达他们的感情。

用编程语言是一样的。

你知道 JavaScript 吗？学 C 你懂 C？学围棋。你知道围棋吗？学习 Python。对于在实际系统中使用的语言，有很好的选择，可以解决实际问题。他们都有他们的价值。

## 性能测试

您可能已经看过一些性能测试，或者一个库如何优于其他库，或者一个语言框架如何提高系统的性能。

但是你自己检查过吗？你测试了吗？

是一种很好的练习方式。你将继续练习你最喜欢的框架或库，甚至可以学习如何以其他方式使用它。

你可以学习其他的方法来改进你的代码，其他的算法。或者说[时空复杂度](https://www.hackerearth.com/practice/basic-programming/complexity-analysis/time-and-space-complexity/tutorial/)到底是怎么运作的。

## 调试您的代码

开始调试您的代码。我是说，真正的调试。不是用`printf`或者`console`，而是真正的调试工具。

每种主要语言都有调试工具。了解如何使用它，并开始调试您的代码。

在每次循环中检查变量值。在执行过程中检查程序的总大小。检查是否所有的行都被执行了，或者什么时候没有被执行。

这样做，您还将学习如何在项目或 bug 修复期间调试代码。这是你肯定需要的技能。

## 向他人学习

就像你通过阅读来学习更多关于你的语言的知识，以及听别人如何说或发音一样，向其他程序员学习也是很重要的。

问其他开发者的问题。阅读关于如何解决问题或创造事物的教程。做一些关于一个新话题或框架的课程。

你会惊讶于别人怎么会有你从未想到过的解决方案，或者教你一种新的算法。

它可能是你已经知道的一个话题或者你已经熟悉的一种语言。即便如此，编程每天都在更新。总是有学习新事物的空间。

## 更改

尝试对项目进行更改，看看效果如何。

检查它是否提高了性能或代码可读性。检查什么可行，什么不可行，一段代码是否重要。

不要害怕破坏你的程序。它将教你如何调试它，以及如何在需要时修复它。您将会明白什么对您的程序顺利运行至关重要。

不过，保持较小的变化。您需要了解发生了什么，是什么导致您的程序中断。

# 最后的想法

学习如何编程可能很难。掌握它就更难了。

和其他技能一样，你必须不断练习，这样你才不会忘记和提高。所有高水平的运动员和工人都这么做。

记得把练习和真实的游戏分开。作为一名程序员，你需要他们两个一起成长。他们在你的生活中都有其重要性。

不管你在任何领域有多有经验，实践仍然很重要。

你今天练习了吗？
