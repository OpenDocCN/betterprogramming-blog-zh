没有必要讨论技术是否影响世界。确实如此。然而，它如何影响世界的问题从未失去其重要性。像海德格尔这样有影响力的哲学家广泛地研究技术，因为对他们来说，技术是理解世界的关键。我相信这有很多道理。

但是，如果没有当今所要求的高度技术性的专业知识和资格，一个人如何能够集中和有目的地应对技术进步呢？对我来说，关键必须是专家和世界大部分人口之间富有成效的对话。我们都应该有机会参与并理解新技术。这种有意义的参与需要积极的提问和回答。为了促进这种讨论，没有技术背景的人必须克服他们对高度复杂的技术问题的抑制。

也许我们可以从你那里得到一些帮助？

# 开始使用我们理解的语言

使用这种特定编程语言的术语来解释程序能做什么要容易得多。使用日常语言走出这个区域会使在程序设置中传达细节变得更加困难。更糟糕的是，你怎么能把你投入了这么多时间和精力的东西分解给一个对框架一无所知的人。何必呢，对吧？

让我给你三个你应该花时间和精力的理由:

首先，我相信你会更快地发现设置中的小故障。这是为什么呢？被认为是愚蠢的问题让你重新思考设计本身——每次你不得不调整你的语言，你将不得不找出一种新的解释方式，这样做，你将重新思考设计。

第三，你的论点变得尖锐，在与其他专业人士的讨论中，你会更加激烈地为自己的项目辩护。为什么？嗯，你已经考虑了代码行的每个方面。它们是有意义的，而且你知道，在任何人赞扬或批评你的工作之前，你已经引出了赞成和反对的意见。

# 告诉我们更多，详细阐述，更详细地解释

是的，一遍又一遍地重复同样的事情是非常令人讨厌的。但对我们来说，这听起来可能根本不是同一个原理。相信我。我花了一段时间才理解比例积分微分(PID)控制器的任务。不，我碰巧不知道这些数学术语。你可以这样解释，比如 PIDs 比较设定值和测量输出，看它们与测量值相差多远。但最好是这样解释事情。

对你来说，一个特定的数学原理是如何工作的或者编程语言是如何构造的是显而易见的。这不是给外人看的。我发现有更多的背景和详细的阐述或者一个疯狂有趣的事实是非常重要的(你打赌，我们肯定会记得这个疯狂有趣的事实——其余的，可能没有那么多，抱歉！).对你来说，他们可能看起来无足轻重，但对局外人来说，这给了他们一个想法，一些他们可以联系起来的东西。要理解 PID，你必须掌握反馈回路的机制。非常抱歉，但是没有人每天使用比例-积分-微分控制器这个术语。而且我也不记得上一次在不经意的对话中说导数是什么时候了。老实说，你能吗？你可能在工作的时候说过。我们根本没用过。*一点都不。*

因为技术世界非常复杂，所以我可以在这一部分进一步阐述更多的例子。然而，我坚信，在向程序员提问之前，我们必须首先了解自己，并对我们需要知道的事情有所了解。我们真的准备好提问了吗？我们的任务是尝试并理解解决具体项目或问题的工程方法。他们可以给我们带路，但我们必须愿意走。

# 倾听我们的恐惧

当你对技术的可能性了解得更多，对我们的未来有了更清晰的概念时，不要忘记倾听我们的恐惧。如果一个社会仅仅因为我们不知道我们在和一个机器人还是一个真人说话而停下来打电话，会发生什么？让你的机器人给你最喜欢的餐馆打电话订座是不是很棒？技术人员将它作为一项节省时间的发明出售，但真的是这样吗？是的，我可能不得不从我的时间表中腾出五分钟来打电话，但另一方面，如果我不这样做，我可能会失去与我最喜欢的服务员在一起的五分钟(我们最喜欢的餐厅之所以是我们最喜欢的，不仅仅是因为食物，对吗？).现在，我可以看到这可以派上用场的情况。我能把打给保险公司的电话外包给机器人吗？

既然我刚刚破坏了我的观点，那我再举一个例子:面部识别。当他们从社交媒体上撤下我的照片时，没有人问我——告诉我，我怎么知道我在社交媒体上的照片最终会出现在数据库中，用于面部识别目的？詹姆斯·奥利弗最近做了一个关于它的节目。他向更广泛的受众解释了这些有缺陷的系统背后的机制。他这么做是因为这令人担忧。

最让我困扰的就是这个。为什么有人认为这种技术修复是必要的？普林斯顿大学的 Ruha Benjamin 博士总结道:“解决社会问题的技术手段不仅仅是关于技术对社会的影响，也是关于规范和价值观如何塑造了最初认为必要的工具。”

意思是，我们在一起。

# 我们需要两个世界

也许我的三个愿望涵盖了我所说的我们在一起的核心意思:说共同的语言，指出背景，认真对待警告信号和防御态度并不矛盾，而是在形成，巩固或推翻社会规范的背景下。

尤其是在今天，民间社会再也不能将部分人口排除在这些讨论之外。正如每一次剧变一样，它需要提出问题的勇气，以目标导向的方式回答这些问题的能力，以及达成有利于未来的妥协的共同意愿。
