# 1.vue-lazy 负载

Vue-Lazyload 是一个很棒的库，不仅可以延迟加载图像，还可以延迟加载组件。

延迟加载是一种编程实践，在这种实践中，只有在第一次需要时才加载或初始化一个对象。在 web 开发中，延迟加载通常用于在图像进入视口之前加载图像。

这可能会大大提高性能，尤其是在应用程序中有大量图像的情况下。

Vue-Lazyload 库还能够使用惰性容器来惰性加载组件，这使得它能够在一个容器中加载一组图像。

在 GitHub 上有超过 5k 颗星，Vue-Lazyload 是一个非常受欢迎的延迟加载库。

# 2.真空头

vue-head 库使得操作 head 标签中的信息变得轻而易举。

例如，通过向组件添加一个`head` 对象，您可以更改页面的标题。这个库在添加元标签来优化搜索引擎(SEO)应用程序时也非常有用。

vue-head 库在 GitHub 上有差不多 1k 颗星，已经存在好几年了。

# 3.BootstrapVue

[BootstrapVue](https://github.com/bootstrap-vue/bootstrap-vue) 库正如其名称所描述的:用 Vue 构建的引导组件。Bootstrap 是一个用于 HTML、CSS 和 JS 开发的开源工具包。这是世界上最流行的前端框架。

BootstrapVue 的伟大之处在于，它为 Vue.js 提供了 Bootstrap v4.3 组件和网格系统的最全面的实现之一。这包括广泛和自动化的 WAI-ARIA 可访问性标记。

这个库非常适合构建响应迅速、移动优先的应用程序，这就是为什么它是 Vue.js 应用程序 UI 基础的绝佳选择。

# 4.vue-多选

如果你需要比 HTML 标准`select`更多的东西，那么 [Vue-multiselect](https://vue-multiselect.js.org/) 库是一个不错的选择。

除了传统的 single `select`，这个库还必须提供多重选择、异步选择、标记、在选项中搜索的能力等等。

这个库允许定制配置，这使得它具有高度的可定制性。如果您需要在应用程序中使用某种选择，我强烈推荐这个库。

这个库在 GitHub 上有近 5000 颗星星，根据他们的网站，可能是 Vue.js 最完整的选择解决方案，没有 [jQuery](https://jquery.com/) 。

# 5.元素 UI

另一个流行的 UI 工具包是[元素 UI](https://github.com/ElemeFE/element) 。GitHub 上有超过 40k 颗星，Element UI toolkit 绝对要在这个列表上。它是 Vue.js 生态系统中最受欢迎的项目之一，该生态系统提供了各种各样的 UI 组件。

它有很好的浏览器支持:现代浏览器和 Internet Explorer 10+都支持。

这个库有太多的组件，无法一一列出，所以我建议您查看元素 UI 本身。

# 6.vuelidate

[vuelidate](https://github.com/vuelidate/vuelidate) 是一个为 Vue.js 提供简单、轻量级、基于模型的验证的库，虽然在 Vue 生态系统中存在非常相似的库，但是这些库和 vuelidate 之间有很大的区别。

大多数其他库要求您在模板中编写验证规则。这在你的模板中引入了很多噪声。人们也可能认为模板不是声明应用程序逻辑的最佳地方。vuelidate 中的验证与模板完全分离。

如果您遇到了在客户端验证表单的问题，我强烈建议您去看看 vuelidate。

# 7.Vue。可拖动的

当您的应用程序需要拖放功能时，除了 [Vue，您别无选择。可拖动](https://github.com/SortableJS/Vue.Draggable)。

这个库基于 [Sortable.js](https://github.com/RubaXa/Sortable) ，它提供了所有的特性。Vue。Draggable 帮助您构建复杂的拖放界面，并专注于获得正确的拖放交互，将其视觉方面留给您。

这个库也支持触摸设备，并且不依赖 jQuery。

# 8.阿波罗万岁

GraphQL 时下大热。周围议论纷纷。如果您正在使用 Vue.js，并且也想加入 GraphQL 的行列，那么这是一个好消息。

[Vue Apollo](https://github.com/vuejs/vue-apollo) 库让 Vue.js 和 GraphQL/ [Apollo](https://www.apollographql.com/docs/intro/platform/) 一起使用变得流畅愉快。

如果您已经熟悉 GraphQL 和 Apollo，那么让它们运行起来应该不会太难。如果您想在应用程序中使用 GraphQL，这个库是必备的。

# 9.Vue-i18n

如果你需要一个多语言应用，那么你肯定需要 [Vue-18n 库](https://github.com/kazupon/vue-i18n)。

Vue-i18next 是一个强大的 Vue.js 国际化插件，基于 [i18next](https://www.i18next.com/overview/supported-frameworks) 。它为您提供了一个完整的解决方案，将您的产品从 web 本地化到移动和桌面。

i18next 库具有翻译功能，如复数、格式化和插值。最重要的是，这个库是面向组件的，这意味着您可以在单个文件组件上管理本地消息。
