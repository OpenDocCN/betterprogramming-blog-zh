# 选择一门学科并坚持下去

我选择 web 前端工程作为我的第一个学科，因为我喜欢我所做的事情如何虚拟地反映出来。所以，我选择 JavaScript 作为我的主要语言，直到现在。

去年我花了大约三个月的时间参加数据科学训练营。我学习了 R 和一些 Python，这很有趣，但它们并没有很好地留在我的脑海中，因为我知道我对数据科学没有热情。我参加了新兵训练营，所以如果我找到一份数据科学家的工作，我会得到更高的薪水。我也在我家的公司做过一年左右的项目经理助理，和软件开发无关。

最后我更倾向于前端开发，于是我又回到了我当初做前端开发的道路上。长话短说，我花了太多时间在我热爱的领域上。

选择你的第一门学科，选择你的第一门语言。坚持他们。掌握它们。

# 了解软件工程原理

你应该学习一些工程原理，你可以用它们来指导你写代码。以下是一些可能帮助你开始的原则:

*   干(不重复)
*   啊哈(避免草率的抽象)
*   单一责任原则

此外，了解一些编程风格的基本知识也是值得的，比如面向对象编程和函数式编程。

# 了解 O 符号

简而言之， [O 符号](https://en.wikipedia.org/wiki/Big_O_notation)表示一个函数的运行时完成。在实践中，这个概念会给你一个不同的角度来看待你的代码性能。

# 强迫自己阅读文档

对于初学者来说，文档可能会令人害怕和困惑。请强迫自己阅读文档，即使你很难理解它。耐心点。如果您正在自学 React，请阅读文档。如果你想学习 Redux，请阅读文档。您可以找到大量的视频或文章，但是请相信我，关于库或框架的最佳知识来源是文档。

# 在 Github 上查找代码示例

Github 是惊人代码的伟大来源。我建议看一看您使用的库/框架的源代码。你会惊讶于有经验的人是如何编写代码和解决问题的。

# 意识到良好的发展习惯

软件开发不仅仅是写代码。实际上，您将与其他软件开发人员合作，并且可能还必须处理各种项目管理工具和实践。每个公司可能有不同的实践和习惯，但是我见过有很好的开发习惯的公司和没有的公司。

在一家公司工作时，你应该注意以下几点:

*   良好的代码审查习惯
*   对代码质量的关注
*   良好的项目管理实践

如果你工作的公司(或者更确切地说是公司内部的人)具备这些品质，那你就来对地方了。相信我，作为初学者，你会从良好的代码审查中受益匪浅。此外，采用良好的项目管理实践过程的公司对你来说也是很好的，因为它们会迫使你变得沟通、透明、开放和协作。

公司内部的实践对新手来说很重要(我不是说如果你找不到理想的公司，你就注定要失败)，因为你的环境会对你早期的个人发展产生重大影响，无论是在技术方面还是非技术方面。

# 谦虚点，问些愚蠢的问题

我相信实际上没有愚蠢的问题，除了帕特里克·斯达的问题。没有对他不尊重；他绝对很搞笑，但大部分时间都是哑巴。

我不能否认，有时候我害怕问问题，因为我不想让人们认为我无能。更糟糕的是，我曾经拒绝提问，因为我认为我比对方优秀，所以问我不懂的事情没有意义。

为了能够问这些“愚蠢”的问题，你需要一定程度的谦逊和接受。

不懂事是绝对正常的。所以，继续提问吧。你可能会惊讶于你能学到多少东西。

# 结论

每个人都是从初学者开始。完全没问题。重要的是，你开始做你喜欢并充满激情的事情。请帮我一个忙，坚持下去！

快乐学习！
