以下六件事帮助我成为一名全职软件工程师。

# 1.弄清楚基本情况

甚至在你开始实习之前，研究你将要工作的技术，并开始学习它。了解你将要从事的技术的基础。

对 OOP 原则有很好的了解，比如封装、抽象、继承和多态。这些概念在所有编程语言中保持不变。提前学习代码编辑器的基础知识也会给你一个良好的开端。

在这个阶段你可以跳过学习框架，但是如果你的基础已经很清楚了，在实习期掌握它应该会很容易。一旦你掌握了基本知识，你就能更清楚地理解这个问题。这也会让你的资深同事更有信心把复杂的问题分配给你，而不是自己解决。

# **2。随身携带笔记本**

在你实习的最初阶段和入职过程中，会有很多信息抛给你。你很有可能会错过重要的事情。这时候记下事情会有所帮助。

写下来也有助于将一个复杂的问题分解成更小的任务，就像你要解决的问题的小清单，否则你可能会忘记。这也有助于你和其他同事一起集思广益，验证你解决问题的方法。

# **3。提问**

![](img/6783f561cbb73fe05f6a70885402ae45.png)

乔恩·泰森在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

作为一名实习生或新人，你应该有很多关于你工作场所的产品和流程的有趣问题。不要害怕或羞于问这些问题。

要明白工作需要团队合作，而不是个人努力。你可能一整天都被困在同一个问题上——如果你只是简单地寻求帮助，五分钟内就能完成。

也就是说，不要问那些你很容易在网上找到答案的问题。框架和 StackOverflow 的文档在这里应该会派上用场。

**4。读取、写入和重构**

不像在你的学校或训练营，你实习的大部分时间将花在阅读别人的代码上。即使你被分配了一个全新的模块或功能，也要尽可能多的阅读。这将确保无论何时你做出贡献，你都坚持组织或团队的标准。

一旦你熟悉了你的团队所遵循的编码实践，你就可以开始做出贡献了。编写干净、可维护、可扩展的代码。不要忘记测试你的代码。遵循您的团队制定的最佳实践，但是您也可以添加自己的最佳实践！

有时候在制作上，你必须快速做事。一旦你的代码被部署，你有一些空闲时间，重新审视你的代码，并尝试提高其效率。

# **5。主动提供帮助**

![](img/eac4c725c77c48436b5e20bff103cf57.png)

由[安妮·斯普拉特](https://unsplash.com/@anniespratt?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

一旦你开始按时完成你的职责，就要多做一点。帮助你的队友和其他实习生。通常同一批实习生可能会面临和你类似的问题。帮助他们，互相分享你的学习成果。

了解如何将您的代码部署到生产环境中的过程，并帮助部署团队进行部署。

# **6。主动沟通**

随着你实习的进展，经常会有这样的时候，你在紧张的期限内工作，而你低估了某个特定的特性。如果您正在开发的功能比预计的时间要长，请及时与您的报告经理沟通。这也有助于您的经理与外部利益相关者沟通。

提前做好休假计划。尽早告知您的可用性有助于您的经理为您规划未来的交付成果。

明白你的沟通技巧和你的技术知识一样重要。
