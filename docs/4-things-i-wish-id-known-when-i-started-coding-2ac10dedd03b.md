## 初学编程的人可能对自己太苛刻了，这篇文章提供了一些安慰

![](img/537db824a063b2a9a98a4d29c1a159bf.png)

在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的 [ThisisEngineering RAEng](https://unsplash.com/@thisisengineering?utm_source=medium&utm_medium=referral)

15 年来，我一直从事专业水平的编码工作，一生都在涉猎代码和技术。在这篇文章中，我将分享一些我希望在开始编码时就知道的事情。

# 1.最佳实践通常是见仁见智的事情

程序员通常对正确的做事方式持有强烈的观点。当我们接受一个人的“最佳实践建议”作为不容置疑的真理时，这可能是有问题的。

当我刚开始写代码时，我对网上表达的这些教条式的观点非常敏感。这导致了一种焦虑感，我必须不断地做*一些*错误的事情。

我希望有人告诉我，所有这些意见都需要根据当前项目的需求来考虑，并且我们通常需要相信自己的判断。

# 2.寻求帮助是可以的

作为一名初学编程的人，我浪费了大量的时间来自己解决问题，而我应该向别人寻求帮助。

从这个角度来看，我们生活在软件开发的黄金时代，我们可以在网上发布一个问题，有时在几分钟内就有许多高质量的回答和讨论。

这并不是说我从来没有问过问题；我只是没有经常问问题或者问得不够快。

现在，没有借口了。注册[栈溢出](https://stackoverflow.com)，如果你陷入困境，寻求帮助。

# 3.有人会付钱让你做你喜欢的事

从我真正精通编程到我开始获得报酬之间有三年的时间。

这在一定程度上是因为缺乏信心，但也是因为我没有意识到有人会付钱让我去做一些工作。

我希望我已经理解了对拥有广泛技能和经验的程序员的巨大需求。

经常被有编程技能的非程序员同事问到“我如何成为一名程序员？”在很多情况下，他们已经有了技能，只需要开始申请角色。

# 4.采取快速简单的选择是可以的

我们在一个奖励专业技能的领域工作。克服高难度技术挑战的能力令人钦佩。

这当然很好，但是它也可能导致一种感觉，如果你偷工减料或者以简单的方式做事，你就不是一个真正的程序员。当我开始的时候，我经常觉得我必须用困难的方式来证明自己。

我希望我知道有时候一起编写代码来更快地构建一些东西是可以的，尤其是在项目的早期阶段。

如果你不得不偷工减料来按时发货，不要对自己太苛刻。你总是可以在以后整理和重构。

我希望这一见解有用。如果任何人有任何他们希望作为一个初级开发人员知道的智慧，我很乐意在评论中阅读它们。
