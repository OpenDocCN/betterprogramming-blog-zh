## 消除关于如何成为一名科技博客作者的常见误解

我于 2020 年 12 月开始在 Medium 上写作，但我进军博客可能需要数年时间。很长一段时间，我认为我会喜欢写作和与他人分享内容。然而，我花了这么长时间才进入其中，因为我想出了一个又一个理由，为什么我不应该试一试。

在我开始写作的过去一年里，我有机会和我周围的技术人员谈论写作，他们中的许多人表达了我在开始写作之前的类似疑虑。今天，让我们来解决一些你可能也有过的担忧，如果你正在考虑进入科技博客的话！

# “我对技术的了解还不足以写这方面的东西。”

在一个冒名顶替综合症猖獗的领域，觉得自己不合格是成功的最大障碍之一。每个人在某种程度上都有这种感觉，把自己放在那里会很可怕。问题是，无论你在科技职业生涯的哪个阶段，你都是某方面的主题专家；你只需要利用它。

## 专注于你熟悉的领域

我喜欢写我在科技领域的个人经历，以及针对早期技术人员的职业建议，因为这是我所知道的。并不是每个科技作家都需要关注某个细分技术领域的令人疲惫的技术细节。如果你的经验允许你写那些东西，那很好，但是，要坚持你熟悉的东西。在一个问题上，你不必是唯一的声音；不管你的背景如何，你仍然可以提供独特的视角。

## 不要害怕犯错

如果你决定写更多的技术方面的东西，最可怕的可能性之一就是有人可能会指出你写了一些糟糕的代码，不是吗？听着，这可能是一个可怕的想法，但是犯一个错误并得到建设性的反馈可能是一个很好的成长机会。知道自己有错比继续沿着那条路走下去要好得多。归根结底，发布技术教程不是脑外科手术，少数错误不会结束你的技术生涯。

# “作为一名技术人员，写作对我没有任何帮助。”

在像技术这样竞争激烈的领域，当你可以通过项目和其他技术追求来磨练你的技能时，你为什么要花时间写作呢？

在我大学一年级的时候，我完全不知道我想在技术领域追求什么，我抓住了很多机会与各种行业的专业人士谈论他们是如何做出职业选择的。老实说，许多人讲述了类似的故事，并给出了类似的建议，要在大学里花时间探索不同的道路。那一年，我有机会会见了一家大型科技公司的开发者支持者，他们给了我一条特别的建议，这条建议在我的记忆中非常突出:“创建一个科技博客。”

写作提高了你作为技术专家的可信度，因为其他人可以找到你的一整部作品，讲述你在技术方面的经历。你的博客可以成为你个人品牌的一个重要组成部分，它可以让你建立一个网络，推进你的职业生涯。它也可以作为深入挖掘技术主题的灵感，这样你就可以确保你总是在做出高质量的工作。

# “我太忙了。”

我一直主张，你可以决定什么时候你的工作已经满了，不能再增加你的负担，所以如果你真的觉得写博客会让你不知所措，我会告诉你在这个问题上要听从你的直觉。然而，我也要说写博客并不像有些人说的那样耗时。

特别是在 Medium 上，看到所有关于你需要每天写作或每周发表多次才能成功的建议可能会令人生畏，但这是因为许多文章是为希望写作成为他们主要工作或主要副业的目标受众写的。没人说你必须一直发表文章才能称自己为博主。

我给自己定了一个目标，每周发布一次，因为这让我保持一致，而且对我的时间表来说也是合理的。我每天写博客的目标只有五分钟；我经常这样做，但我知道在忙碌的日子里，我总能找到五分钟的时间写下一些话。我见过有些人只有在灵感来临时才会发表一篇文章，这也是开始写博客的有效方式。你没必要为了适应写博客的时间表而弯腰。你可以根据自己的需要定制你的博客习惯。

# “没人会看我写的东西。”

嗯，我不是一个出色的营销人员，所以我没有快速建立大量受众的方法，但是有很多比我更有资格的人写过这方面的文章。但是我知道，如果你从来不把你写的东西拿出来给别人看，几乎没有人会去读。

没有人会在一夜之间拥有追随者，所以很少有人会看到你的前几篇博客。在我写这篇文章的时候，我的博客的浏览量从 11 到近 9000 不等。分享你的作品是所有人看到它的唯一方式。此外，不是每个博客都会大获成功，但是浏览量少并不会降低博客的重要性。你仍然可以从阅读量较少的博客中获得一些很棒的东西，它可能会对少数阅读它的人说话。

# “我写得不好。”

我总觉得有一种普遍的误解，认为某些艺术追求，如视觉艺术、唱歌和写作，完全依赖于天赋。这给人一种感觉，如果你现在不擅长写作，你永远也不会擅长。

事实上，在提高方面，写作和编程并没有太大的不同。写作和编程一样，需要练习才能变得更好。你肯定不是一个好的程序员，那么为什么你会认为你需要在第一天成为一个好的作家呢？

我发表了很多我不喜欢的博客，但我也将写作范围缩小到我真正喜欢写的话题，这有助于我发表更多的个人博客。你写得越多，你就越能找到你写作的声音。

# 最后的想法

作为一名技术人员，写作是我经历的很大一部分，我认为很多人会从博客成为他们技术旅程的一部分中受益。如果你一直想写博客，并且正在寻找一个信号，这就是了！试一试，你会发现你真的很喜欢它。

感谢阅读！
