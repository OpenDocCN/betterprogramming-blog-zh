大约两年前，我获得了软件开发专业的技术文凭(也称为大专文凭)。

我不认识任何一个毕业于计算机相关领域的人，这意味着我不知道自己到底在干什么。

现在我已经毕业进入这个领域，并在技术领域找到了一份工作，我觉得我应该让现在和未来的计算机专业学生知道一些我希望在主修软件开发之前就知道的事情。

# 1.我也许可以免费做这件事

回顾我所学的课程和学到的技能，我知道我本可以免费获得同样的教育。

考虑到学费的成本——更不用说自己在大城市的生活成本了——我不禁后悔，我本可以在父母舒适的家中免费学习我所做的一切。

由于许多软件开发人员的慷慨和独创性，谷歌和 YouTube 上可用知识的数量和质量令人震惊。毫无疑问，我可以说，你可以在自己舒适的家中，零美元地学习软件工程或计算机科学课程中的所有内容。

不过，有一个很大的警告。有两件事是你从自学课程中学不到的，而从大学证书中学不到的:

1.  雇主尊重的一张非常花哨、非常昂贵的纸。
2.  一个由同学、校友、导师和行业专业人士组成的网络可以帮助你找到工作。

是的，有很多成功的故事，一些人住在他们妈妈的地下室里自学编程，并立即被一家白金级公司聘用。

然而(这是一个很大的然而)，这些人是例外——而不是规则。科技行业的不幸之处在于，尽管它走在了时代的前面，并对雇佣没有大学文凭的人开放，但仍然有人喜欢并强调雇佣大学毕业生。这就是非常奇特、非常昂贵的纸张出现的地方。

第二件事是，那些在没有任何正规教育的情况下在顶级公司找到工作的程序员是那些刚刚从前门走进去的天才。事实是，现在许多初级职位都是通过关系网获得的。这实际上是我如何得到我的第一份技术工作的。

需要指出的关键点是，是的，我本来可以免费做这件事。然而，上大学给了我两件非常重要的事情，它们让我找到了工作。出于这个原因，虽然我后悔花了这么多钱，但我不会改变我的经历。

# 2.学习编码完全不同于你以前可能学过的任何东西

每一天你都会感觉像被迫从消防水管喝水，每次你回家过节，你都会看起来像是被夹在蜜獾和鬣狗之间的战斗中。

玩笑归玩笑，如果你以前没有涉猎过一点代码，第一次学习编码会很痛苦。

你过去所做的事情很少会与学习如何编码的过程相关。即使知道多种自然语言也不会总是有帮助(我是通过艰难的方式学到这一点的)。因此，如果你想进入一个与计算相关的领域学习如何编程，最好的经历就是拥有强烈的职业道德和坚持到底的态度。

当学习编码时，你首先要学习如何失败。编码就是失败，然后从错误中学习如何改进。这对于第一次编程的人来说可能很难理解，尤其是那些不习惯失败的人。然而，从长远来看，在这些失败后坚持不懈会让你成为一名更好的程序员。

此外，偶尔一点失败对每个人都有好处。

# 3.你所学的软技能课程和你所学的技术技能课程一样重要

我在大学学到的最重要的东西不是如何编码。

我在大学学到的最重要的东西是与代码无关的一切:表达技巧、解决问题、沟通、团队合作和项目规划。这些技能比任何技术技能都让我受益匪浅。

我对任何现在或未来的科技学生的最好建议是不要在软技能课程上吝啬。当你可以学习一些很酷的新编程语言时，这似乎是浪费时间，但相信我，软技能将使你从数百名求职者中脱颖而出。此外，软技能会让你在工作中获得晋升，并提升你在公司的管理职位。

在我两年的副学士学位课程中，大约 25%的课程是商务沟通和组织行为学，这给了我上面列出的所有技能。有趣的是，这些软技能实际上让我在技术领域找到了第一份工作。

不相信我？你所要做的就是看看一些科技招聘广告，就能意识到软技能对任何想找科技工作的人来说是多么重要。十有八九，你会发现雇主需要具备顶尖软技能的个人。

# 4.你不能保证有工作

他们在大学里告诉你的最大谎言是保证你一毕业就能找到工作。导师们喜欢描绘这样一幅画面:那些排队向你提供工作机会的雇主。

最糟糕的是，社会支持这种说法，告诉新毕业生，通过学习编程，他们可以保证找到工作。

没有什么比事实更离谱的了。

是的，理工毕业生(软件工程，软件开发，计算机科学等。)的需求量很大，并且是世界上失业率最低的国家之一。

然而，他们没有告诉你的是，尽管如此，许多人仍然很难找到工作。

这可能是由多种因素造成的。

对我来说，这可能是因为我毕业时陷入了衰退。或许是因为我没有传统的大学四年制学位。这也可能是因为我毕业的那个省正在尽一切努力把高科技带到这个省，除了实际上把高科技带到这个省。

我最终还是在技术领域找到了一份工作。然而，对于许多科技毕业生来说，情况并非如此。

这里的要点是，不要指望你毕业时能得到一份工作。相反，你需要像任何人一样努力工作才能找到第一份工作。

# 5.我希望我主修别的东西

我的秘密是:如果我的成绩好一点，我会选择软件开发以外的专业。

高中的时候，我很努力，成绩很一般。我不知道为什么，但我努力成为那种能在考试前 10 分钟浏览考试信息，然后全部掌握的学生。相反，我必须学习几个小时，只是为了拉 c，Bs，偶尔 a。

正因为如此，我的成绩达不到他们在我们省的一流大学主修任何专业的要求。

这让我别无选择。幸运的是，其中一所技术学院有一个软件开发副学士学位项目，不需要很高的 GPA。长话短说，我申请了，被录取了，我的专业是软件开发。我最终以优异的成绩从该项目毕业，老实说，鉴于我并不出色的高中经历，这是一个很好的自信提升。

不要误解我，我不后悔接受软件开发教育。这是一个工具性的专业，无论我选择什么职业，我都可以用它。这也将为我打开许多大门，也可能是我的薪水比同事高的原因。

然而，回过头来看，如果我能够主修科学、工程或兽医，我可能会更开心。

建议:花些时间决定计算机相关专业是否适合你。人们很容易被六位数的薪水、工作保障和在世界任何地方工作的承诺所动摇。虽然主修技术永远不会出错，但在申请之前做一些自我反省，找出你真正想要的是什么，也不会有什么坏处。

# 最后的想法

这些只是我希望有人在我主修软件开发之前告诉我的一些事情。虽然我不得不艰难地学习所有这些东西，但你不需要。

我总是建议人们，如果他们正在寻找工作保障，一个将永远让他们学习的职业，以及对世界做出一些真正改变的机会，他们应该主修计算机相关领域。

话虽如此，我也想让人们睁大眼睛进入这个领域。这是一个与众不同的领域，所以在冒险尝试之前，了解所有的事情是非常重要的。虽然你可能不会后悔，但你总是想跳下悬崖，并确切地知道你将进入什么。
