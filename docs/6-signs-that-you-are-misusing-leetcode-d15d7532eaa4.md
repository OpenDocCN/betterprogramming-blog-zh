# 你滥用 LeetCode 的 6 个迹象

> 原文：<https://betterprogramming.pub/6-signs-that-you-are-misusing-leetcode-d15d7532eaa4>

## 避免这些错误，提高你在编码面试中获胜的几率

![](img/7ea2e56fa1e62720863374ef4af74f31.png)

让我们面对它；每个人都害怕编码面试。通过这一轮面试的可能性取决于你在 2-3 个编码问题上的表现。

不想在面试官面前出丑，你可能已经决定求助于 LeetCode 这样的网站，像一个热爱代码的饕餮者一样沉溺于编码问题。

不幸的是，当一个人过度沉迷于这样的编程平台时，他们会冒不必要的风险，即偷工减料，而不开发在编码回合中获胜所需的技能。

以下是你在 LeetCode 上浪费时间的 6 个迹象。

# 1.你不遵守时间限制

![](img/9c438faf8fd10fa7c3b820a8e7dde8d8.png)

照片由[迈克](https://www.pexels.com/@mike-468229?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels)从[派克斯](https://www.pexels.com/photo/hourglass-on-brown-wooden-frame-1178684/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels)拍摄

我相信你听过这样一句话:学习编码的最好方法就是编码。

这同样适用于编码面试。

在这一轮中，你希望提升自己表现的唯一方法是，在你接受编码挑战时，模仿面试的场景。这首先需要给自己每个问题设定一个时间限制。

在面试中，你不会有闲适地思考一个问题的解决方案，也不会在中间休息或者直接跳到另一个问题。

给自己一段特定的时间来解决问题，在这段时间内不要偏离解决问题的方向。

例如，你可以花 20-30 分钟解决简单级别的问题，30-40 分钟解决中等级别的问题，40-50 分钟解决困难级别的问题。

通过对你的问题设定一个时间限制，你会变得更习惯于在压力下思考、计划和执行，这将提高你在实际面试中的表现。

# 2.你工作的时候不交流

在编码面试中保持沉默是一个可以理解的决定。毕竟，大多数人太习惯于闭着嘴编码，因为他们早就养成了这样做的习惯。

有些人在面试中表现类似，只在别人跟他们说话时才说话。这是一个很大的错误，很容易破坏你给面试官留下深刻印象的机会(即使你的回答很充分)。

这是因为两个原因。

首先，如果你在解决问题时逻辑错误或走进死胡同，面试官将无法提供任何指导，因为他们甚至不知道你在想什么。

其次，即使你用复杂的代码成功地回答了这个问题，但未能恰当地解释你的方法将表明你不善于向他人解释你的工作，这对大多数希望通过新的补充来加强团队的人来说是一个危险信号。

要明白，你不仅被评价为一个编码者，而且被评价为一个沟通者。

在做 LeetCode 挑战时，通过表达自己的想法来习惯发声。

当你遇到问题时，不要直接跳到编码。说出你最初的想法、方法和测试案例。当你编码时，继续评论你的工作和思考过程。

# 3.你太快放弃，奔向解决方案

![](img/8ef2720746ff698f839123cb0c0cea2b.png)

来自 [Pexels](https://www.pexels.com/photo/woman-sitting-on-chair-while-leaning-on-laptop-3791136/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels) 的 Andrea Piacquadio 的照片

像 LeetCode 这样的编码平台之所以受欢迎，主要是因为它们的实用性、易用性和方便性。他们庞大的编码问题目录和他们解释透彻的解决方案证明了付费订阅的合理性。

不幸的是，这些特征会让*对*你不利。

当你正在努力解决一个问题时，知道解决方案就在一个点击之外可以很容易地提供一个激励，减少对当前问题的投入，直接跳到解决方案。

如果你放弃得太快或者直接奔向解决方案，你就没有适当地训练自己在压力下的行为。你也没有为面试的条件做好准备。

毕竟，当你在面试中挣扎的时候，你没有机会问问题的答案或者要求暂停。

努力解决问题，只有当你没什么可补充的时候(或者时间到了)才停下来。

# 4.你不会考虑解决方案的更好的替代方案

我在 LeetCode 上得到的最满意的感觉之一是当我的提交通过了所有的测试用例。在彻底的计划和编码后解决一个问题是耗费脑力的，所以一旦你完成了挑战，你可能会忍不住拍拍自己的背，然后继续前进。

![](img/c8637fab4bb5da5bfd2229c8fcd56ae2.png)

来自 [Pexels](https://www.pexels.com/photo/man-in-black-suit-achieved-an-accomplishment-3779409/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels) 的 Andrea Piacquadio 的照片

然而，这样做将意味着你忽视考虑任何替代解决方案。

不要转移到其他问题上，考虑是否有任何方法来改进代码或任何其他有效的方法来达到解决方案。面试官经常会问候选人他们是否认为他们的代码可以改进，或者一种不同的方法是否更适合，所以要习惯访问你自己的工作。

时刻评估自己回答的时空复杂度，考虑改进的可能性。

在你考虑了这些因素后，看看网站上发布的解决方案，问问自己:

*   哪种解决方案符合您的方法？
*   哪些效率最高？
*   你的答案与最优解相比如何？

如果你发现一个解决方案比你的更有效率，那就花时间去研究它背后的逻辑和推理。

如果您理解解决方案中展示的方法，您就更有可能在不同的问题中实现它。

# 5.你不会主动考虑测试用例

一个测试用例可以决定一个成功或不成功的提交。因此，在开始编码之前，花时间考虑各种可能的测试用例是很重要的。

仅仅依靠 LeetCode 提供的测试用例可能会对你的成功不利，因为这些测试用例并不能说明所有的场景。

养成为问题提出多种可能输入的习惯，并为每一种输入做好计划。

例如，如果一个问题提供的测试用例有正整数作为输入，考虑一下代码在有 0 或负值的情况下会如何执行。

要明白，在面试中，你不会每次想到一个新的案例就提交一个新的尝试，所以一定要在一开始就明确这一步。

# 6.你没有反思自己的弱点

![](img/bd93be282fb77c0eb776c254e5dd2112.png)

当在相当长的一段时间内承担编码问题时，很容易在犯错误后变得固执和自满。

我对此感到内疚。

当我答错一个问题时，我会看到答案，并告诉自己下次我会答对的。我通常不会。

这是因为我很少思考为什么我不能在第一时间找到解决方案。有时是因为语法错误或推理中的漏洞，但其他时候是因为缺乏对重要概念或工具的理解。

为了从 LeetCode 中获得最大收益，自我反思是必不可少的。了解是什么阻碍了您找到问题的正确/最佳解决方案。

你是不是仅仅追求错误的逻辑？

是否对某个数据结构或算法缺乏足够的了解？

如果你的苦恼源于对一个概念缺乏理解，降低该主题编码问题的难度，或者刷新你对该主题的记忆，直到你有足够的信心再试一次。

# 结论

保持警惕，如果你犯了这 6 个习惯中的任何一个，就大声呼唤自己，以免养成任何坏习惯。

不要以为订阅 LeetCode 就一定会成功。Leetcode 是一个很棒的工具，但是和任何工具一样，关键在于你如何使用它。

祝你的编码面试好运！
