大多数前端和全栈开发人员讨厌写 CSS。这有很多原因:

*   他们仍然无法让 CSS 准确无误地完成他们想要的任务。
*   可能体积比较大。通常，即使是最简单的样式也需要太多的属性。
*   一些 CSS 规则并不简单(例如，伪元素需要内容属性才能出现)。
*   对于类命名约定、何时使用 id 和类、嵌套深度等，没有标准的方法。不是每个代码库都使用 BEM、SMACSS、styled-components 等。
*   CSS 在变化和发展，不同的属性变得流行，而其他的属性则逐渐消亡。
*   没有两个浏览器在如何呈现相同的 CSS 上意见一致。尽管有些人接近达成一致，但在编写 CSS 时仍然必须考虑到至少四种不同的浏览器(更不用说响应性 CSS 了)。

越来越多…

尽管如此，我喜欢玩 CSS，我真的喜欢——如果我管理代码，而不是 CSS 管理我。

在过去的几天里，我用[引导程序 4](https://getbootstrap.com/) 标记了一个登陆页面，并在途中切换到[引导程序间距实用程序方法](https://getbootstrap.com/docs/4.0/utilities/spacing/)，它允许你

*   拥有更少的 CSS 类
*   根据网格将元素相对于彼此对齐
*   使用集成或自定义网格断点
*   控制引导布局中的水平和垂直间距
*   保持代码整洁，不要过于复杂
*   保持用户界面外观一致
*   有能力改变整个应用程序的间距，只需触摸一个文件

Bootstrap 4 为边距和填充提供了内置的实用响应类，您可以在各种元素中轻松使用这些类来管理间距。
