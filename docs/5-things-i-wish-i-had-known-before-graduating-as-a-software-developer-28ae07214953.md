自从美国在 20 世纪 60 年代启动太空计划以来，科技行业从未像其他行业那样让所有人眼花缭乱。吸引了这么多希望进入这个新兴行业的人的注意力和欲望，技术保持了 21 世纪职业的声誉。

虽然它也吸引了我的注意力，但我觉得在一头扎进去之前，每个毕业生和未来的科技员工都应该知道五件重要的事情。这些事情中的一些会帮助你为你的职业生涯做更好的准备；其他的将帮助你更好地规划你的职业生涯，其余的是作为思考的食粮。

# 1.雇主寻找的大多数技术你都不会在学校里学到。

在大学里，你可能会学习使用 C 或 Java(如果你的大学有狂野的一面，可能是 C++或 C#)等基本编程语言来编码。这些都是很好的基础语言，可以帮助你巩固基本的编程概念和最佳实践。

最棒的是，Java 通常是雇主希望你了解的语言之一。

最糟糕的是，雇主希望你知道的语言/图书馆通常只有四到十种中的一种。

论点往往是编程语言容易学习；强大的软件开发技能是可以转移的。虽然我在某种程度上同意这一点，但我也承认学习新的编程语言需要时间，而雇主并不总是愿意给你时间来帮助你迅速入门。

因此，在大学期间，你应该学习雇主正在寻找的所有技术，以节省你的时间和毕业后的担忧。搜索工作页面，寻找最常出现的技术。然后，每周花几个小时用这些技术进行实验、构建项目和解决编码面试问题。

# 2.所谓的程序员“赤字”并不像人们所说的那样真实。

在获得软件开发的副学士学位后，我得意洋洋，因为教授、公司和头条新闻告诉我，通过学习编程，我可以保证得到一份工作。

想象一下，当我申请了 100 多个开发人员的职位后，却空手而归时，我有多惊讶。

我将在下一部分讨论这背后的一个主要问题，但现在，我想解释一下微软在 2012 年发布的一份报告，该报告导致世界认为 2010 年至 2020 年将缺少 120 万名开发人员。

在微软发布他们的报告的同一年，经济政策研究所(EPI)发布了一份对微软报告的反研究，指出了微软对他们声称的开发者短缺的推理中的一些主要缺陷。

EPI 的报告解释说，微软如此严重地扭曲了他们的结果，因为他们认为只有计算机科学专业的毕业生才能从事与计算机相关的工作。EPI 用职业信息网络的数据支持这一点，该数据概述了 47%的网络开发人员没有四年制学位，更不用说计算机科学学位了，41%的计算机系统分析师也没有四年制学位。因此，微软认为填补空缺所需的开发人员需要计算机科学学位是不准确和不合逻辑的。

EPI 的报告进一步解释说，查看工资趋势数据表明，如果开发人员短缺，开发人员的工资将会大幅增加——但事实并非如此。

总而言之，微软的报告试图鼓励国会提供更多的非移民客工签证，以便他们可以雇用更多的外国计算机科学毕业生来填补空缺职位，结果告诉美国更多关于微软如何致力于保持低工资和本土人才失业的信息。

根据 EPI 报告的数据推断，世界上很少有地方出现开发者短缺。虽然也有例外，一些国家可能需要引进外国人才，但总的规则是，那里有开发人员的职位，有开发人员可以填补这些职位。

实际填补这些职位的开发人员之间似乎存在脱节。

# 3.雇主仍然希望他们的开发人员拥有学士学位。

我不同意任何人，无论受教育程度如何，都很容易进入科技行业的观点。

在 Indeed 上快速搜索一下，显示出雇主对软件开发候选人的要求是 90%持有学士学位。

这如何转化为自学成才的开发人员可以进入的领域，不管他们的教育水平如何？

即使有一个大专文凭或大学证书也不足以让你迈出第一步。由于如果候选人不完全符合标准，确实不会将申请转发给雇主，所以任何没有学位的人的简历都很难放到招聘人员的办公桌上。

这样做的问题是，有成千上万的合格候选人必须做出选择，要么为了让简历通过系统而谎报学历，要么去其他地方寻找对候选人没有这么严格要求的公司。

我知道雇主们现在正努力在程序员的海洋中挑选开发人员，但这在最初的选择过程中是做不到的。在面试和技术面试过程中，从编码人员的海洋中挑选出开发人员。

拥有学位并不是一个人成为有能力的开发人员的最终标志。拥有一个学位只是向雇主表明你能够学习，并且你在生命中的四年里坚持做某事。

# 4.20 多岁的时候，概括才是王道。30 多岁，专精。

五年前，当我第一次选择大学专业时，父亲给了我一些最好的建议。

“让他们很难对你说不。”

作为一名应届毕业生，你将与数百名非常合格的候选人竞争同一份工作，有时这些候选人资历过高。

你的工作就是让雇主很难对你说不。

这意味着你需要概括。这意味着，在你 20 多岁的时候，你需要成为一个万事通，而不是一个大师。

你在一般化和特殊化的争论中的立场是个人原则问题。在科技界，这一争论非常激烈，两边的人都相信他们已经找到了成功职业的答案。

我的信念以一种时间敏感的方式落在辩论的双方。20 多岁的时候，你应该是个多面手。一旦你到了 30 岁，你应该专攻。

我并不是说成为一名多面手意味着你应该以成为一名全栈开发人员为目标。真正的全栈开发者在他们所有的技术上都是有能力的，他们是独角兽。优秀的全栈开发者其实并不存在。

不，我是说要成为一名多面手，你应该在一个特定的技术领域进行概括。如果你想成为一名前端开发人员——尽可能让自己沉浸在相关技术中——这样你就可以投入到你遇到的任何前端开发人员角色中。如果你想成为一名后端开发人员，了解后端开发的所有堆栈，并对它们了如指掌，这样你就可以为无数的公司工作，并迅速进入状态。

然后，在你毕业，获得第一年的工作经验，并作为初级开发人员崭露头角之后，你可以开始缩小你的关注范围，专攻某些技术。

例如，作为一名前端开发人员，你可以专攻 UI 或 UX 开发，或者你可能成为开发 React web 应用程序的专家。或者，作为一名数据分析师，你可以专注于成为更多的机器学习或人工智能工程师。

在职业生涯早期进行概括的好处是，你可能会更快找到工作。在你职业生涯的那个阶段，雇主寻找的是那些知道的不多也不少的人——那些可以马上进入角色，但又不会专业到拒绝这份工作的人。

一旦你在战场上呆了几年，并且正在寻找下一份工作，雇主就会寻找专家。他们需要能够胜任中级和高级职位、公司内部不同职位的人，以及有经验满足特定需求的人。

# 5.它不全是便装、办公室里的狗、午餐、无限假期和股票期权。

科技行业并非没有缺点。

尽管被誉为最具“包容性”和“新时代”的行业之一，但由于性别歧视和多样性等问题，科技行业还有很长的路要走。

办公室政治和工作不满在科技员工中也很普遍，这使得它看起来不太像它经常被描述的独角兽。

雇主提供的休闲工作服、办公室里的狗、免费午餐、无限假期和股票期权等承诺，往往掩盖了长时间工作、高压环境和团队成员之间的残酷竞争。

在上面列出的所有方面，科技行业与任何其他行业都没有什么不同，但当你被该行业不断推动的所有独特优势分散注意力时，它就不会以同样的幅度出现。

# 结论

科技行业是未来，每个人对进入这一领域的狂热痴迷证明了这一点。

然而，这五个较少被讨论的细节都是我希望在毕业进入这个行业之前能够更多意识到的。该行业的开发人员需要不断分享他们的经验，以便未来的员工可以帮助将技术空间塑造成该行业最初设定的目标:多样化、包容性和处于人类最佳创造的前沿。
