# 我学习一门新编程语言的一个很好的技巧

你准备好了吗？这很简单。你确定吗？嗯，这就是:

与其专注于学习一门新语言，不如专注于解决一个问题。

如果这对你来说很有意义，那么我想你可以停止阅读这篇文章了。但是对于你们其他人(我希望是你们所有人)，我想解释一下我的意思。

# 1.不要专注于学习一门新的语言

![](img/1ca8359eb0a3b65d1987bc586f2b43fb.png)

雅各布·萨普在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

让我们稍微分解一下。第一部分至关重要:“而不是专注于学习一门新语言。”

这一点如此重要的原因是，如果你只是想学习一门语言，你可以拿起手册或阅读该语言的文档，记住所有的语法单词，并让它“学会”。然而，这对你的帮助并没有你想象的那么大。

很可能，你学习这种编程语言是为了做一件事:*编程*。如果你所做的只是记住组成语言的单词，你就不知道如何使用它！

如果你不熟悉编程，那么你可以将完全相同的逻辑应用于口语。我已经学了很多很多年的西班牙语，但我还远未达到流利的程度。

当我在国外旅行时，我学到了更多关于说西班牙语的知识，并且有机会练习我所学到的东西。

这里有一把钥匙，你可能漏掉了。乍一看，你可能认为我自相矛盾。听起来好像我在说学习词汇很重要？

那是真的！如果我不知道一些西班牙语单词，我就会迷路了。

不过，这里还有另一个关键点。我说……*有的*……我没说*所有的* 这些话。我学到的足够应付过去了。

在 Tim Ferriss 的书《每周 4 小时工作制》中，他解释说前 300 个英语单词占了所有书面交流的 65%。今天常用的英语单词大约有 170，000 个。

这意味着如果你想有效地学习英语，你只需要记住 0.17%的词汇！

我再重复一遍:*学习英语(至少是书面形式)你只需要 0.17%的英语词汇量！*

编程也是如此！

当你开始更多地了解编程世界时，你会发现一种叫做 API 的东西。你可以把 API 想象成一个新的词汇列表。当*只有一小部分*最终被需要时，没有理由去学习所有这些词汇。

所以，总结第一点:不要担心为你的新编程语言学习整个 API。

你所需要的只是一些开始的关键字来启动这个球。事实上，你甚至根本不需要记忆任何东西。这就是编程的好处:不会说！

你可以在你的电脑上打开所有的词汇，并在需要的时候参考它们。你最终会不小心记住它们！

# 2.专注于解决问题

![](img/1632d551172b3b2b8560d8abf177cbdc.png)

照片由[Olav Ahrens rtne](https://unsplash.com/@olav_ahrens?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

这就引出了我的第二点:“专注于解决一个问题。”

这一点有点模糊，开放式。我是故意这样做的。“问题”可以是任何东西。这并不意味着你正在用编程解决世界饥饿问题。

如果有那么简单，早就完成了。或者会吗？让我来定义一下我在这里所说的“问题”是什么意思:

问题是你想通过编程构建的任何东西。

这可能是一个促进你写作的新网站，可能是给一个无人机或机器人编程，可能是你想建立一个新的食物食谱应用程序，或者可能你只是想建立一个简单的预算工具。这个清单可以一直列下去。

我的观点是:找一个你热爱的项目！

这将是你的主要动力。我不打算美化它，有时编程是一件痛苦的事。项目中最难的部分之一是开始，这是大多数人放弃的时候。

然而，一旦球开始滚动，你获得了动力，就没有什么能阻止你了！克服开始的部分会很困难，但是如果你有一个你很想建立的项目，那就没关系了！

![](img/1ebe799e6ad05920bfc337adc4323b8c.png)

伊恩·施耐德在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

好了，就是这样。那是我的小费！这就是我学习任何新语言的方法。你从你的项目想法开始，并开始研究如何建立它。

你将读到你的编程语言的不同部分以及如何构建你的项目的不同策略。很快，你将不用思考就能掌握这门语言！

一个额外的提示:如果这是你的第一门编程语言，帮你自己一个忙，不要选择一个听起来*很难的项目。如果你对它充满热情，那就去做吧！*

但是我会建议选择一些听起来有趣但听起来不太可能的事情。

我希望我的建议至少能帮助到你们中的一个人！现在，开始编程吧！
