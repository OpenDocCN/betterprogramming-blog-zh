## 这就是社区能为你提供的

这方面的结论相当一致:

**作为一名开发人员，加入社区是对你的成长和职业生涯最好的事情之一。**

开发人员(和非开发人员)社区的列表每天都在增长，如果你仔细想想，这并不奇怪。

开发人员社区在性格和个性上类似于学术界或科学界的其他团体。社区内的人们真诚地希望互相帮助和提供资源。

如果你像许多人一样(包括我)，你可能会发现很难相信像编程这样一个利润丰厚的领域会产生这么多慷慨的参与者。事实是，与其他开发人员合作有很多好处。

**以下是你应该加入开发者社区的几个理由:**

# 一路上的帮助

这一点相当明显，但也非常低调。无论你给开发者社区提供多少，它总会给你更多的回报。

*(我现在最喜欢的网络社区是*[*indie hacker*](https://www.indiehackers.com/)*)。)*

当我第一次开始对代码感兴趣时，我通过一个共同的朋友认识了我所在公司的一名开发人员。我表达了对学习代码的兴趣，他马上提供了一些建议。

他建议我尝试一些资源和学习平台。在我的整个学习过程中，我一直和他保持着联系，当我对我所学的基础知识(当时是 HTML 和 CSS)感到满意时，他提出让我接一些他的小型自由职业工作。

这是巨大的！

即使在今天，当我已经转向新技术，正在学习不同的东西时，我的朋友继续提供建议和指导。最近，我一直在学习 Laravel，并且接触了一个我一直碰到的 bug。我们接到了一个电话，他帮我修复了我遇到的问题。

在我学习代码的过程中，他给我带来了巨大的价值。我可以诚实地说，如果没有他的帮助，我不会有今天的成就。

而且那只是一个人！开发者社区充满了像我朋友一样渴望帮助和提供指导的人。

我知道，像他这样的人以及我在 IndieHackers 等在线社区中遇到的其他人，将继续成为我学习和成长为一名开发人员的帮助来源。

# 产品感

你有没有注意到，最好的产品都有“它”的因素？通常很难看出“它”是什么，但你可以肯定地说“它”使一个产品比下一个更好。

这叫产品感！它是看到、理解和创造人们会喜欢的产品和工具的能力。

*(我最喜欢开发一些产品感的地方是* [*产品猎取*](https://www.producthunt.com/) *。这是一个快速查看目前哪些产品受到关注的地方。)*

当你开始和其他开发者接触时，你经常会看到他们在做什么。在这个过程中，你会看到一些非常好的产品和一些非常差的产品。一定要注意这一点。看看伟大的产品，记下是什么让它们伟大。对不太好的产品也是如此。尽量避免他们正在犯的错误。

你在开发者社区中参与的越多，你就能越快地开发出产品意识。

你会发现你做的东西看起来和感觉起来会更好。他们会有更好的设计，更好的用户体验，更好的系统来支撑他们。

产品感的美妙之处在于，它是而且应该一直在进化。文化在改变，技术在进步，人们对不同的事物感兴趣。只要你和一个社区保持联系，你总是能够看到人们在建设和消费什么。

# 比平时更早开始的信心

这个很大。

技术和产品领域的一个大概念是 MVP(最小可行产品)。这本质上是你的产品最基本、最精简的版本，也是大多数成功的创始人会告诉你的开始。

新创始人面临的最常见的障碍是，他们不知道什么时候他们已经建立了足够的东西来称之为 MVP。大多数时候，在最终发布之前，他们的构建远远超出了他们的 MVP。这显然把他们推回到他们本可以为客户服务的时候。

人在学习编程的时候也是如此。当我第一次开始学习代码时，我经常发现自己被外面的大量信息淹没了，总是觉得在我真正开始构建东西之前，我需要学习更多的东西。

加入一个社区可以帮助你避开这个障碍。你不能低估与比你更远的人交流的价值。

当我从其他开发人员那里得到鼓励时，我开始比以前更快地构建项目。事实上，我发现这有助于加快我的学习过程。

现在，你可能知道的比你想象的多，你也比你想象的更有能力。

# 你原本没有的机会

如果你还不知道，市场是建立在关系之上的。

没错，你要有知识，要有能力，但俗话说的好，“重要的不是你知道什么，而是你认识谁。”

无论组织或项目有多大，人是推动事情向前发展的力量。人是对组织做出决策的实体。人们让事物变得独特和有创意。人是优秀产品和组织与伟大产品和组织的区别。

人们与你在职业生涯中获得或错过的机会有着千丝万缕的联系。这就是为什么你必须和你周围的人建立联系。

就像这个等式一样简单:

**社区=机会**

到目前为止，我能够涉足的机会来自于我与同龄人的关系。这些关系不仅仅是为了机会。你只会从真诚和相互投机的关系中获益。

我相信，当你在职业生涯中学习和成长时，加入开发人员社区将对你有益。我鼓励你投入到社区中，无论是通过与开发伙伴的对话，还是潜伏下来，从你周围发生的对话中学习。
