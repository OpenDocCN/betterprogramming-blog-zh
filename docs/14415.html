成为全栈玩家是我最佳职业选择的5个原因

原文：https://betterprogramming.pub/5-reasons-why-being-full-stack-was-my-best-career-choice-5606ce1a74b1?source=collection_archive---------4-----------------------#2022-12-12

根据我在几家初创公司工作的经验

作为一名全栈工程师，典型的一天就是盯着装满React、Ruby on Rails和大量遗留代码(如十年前编写的AngularJS)的多个代码编辑器。从设计新的API和数据模型到构建丰富的用户界面，我学到了构建端到端功能所需的所有要素。

技术工作只是故事的一半。我工作的一个关键部分是需要与其他团队建立关系，以确保我编写的代码符合他们系统的标准。

对我来说，这种类型的工作既令人兴奋又让人不知所措。我喜欢广泛的职责范围和它提供的许多学习机会。

这里有五个理由说明为什么“全栈”是值得努力的，并且对我的个人和职业发展是非常宝贵的。

1.我移动得更快

如果缺乏对组织内的系统和工具的更广泛的理解，即使一个完整的工程师团队也可能比一个完整的工程师移动得更慢。

发生这种情况的原因如下:

  • 零散的知识——发现和撰写技术规范可能需要更长时间，因为需要多名工程师参与。
  • 更多会议——在开发开始之前，经理可能需要召开多次会议来协调工程师。
  • 受阻工程师——通常，一名工程师可能会无所事事，或将注意力转移到另一个领域，等待另一名工程师完成他们的工作。

拥有全栈功能使我能够避开这些低效问题，快速行动。

例如，在一家中型创业公司学习了技术堆栈后，我构建了一个端到端的功能，需要接触五个不同的代码库，前端+后端，使用两种不同的编程语言，几个月后就开始产生收入。

这引起了高管们的注意，他们惊讶于我能独立完成这个产品。根据他们的经验，构建类似的东西需要一个完整的团队，需要更多的协调，并且需要更长的时间。

2.我找到了更多解决问题的方法

全栈意味着鸟瞰各种系统如何协同工作。这种视角经常让我找到别人看不到的答案和机会。

我这样做的一些方法包括:

  • 复兴一个停滞的项目——我从不同的角度帮助失败的项目重新启动。有没有更简单的前端代替后端的解决方案？用React编写或者作为一个简单的Rails视图会更快吗？我们能在不同的代码库中实现它吗？
  • 在团队之间交叉传播想法——在代码库A中工作时，我可能会找到我认为对代码库b有用的模式。这样，我将充当团队之间的桥梁，并倡导可扩展的实践。

3.我变成了一个强有力的沟通者

根据我的经验，找到并与领域专家建立关系是快速学习并为堆栈的每个部分做出贡献的关键。

为此，我学会了与不同的团队沟通并建立良好的关系。

这意味着向其他团队介绍我自己，并就如何做出最佳贡献寻求指导。通常，我会重点问这样的问题:

  • 你的代码审查过程是什么样的？
  • 你的代码库的长期目标是什么？
  • 你在哪里有没有时间解决的技术债务？

最后一个是我的最爱；我通过为不同的团队解决技术债务来建立他们之间的友好关系。这让我以后需要帮助合并公关的时候，生活变得简单多了！

4.我更了解这个行业

正如产品和营销团队通过漏斗可视化用户旅程一样，全栈工程师可以通过技术栈跟踪用户旅程。

从注册到付款的用户路径映射如下:

  • 用户登陆营销网站—盖茨比静态网站应用程序
  • 用户登录/注册— NextJS应用程序和移动应用程序
  • 创建新的用户入口Ruby on Rails中内置的GraphQL API
  • 用户提交订单—移动应用程序
  • 用户通过支付流程React计费应用程序

将用户漏斗覆盖到技术堆栈上是一种超级力量。这样做有助于我清楚地看到我需要深入栈中的哪个位置来改善用户下降。

例如，如果你问我我们如何提高注册转化率，我会建议在营销应用程序上尝试更好的CTA来路由注册，改善我们的GraphQL API的注册请求响应时间，或者简化我们的NextJS应用程序中的注册表单。

了解整个堆栈是将业务问题转化为技术解决方案的有效方法，并为构思表带来独特的视角。

5.我被要求扮演领导角色

有效地解决问题会引起人们的注意。我被要求对技术提案提出意见，指导工程师，并根据我在全栈开发方面的成功接管项目。

一些最有趣的领导机会包括:

  • 做“粘合工作”——这实际上意味着通过为工程师提供合适的资源来加速他们的工作。
  • 改善跨团队的文化——在我看来，全栈意味着在任何系统中工作，包括人事系统。我经常扮演管理角色，参加单口相声，计划会议，并提供可以在团队之间建立一致性的文档。

最后的想法

业务问题很少考虑人们喜欢使用的代码库的边界。这就是为什么我开始培养在任何系统中工作的心态和技能。

总之，这些是我看到的主要好处:

  1. 移动得更快(一旦你加速)
  2. 寻找一个问题的多重解决方案
  3. 成为伟大的沟通者
  4. 更好地了解业务
  5. 成为领导者

那么，今天你应该离开并成为一名全栈开发人员吗？看情况！

全押有很多不利之处。这很耗时，你没有领域专家的知识深度，而且你可能经常独自工作，缺乏指导。

无论你是为一家小公司工作，还是为一家大公司工作，我相信每个人都能从哪怕是一点点的完整堆栈中受益。
