## 厨师多了烧坏汤

我记得不久前读过或听过一个关于选择太多的问题的故事。

有一家公司发明了一种由蔬菜和水果组成的饮料。出于各种原因，每个人都喜欢这种饮料。有些人喜欢它，因为它味道好，有些人喜欢它，因为它会让他们的母亲摆脱整个“吃你的蔬菜”的事情。

每个人都知道这种饮料是关于什么的，公司是关于什么的，他们的销售是稳定的。

然后，事情发生了变化。

管理层决定扩大他们的饮料范围，并开始加入偏离原始健康饮料形象的口味。人们开始对那家公司感到困惑。销售受到影响。

这就把我带到了今天的主题:JS 框架的丰富性以及由此产生的问题。

起初，有 jQuery。不要误解我的意思，我明白为什么随着应用程序变得越来越大，SPAs 成为一种事物，有必要改进 jQuery。起初，有 angular js——除了 jQuery 之外，我使用的第一个 JavaSscript 框架——我对它很满意。

大约在同一时间，ReactJS 也出现了，它在某些方面比 AngularJS 更好，但一开始写起来更复杂。你也有淋巴结。JS，它提供了有史以来第一个(或者说是第一个)异步服务器端编程的可能性。

然后，防洪闸门打开了。

快进到 2021 年，你拥有的 JS(或 TS)框架的数量…老实说，有点疯狂。以下是他们中的一些人在*的名单:*

*   有角的
*   反应
*   VueJS
*   KnockoutJS
*   烬。射流研究…
*   苗条的
*   NuxtJS
*   ExtJS
*   节点。射流研究…
*   盖茨比（姓）

相信我，这份名单还在继续。

现在，有些人认为这些框架各有不同的用途。他们将它与现有的零食口味进行了比较。他们的理由当然是每个人都喜欢不同口味的小吃。

现在，我有点明白这个论点了——至少是第一部分，每个框架为不同的网站服务于不同的目的。

问题是疯狂数量的框架为开发者创造了。具体来说就是想换工作的人。假设你是一个有棱角的开发人员，想换工作，比较有意思的是在 React 或者 Vue。任何明智的开发人员都知道，虽然在语法方面会有一个学习曲线，但是 Angular、React 和 Vue 的基本原理是非常相似的。

除了，在技术面试之前，是人力资源面试。有很多公司在复选框上工作。我这么说是什么意思？嗯，他们的工作要求可能包括一定年限的 React 工作经验。如果你没有它们，那么，很不幸。

当然，有些人可能会认为开发人员需要跟上他们工作领域的最新趋势。对此我说:当然，我会学习一个新的框架。除此之外，谁能知道在一到两年的时间里哪个框架仍然是相关的呢？谁说不会出现一个新的，会被许多公司采用，你不知何故错过了学习它的起点，因为你不认为它会变得受欢迎？

你不可能学会所有的框架。老实说，从长远来看，给你的大脑注入如此多的信息和多巴胺会破坏你的心理健康。在这一点上相信我，我记得在 2019 年换了一份工作，无法适应新的工作场所，因为人们更放松，而我的工作中普遍的共识是，在每一分钟，你都需要做一些事情。

因此，我的大脑如此习惯于在三年的相同职业道德之后，由于它习惯于高水平的多巴胺，我真的不知道发生了什么，我失去了控制，直到我不得不放弃或失去理智。我最终放弃了，花了整整一年才明白为什么会这样。

这就是我的看法。我觉得这个问题没有得到足够的重视。我的意思是，在一个基于此的公司中创建工作是很容易的——只要每两年在一个新的框架中重写一些东西。但是我觉得这会对一个人的心理健康造成损害，当有人在周末决定创建一个新的 JavaScript 框架时，需要考虑到这一点。
