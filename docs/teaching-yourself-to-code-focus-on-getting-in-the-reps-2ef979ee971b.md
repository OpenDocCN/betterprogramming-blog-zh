## 如何通过更多、更高质量的练习来取得显著的进步

在我成为一名开发人员的过程中，有一条建议确实比其他任何建议都更相关、更有帮助。在我参加的编码训练营中，我的指导老师说过这样的话:“集中精力做代表。”

学习编码的真正工作并不需要理解离散数学或操作系统的内部工作原理——如果你感兴趣的话，当然要学习这些，但是没有它你也可以做很多事情！你最初的工作不应该是学习计算机科学中最复杂的部分。学习编写有效代码的基础知识。

学习编写有效代码的唯一方法是编写代码，直到它有效为止。

当我学习一个新的编码概念时，通常是第一次尝试，我可能会看到发生了什么，它似乎是可行的，但我并没有真正理解。也许第二次或第三次，我*还是*不明白。然而——这是真正令人惊奇的部分——最终，我以正确的方式看待它，突然，这个小拼图与我已经拥有的一些其他知识合二为一了。这需要投入销售代表并获得这些经验才能向前推进。

那么，有什么方法可以让你获得更多的销售代表呢？我有一些想法！

# 获得更多编码代表的 5 种方法

## 1.做教程——我的意思是，真的做

有大量的教程，而且很多都是免费的！继续寻找适用于你学习目标的教程和方法资源，并真正花时间[充分利用它们](https://medium.com/better-programming/7-ways-to-actually-learn-something-from-coding-tutorials-b6cdddace26f)。

教程是通过简单的本地开发环境或集成 ide 提供的一些训练轮来介绍主题的好方法，也是获得新概念的最初几次重复的好方法。

## 2.去 meetups 获取代码示例

甚至在我所在的密苏里州哥伦比亚附近的一个较小的社区中，也有几个技术聚会小组，他们不时聚在一起讨论开发主题和项目。随着所有疫情疯狂的进行，一大堆聚会已经开始变得遥远！这提供了很多参加非本地讲座的机会。

当我参加这些讲座时，我会做很多笔记，包括确保记下我想做更多研究并试图学会自己做的项目。我总是确保询问演示者是否愿意分享他们的代码——有例子是很好的，当我从事我自己的项目时，我经常会回到他们身边。然后，我可以通过通读并重写部分代码来检查他们的代码，以查看运行的代码是如何变化和工作的。

## 3.帮助社区活动，通过教导他人来巩固你的知识

让其他人参与技术并让自己学习一些新东西的一个方法是帮助组织一次社区编码活动！在我们的社区中，我们有几个不同的小组，为孩子、女孩、学生和普通大众创造机会一起学习编码——他们总是在寻找志愿者！他们中的一些人组成一个小组，完成一个 Raspberry Pi 项目，而其他人则做简单的小组教程，如学习 Python 基础知识或 Web 设计入门。

帮助其他新手编写代码对于巩固你的编程基础很有帮助，而且你可能有助于塑造未来开发人员的教育。

## 4.构建自己的小而完整的项目

很容易陷入讲授类似内容的教程的窠臼。你只能学习如何写一个`for`循环这么多次。想出一个小的、具体的、包含一些你已经掌握的知识的想法。然后去造！让它达到可以工作的程度，然后用新特性迭代你的 MVP。

将你所学的代码片段应用到你可以与之互动的网页或控制台应用程序中，你学到的不仅仅是一遍又一遍地重写相同的教程代码。

## 5.扩展他人的项目

例如，[我喜欢这套 JavaScript 教程](https://javascript30.com/)。它们很有趣，看起来很漂亮，让我接触到了很多我从未见过的前端概念。对于其中的几个教程，我会将教程的一部分或全部放到一个网页上，我可以在网页上练习样式和添加新的想法。其中一个项目是建立一个 HTML 画布，你可以在上面画彩虹线。然后我把它放在网页上，增加了改变颜色和大小的功能:

# 结论

更好地编写代码的唯一方法是从编程世界中吸收新思想，并将其付诸实践！找到创造性的方法进行更多的练习将有助于你的技能发展得更快，也有助于你学得更深。

对于其他新开发人员，我很想听听你们是如何成功快速升级的！我希望我分享的策略能给你带来一些新的想法，我一直在寻找方法来改进我的学习策略和学习新的编码主题。
