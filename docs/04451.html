不仅仅是String.format

我们都知道String.format，但是还有其他选择。Java有多种格式化、对齐、填充和调整字符串的方法。

字符串格式化的历史

编程语言有格式化字符串的悠久传统。

60多年前，Fortran以FORMAT关键词发布:

     WRITE OUTPUT TAPE 6, 601, IA, IB, IC, AREA
601  FORMAT (4H A= ,I5,5H  B= ,I5,5H  C= ,I5,
   &         8H  AREA= ,F10.2, 13H SQUARE UNITS)

更广为人知的C风格的格式字符串，由printf使用，而Java的String.format起源于BCPL:

WRITEF("%I2-QUEENS PROBLEM HAS %I5 SOLUTIONS*N", NUMQUEENS, COUNT)

Java不支持String.format。直到Java 1.5发布，它才被包含进来。在此之前，java.text.MessageFormat是让弦向你的意志弯曲的方式。

格式指定符

所有格式字符串都以%开头，由多个可选部分和实际转换说明符组成。

通用语法可以分为三个不同的组:

  • 常规、字符和数字类型
    %[argument_index$][flags][width][.precision]conversion
  • 日期和时间
    %[argument_idnex$][flags][width][.precision]conversion
  • 无论据说明符
    %[flags][width]conversion

精确

该行为取决于转换类型。

对于常规参数类型，精度是结果格式化字符的数量。

浮点转换(%a、%e、%f)限制小数点后的位数。

一个特例是%g，它将定义舍入后的结果幅度的位数。

对于其他转换(字符、整数、日期/时间、百分比和行分隔符转换)，指定精度不适用，并将抛出异常。

java.util.Formatter

java.util.Formatter类是printf格式说明符的实际解释器，并且在我们的格式需求中做了所有繁重的工作。

在内部，这一切都归结于java.lang.Appendable接口和一个java.util.Locale。

方法System.out.printf在内部使用一个java.util.Formatter，就像java.lang.String一样。

Java . text . message格式

我们现在已经了解了使用printf格式说明符的不同方法，但是Java有一种更基于自然语言的格式化字符串的方法。

大括号中使用了更简单的说明符，而不是相当隐晦的格式字符串:

{argument index , format type , style }

参数索引是必需的。格式类型和样式是可选的，但是样式不能单独使用。它必须始终遵循格式类型。

结论

现在我们知道了格式化字符串的不同方法。选择哪一个取决于我们的要求。java.text.MessageFormat对眼睛来说更容易，尤其是如果你对printf不流利的话。但是String.format被广泛使用和理解，所以我们不能出错。并且printf格式说明符不仅仅被Java使用，所以我们学习了一个更通用的技能。
