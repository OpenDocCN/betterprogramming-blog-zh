结论

从我的使用案例来看，好友是明显的赢家。由于我不是DevOps专家，设置CI/CD管道在视觉上确实吸引了我，我发现它非常容易配置。

Buddy 也获得了最快的运行时间，但我怀疑人们很快会对我如何改进现有管道提出建议。然而， Buddy 并不像其他服务那样广为人知，因此，在使用他们的服务胜过竞争对手之前，你应该仔细检查他们提供的服务。

其次，我会推荐 AWS CodePipeline ，因为它出奇地容易配置，这并不是我使用AWS的一贯经验。

CodePipeline与其他AWS服务深度集成，这使得它非常强大。如果你已经在大量使用AWS，我会全力推荐这项服务。

我个人发现 GitLab 是所有服务中最难使用的，因为它没有得到很好的记录。 Travis 和 CircleCI 都有很好的文档记录，如果我不使用AWS或者不能使用像 Buddy 这样的小公司，我肯定会考虑这些服务。
