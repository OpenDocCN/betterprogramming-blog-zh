# 在接受一份工作之前，你应该澄清什么？

> 原文：<https://betterprogramming.pub/what-should-you-clarify-before-accepting-a-job-offer-2a5b8a149c9e>

## 在回答“是”之前，先问正确的问题

![](img/53a35dab6335ad34f7e6681d2137f105.png)

在接受工作机会之前，你应该澄清一些要点。此刻很容易错过他们，然后马上接受一份工作邀请。

因此，在这篇文章中，我想谈谈你在接受录用之前应该问的一些问题。我建议你做一个小清单，因为你可能会在面试中忘记它们。

因此，您可以提出如下问题:

1.  如果我发布一个附带项目，公司会要求所有权吗？
2.  公司是否有清晰的晋升路径？
3.  公司给我提供导师吗？
4.  公司允许我参与开源项目吗？

# 辅助项目所有权

你可能想问的第一个问题是，“**如果我发布免费/付费的辅助项目，公司会要求所有权吗？**“有很多公司声称拥有员工项目所有权的例子。即使你在你的空闲时间和你的设备上构建项目，这也可能发生。

因此，你不希望投入你的资源和时间只是为了让一家公司“拿走”你的项目。而且，你甚至可以有法律纠纷，这就更糟糕了。

因此，在接受工作邀请之前，确保公司不会声称对你的项目拥有所有权。

# 清除进展路径

另一个重要的问题是，“**你有明确的升学路径吗？知道你是否能在公司里进步，以及你能得到什么样的支持是很重要的。**

你有多少年的经验并不重要。原因是你可以做一份工作很长时间而不会变得更好。你要努力积极提高。

因此，你的进步可能会停滞不前。这意味着你作为一个开发者和一个人都没有进步，而且你也很难找到另一份工作。相信我；你不想发生这种事。因此，问问公司，他们如何支持他们的员工作为开发人员和人员进行改进。

# 开源项目

如果有机会为开源项目做贡献，你能做到吗？这是你想和你未来的雇主澄清的另一点。

为开源项目做贡献是一种跳出常规、提高编程技能的好方法。只要项目不与公司冲突，他们应该允许你为开源项目做贡献。当然，只要你在空闲时间做这些事情。

总之，不要忘记问你未来的雇主他们是否允许你为 OSS 做贡献。

# 指导

有一个导师对你来说是超级有益的，尤其是在你职业生涯的初期。当你挣扎的时候，能够去找一个人是非常重要的。然而，不要把导师和应该填鸭式教育你的人混为一谈。

导师应该在那里给你指点和指导。导师不是解决你问题的人。然而，如果指导是可行的，并且做得正确，这对双方都有好处。

因此，确保公司为你提供指导。适当的指导有助于你更快地发展。

# 结论

文章的问题并不详尽，可以补充更多的问题。其实我更想听到和发现更多的问题。你对合同越小心越好。

在此之前，不要忘记询问:

1.  你的副业项目的所有权
2.  晋升途径
3.  师徒制
4.  为开放源码软件做贡献
