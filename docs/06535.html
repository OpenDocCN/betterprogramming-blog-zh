如何在训练营结束后找到一份工程工作

作为一名新工程师，如何看待这个世界

找工作没有神奇的步骤。我没有什么特别的秘密来保证一个提议。然而，我花了大量时间在求职过程中评估这个问题，并在事后反思这个问题。

我的LinkedIn收件箱不断收到来自应届毕业生的请求，讨论我的旅程，我总是很乐意这样做。在我看来，当我刚起步时，人们为我做了这些，我们都有责任付出善意。我继续接受进一步谈论这个话题的请求，但对于那些我不会亲自与之交谈的人，这篇文章概述了我全心全意相信是成功找到第一份软件工程工作的关键。

致每一个读到这篇文章的人——抬起你的头，把你的眼睛放在奖品上。忽略悄悄潜入你脑海的忧郁的质疑声，继续磨练和完善你的方法。每天你都在一点点靠近，即使感觉不是那样。

设置场景

你刚刚从软件训练营毕业，祝贺你！这是一个非常令人兴奋的成就。你有一套新的技能，你已经准备好融入这个世界。你的训练营可能已经为你安排了一位职业教练，给了你一份可以使用的简历模板，并为你的求职提供了一个宽松的框架。

但是非常清楚的是，你拥有一套所有其他训练营毕业生都拥有的新技能，你使用的简历模板所有其他训练营毕业生都在使用，你建立的框架与所有其他训练营毕业生完全相同。最重要的是，你申请的是和其他训练营毕业生完全一样的工作。你明白我的意思了。

我说这些不是要打击你，而是要让你重新聚焦。毫无疑问，你已经被告知，简单地点击应用和交叉手指是一个无效的策略，即使你这样做1000次。尽管如此，上面应该描绘出这种方法是多么的无效。

不仅仅是无效——这个词意味着它在某种程度上是有效的，也许如果你做的次数足够多，就会有所收获。我不相信这是真的。当然，有些非常非常小比例的人会在训练营刚开始时就有这种运气。但是据我所见，提交了多达700份申请的“初级开发人员”角色中，只有一人能够得到这份工作，你不能依赖这种策略。你不会找到工作的。

与众不同

读完上面的内容后，看到这一点在列表的最顶端应该是不足为奇的。你必须让自己从其他训练营的毕业生中脱颖而出。你做的每一个决定——你提交的每一份工作申请，你写的每一封电子邮件，你在LinkedIn上的每一篇帖子——都以这个问题为导向，“我如何才能脱颖而出？”

这种与众不同的想法将会渗透到这个列表中的所有内容。一切都回到这一点，但有两个非常具体的方面，你必须与众不同:你的技能和你的简历。招聘人员会将这两者与其他申请人进行比较，这通常是他们第一次向你介绍候选人。

新技能

花点时间看看几个不同训练营的课程。我几乎可以保证，他们的工程课程是由教授带框架的JavaScript(可能是React或Angular)和带框架的Ruby或Python(两种极其相似的语言)(像Rails或Django)组成的。去吧，看看。我将把几个最受欢迎的放在下面:

  • 熨斗 : JavaScript，Ruby
  • 总装配 : JavaScript，Python
  • 跳板 : JavaScript、Python
  • hack reactor:JavaScript
  • Codesmith : JavaScript

有一些教授Java的训练营。网等。，但在大多数情况下，你会发现上述内容。

现在是的一个原因——这些语言被广泛使用，对于专业开发人员来说非常有价值。但应该清楚的是，这是一个非常千篇一律的模式。你现在是一名开发人员——这意味着你已经承诺终生学习。现在就开始这段旅程。自学新的技能，深入学习你已经掌握的技能。

您已经在训练营中学习了React，但是您曾经完整地阅读过文档吗？能不能用钩子，或者解释一下类和功能组件的区别？你和Node合作过，但是你能确切地说出它是什么吗？寻求加深对你已经熟悉的话题的理解。这将使你在面试中谈论代码时脱颖而出。

除了加深理解，还要学习新的东西。参加AWS的课程，或者使用MongoDB构建一个应用程序。你不会成为另一种语言的大师，但你会花时间去尝试一种静态类型的语言，比如Java，因为训练营之间的另一个相似之处是动态类型语言的教学。

如果你不知道我说的静态类型和动态类型是什么意思，你可以从这里开始。学习关于语言以及为什么一种语言比另一种语言更常用。练习写作测试，这是职业发展的一个重要部分，但很少在训练营中教授。研究互联网本身——实际的物理互联网——以及它是如何工作的。我对此的看法是 Udemy 。

最后，阅读。读，读，读，再读一些。看博文，看书，看文档，看StackOverflow。做一块海绵。尽你所能消耗一切，即使感觉它完全超出了你的能力范围。阅读有助于你熟悉谈论代码所用的语言，并且它扩展了你思考工程的方式。

简历

希望你技能的扩展能帮助你的简历脱颖而出，但不要仅仅如此。你的简历在美学上看起来都是一样的。这就像一个“缺少什么”的游戏——只不过玩游戏的人实际上并不太关心找出不同之处，他们更关心的是翻看桌子上的一堆简历。

训练营会鼓励你使用他们的模板，但我不同意正是因为这个原因。不代表他们没有有价值的指导，但不要只是把他们的占位符信息填成你的就收工了。使用不同的字体。调整布局、配色方案、要点——任何能让它更独特同时保持专业性的东西。此外，在过去的经验和工程之间建立有意义的联系。

最后，突出你的成就，并让他们容易获得。您应该已经部署了任何想要呈现的应用程序，并且可以与之交互。你构建的任何东西都可以通过Heroku轻松免费地部署。

建立你的形象

在找工作的大多数情况下，介绍一个人在网上出现在实际介绍之前，我们都知道良好的第一印象是多么重要。你的网上形象应该得到磨砺和精心打造，以反映你作为一个人以及作为一名专业人士的身份。它应该展示你的职业道德、知识、技能和对软件工程的热情。

开源代码库

你的GitHub活动网格应该是漂亮的亮绿色。在采访过程中，我多次听到这样的话:“我可以看到你在GitHub上非常活跃，这很好。”团队在招聘时确实会考虑到这一点，所以在致力于自己的项目和为开源库做贡献时要始终保持积极。第一次为开源代码做贡献可能会令人生畏，但是有大量的资源和指南可以帮助引领方向。从读这个开始。

商务化人际关系网

在LinkedIn上活跃起来。加入群，发帖，和别人的帖子互动。这种接触是展示你的兴趣和你如何支配时间的好方法。

LinkedIn上的另一个注意事项——它是一个职业社交网站。求求你，求求，来一张专业的照片。你不必穿西装，但你应该看起来像样。

博客

博客是学习新技能和展示新技能的绝佳工具。每当我学到新的东西，我都会写一篇博客。教学是一种行之有效的学习方法。再说一次，在我的多次采访中，博客被认为是给我带来优势的东西。团队实际上花时间阅读了我写的一些东西，我的写作能力说明了有效沟通的能力。

回到我们潜在的差异化主题，写博客是展示你的技能和兴趣的绝佳机会。如果你对机器学习、密码或安全感兴趣——所有这些都是非常高级的领域，在你的第一份工作之前完全学习可能不太现实——写一篇关于它的博客吧！尽你所能学习，并分享你所学到的。

与人交谈

软件工程最容易被误导的模式之一是它不是一种社会实践。别搞错了——它非常社会化。作为一名软件工程师，就像写代码行一样需要表达和交流(顺便说一下，这也是表达和交流)。

一个附加的团队成员审查其他人的代码并提供有意义的反馈，合作解决问题和排除故障，包括关于如何与代码库交互和为代码库做出贡献的雄辩的、人类可读的指南，有效地讨论思维过程、编写的代码、前方的障碍和技术决策，提问并有益地回答问题…你明白了。

向前人学习

每个专业工程师都曾经是一个寻找第一份工作的非专业工程师。他们中的每一个人都可能遇到“你需要有经验才能获得经验”的悖论——那些面试过程最终会以“没有足够的经验”拒绝你，即使手头的职位是入门级的，根据定义，是为即将进入职场的人准备的。

如果你以专业人士的身份与他们交谈，那么他们也已经知道如何跨越这个障碍。问问他们是怎么做到的。询问他们申请了什么样的工作，他们如何展示自己，他们如何着手申请，什么策略对他们有用，什么没用，为什么。每个人都是不一样的，求职也不是一个“一刀切”的过程。倾听和学习他人的经验，并利用它找到适合你的方法。

与他人联系是而不是个人雇佣你。这是关于建立真正的关系和扩大你的网络。这是向前人学习的问题。俗话说，“求工作，得建议。征求意见，你就能找到工作。”

言语流畅

你将不得不在整个面试过程中谈论代码，谈论代码和编码完全不同。你能解释一下为什么你用你的方式写代码吗，或者你的答案只是，“因为我就是这样学会做的？”你知道用来讨论你所使用的工具的正确语言吗？你能阐明你所做的决定及其实施过程吗？

谈论代码本身就是一门艺术，相信我，一开始你会把它搞砸的(我肯定会)。每次你与另一位工程师交谈时，你都有机会在这些交流中变得更加流利。通过仔细倾听，这些对话会变得越来越自然。密切注意用词，查找不熟悉的术语，并参与其中。你会发现，如果一个新术语出现一次，你很可能会再次听到它。熟悉它，这样如果它在面试中出现，你就不会僵住。即使只是知道某样东西是AWS产品，或者是一种数据库，或者是你已经知道的东西的另一种说法(发生的比你想象的要多)，在面试中也会有很大的价值。

如果你生活中的某个人愿意，不管他是不是工程师，当你展示你写的代码或者解释高级概念的时候，请他倾听。通过这样做，你将摆脱那些笨拙的，“哇，我实际上刚刚说了，”的时刻，在一个安全的空间，以及开始了解你在谈论代码时的痛点。你不知道如何展开的话题会很快变得明显，让你有机会在面试前纠正这一点。

离开你自己的路

我相信你听说过“冒名顶替综合征”冒名顶替综合症不仅是真实存在的，而且如果不加以检查，它还会在求职过程中造成切实的损害。所以现在检查一下。

选择要申请的职务

找零经验的训练营毕业生是找不到工作的。你不是。你会找到需要至少一年工作经验的工作，很可能需要三年，你不知道的技术会和你知道的技术放在一起。那么，在最大化你的胜算的同时，你该投哪一球，该传球哪一球呢？

同样，这肯定没有一个有限的方程式。然而，我发现对自己有用的是:

更大的公司，有更大的，更成熟的工程部门，有更广泛的工作列表和更具体的要求。换句话说，他们更有可能有一个寻找具有三到五年经验的中级工程师的列表，但也有一个寻找具有一到两年经验的初级工程师的列表。一些较大的公司甚至为新工程师提供学徒机会。在寻找更大的公司时，更多地从表面上看他们的要求，寻找包含“初级”、“学徒”或“入门级”字样的职位。

创业公司则相反，空缺较少，要求更加模糊。他们可能在寻找一个有三到五年经验的工程师，但他们可能愿意冒险聘用一个感觉非常合适并愿意努力工作的新人。对于创业公司来说，我认为任何列出三年或更长时间的经验并且不包括“高级”这个词的东西都是公平的游戏。如果你对他们的所有东西都不熟悉，那就在任何对话之前主动温习一下问号。

不管情况如何，我制定了一个规则，那就是我不会去申请那些我不会花时间去写求职信和联系公司人员的工作。最重要的是，这条规则非常适合我。

底线是，如果你只申请那些你符合所有要求的工作，你将永远找不到任何工作，并且会在很多好机会中自我选择。考虑到你的简历通常不会完全符合要求，人际交往对你的机会至关重要。

展示自己

是。自信。

我可以就这两个词离开这一部分，感觉我已经表达了我的观点。对你不知道的事情要诚实(显然，不要对你不具备的技能撒谎)，但要充满自信地说出你对团队的贡献。面试官知道你没有经验，但还是选择和你谈，所以不要再想了。告诉他们你将给团队带来的一切，不要低估你的技能 。

我不能过分强调这一点。不要因为不安全感或害怕无法提升而贬低自己的技能。正如我所说，只要你对自己的技能保持诚实，你就没什么好害怕的。公司知道雇佣你能得到什么。他们比你更清楚自己需要什么，也更有资格做出决定。不要为他们做。

全力以赴

打磨每一件反映你的小事。你必须有一个部署好的、专业外观的应用程序，当机会出现时，它能很好地显示出来。尽管从小就被告知不要这样做，但我们都以貌取人。如果当有人打开你的应用程序时，它看起来很糟糕或者损坏了，那么这个第一印象将是很难消除。花点时间挑选一些字体和基本的调色板，让它看起来更漂亮，并且便于移动使用。在像 Heroku 这样的地方免费部署。

当你接到一个涉及构建一个小应用程序的编码挑战时，同样，让它看起来更好。使用洛蒂动画制作一个有趣的装载旋转器，并尝试复制公司的美学。这些事情会让你在编码挑战提交的海洋中脱颖而出。

让激情引领

如果你不能清楚地说明你为什么想要一份特定的工作，那么就不要申请它。如果你不能解释你为什么想为一家公司工作，那么他们为什么要你为他们工作？

有些人说“好吧，但事实是我对这里(插入看似平凡的行业)不感兴趣。”那很好。我不相信在克拉特工作的每个人都对存储充满热情，或者Robinhood的整个团队都受到了华尔街的启发。但是看看他们的使命陈述，看看他们实际上在做什么，他们是谁，他们看重什么。克拉特“致力于让人们的生活变得便利，这样他们就可以体验更多他们喜欢的东西”，而罗宾汉则致力于“让所有人的金融民主化”。我不知道你怎么想，但这两个听起来都很不错。

有些使命陈述仍然不能引起你的共鸣，这没关系！向前看。不要浪费你(或他们)的时间。如果你发现一家公司的基本价值观和驱动力与你自己的一致，你就更有可能在面试过程中建立真正的联系，传达出你有一些有意义的东西可以拿出来，如果有机会加入团队，你就能取得成功。

这是一次艰难的旅程。确保照顾好你的精神和身体健康，并且每天提醒自己自从你第一次开始编码以来你已经完成的所有事情。接受新的挑战，同时陶醉于那些你已经面对和征服的挑战。为自己设定现实的期望，并不断反思自己迄今为止所做的决定。

在这个过程中你不是一个人，所以找到与你同舟共济的其他人。希望有一天当你回首往事时，一切都是值得的。
