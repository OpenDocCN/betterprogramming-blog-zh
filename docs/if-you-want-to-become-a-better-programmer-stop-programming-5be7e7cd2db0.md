从最简单的原则和实践来看，编程包括解决问题和批判性思维。

如果你思考是什么让一个人成为优秀的批判性思考者和问题解决者，那就是他们想象和应用创造性的解决方案来解决特定问题的能力。

换句话说，真正把一个伟大的开发人员从一个不那么伟大的开发人员中剔除出来的是一件简单的事情:思考的能力。

我不是来给你讲那些你可能已经知道的导致开发的显而易见的事情，比如计划阶段或者架构阶段。我也不会告诉你休息有多重要，这样你就可以让你的潜意识在背景中思考问题。

你已经知道这些事情了。

相反，我要向你揭示的是这个行业中初露头角的程序员经常忽略的东西。不太明显的。对加速我的编程生涯贡献最大的一件事。

# 堵塞创造性思维之路的驼鹿

如果你认为随着时间的推移越来越多的编程会让你成为一个更好、更进步的程序员，那你就对了。肯定会的。

经验和智慧几乎是你做任何事情的重要因素。不断的实践和失败正是强化和嵌入我们头脑中的关于什么可行什么不可行的想法。

但是，它会让你成为一个更好的全面思考者，更善于创造新问题的创新方法吗？一点点。

编程实践本身并不是成为一名真正伟大的开发人员的唯一因素——这实际上适用于所有行业。

如果您只关注编程的应用、围绕它的概念和理论、最佳实践以及要遵循的最佳原则，如果您不能立即创造性地思考和应用这些东西，要真正弥合这一差距将会困难得多。

如果你真的想提高，你需要超越眼前的事物，让自己接触那些能让你跳出框框思考的想法。

在一个领域的全面掌握不能通过将你所有的精力完全集中在无休止的循环中反复练习同样的事情上来实现。

相反，你应该通过探索超越你的技艺来影响掌握。

# 创新者与其他人的区别在于

在他的书 *Clean Code* 中，Robert C. Martin 将报纸文章的结构方式与编程联系起来，作为一种干净地构造函数和类的方式。

如果有一种方法可以提高你的创造性思维能力和效率，那就是探索与你关注的主题无关的话题，并在这些看似无关的事情之间建立联系，以充分利用你的技能。

简单回顾一下罗伯特·格林的书《掌握[](https://www.youtube.com/watch?v=j-QUDT4KOUg)*中的一个观点，你必须让学习本身成为一种奖励。重新发现你孩子般的好奇心，让这种好奇心培养和发展新的想法，最终在看似不相关的主题之间建立联系。*

*一个很好的例子是国际象棋选手和柔术竞争者 Josh Waitzkin，他能够以比所有同龄人更快的速度掌握柔术，因为他能够应用他从下棋和掌握国际象棋中学到的策略。*

*另一个受欢迎的例子是有史以来最伟大的博学家莱昂纳多·达·芬奇，他利用对自然的观察、对解剖学的研究以及对所有学科的强烈好奇心，影响了他对工程突破和艺术杰作的富有远见的构建。*

*你在生活中接触到的每件事都会以这样或那样的方式有益于你进行的每项活动。*

*为了更好地思考，成为自称的天才，你必须学会如何[将点点滴滴](https://www.brainpickings.org/2013/05/23/uncommon-genius-stephen-jay-gould-connections-creativity/)串联起来。一旦你能在不同的主题中看到一个相关的模式，你就能在你的工作中运用新的和原创的想法，这样你就能更有创造性和更有效率地运用它们。*

> *创意常常在于把以前没有被怀疑过的想法联系起来— W. I. B .贝弗里奇， [*科学调查的艺术*](https://www.brainpickings.org/2012/05/25/the-art-of-scientific-investigation-1/)*

# *如何通过不编程来招募更好的想法*

*现在，我并不是说你必须完全停止编程，把自己完全从这门手艺中分离出来，以尝试培养新的想法。*

*不过，我要说的是，你必须从字里行间寻找答案，并愿意以非正统和意想不到的方式填补这些知识空白。*

*也就是说，在你自己的时间里，阅读书籍，探索你感兴趣的、可能与编程没有直接联系的主题。*

*阅读心理学，这样你就知道如何在充分理解人们如何工作的基础上构建你的产品。阅读哲学，这样你就可以应用哲学概念来影响需要在代码中做出的批判性思维决策的轨迹。阅读关于建筑和 T2 科学的书籍，这样你就可以像自然界中的建筑和化学键一样，以内聚的方式构建你的代码。学习冥想，这样你就可以平静地做决定，并把自己从制作漂亮代码的艺术中分离出来。或者阅读您感兴趣的任何主题，并在您正在探索的主题和软件工艺之间建立自己独特的联系。*

*在 John Somnez 的书*完整的软件开发人员职业指南*中，他甚至提到，拥有与编程无关的领域知识的人比那些只学习编程的人更快地成为更好的程序员，因为他们可以将他们在另一个领域学到的知识应用到他们正在构建的东西中。*

*任何事物之间都可以建立联系。你在编程中所做的一切都会受到其他研究领域中引入的思想的影响。想想人工智能以及计算机科学和神经科学的结合。或者建造火箭飞船，将计算机科学与物理、数学和工程结合起来。这正是真正的创造力形成的地方，也是让你从一个优秀的程序员走向伟大的程序员的桥梁。*

*有意识地做所有这些事情正是让我比大多数其他开发人员更快地利用我的编程能力的原因。快速学习、建立直接联系和更好思考的能力。*

# *如果有什么东西你应该带走，那就是这个*

*保持好奇，不断探索，不要让自己与一种独特的做事方式结合在一起。*

*如果你在追求提高编程技能的过程中发展了狭隘的视野，你将生活在迷雾中，这将阻碍你看到更大的图景或产生任何真正创新的东西。*

*你会陷入一个习惯性的循环，只做别人期望你做的事情，只应用那些你可以直接通过理论知识获得的东西。*

*试着重新发现你孩童般的好奇心，重新点燃你头脑中那微弱的光芒。你可能会对自己发现的伟大想法和思维模式感到惊讶。*
