不管你是否同意，技术面试需要充分的准备。为了确保你为关于[数据结构和算法](https://www.geeksforgeeks.org/data-structures/)、[系统设计](https://www.theseattledataguy.com/passing-the-system-design-interview-for-software-engineers/)、 [ETL 设计](https://www.theseattledataguy.com/etls-vs-elts-why-are-elts-disrupting-the-data-market-data-engineering-consulting/#page-content)、统计、编程和机器学习的连续问题做好准备，需要涵盖许多主题。这完全取决于面试官的角色、公司、团队和心情。

每次技术面试都是独一无二的。例如，如果你是一名数据科学家，一次面试可能会要求你进行统计、SQL 和实验设计，而其他人可能也会涉及数据结构和算法。

只是为了让我们的生活更加沮丧。

所以。

如果你不确定你将面临哪种类型的面试，我们建议询问招聘人员。一些公司非常擅长保持面试的一致性，但即使这样，团队也可能会根据他们想要什么而偏离。

这里有一套通用的步骤，你可以用来准备下一次技术面试。

# 弄清楚学什么

每个公司都有自己特定的关注点，这也可能因职位不同而不同。确定你需要学习什么的最简单的方法是和招聘人员谈谈，了解他们想要什么。这应该给你一个基本的想法开始，但不要忘记做自己的研究。

当你决定你需要学习什么的时候，列出一个能帮助你学习的资源清单。

HackerRank 是一个对编码很有帮助的网站。LeetCode 是另一个在过去对我帮助很大的网站。这两个网站都给了你大量的问题，但是不要认为这些问题会和你的技术面试中的问题一样。相反，试着吸收他们提供的知识，并在你的面试中使用。

系统设计是你在[大多数面试](https://www.youtube.com/watch?v=kAoNrYJk6u8)中想要关注的另一个领域，所以你会想要查看关于这个主题的视频。

YouTube 是所有技术的另一个重要资源。你可以在那里找到很多人来解释你可能还没有完全掌握的概念或技术。通常，在视频中看到这种做法会在你脑海中形成印象，这样你就可以在面试中使用这些信息。

# 创建学习计划

为了确保你确实涵盖了你需要学习的所有内容，制定一个学习计划是个好主意。你可以[以我的为灵感](https://docs.google.com/spreadsheets/d/1GOO4s1NcxCR8a44F0XnsErz5rYDxNbHAHznu4pJMRkw/edit#gid=0)或者从头开始创造你自己的。基本上，你要列出你需要关注的各个领域，比如 SQL、数据库、Hadoop、算法、数据结构等等。同样，这取决于你申请的公司。

学习资源如此之多，很容易让人感到不知所措，这就是为什么你需要一个学习计划。花点时间写下你在面试中可能会遇到的问题，然后找一个资源来了解答案。这将简化整个过程，让你更容易学会所有你需要学习的东西。你可以以我的学习计划为基础，根据你可能会收到的问题来制定计划。

你也应该记下你已经完成的部分。这也会对你面试前有所帮助。

# 对自己进行一次预测试

一旦你为你的学习指南收集了一些问题，在每一部分，是时候做一个预先测试或预先面试了。这是记录你真正需要集中学习的地方的最好方法。

就学习计划的各个部分对自己进行测验。你已经精通哪些领域？你真正需要努力的领域是什么？预面试不需要涉及其他任何人，但这是一个测试你知识的好方法。完成学习计划后尝试一下，看看自己提高了多少。

请记住，在实际的面试中，你无法访问谷歌，所以试着只用你已经知道的东西来解决问题。你应该把这些知识保存在你的大脑中，这样当你没有电脑的时候，你就可以随时利用它们。当然，在实际工作中，你可以使用电脑和互联网，但对于面试，你只需要使用你的大脑作为资源。

# 进行模拟面试

模拟面试是一种很好的真实练习方式，可以让你更好地了解自己。你应该尽可能多地进行模拟面试，用一个朋友或顾问作为面试官。

首先，你可以让你的朋友问你学习计划中的问题，但是最好的学习方法是接受一个你没有练习过的问题。如果你和同样学习技术的人一起工作，试着从你自己的学习清单中问对方问题。一个不熟悉的问题会让你有点困惑，但这是学习如何恢复并当场给出答案的绝佳练习。

虽然模拟面试是很好的练习，但是没有什么比真正的面试更能获得宝贵的经验和反馈了。不幸的是，如果你刚刚毕业，你会发现自己要为各种公司做大量的面试。这是职业选择的一部分，虽然你肯定会犯错误，也会在一些面试中失败，但它们都是有价值的练习。

# 不要忘记产品感

产品感是数据科学家、软件工程师和数据工程师的一项重要技能。了解新产品的衡量标准和衡量原因有助于确定产品是否表现良好。

产品感部分的一个很好的例子是面试官询问衡量标准。

指标对于整个技术团队来说扮演着重要的角色。度量有助于告诉软件工程师他们的特性是否表现良好，它会影响数据工程师，因为他们需要知道跟踪什么，而数据科学家需要知道分析什么。

有趣的是，有时某些指标按照你希望的方式发展可能并不总是好的。人们在你的网站上花费更多时间的原因可能是因为网页加载时间更长或者面临其他类似的问题。

反过来，面试官通常会问很多产品感问题，因为他们可以帮助看出开发者的想法。他们是否完全理解用户如何与产品互动？他们了解计数器指标吗？

他们能推动产品开发吗？

# 注意你的弱点

每次面试后，不管是练习还是真实的，记下你的表现。哪些问题让你栽跟头？哪里彻底失败了？这些都是你的优点，你会想要更多地研究这些方面，以确保你能在下次面试中得到它们。

每一次面试，你都会学到更多，看到更多的弱点。这只是过程的一部分，所以不要气馁。把每一次失败都看作是一次学习的经历，在下一次失败中变得更加强大。
