语句、表达式、文字和运算符

当我学习编程时，我记得我被术语轰炸过。很多教程和在线讨论似乎都认为我已经理解了这些单词的意思。像“陈述”、“表达”和“字面”这样的词被使用过，但从未被解释过。

在这篇文章中，我将讨论四个常用的编程术语，不管你使用什么语言，理解它们都是有用的。

文字

让我们从最简单的开始…

编程中的一个文字是一段表示源代码中固定值的文本。文字的例子有:

123     // an integer literal
1.23    // a floating point literal
"123"   // a string literal

请注意，文字不同于常量。思考这个问题的一个好方法是，所有的文字都是常量，但不是所有的常量都是文字。

const x = 123 // x is a constant initialised with the literal 123
const y = x   // y is a constant initialised with another constant

在某些语言中，复杂对象也可以是文字。例如，JavaScript有对象文字。

var a = {
    x: 1,
    y: 2
};         // The construct { x: 1, y: 2 } is an object literal

经营者

编程语言中的运算符是一种符号，它指示解释器或编译器执行语言规范中定义的数学、逻辑、关系或其他运算。

常见的运算符包括:

  • 算术:+、-、*、/
  • 任务:=
  • 逻辑:!、&&、||
  • 相等测试:==，!=
  • 函数调用:( )
  • 增量/减量:++，--
  • 成员选择:.
  • 顺序关系:<、<=、>、>=

使用运算符往往等同于调用函数，例如:add(x, y)等同于x + y。然而，情况并非总是如此。例如，赋值运算符=，右操作数的值被赋给作为左操作数给出的变量。

一个运算符也可以是一个单词。例如，C有一个sizeof操作符，它返回分配给给定数据类型的存储空间。

接受一个参数的运算符称为一元运算符。一元运算符的例子包括++(增量)和!(逻辑否定)。

接受两个参数的运算符称为二元运算符。二元运算符的例子包括+(加法)和<(小于)。

声明

“语句”的确切定义因语言而异，但一般来说，语句是不返回值的可执行代码的单元。语句可能包含变量、运算符、文字和其他语言关键字的混合。

简单陈述包括:

  • 任务:a = b
  • 返回报表:return a

更复杂的语句包括:

  • if报表:if (a < b) return c;
  • for循环:for (a = 0; a < b; a++) c++;
  • switch报表:switch(a) { case 0: b++; break; }

一个完整的代码块也可以被认为是一条语句，而这条语句又可以包含其他语句。

公式

语句和表达式之间的区别是微妙但重要的，这很容易被忽略。

与语句不同，表达式是一个可执行代码单元，它由返回值。最简单的表达式需要对文字或变量应用操作符(如上所述)。

  • 1 + 2:将二元运算符+应用于1和2返回3
  • a * b:将运算符*应用于a和b返回a乘以b
  • !a:将一元运算符!应用于a，返回一个布尔值

表达式和语句可以组合在一起，例如:

a = b + c;

所以b + c是一个表达式，a = b + c;是一个语句。该语句的副作用是更新a的状态以包含b + c的结果。

看待这个问题的一种方式是执行语句并计算表达式。

在理解一种给定的语言如何定义为表达式时需要小心。例如，在许多语言中，赋值不返回值，会被认为是语句，而不是表达式。然而，在C中，赋值返回赋值的值。

这可能会导致错误。例如(在C语言中):

if (a = 10)
{
    deleteFile();
}

这里，程序员不小心把a == 10写成了a = 10。代码仍将编译和运行，但不是预期的效果，10将被分配给a并且新的值a将被返回，这意味着deleteFile()总是被调用，并且a已经被修改而不是简单地与10相比。

总结:运算符优先级和结合性

运算符优先级是一组规则，用于确定运算符在复杂表达式中的应用顺序。误解运算符优先级是另一个常见的错误来源。

考虑下面的表达式:

(2 - 1 ? 0 : 1) == 0

会返回真还是假…？运算符优先级规则告诉我们，—将首先被求值，因为它具有最高的优先级。因此表达式将返回true。然而，程序员可能已经假设?具有更高的优先级，在这种情况下预期的结果将是false！

因此，为了避免错误和意外行为，通过小心使用括号使优先级和结合性变得清晰非常重要。

所以:

((2 - 1) ? 0 : 1) == 0

尽管额外的括号在技术上是不必要的，但是它的意图是绝对清楚的。
