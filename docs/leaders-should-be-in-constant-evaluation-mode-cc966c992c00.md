## 自省从来没有不好的时候。

当我们跟踪项目时，我们会考虑数据:

*   完成了多少工作？
*   以什么速度？
*   比上次多还是少？
*   比上次好还是差？
*   更好、中性还是更差的结果？(*我认为这是最重要的*)

但是，我们有多频繁地对我们的团队、我们自己以及我们的整个组织使用同样的镜头呢？

作为工程管理者和领导者，我们应该在不断的评估模式中:

1.  [我们自己](#17b6)
2.  [我们的流程](#b7c1)
3.  [我们的团队](#4360)
4.  [我们的工作](#3aa1)
5.  [我们的工具](#3caa)

下面我列出了我在日常工作中或者在一周/一个项目/一个季度结束时经常问自己的问题。让我知道你认为哪个最有用！

# 1.我们自己

作为领导，不全是你一个人的事，但很多时候，我们才是问题的根源。

*   我是否以正确的方式出现在我们的团队中？
*   我是否提供了足够的支持？
*   在以下四个类别中，我说得够清楚了吗？
*   我是否给了我们的团队适当的自主权？
*   我每天都在哪里挣扎？
*   我哪里做得不够，我如何改进我失败的地方？
*   我现在最不擅长什么？
*   我在哪些方面做得好？

花一些时间进行自省，以确定在你的日常管理中是否有可以调整的东西，这对随后的所有领域都是有益的。

如果需要的话写日记，但是至少每个月问自己一次以上的问题，并且诚实地回答。

# 2.我们的流程

当实施新流程或巩固旧流程时，我们为什么要这样做？为了什么？有什么帮助？询问一些关键问题有助于评估过程。

*   这个过程对谁更有利(我自己还是我的团队)？
*   我们的流程是完成工作的障碍吗？
*   我们可以调整当前的流程吗？
*   我们流程中最令人沮丧的部分是什么？
*   我们流程中最有益的部分是什么？

一个好的开始是写下在一批工作中进行的所有仪式，然后对上述问题进行评分。完成后，让队友进行相同的练习，以推动对话和改变。

# 3.我们的团队

我*喜欢*考察团队动态。项目与项目之间，通过以下问题关注哪些方面可以改进:

*   谁合作得好，为什么？
*   这个团队有哪些突出的优点或缺点？
*   我们的资历水平合适吗？
*   我们需要从外部填补哪些漏洞？
*   我们可以提供什么样的培训来巩固当前的团队？
*   我们的团队规模是否适合我们要实现的目标？
*   我们能支持一个更大的团队吗？
*   我们能和一个更小的团队一起工作吗？
*   团队中有哪些原型，我们是否恰当地利用了它们？

对于团队来说，有太多的因素和变量。如果你没有处于持续评估模式，很容易让事情滑向失控。成为你最好的问责伙伴，监督变化或转变。

# 4.我们的工作

在一些组织中，这将是你控制最少的领域，尤其是取决于你作为领导者的角色和级别。参与工作/项目决策时，评估以下内容:

*   我们为什么要做这项工作？
*   这是否符合我们的年度/季度/月份的 okr 或目标？
*   因为这项工作，我们*没有*在做什么工作？
*   如果我们没有期限和限制，我们会做什么？
*   我们想要达到什么样的结果？
*   我们如何确定我们的工作是否成功？
*   我们是否专注于微小的工作，而不是那些稍后会打开大块工作的工作？
*   我们做这项工作仅仅是因为它在我们的专业领域吗？
*   我们将如何在这项工作中成长？

仔细检查每一项工作，为你自己和你的团队确定一个评估策略。某种可行性矩阵或分析也可以有效地确定一个项目的优先顺序。

# 5.我们的工具

我是一名工程领导者，所以当我想到工具时，它特别与技术有关。构建与购买是评估技术的一个很好的例子。不久前，一位首席技术官告诉我，他们只致力于让自己与众不同的东西，其他东西都是可以买到的，这一点我仍然深信不疑。

*   这是正确的技术堆栈吗？
*   改变我们目前使用的价值主张是什么？
*   有我们应该探索的新技术吗？
*   是否有我们应该弃用/淘汰的旧技术？
*   如果我们继续使用当前的技术，一年或五年后会有什么帮助或伤害？
*   我们能做些小的优化吗？
*   我们在整个堆栈中是否有足够的专业知识？
*   有我们不太关心/更关心的领域吗？

这些都没有错误的答案。随着每个问题的提出，我倾向于发现自己对某些事情的感觉更加强烈，并质疑自己对他人的立场。

# 最后的想法

应该每天、每周和每月对上述每一项进行检查，以发现趋势。我们击中目标了吗？我们还能做得更好吗？我们在哪些方面超出了预期？

首先找到一个基线，然后继续跟踪，看看事情如何随着时间的推移而演变。随着新变化的实现，观察演进的过程。事情有好转吗？更糟？通过设定基线和不断评估，可以做出调整，让事情回到正轨，或者进一步向有益的变化倾斜。

问问你自己上面的一些问题，并在评论中发表你的想法！
