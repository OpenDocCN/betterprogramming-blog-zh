每个程序员最终都会学习循环，最有可能从`while`循环开始，最终到`for`循环，甚至递归。理解这些概念是非常有用的，但是有一些类型的循环是非常规的，比如`for ... of`循环。

让我们看看 JavaScript 中的常规和非常规循环。

# 1.在…期间

许多开发人员在学习期间会遇到的第一个循环是`while`循环。这是一个非常简单和最小的循环，如果一个特定的测试返回`true`，它就会循环。如果值是`false`开始，它根本不会循环。

# 2.做什么

一个非常相似但略有不同的循环是`do ... while`循环结构。像`while`循环一样，当测试返回`true`时，它将在一个代码块上循环。与`while`循环不同，这个循环将至少执行一次，并在循环结束时而不是开始时进行检查。

# 3.为

`for`循环是最广泛使用的循环之一。它用于各种任务，比如迭代一个数组或特定次数。与`while`循环不同，您不必自己更新测试值。

# 4.为了…在

`for`回路的一个微小但有用的变化是`for ... in`回路。这个循环对于迭代一个数组或另一个可迭代对象以及对该可迭代对象中的每个值做一些事情是很有用的。它将创建一个您可以使用的项目变量，而不是索引变量。

# 5.为了…的

`for`循环的另一个细微变化是不太为人所知的`for ... of`循环。其使用方法与`for ... in`循环相同，但其用途略有不同。

# 6.为每一个

一些可迭代的对象，比如数组，包含一个内置的函数来迭代它们。与`map`和`filter`一样，`forEach`函数需要对 iterable 中的每一项执行一个函数。

# 7.递归的

在一个数组中循环或者循环一定次数的“最难”的方法是递归。递归是计算机科学中的一个话题，泛指一个函数将调用自己，直到一个特定的迭代。这个概念一开始可能很难理解，但是我希望一个代码片段可以解释清楚。

# 结论

循环可以通过多种方式完成。其中一些只对特定的任务有用，而其他的可以用于所有的用例，如果你正确实现它们的话。
