简单对复杂对复杂对混乱

原文：https://betterprogramming.pub/simple-vs-complicated-vs-complex-vs-chaos-737b5964849d?source=collection_archive---------10-----------------------#2020-03-03

这些与系统和软件项目有什么关系？

  • “任务很简单。”
  • "这个计算很复杂。"
  • “软件项目很复杂。”
  • “队伍乱了！”

你听过这些短语吗？你知道每一个都是错的吗？

当我们谈论术语简单、复杂、复杂和混乱时，我们指的是系统中的复杂性类别。

系统中的复杂性类别描述系统中已知的 或 未知的已知和未知。当一个人想要学习或理解一个系统时，这是很重要的。

每个复杂性类别都是相互独立的，一个系统只能被分配到这些类别中的一个。

+------------------+-------+---------+--------+----------+
| Complexity Class | Known | Unknown | Knowns | Unknowns |
+------------------+-------+---------+--------+----------+
| Simple           | ✓     | x       | ✓      | x        |
| Complicated      | ✓     | x       | ✓      | ✓        |
| Complex          | x     | ✓       | ✓      | x        |
| Chaotic          | x     | ✓       | x      | ✓        |
+------------------+-------+---------+--------+----------+

已知、未知、已知和未知

在一个系统中，什么是已知的、已知的、未知的和未知的？

说实话，这个话题有点抽象，可能很难理解。但是让我试着解释给你听。我们先来看看这张表:

+---------+----------------+------------------+
|         | Knowns         | Unknowns         |
+---------+----------------+------------------+
| known   | known Knowns   | known Unknowns   |
+---------+----------------+------------------+
| unknown | unknown Knowns | unknown Unknowns |
+---------+----------------+------------------+

这个矩阵显示了已知、未知、已知和未知的所有可能性。

唐纳德·拉姆斯菲尔德曾经说过:

“这里有已知的知识；有些事情我们知道我们知道。我们也知道有已知的未知；也就是说，我们知道有些事情我们不知道。但也有未知的未知——有些事情我们不知道我们不知道。”

我举个例子解释一下拉姆斯菲尔德的引用和上表。想象一下，你拥有一家卖饮料的小店。

  • 众所周知，人们会在你的店里购买饮料。
  • 你已知的未知数是你在月底会卖出多少，但只需要一些时间就能弄清楚。
  • 当你知道一种新的饮料可能会流行起来，但你不知道什么时候会有，是否会有，这就是未知数。
  • 未知就是未知。这些是你还没有意识到的未知风险，只有当它们发生时，你才能识别它们。

简单的

在一个简单的系统中，我们知道已知。

在这样的系统中，一切都是可以预测的。如果你做X，期待y，简单的系统是最容易理解的。

例子

想象一下棕色木箱上的一个简单的红色按钮。每次你按下按钮，一个小丑的形象就会从盒子里跳出来。

我们知道已知的，没有什么是未知的。这个系统很简单。

复杂的

在一个复杂的系统中，我们知道已知和未知。

这仍然是“如果你做X，期待Y”的规则。现在我们加上“如果Z发生，做X。”这导致了一个因果链:“如果Z发生，做X，然后期待y。”

要知道未知，需要分析。

简单和复杂的系统是可测量的。建造一个乐高房子可能很复杂，但你可以测量每一部分，从而提前计划。这意味着简单和复杂的系统是可复制的 ，并且大部分是可扩展的，除非你遇到物理限制。

例子

让我们拿着我们的棕色木箱，添加一条规则。如果你按下按钮五次，小丑就会跳出来。

你知道你必须按五次按钮，小丑才会跳出来。从未知点看盒子的其他人不会知道你必须按下按钮五次。这种知识基于分析和经验。

这就是已知未知的T3，从而也是一个T5的复杂系统。

复杂的

在一个复杂的系统中，知识是我们未知的。这意味着我们意识到了自己的不确定性。

我们不能为系统本身制定计划，但我们可以为可能发生的某些事件提出建议或计划。反过来，我们只能知道发生过的事情。

但是这并没有使一个复杂的系统变得复杂。这只是对复杂系统的回顾，有助于你更加了解未知的已知。只要未知的已知没有发生，它们就仍然是未知的未知。

例子

我们带有红色按钮的棕色木箱得到了升级。它和另一个棕色的有红色按钮的木箱相连。所以如果有人按下按钮，两个盒子的状态都会增加1。而这个盒子将会给位于另一个国家的另一个人。

你现在处于一种你不知道已知 的 小丑会蹦出来的状态，但你永远不知道他什么时候会真的跳出来。

这是一个复杂的系统。

混乱的

混沌系统是一个未知的未知系统。你永远无法确定你的分析 和 知识是否能适用于未来的任何事件。你处于一种完全迷失和不断变化的状态。你前进的唯一方法就是不断地做出反应。

例子

现在我们改变这种行为。小丑完全随机地从盒子里跳出来，你只能通过按下按钮来关闭盒子。别忘了还有一个盒子的按钮和你的相连。你的目标是保持按钮关闭。

你永远不知道小丑什么时候会突然出现，你必须不断做出反应。但就算你反应过来，拿着另一个盒子的人可能已经反应过来了。

没有办法分秒必争或者靠感觉。你不知道未知的事物 ，因此你处于一个混乱的系统中。

观点

也就是说，系统的分类可以不同。例如，一个软件项目整体上是一个复杂的系统，因为人类总是给系统增加不确定性。

如果你把你的视角局限在整个软件项目中的一个功能上，这个系统就会变得复杂。如果你把它限制在函数中的一个变量上，这就变成了一个简单的系统。

总之，我们可以说较高复杂性的系统可以由较低复杂性的系统组成。但这并不意味着你可以把一个复杂的系统分解成简单的系统。

它总是取决于你的视角。
