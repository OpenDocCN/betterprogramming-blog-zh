## 了解概念验证、原型制作和 MVP

你对下一件大事有想法。你带着喜悦开始工作。你为它工作了很长时间，随着时间的推移，你慢慢地放弃它，转向下一个想法。再说一次，你在一个废弃的项目上花了很多时间。

原因很简单:你想先有一个完全工作的产品。你试图在脑海中构建每一个功能，并且从第一天开始就过于关心工作的质量。建造完美的东西需要很多时间。结果，随着时间的推移，你失去了兴趣，这个项目变成了一个越来越大的废物堆。

即使你完成了，也很有可能以一个不成功的产品而告终。

这并不意味着无论如何你都应该工作到完成为止。做正确的事情很重要。如果你从来没有事先验证过这个想法的可行性，你可能会花几个月甚至几年的时间在错误的事情上。你头脑中的想法和现实是不同的。

有很多方法可以解决这个问题，而且非常简单。你只需要遵循这三个核心步骤:

*   概念证明
*   样机研究
*   最低价值产品

所有这些步骤都有其自身的价值，并尽早防止我们走错方向。

让我们深入研究一下，并学习如何利用这些概念为我们带来好处。

# 概念验证

第一步是验证你想法的可行性。这一步是为了我们自己。首先，我们需要看看我们是否能实现这个想法，以及它是否能按照我们的设想发挥作用。

借此机会实现你的想法的一个非常粗略的版本。只是它的核心，看看你如何实现它。

建立一些显示项目可行性的东西。不需要有用或者像样。让它丑陋，让它无用。从这一步你将获得的最有价值的东西是知识。其他的就不要想了。

是学习的问题。弄清楚如何构建它，以及它是否如你所愿。你可能会发现事实并非如此。这样，你可以尽早找到正确的解决方案。

另外，看看你是否想继续努力。如果你没有兴趣，也许继续前进更好。在浪费几个月在这个项目上，却发现它对你来说不够有趣之前，这样做要好得多。

# 样机研究

这就是乐趣的开始。你已经验证了你的想法的可行性，并决定尝试一下。现在是时候实现一些工作和像样的东西了。

概念验证和原型之间的主要区别在于可展示性。有了 PoC，你不需要任何像样的东西。但是对于一个原型，最重要的是从人们那里得到反馈。它需要以一种显示你的想法的价值的方式可用。

最重要的部分是创造出能正常工作的东西。记住，这不是最终产品。如果它有很多错误或者不是你做过的最高质量的作品，这完全没关系。不需要超级花哨。试着尽可能快地构建一些东西，以获得对你想法的反馈。

这都是为了获得一些反馈，看看人们是否对此感兴趣。这将允许你和真实的人一起验证你的想法。您将利用这些反馈来塑造您的最终产品。

放手也很重要。发现这个想法在现实生活中并不像在你头脑中那样有效，这完全没关系。你可以从人们那里得到完全意想不到的反应。

即使面对现实有时很残酷，但这就是原型设计的全部意义。如果你能利用反馈来改进你的想法，那就去做吧。否则，就随它去吧。

我不是说你应该放弃你的梦想，仅仅因为一些人对它说了负面的话。但是如果你明白他们是对的，你看到事情并没有像你计划的那样发展，那么就没有必要在这上面花费更多的时间。

# MVP——最低价值产品

如果你已经走到这一步，你应该感到自豪。你已经用真人验证了这个想法的可行性和价值。你有可能有用的东西。现在是时候打造一款产品进入市场了。

这是你需要创造出高质量、新奇和适销对路的东西的一步，因为这是你要向人们要钱的时候。

您将在这里看到 PoC 和原型制作的好处。你已经知道如何实施，最重要的是，已经有了一些反馈。这将有助于你加快进程，并为你的潜在客户做一些有价值的事情。

人们经常会弄错这一步。你很有可能听到有人说，“让我们创建一个 MVP，MVP-1，MVP-2，等等。”人们经常将它与 PoC 和原型互换使用。其实根本没有 MVP-1，2 什么的。只有 MVP，是指为市场而投产的东西。

即使你需要为市场构建一些东西，那也不意味着你需要构建所有的东西。MVP 意味着最低价值的产品，这是有原因的。它是关于识别你的产品的核心特征。

等待完成产品的每一个环节再上市并不明智。达到这一点可能需要数月或数年时间。因此，最好是确定你的产品的核心功能，让它在市场上可用，并在人们已经在使用它的时候继续工作。这就是 MVP 的全部意义。让它尽快可用。

# 结论

了解所有这些概念以及如何从中获得价值是很好的。确保你在一个项目中至少一次尝试遵循每个步骤。你会尽早看到事情是如何变得明朗的。每一步都将帮助你为下一步或下一个伟大的想法节省时间！
