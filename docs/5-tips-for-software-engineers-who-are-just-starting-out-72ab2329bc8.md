## 这可能令人望而生畏，但这里有一些建议可以帮助你在混乱中导航

刚开始编程的时候，总是很迷茫。我经常感到失落和不知所措，这迫使我在学习不同的编程语言、不同的编码教程和不同的课程之间跳跃。我觉得我应该做所有的事情，但我不能完成任何一件事情。最终，我开始意识到这是一场灾难。我开始一次改变一种成分，最后，我能够成为一名优秀的软件工程师。

这里有一些对我有帮助的提示，我希望你也能发现它们有用。

# 1.编程语言并不重要

编程语言很重要，因为它是你与计算机交流时使用的语言，但它不是最重要的部分。你可以学习 Java、Python、Javascript、C++，但最终，这些语言之间的差别并不大。你很快就能发现它们。不要担心哪种编程语言是 2021 年最热门的，也不要担心 2025 年最热门的是什么。一旦你学会了一种编程语言，转换到一种新的编程语言是相当容易的。

这几乎就像学习英语、德语或西班牙语。一旦你知道当你*向某人*打招呼时会用到一个*问候词*，那么在任何语言中这样做都是相当简单的。每种语言都有问候语: *Hello，Hallo，或 Hola* 。编程也是如此——基础是最重要的，而且基础总是与语言无关的。

一旦你学会了编程语言的主要结构，比如赋值变量、循环、条件、方法、过程、数据类型——所有其他编程语言都有类似的东西。所以就挑一个开始吧。

# 2.找一个导师

纵观历史，导师的有效性证据随处可见。在大学系统接管之前，学徒制是教育的主要形式。一些世界上最伟大的艺术家和创造者师从他人。

导师会帮助你更好地理解材料，并打下更坚实的技能基础。当你与他们互动时，他们会在你当前的理解中戳洞，并对你的能力进行压力测试。他们有经验帮助你理解难懂的概念，当你遇到困难时，他们能给你指出正确的方向。

在最初的几年里，我有机会和那些会提供反馈、就各种话题提供指导的人在一起。我当时工作的地方有一个很好的导师制度，每个人都有一个导师。你不仅可以获得编程技能方面的反馈，还可以获得软技能、演示技能、如何撰写技术文档、与其他工程师有效合作等方面的反馈。它帮助我意识到并专注于我认为从未有过的问题。

# 3.不要和任何人比较

跟我重复:*我不会拿自己和其他开发者比。*

将自己与他人进行比较是普遍接受的自然行为，因为我们是以社会为导向的动物。但是一切适度都是好的，如果你发现自己在 LinkedIn 上查看其他人的个人资料，你需要停下来。总会有比你更聪明、更成功的人。你需要避免绝对会导致你失败的[有毒心态](https://levelup.gitconnected.com/5-mindsets-of-unsuccessful-developers-5d9bd5e4f700)。

通过观察他人获得灵感和动力是完全可以的，嫉妒就不行了。这两种情绪之间只有一线之隔。你永远都要记得尽你最大的努力。如果你每天都这样做，你就是金色的。

瞧，我给了你成功的秘诀！

# 4.学习在没有 IDE 的情况下编码

是的，这是你在工作中永远不会做的事情，但是请相信我，当你在没有 IDE 帮助的情况下编写代码时，你将自己置于一个更困难的境地。编写代码要困难得多，你要花很多时间来修改导入、缩进或语法。但这只是开始的时候。一旦你抓住它，它就会变得像奶油一样光滑。

此外，您还必须了解 IDE 能为您做什么，它如何编译代码，文件在文件系统中是如何组织的，以及依赖关系管理是如何工作的。

我绝对不是要求你永远不要使用 IDE，但是当你开始时，你可以考虑在文本编辑器中编码。就像学开手动挡的车，然后换成自动挡。希望有意义。

# 5.最后，不要急

如果你正在努力学习 Java，慢慢来，不要急于学习。花时间学习核心概念，不仅仅是编码语法和数据结构，还有指导语言的设计原则。你需要意识到 Java 花了 25 年才达到今天的水平，而且有一些非常有才华的人开发了这种语言。你不能只花一个月就说你懂 Java。你可以很快学会基础知识，这将帮助你解决编码问题，但从长远来看这是不够的。当你有一件事要做的时候，把你的时间和精力放在开始总是比较好的。很快你会有更多的责任，你将永远无法给它足够的时间。到时候知识的漏洞会一直困扰着你。这就是[冒名顶替综合症](https://www.google.com/search?q=imposter+syndrome&oq=impost&aqs=chrome.1.46j0l3j69i57j0l2j69i60.2834j0j7&sourceid=chrome&ie=UTF-8)如此普遍的唯一原因。

所以一定要深入挖掘。多花一点时间和精力去多了解一点。相信我，这将对你的职业生涯产生巨大的影响。
