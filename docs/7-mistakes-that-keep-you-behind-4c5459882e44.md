## 帮助你更有效学习的小贴士

![](img/928040f4cc5e30e962566d747e3f814a.png)

马库斯·斯皮斯克在 [Unsplash](https://unsplash.com/search/photos/code?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

学习 JavaScript 并不是每个人都感兴趣，尤其是那些没有耐心的人。有时候，人们会开始质疑这是否是他们正确的职业道路。

但是 JavaScript 从来就不是在公园散步。

好消息是，你的学习过程可以通过做出更好的决定和制定更好的学习方法来改进。*坏*消息？这不会自己发生的。唯一能改变这一切的人就是你。

本文将讨论开发人员在学习 JavaScript 时可能会犯的 7 个错误。

# 1.快速浏览概念

![](img/f613f5b2b902e70cbb70432af86cced5.png)

阿曼达·琼斯在 [Unsplash](https://unsplash.com/search/photos/alarm-clock?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

一旦你开始理解像[闭包](http://javascriptissexy.com/understand-javascript-closures-with-ease/)或[高阶函数](https://codeburst.io/here-are-7-ways-higher-order-functions-can-improve-your-life-a392aa6b29d2)这样的概念，在没有获得一些实践经验之前，跳到下一课几乎从来都不是一个好主意。

我知道，转移到下一个话题可能很诱人——这种事每个人都会遇到。但是相信我，如果你先花一些动手的时间编码，哪怕只有五分钟，它会让将来构建项目变得更容易。通常情况下，对一个概念进行编码的感觉会留在你的脑海中，帮助你长期记忆。

如果你匆忙地浏览概念，你最终会忘记，不得不通过搜索旧材料来刷新你的记忆。当这种情况发生时，当你已经花时间学习该科目时，失去宝贵的时间是令人沮丧的。然后那些消极的想法开始堆积在你的脑海里…

这很重要，因为人们经常说他们[因为*时间*不足而](https://javascriptissexy.com/a-significant-number-of-students-quit-their-programming-education-find-out-why-before-you-sign-up-for-a-program-and-suffer-the-same-fate/)退出了他们的编程生涯。

当一个概念对你来说是小菜一碟时，你应该准备好继续前进！

# 2.不拥有你的代码

![](img/869a01a115cd9c40f1b645741634d67a.png)

照片由[丹尼尔·里卡洛斯](https://unsplash.com/@ricaros?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText)在 [Unsplash](https://unsplash.com/search/photos/coding?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄

我今天仍然看到的一个常见现象是人们变得过于依赖他人的工作——无论是从教程中复制另一个人的文件，还是使用 lodash 库做任何事情。虽然在很多任务中使用 lodash 并没有什么错，但是如果你自己没有应用过这些概念，从长远来看，你也可能会让自己落后。概念是*将会坚持其他工具/框架/语言*的东西，因为它们是*在编程中思考的方式*。

这就是为什么很多面试仍然给出编码挑战的原因——他们想看看你是如何思考的，或者你是如何将概念应用到开发中的常见问题上的。您可以单独使用组件库来创建最好看的用户界面，同时让 lodash 为您处理应用算法，但是当问题出现时，它们需要您准备好应用概念并解决它们。他们不期待 npm 安装另一个库来解决他们的问题。他们需要你。

你不希望在应用已经学过的概念时遇到困难。所以花一点时间自己练习是个不错的主意。*拥有自己的代码*！

# 3.过度思考一切

![](img/7cde53611a490d1ff48d345d565e1287.png)

盖尔·马塞尔在 [Unsplash](https://unsplash.com/search/photos/piles-of-books?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

有了网上可以找到的所有信息，在学习 JavaScript 的下一个主题之前，坐下来收集所有你需要的资源是很有诱惑力的。

但是我不认为我曾经听任何人说过那是学习和变得更好的方法。我们似乎有这样做的冲动，因为我们希望所有的信息都在我们面前，所以我们觉得今年剩下的时间都准备好了。

*都在你脑子里*。

只要你觉得自己是 web 开发之王，就不要忙于收集 80 个教程、6 本电子书、超过 50 个资源书签，并试图开始一起解决它们。

如果你正在学习 JavaScript，动手学习吧——刚开始写东西。利用你在过去一个小时中学到的知识，为自己举一个小例子。这些活动会比你收集的马拉松式的学习材料更长久地留在你的脑海里。

# 4.用太多的信息淹没自己

![](img/af3fd96338f927c62f9bcc863c8983f9.png)

[吕山德元](https://unsplash.com/@lysanderyuen?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText)在 [Unsplash](https://unsplash.com/search/photos/complicated?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍照

当你花大量的时间来获取信息，而没有在中间编码时，你实际上是让自己在以后回到材料时忘记了。

你会觉得自己在学习。但是几个月后当你最终忘记的时候会发生什么呢？你会发现，由于某种原因，你无法应用几个月前学过的概念和材料。

如果你需要一个指导方针，每消耗一个小时的信息，至少花 15 分钟编码。

# 5.将自己与更高级的开发人员进行比较

![](img/27f6ca29cf54e25eeb9419e608b00d77.png)

由 [Valeriy Khan](https://unsplash.com/@valeriydmi?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 在 [Unsplash](https://unsplash.com/search/photos/programmer?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

当你将自己与他人比较时，你不会去想他们是如何到达那里的——只是想他们现在处于什么位置。

你是否认为所有成功的程序员都是从编写干净、棒的代码开始学习的？

继续关注你现在正在学习的东西，你会不断进步的。很快你就会处在一个人们开始怀疑你是如何到达那里的位置。

# 6.学习基于 JavaScript 构建的工具和框架

![](img/e41ecc064ece77f767843419bdfa5090.png)

法比安·格罗斯在 [Unsplash](https://unsplash.com/search/photos/coding?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

你应该学习 JavaScript，但是你却马上学习了 React/jQuery/Angular/Vue。当一种新的革命性工具问世，而你做出了改变，会发生什么？你最终会依赖别人来找出这些巧妙的技巧，并希望有人分享它们，因为他们学会了 JavaScript，而你没有！

如果你还没有牺牲大量精力来学习普通的 JavaScript，我强烈推荐你*真的*学习这门语言。如果你学习这门语言，那么你应该理解在它之上构建的工具，*为什么*它们被构建，以及*它们解决什么* *问题*。学习 JavaScript 中的 *why* 也将帮助您避免做出糟糕的代码设计决策。

我职业生涯中犯过的最遗憾的错误之一就是在开始学习 JavaScript 之前就学习 jQuery。当 React 出现时，我也跟风开始学习 React。然而，在我意识到我不知道为什么我必须使用 react 或者为什么人们需要它之后，这变成了一个非常尴尬的学习过程。它突然变成了一个记忆 react 代码的游戏。在 JavaScript 工具或框架中，许多重要的部分都被隐藏起来了。

# 7.不要把概念分成更小的部分

![](img/9598a9986c781ab3ac9f9477d9a99b1c.png)

阿尔瓦罗·雷耶斯在 [Unsplash](https://unsplash.com/search/photos/plans?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

学 JavaScript 就像小时候学数学一样。一旦你学会了加法、乘法、除法、减法、[的基础知识，你就可以继续使用 *>(大于)**<(小于)*【长除法】、基本代数等来比较数字。当你发现自己难以理解某件事时，往往是因为你跳得太远了，而不是迈出一小步。不先学习基础知识，直接学习代数是不可能的！](https://www.understood.org/en/learning-attention-issues/signs-symptoms/age-by-age-learning-skills/math-skills-what-to-expect-at-different-ages)

通过采取小步骤，当你遇到一个对你来说没有意义的概念时，你会更容易获得帮助，因为你已经缩小了主题的范围。
