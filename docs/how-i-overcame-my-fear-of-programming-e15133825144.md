# 我如何克服对编程的恐惧

## 给 18 岁的自己的一封信

![](img/53752f106bc04958c385fb9289f0a3b0.png)

萨法尔·萨法罗夫在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片。

2015 年我写了第一个 C 程序。我仍然记得，一旦我完成了，即使是最基本的程序，我也会感到欣喜若狂:

```
#include <stdio.h>int main()
{
 printf(“My name is Derny”);return 0;
}
```

我们的老师在这个领域有 40 多年的经验，我仍然记得他们给我们读了几页由 Brian Kernighan 和 Dennis Ritchie 写的 C 编程语言的*。在这学期的大部分时间里，我们学习了数组、指针、流控制、字符串以及所有这些好东西。这个理论不成问题。对我来说，记住题目并在考试中解释它们总是很容易。当我们有我必须编码的项目时，真正的问题出现了。*

现在想想，这些项目非常简单。然而，在那个时候，我没有现在这样的经历。我差点挂了那门课，因为我写不出一个有意义的函数。

这个实验让我反感代码。一有机会，我就尽量避免上编程课。我找到了对[网络](https://www.cisco.com/c/en_ca/solutions/small-business/resource-center/networking/networking-basics.html)的热情，并告诉自己不需要编程技能。

今天，我为那些因为不相信自己能成功而感到绝望的年轻开发者写这篇文章。我是来告诉你一切都会好起来的。

# 走出你的舒适区

学习新事物的唯一方法是走出你的舒适区。无论你是想去掉自行车上的训练轮，还是想成为谷歌的首席工程师，你都必须采取行动，走出你的最佳状态。在您的代码编辑器中，可能发生的最糟糕的事情是什么？终端出错了？引发了异常？如果你犯了一个错误，世界不会停止转动。

网络相关的课程对我来说很容易，我觉得很舒服。编程项目不是。我害怕他们，甚至没有花时间去理解发生了什么。我的手心出汗了，我不敢问问题，看起来像个哑巴。

在我的第二年，我开始意识到一些事情:最好和最坏的可能结果是什么？最好的是我终于懂了 C 语言，最坏的是我用问题惹恼了老师和朋友。我带着我的骄傲，把它深藏在我的背包里，迎接挑战。

> "经验是每个人给自己的错误起的名字。"——[*王尔德*](https://www.goodreads.com/quotes/7201477-experience-is-the-name-everyone-gives-to-their-mistakes)

# 经历失败

我想告诉年轻时的自己的一件事是，学习来自错误。每个程序员都会犯错误。这是生命的循环。你犯了错误，从中吸取教训，然后变得更强。

那时候，我有不同的心态。我不能评判我的失败，而是停止尝试。我没有花时间一头扎进堆栈溢出论坛，阅读代码，并试图理解。我什么都没做。这是拖延的一种形式。我告诉自己，“如果我不尝试，我就不会失败，我会保持快乐。”然而，我可以向你保证，没有比找到一个解决你几个小时的 bug 的方法更令人满意的了。

从失败中学习在我们这个世界上很普遍。看看一些亿万富翁的故事就知道了。你很少第一次就有所收获。关键是要走出失败是一件坏事的想法，去写一些代码。通过练习，你会变得更好，通过记下让你成功的系统，将它重新应用到其他元素，会更容易找到其他解决方案。

# 专注于基础

世界上有将近 300 种编程语言。其中一些是为银行界而创建的，另一些是为 UI 和组织页面而创建的，还有一些是为了随意使用(比如[空格](https://en.wikipedia.org/wiki/Whitespace_(programming_language)))。到今天为止，我可以说我精通几乎五种语言和一些框架。

当我开始上大学的时候，我学习了 Java。OOP 的概念对我来说是一个谜。一旦我开始编程，我意识到我缺乏基本的编程知识。这需要一些时间和大量的练习，但是我很快认识到所有的编程语言都关注这些概念:

*   数据类型和数据抽象
*   控制结构
*   句法
*   循环和迭代
*   功能

一旦你有了这些想法，学习一门新语言应该是小菜一碟！

# **最后的想法**

我对编程的恐惧让我无法完成伟大的事情。通过走出我的舒适区，拥抱失败，专注于基础，我现在可以在技术领域发展了——你也会的！
