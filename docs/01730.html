我希望在开始编码之前就知道的10件事

原文：https://betterprogramming.pub/10-things-i-wish-i-knew-before-i-started-coding-c35a58855b7e?source=collection_archive---------0-----------------------#2019-10-09

我知道我不是唯一一个因为不知道从哪里开始而耽误了软件工程之旅的人。

过多的资源提供了一个不可思议的机会，但也可能使迈出第一步变得令人生畏。对于那些正在考虑迈出第一步的人，或者已经迈出第一步的人，这里有10件事是我希望在开始之前就知道的。

1.选择一种语言很重要

外面有许多编程语言。每种语言都有它的长处、短处和最适合的场合。

虽然你的第一种语言几乎肯定是最难学的，并且许多最基本的基础知识在不同的语言中是相似的(变量、迭代等)，但花时间和精力做一些研究并决定你想从哪种语言开始是值得的。

编程的便利性是一个必须考虑的因素。诚实地告诉自己你有多自信或害怕。如果选择一门更复杂的语言最终会阻止你继续学习，那么选择一门你一开始能更容易获得成功的语言可能会更好。

另一个重要的因素是你为什么学习软件工程。如果你有一个想要开发的应用程序的想法，从在早期阶段和大规模地思考该应用程序的逻辑开始，并研究哪种语言和技术最适合你的需求。

如果你希望以软件工程师的身份加入一家公司，列出10-20家你想加入的公司，找到他们招聘软件工程师的职位，注意他们使用的语言。

2.所有语言都有官方文档

语言不仅仅是被创造出来，并被送到生态圈，希望它们能被理解。从最基本的概念到最复杂的应用程序，语言、框架和库在网上都有完整的文档。我在下面列出了一些链接，这样你就可以看到它们的样子了:

【Python.org T4】

RubyonRails.org

Reactjs.org

3.谷歌是你的朋友，你们只会越来越亲密

每个软件工程师在遇到困难时都会使用谷歌。这不是一个初学者该做的事情，也不是一件该走的事情。很可能，你不是第一个收到你所面临的奇怪的错误信息或错误的人，如果幸运的话，那些在你之前的人已经去过StackOverflow.com或类似的地方获得反馈并解决他们的问题。

没有谷歌和其他资源，从A到B在软件工程中毫无意义。只有使用干净的代码、最佳实践和可伸缩的算法才能有效地到达B。

4.这不是魔术——这是解决问题，而且是可以学习的

很多人看着web应用程序，觉得它们很神奇。

他们不是。它们是成千上万行精确编写的代码，用来执行单个功能，并在整体中完美运行。

像任何东西一样，像任何其他语言一样，它是可以学习的。这一开始毫无意义——就像法语或汉语一开始对非母语人士毫无意义一样。但是就像这些语言一样，你会一点一点地学会如何用它的语法和措辞来表达你的信息，并学会通过它来交流你想要的任何东西。

5.你永远不会知道一切

你永远也不会知道关于软件工程或者任何一种语言的所有知识。

这在一定程度上是因为要知道的东西实在太多了。但这也是因为要知道的东西总是在变化。

新版本的语言会定期发布不同的功能和更新。新的框架和库出现了，成为当前的 it 框架，每个人都想使用它。发现更好的实践来代替当前的最佳实践。

软件工程唯一不变的是它总是在变化和发展，最好能马上适应这个事实。

6.GitHub是一个东西

GitHub 是一个远程版本控制系统，与本地版本控制系统Git协同工作。

你应该从一开始就有一个帐户，并应熟悉和舒适地使用这两者。

在整个项目中，经常会有这样的时候，你需要解决一个问题，但不知道如何解决，或者只是想尝试一下，看看感觉如何。

Git允许您这样做，而不会有破坏工作代码的风险。它允许你从主项目中分支出来，本质上是创建一个单独的代码版本，你可以在离开主项目的同时，完整地处理分支，还允许你选择你实际提交给项目的内容。

您可以尝试解决问题的这种方法和那种方法，文件会被保存，但是在您实际执行git提交之前，它不会被保存在主git分支中。

GitHub只是您的本地Git存储库的远程存储系统(这也意味着，是的，对于每个项目，您需要一个本地Git repo和一个GitHub repo)。把它想象成代码的DropBox。这样，如果你的电脑出了什么问题，你写的所有珍贵代码都不会丢失。

7.HackerRank、LeetCode和Codewars也是东西

这三个站点(当然还有其他站点)有无尽的代码挑战，允许您磨练自己的技能。

一旦你觉得可以轻松地编写基本代码，你就可以开始学习如何使用代码解决问题。它们不仅有助于磨练你的语法和语言写作知识，还有助于你解决问题的技能。

我个人很喜欢LeetCode，因为它给了你关于解决方案质量的真实反馈，而不仅仅是你是否解决了它。不过，我也经常使用 HackerRank ，只是因为我喜欢它提供的挑战。

8.熟能生巧——或者至少是一个能干的软件工程师

你永远不会完美，也永远不会无所不知。但是我不能过分强调练习，练习，练习的重要性。

和其他人一样，有时候我觉得我在努力学习一些东西，但就是学不进去。这会让你感到沮丧，让你想放弃，继续下一个目标。但是我保证如果你坚持下去——即使感觉毫无进展也要坚持练习——最终你会有成功的一刻。突然间一切都变得有意义了。去那里可能很难，但这是值得的。

9.语言不同于框架，框架不同于库

你会在极其相似的情况下听到术语语言、框架和库，但它们不是一回事。语言是最原始的编程形式(除了二进制)。它最终被编译成1和0。语言的例子有Ruby、Java、Python和JavaScript。

库和框架建立在语言之上并使用语言，但它们也不是一回事。它们都抽象了构建它们的语言，以提供解决常见问题的解决方案。这通常是以提供通用方法的形式，这样就不必每次都编写它们，或者提供命令行选项来生成新应用的框架。主要区别在于谁在控制。

使用一个库，软件工程师保持控制并决定他或她想在哪里使用这个库。他或她控制着应用程序的流程。库的例子有React.js和Redux，它们都是JavaScript库。有了框架，应用程序的流程是预先确定的。Rails是一个常见的Ruby框架，对于Python来说，Django也是如此。

10.这不仅仅是记忆语法

软件工程不仅仅是记忆和知道语法。

编码，当它按预期工作时，是容易的。当你可以发誓你写的代码是正确的时候，知道如何调试。这是在解决一个复杂的问题。这就是挑战和乐趣所在。

要做到这一点，你不仅要理解a + b = c，还要理解 why a + b = c。如果你不理解，总有一天你会把a和b加起来，得到d，却不知道为什么，也没有工具来找出原因。

每种语言的细微差别以及一行代码是如何工作的是如此重要，因为它有时会导致意想不到的后果。
