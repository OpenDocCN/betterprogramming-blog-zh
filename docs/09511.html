摆脱技术问题的框架

所以你得到了你梦寐以求的软件工程实习。完成了所有的在线评估和面试后，终于到了展示你的技能的时候了！

但是，像生活中的所有事情一样，实习旅程并不总是阳光和彩虹。除非你是某个特别的软件，否则你可能会偶尔遇到看起来不可能克服的问题。

在我最近的实习中，我确实做到了。下面是我用来克服我自己的许多技术挑战的七步框架。

问题解决框架

  1. 识别子问题
  2. 寻找类似的例子
  3. 钻研源代码
  4. 刚开始写
  5. 休息一下
  6. 考虑你的背景
  7. 伸出援手

识别子问题

当面对一个似乎无法解决的问题时，试着把问题分解成容易理解的部分。

手里拿着笔和纸，问自己一些能揭示任务/问题总体目标的引导性问题。这些问题应该从简单和高水平开始。

“我想要构建什么？”

“我需要具备什么样的开始条件？”

“我需要哪些零件来实现这个任务？”

高层次的计划让我们能够冷静下来，并确定我们任务的重要方面。我们的目标是在我们更大的问题中定义清晰的、可行的、简单的子任务。

当我们确定更多的任务时，我们要考虑它们是否可以实现。如果没有，那么重复同样的一系列步骤，将子任务分解成更小的子任务。

在这个过程的最后，你应该会从我们最初的任务中得到一个子问题树。我们树上的叶子代表了我们可以轻松完成的所有子任务。把这些扔在看板上，然后开始吧！

寻找类似的例子

当构建新的软件组件时，示例会非常有用。努力搜索与您试图构建的模块的功能相匹配的类似代码片段是值得的。

帮助您搜索这些代码片段的一些资源包括:

  • 贵公司的代码库(我喜欢VScode搜索功能。)
  • YouTube教程
  • 堆栈溢出
  • 文档(编程语言、库、框架)
  • GitHub
  • 塔布宁搜索代码

以他人为榜样有很多好处。首先也是最重要的，它可以很容易地让你从问题中解脱出来，加速开发。

然而，我发现示例是了解不同库的最佳实践的好方法。把它作为学习实现不同特性的新方法的机会。

深入源代码

使用新的库/框架时有时无。如果您足够幸运，能够使用文档良好的库，开发会非常顺利。然而，我们并不总是那么幸运。

在我的最后一次实习中，我发现自己在使用一个不太为人所知的GraphQL框架。文档是乏味的——这使得开发令人沮丧。

对于这种情况，查看库的源代码来寻求帮助是非常有用的。这听起来可能很可怕，但源代码最终是事实的来源。

我们可以使用VScode的定义搜索(右键单击一个函数/类名)直接跳转到相关函数/类的代码中。

通过理解我们使用的函数/类的内部工作方式，我们甚至可以开始操纵它的功能来更好地满足我们的需求！

开始写吧

听起来很老套，开始是开发过程中最难的部分。盯着屏幕和头脑风暴可以帮助我们想出有效的方法，但我们最终还是要开始编码。

如果你发现自己没完没了地计划、阅读文档，并对从哪里开始感到紧张，那么可能是时候进行一次大胆的尝试了。您可以采取的几种方法是:

  • 将打印语句添加到代码库中，以了解更多关于系统中信息流的信息。
  • 复制类似的代码示例，并慢慢调整它们以适应您的任务需求。
  • 在空白项目目录中构建您的软件组件。这有助于简化问题。之后，您可以将它移植到代码库中。

我们希望遇到新的信息，这些信息可能会带来解决问题的新想法和新方法。有时候获得这些线索的唯一方法就是弄脏我们的手。

休息一会儿

如果你没有取得很大的成功就达到了框架的这一点，是时候休息一下了。花几个小时苦思冥想同一个问题会变得极其消极和低效。

休息并不意味着不工作。当我对一个问题感到厌倦时，我喜欢开始做一个不同的任务。这种转变让我释放了很多压抑的压力。

在做一个单独的任务时，最好是一个比较轻松的任务，我喜欢下意识地继续思考之前的问题。想到的任何随机笔记，我都会记在一张纸上以备后用。

通过利用有计划的拖延，我们将有希望提出创造性的想法和解决问题的方法。

当我们最终准备好回到之前的问题时，我们会有一双新的眼睛和一个可能的线索列表。

考虑你的背景

我们的背景也许能为新的解决方案提供额外的见解。

如果你学过很多计算机科学的课程，也许可以考虑不同的数据结构或算法如何帮助你从新的角度解决问题。

如果你在另一个不同的领域有经验，想办法将你遇到的其他系统运用到当前的问题集中。

在我最后的实习中，我不得不在球形物体上做一些有趣的坐标转换。这立刻引发了我内心的数学系学生。我能够利用我目前的研究提出一个新颖的轻量级解决方案！

伸出援手

如果其他方法都失败了，是时候寻求帮助了。

现在，大多数实习会给你提供一个导师或经理，但我有几个建议可以先尝试一下。

论坛式的网站是很好的资源。像“堆栈溢出”和“T2”这样的地方，有很多有经验的人愿意帮忙。只要确保在适当的subreddit (Reddit)发帖，尽量不要重复提问(栈溢出)。

当使用特定的库时，考虑联系维护人员。一些图书馆的GitHub页面有一个讨论标签，用户可以在这里提问，展示项目，以及讨论与图书馆相关的话题。我发现在这里提问也很成功。

最后，如果你足够幸运有一个实习队友，确保你接触到他或她！

无论你向谁求助，都要试着理解他们是如何得到答案的。我们想了解其他人是如何解决难题的。

这个解决问题的框架帮助我在上一次实习中取得了成功，我相信它也能帮助你。

祝你好运，记得玩得开心，享受实习经历！
