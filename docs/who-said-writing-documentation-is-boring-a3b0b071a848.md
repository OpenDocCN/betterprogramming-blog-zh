文档通常被认为是我们工作中枯燥、耗时和困难的方面。但这很重要！

在这份每日文摘中，我们将浏览一些指南、工具和技巧，让写作和阅读文档再次变得有趣起来！

[数据文档困境？这是一个框架](https://towardsdatascience.com/data-documentation-woes-heres-a-framework-6aba8f20626c)——一个老数据团队成员决定退出的那一天， [Prukalpa](https://medium.com/u/38a5ef6ab673?source=post_page-----a3b0b071a848--------------------------------) 感到震惊。更重要的是，因为团队成员知道他们使用的数据的所有上下文。用她自己的话来说:“*当我第二天早上醒来时，我记得我对自己做了一个承诺——我再也不会给一个人把我们置于如此脆弱境地的权力。”。*这一事件标志着 Atlan“装配线项目”的开始，这是一项使数据团队尽可能敏捷和有弹性的计划，从而培养了文档优先的文化。

[![](img/4a238d14b1fd0acfdfceb1ebe411e0f5.png)](https://towardsdatascience.com/data-documentation-woes-heres-a-framework-6aba8f20626c)

接下来，阅读 [Trevor-Indrek Lasn](https://medium.com/u/ce572601b7e?source=post_page-----a3b0b071a848--------------------------------) 的[指南，使用 DocZ 工具](/docz-its-never-been-easier-to-document-your-project-8026ed0a5097)编写和发布漂亮的交互式代码文档。由 Gatsby 提供支持的 DocZ 速度惊人，让你可以创建和使用真正的可定制主题。它还内置了对 TypeScript 的支持。

如果您希望添加图表以使您的文档在视觉上更具吸引力，那么可以使用 Mermaid——一种简单的类似 markdown 的脚本语言，用于通过 JavaScript 从文本生成图表。[阅读本指南，了解如何使用它](/mermaid-create-charts-and-diagrams-with-markdown-88a9e639ab14)。

当然啦！在编写文档时，Markdown 是开发人员事实上的工具。尤其是因为你可以使用 VS 代码扩展来使你的编辑更快。从添加预览、键盘快捷键、自动预览到使用 PlantUML 来粘贴图像和表情符号，这里有五个 VS 代码扩展[，它们将大大提高你的降价幅度](/top-5-vscode-extensions-to-boost-productivity-when-using-markdown-b4d1bbeb7175)。

此外，您可以通过使用 Remarkables 在 Markdowns 中突出显示您的代码，remarkable s 是一个 web 组件，有助于漂亮地显示代码，同时还允许您设置自定义解析器来动态突出显示代码。[这里有一个由](/highlight-markdown-code-with-remarkable-and-a-web-component-eaf4d69e19b6)[大卫·达尔·布斯科](https://medium.com/u/94f0c8061324?source=post_page-----a3b0b071a848--------------------------------)撰写的如何让你的减价代码“不同凡响”的指南。

需要更多灵感来驾驭文档中的代码？ [Victoria Drake](https://medium.com/u/d838624534b3?source=post_page-----a3b0b071a848--------------------------------) 向我们展示了[通过使用 Go 和 GitHub 动作](/go-automate-your-github-profile-readme-d0ce2252f8aa)创建一个动态降价文件来自动化 GitHub profile README。

如果有一个“顶级降价编辑器”列表， [bshelling](https://medium.com/u/e8011fe483e6?source=post_page-----a3b0b071a848--------------------------------) 建议[向其中添加 Visual Studio 代码](/if-theres-a-top-markdown-editors-list-add-visual-studio-code-to-it-ae68f8a4adb4)。

为了提高 LWC 的文档质量，使用 JSDoc 库，正如 Sebastiano Vierk 在他的故事中所展示的。

还觉得编写代码很耗时吗？James Briggs 向我们展示了如何为任何 Python 项目构建自动文档工具。不要错过他的 Python 文章的[自动文档。](https://towardsdatascience.com/auto-docs-for-python-b545ce372e2d)

说到 Python，你可以按照[路易·德·布鲁因](https://medium.com/u/eb6696b15003?source=post_page-----a3b0b071a848--------------------------------)的指导开始[编写 Python 文档字符串](/the-guide-to-python-docstrings-3d40340e824b)。或者前往[崔永](https://medium.com/u/88ff1e2545d0?source=post_page-----a3b0b071a848--------------------------------)的[定义函数文档串的最佳实践](/how-to-write-proper-docstrings-for-a-python-function-7c40b8d2e153)。

很难忽略 Julia——尤其是当 Python 成为我们讨论的一部分时。使用 [Emmett Boudreau](https://medium.com/u/ea170050148c?source=post_page-----a3b0b071a848--------------------------------) 指南[用 Documenter.jl](https://towardsdatascience.com/how-to-automate-julia-documentation-with-documenter-jl-21a44d4a188f) 自动化 Julia 文档。

你知道苹果公司通过 DocC——一种从 Xcode 编写文档的工具——使文档编写变得更加容易吗？查看 Amit Samant 的指南首先看看有前途的新苹果工具。他还写了第二本指南，展示我们如何通过在网络上托管 DocC 文档来与世界分享我们的文档——并在网站和代码库之间同步文档。

如果你来自 AWS 背景， [Allen Helton](https://medium.com/u/506242edfbaf?source=post_page-----a3b0b071a848--------------------------------) 带我们了解[如何使用异步 API 规范](/aws-websockets-writing-documentation-using-async-api-spec-6c4ccc77f20)正确地记录事件驱动架构。“由于异步 API，我们能够清楚地说明我们的事件模式是什么，各种通道是什么，消费者可以发布或订阅哪些事件，以及发布或订阅需要什么服务/介质(EventBridge 或 WebSocket)”。另外，如果你喜欢智能合同，请看[如何轻松地记录你的可靠性代码](https://medium.com/coinmonks/how-to-document-solidity-smart-contracts-with-no-sweat-e78936ede833)。

还觉得懒写文档？让人工智能替你写。Hahnbee Lee 和她在 Mintlify [的团队引入了 AI Doc Writer，这是一个 VS 代码扩展，它通过让工具来处理文档来努力实现自动化。只需在 mac 上突出显示代码并按下`Command + .`即可生成该功能的文档。我在 GitHub Copilot 生成的函数上测试了它，结果还不错。](/too-lazy-to-write-documentation-let-the-ai-write-it-for-you-8574f7cd11b2)
