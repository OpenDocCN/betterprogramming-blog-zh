公司断然拒绝你的申请而不给你任何暗示是很常见的。然而这一次，我得到了真实的反馈。

作为一个相当新的开发人员，来自非技术背景，我的学习曲线非常陡峭。

我曾经在一次交接技术测试后的选拔过程中被拒绝。我被要求创建一个全栈应用程序，它读取 25 条销售记录的列表，并返回一个可过滤的分页表(一次 6 条记录)。

主要要求是:

*   用 Node.js 和 Express 创建一个后端
*   用我喜欢的 JavaScript 框架创建一个前端
*   有一个持久层

这是小菜一碟。我相当快地创造了功能性的东西。经过一些彻底的测试，并确定它几乎像素完美，我交付了它。

一周后，我被拒绝了。

# 主要的教训

如果你已经迈出了谈话的第一步，很有可能公司已经知道你会编程了。他们可能已经看过你的工作经历和 GitHub 档案。重点不仅仅是让最终产品发挥作用，而是实际可行的部分。

我并不是说编码很容易，但是很多人都能做到。这不一定是你会脱颖而出的地方。

关键是，你能可持续地编码*吗？你能创造一个经得起时间考验的结构吗？你能以这样一种方式组织你的项目吗？今天有 1，000 人访问，但是几个月后会有 1，000，000 人访问？*

嗯，我不能。

# 为什么我被拒绝了

## **我没有创建自动化测试**

这是特别痛苦的一个。

我在过去学过如何做到这一点，但最终只是打印/控制台。记录事情看起来简单多了。当你考虑团队的时候，这并不是最优的，但是像我在早期开发时那样作为一只孤独的狼来开发，这就变成了一个简单的解决方法。

不管是哪一种，我都忍不住觉得有点固执回来咬我了。我不得不再次学习它，并开始应用它，这样我就不必重复这个过程。

确保你练习了自动化测试，并且用它们完成了你的课后作业。

## 我没有适当关注软件架构模式

我还得提一下，我从来没有真正学过这个。这大概是大多数像我这样自学成才的开发者的一个盲点。

当我 12-13 岁的时候，我开始将编程作为一种爱好，但是生活的机遇让我获得了工商管理学士学位和数据分析的职业生涯(你可以在本文中查看我是如何成为一名开发人员的[)。再多的自动化 Excel 表格或在 web 应用程序中显示图表都没有让我为真实的健壮开发世界做好准备(没什么好惊讶的，是吧？).](https://towardsdatascience.com/i-switched-careers-mid-pandemic-twice-heres-what-i-learned-567715ac405c)

特别是在这种情况下，架构模式是这家公司所期望的很大一部分。从他们告诉我的情况来看，只要我遵循一个已建立的架构，它就不必是一个特定的架构。

有机会的话，学习 MVC(模型-视图-控制器)之类的东西，确保应用。

# 结束语

有时候，让事情发生会让你觉得你已经为找到一份工作和/或提高效率做得够多了。虽然这是一个惊人的壮举，但你可以肯定的是，总有改进的空间。

在软件开发中有很多东西需要学习。语言和框架来来去去。每六个月就有一种新的趋势技术，人们会声称它是下一个伟大的东西(还记得 Deno 吗，甚至一年前？).

我的观点是，不管你是做两天还是五年，当一个崭露头角的新技术开始站稳脚跟时，你必须像其他人一样学习它。

如果你在竞争初级职位，你可能不会遇到和我一样的问题。然而，如果你最近一直在找工作，你就会知道初级职位的机会少之又少。如果你想真正确立自己的地位，确保你知道如何让你的应用程序高效地成长。自动化测试是其中的一部分。

同样，语言和框架来来去去。然而，不变的一点是可持续和可伸缩的软件结构。确保你在关注这些的同时也关注你的编码技能。

祝你好运，继续编码！
