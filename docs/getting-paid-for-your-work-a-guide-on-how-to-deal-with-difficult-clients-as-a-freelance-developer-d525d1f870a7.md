在今天的文章中，我将谈一点在自由职业世界中与客户打交道的事情。这是为那些经营自己的生意或从事自由职业的人准备的。这也适用于那些为别人工作或必须直接与客户打交道的人。我会提到一些你可能会遇到的情况，同时给你一些在这些情况下可以帮助你的提示。

# 廉价客户

让人们意识到他们支付的是什么是非常困难的——尤其是当他们不知道其中包含什么的时候。许多客户不遗余力地试图与你讨价还价，尽可能节省每一分钱。

如果你是一个初学自由职业者，你会有人要求你免费为他们建立一个网站。他们会试图说服你，与他们一起工作会给你更多的经验，让他们的名字或品牌出现在你的投资组合中会对你有好处。尽管如此，你的技能还是有价值的。

一开始我就多次上当，除非他们对你的工作感到惊讶，否则他们不会给你推荐或以那种方式帮助你。对于这类客户，最好的办法是态度坚定，不要让人占你的便宜。你要有自信。不要低估自己。我也不是说你应该对你的服务定价过高。诀窍是根据项目描述有一个最小值。

如果你觉得你没有得到你所提供服务的价值，你应该离开，因为你可能会同意接受这个项目，然后交付一个低质量的产品。不是所有的客户都是这样的。一些客户会同意一个价格，然后试着分析你所有的服务、工作和花费的时间。他们会试图指责你多收了他们的钱或者没有装满你应该装满的东西。你需要创建一个具体的提案，描述工作的确切范围和你愿意拿多少钱来处理这个项目。

永远记住要提前要求押金。一些客户会觉得报价的 50%很多。没问题。收取三分之一作为押金，三分之一作为中点里程碑，最后再收取三分之一。无论你做什么，都要用口头和书面的方式表达清楚。尽早开始处理合同。

# 似乎总是匆匆忙忙的客户

这些类型的客户端通常不知道 Node.js 应用程序中包含了什么。他们想要一个 500 美元的大规模项目在 30 天内完成。这里的问题是你的客户认为他们的项目是你唯一在做的项目。事实上，你正在和另外五个人玩杂耍，就像它希望每天多 48 个小时一样。

这些类型的客户会打电话和发电子邮件来烦你，询问进度和预计到达时间，即使在项目范围中已经指定了。让客户阅读甚至理解你给他们的细节是很难的，因为大多数客户希望事情尽快完成。他们甚至会在下班后或周末给你打电话。

看，对于这样的客户，自由职业意味着你提供 24/7 的支持，而处理他们的唯一方法就是通过沟通。确保你建立了这些界限，给他们一封支持邮件，告诉他们你从周一到周五从 9 点到 5 点或者其他什么时间查看它。总是提供一个延长的 ETA，而不是你完成项目所需要的时间。

另一件重要的事情是指定你愿意修改的次数。这样做的原因是一些客户会试图从你身上得到越来越多的东西。有些人不知道修改一个网站、用户界面或某些功能需要多长时间。

# 不知道自己想要什么的客户

有些客户擅长准备、商业计划、线框和标识。与这样的客户一起工作或为他们工作令人耳目一新。光谱的另一端由没有目标、没有例子、没有内容——什么都没有的客户组成。这些是最棘手的一批人。有些人甚至会要求你在这些方面提供帮助，即使你明确表示不提供商业咨询服务。

对于这类客户，无论是口头上还是书面上，都要非常清楚你提供的服务。

作为开发人员，您需要了解项目的所有关键资产。大致了解有多少页，他们想要什么样的配色方案，诸如此类。如果他们不知道，建议他们提供更多信息，然后让他们给你回电。

如果你自作主张为他们创造一切，很有可能他们最终会讨厌这样——即使他们没有给你足够的信息。他们仍然会试图责怪你，有些人甚至会以此为由不支付服务费用。在开始任何项目之前，确保获得尽可能多的信息和项目资产。

# 期望少花钱多办事的客户

使用 web 开发服务的另一个问题涉及到域名和托管网站。一些客户认为这些都应该包含在开发费用中。他们想支付你从零开始建立一个实体或网站。对于这些类型的客户，你必须非常清楚你将如何处理托管。如果你选择为他们提供服务，你必须包括一个维护包，因为这是一个很大的工作量。所以你必须提供停机支持，设置电子邮件和 FTP，安装 SSL 证书，监控网站。

在这种情况下，我建议明确主机和域名是与你的开发服务分开的。您也可以决定只向他们收取基本的初始设置费用，然后在迁移期间移交凭据。

# 备份客户端

你可能会遇到这样的情况:一切都很顺利，项目已经完成，但客户却不知所踪。他们没有回复你的邮件。他们的电话直接转到语音信箱，他们的最终付款到期了。预先收取一部分费用在这一点上给你带来了优势。这不会完全浪费你的时间。如果他们没有支付最终款项，不要给他们发送完成的项目文件，也不要在客户没有结清余额的情况下将他们的网站迁移到他们的域名。

虽然这令人沮丧，但也有可能出了问题。他们可能遇到了不可避免的紧急情况。在这种情况下，你要非常尊重和理解。在他们联系你之后，你们可以进行讨论，你们可以商定一个不同的付款期限。

# 高科技和低科技客户

随着您在旅程中的进展，您会注意到客户在他们了解什么和不了解什么方面会有所不同——两者都有利弊。如果他们对构建网站或应用程序一无所知，就很难确定项目的价格并证明其合理性。

有些人可能认为开发一个高质量的网站就是将一些元素拖放到页面中。对于这样的客户，你必须做一点教学。要明确这一切都归结于质量。我不是来抨击任何 web 开发服务的，但是质量参差不齐。如果这不是他们想要的，引导他们到上述服务并取消工作。客户必须做出选择。

与可能自己做过一些 web 开发的客户一起工作可能会很有挑战性。有时他们会越俎代庖，告诉你应该使用什么工具或语言，或者开始批评你的部分代码。我发现最好的客户是那些了解网站建设基本流程的人。你不必和那些认为他们能比你做得更好的人一起工作。在这种情况下，我建议你礼貌地走开。记住，不要过河拆桥。

# 抱有不切实际期望的客户

这些客户认为他们的想法会是市场上最好的。大胆梦想是个好主意。然而，客户必须明白，创建一个影响力巨大的帝国——尤其是在科技领域——不是他们一夜之间就能完成的。

有时，当客户向您推销他们的想法时，您可能会意识到已经存在另一个实现。像那样让某人失望变得很难，尤其是如果他们为这个想法投入了大量时间甚至金钱。在这种情况下，如果可能的话，你可以试着推销他们正在寻找的东西的调整版本。要让你的客户明白的主要一点是，光有想法是不够的。你得想好如何成功实施。

# 结论

记住沟通是最重要的。与你的客户进行正确的交流可以避免很多问题和误解。在你同意任何事情之前，进行深入的咨询。讨论工作、价格、时间框架、设计、托管以及任何想到的事情。记录下一切。文件证明了你所说的话。这可能包括定价合同、项目提案、计划、托管协议等任何你能想到的东西。打印出来，贴上你的商标，交给他们。

如果你是一个有这些问题的初学者，不要太沮丧，因为它会变得更容易和更好。一旦你进入最佳状态，你将为几乎所有的情况做好准备。你将建立定价，时间表，提案和其他关键方面，你可以根据要处理的项目进行调整。

保护自己的安心也是好的。如果你有一个难缠的客户，而你觉得你和他们毫无进展，你可以不接受他们——即使这个项目有可能让你赚到一大笔钱。有时候，这个数量不值得你去承受压力和烦恼。所以不要害怕拒绝别人。优先考虑你自己。
