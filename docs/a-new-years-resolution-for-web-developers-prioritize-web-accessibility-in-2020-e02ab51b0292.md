# 全世界有 10 亿人患有残疾

在美国， [26%](https://www.cdc.gov/ncbddd/disabilityandhealth/infographic-disability-impacts-all.html) 的人口，或者说每四个人中就有一个，患有某种类型的残疾。显然，并不是每一种残疾都会直接影响一个人浏览网页的能力，但很多残疾都会。

根据[世界银行](https://www.worldbank.org/en/topic/disability):

> “与非残疾人相比，残疾人更有可能经历不利的社会经济后果，例如教育程度更低、健康状况更差、就业率更低以及贫困率更高。”

残疾人在争取融入社会的过程中面临着巨大的障碍。开发商有一个独特的机会来打破这些障碍，并防止新的障碍被建立。让我们来看看在新的一年里优先考虑网页可访问性的一些简单方法。

# 首先，了解辅助技术

辅助技术使个人能够以适合其个人需求和偏好的替代方式浏览网络。数百万人每天使用这些技术浏览网页。了解这些工具的使用可以增强您构建可访问网站的能力。下面是一些辅助技术的简要概述。

## **屏幕阅读器**

这些软件程序读取浏览器中呈现的文本，可以通过合成器将其转换为语音，也可以通过盲文显示器将其转换为盲文。屏幕阅读器还提供更高级的功能，如定位页面上的特定内容或充当菜单导航工具。

## 语音识别软件

这个软件使计算机能够处理人类的声音指令。苹果的 Siri 或者亚马逊的 Alexa 可能是这个软件最被认可的实现了。虽然这些技术通常被认为是多余的，但这个软件对于许多残疾人来说是绝对必要的。

## 替代键盘

对于那些不能使用标准键盘的人来说，替代键盘是可以定制的计算机输入工具，以满足有运动障碍的人的需要，例如脑瘫或部分失明的人。

# 开发者能做什么？

看起来，这些辅助技术本身就完成了使 web 可访问的工作，但是开发人员可以做很多事情来帮助创建一个可访问的 web。从今天开始，你可以采取以下几个步骤:

## 为图像提供替代文本

屏幕阅读器访问 Alt 属性是为了阅读可视内容，浏览支持语音的网站和/或语音输入软件的人也可以访问 alt 属性。如果图像包含的内容对于理解网页来说是不可或缺的，那么应该提供替代文本。

如果图像是功能性的，作为按钮或链接，也应该提供替代文本。对 alt 属性的需求是逐例的，但是作为一个通用的测试，你可以问自己，如果没有这个图片，我的 web 页面能被理解吗？从那里开始。如果图像纯粹是装饰性的，您可以提供一个空文本替代，例如`alt=””`。

除了使网页更容易访问，这也可以改善你的搜索引擎优化，因为这是一个关键字的机会，但保持有意义的描述。

## 编写有意义的 HTML

编写适当的语义 HTML 可以让你的 web 应用程序更容易访问。简单的旧式语义 HTML (POSH)只是简单地包括使用每个 HTML 元素，就像它被设计用来使用一样。一个`<h1>`标签应该被用于顶层标题。一个`<p>`应该表示一个段落的存在。

这似乎是显而易见的，但是当我们没有花时间去找出为什么我们的代码没有按照我们想要的方式呈现时，偶尔懒惰或随意的 HTML 会使我们的最佳意图落空。我曾听开发人员说过，最好的代码是有效的代码，但是意大利面条式的代码会带来许多复杂的问题——使网络更难访问就是其中之一。

语义 HTML 在页面表现之上提供了一层含义，这种含义可以被屏幕阅读器和其他键盘访问。对于许多使用辅助技术的人来说，HTML 的清晰性意味着清晰。

## 形式

类似的原则也应该适用于表单。表单在 web 交互中起着不可或缺的作用。除了在视觉上被清楚地标记外，表单域和控件必须被正确地标记，从`<label>`元素开始。

可以通过键盘、语音输入、屏幕阅读器和语音输入来访问正确标记的标签。还应向用户提供清晰的说明，以澄清表格的使用和填写。

## 合理组织你的内容

与语义 HTML 携手的是对良好内容结构的需求。那些依赖屏幕阅读器的人通常通过让软件通读标题来获得网页的轮廓。如果没有标题或适当结构的标题，网站可能会不必要地难以导航。

当用户浏览内容、指定标题、段落、有序/无序列表等时，屏幕阅读器也会大声读出 HTML 元素。下面是一个来自 [MDN](https://developer.mozilla.org/en-US/docs/Learn/Accessibility/HTML) 的结构良好的内容示例:

```
<h1>My heading</h1>

<p>This is the first section of my document.</p>

<ol>
  <li>Here is</li>
  <li>a list for</li>
  <li>you to read</li>
</ol>

<h2>My subheading</h2>

<p>This is the first subsection of my document. I'd love people to be able to find this content!</p>

<h2>My 2nd subheading</h2>

<p>This is the second subsection of my content. I think is more interesting than the last one.</p>
```

## 提供音频抄本

与盲人或视力有缺陷的人明显与图像斗争的方式相同，那些耳聋或听力困难的人可以被排除在音频和/或视频内容之外。文本抄本和隐藏式字幕是使网络多媒体内容可访问的必要条件。

文字记录或隐藏字幕应:

*   同步—文本应该出现在伴随的音频旁边
*   等同-标题和文本应该等同于口语
*   可访问性—文本应该是可访问的，并且对于需要它的人是可用的

# 你为什么要在乎？

可访问的 web 应用程序是设计良好的应用程序，可以最大限度地提高所有用户的可用性。根据 [W3](https://www.w3.org/WAI/business-case/) 的说法，可访问性推动创新，提升你的品牌，扩大市场覆盖面，并最小化法律风险。易访问的网站具有更好的搜索结果，降低了维护成本，增加了受众范围。

更重要的是，一个可访问的网络是一个授权的网络。一个可访问的网络是包容性的而不是排他性的——一个邀请式的网络，欢迎任何和所有人以他们能够的任何方式进行连接。
