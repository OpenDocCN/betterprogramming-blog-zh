了解应用程序的构建模块以及它在底层网络上的工作方式

在真正的生产网络上运行之前，开发一个生产应用程序充满了乐趣和游戏。

真实的生产网络存在延迟、丢失和拥塞。应用程序通常在真空中运行良好，但是一旦涉及到拥有真实用户的大型网络，游戏就变了。

对于开发人员来说，学习网络工程的基础知识在过去可能是可选的，但在今天却是必须的。如果您想成为一名更加敏捷、高效的开发人员，您需要了解网络工程的基础知识。

通过将您的代码扔出围栏，让网络团队来处理后续工作的日子已经一去不复返了。如今，无论你去哪里，你都有可能接触到某种形式的操作。

操作现代生产应用程序意味着…

是的，你猜对了。联网。

让我们探索一些基本的网络基础知识、教育资源和提高网络工程技能的一般建议。

基础知识

网络工程领域广泛而复杂。工程师毕生致力于学习复杂的网络拓扑、路由协议和硬件配置。

我在这里的目的是给你一个成功的基础，让你思考网络和它们如何运作，而不是给你灌输OSPF或BGP。

这并不是包罗万象的，如果你愿意，你可以很容易地进入网络的兔子洞。我将以学习指南的形式列出每个领域，每个领域都有一些关键的知识和一些进一步阅读的链接。

开放系统互连模型

维基百科上关于OSI模型的进一步阅读。

  • 了解TCP等元素在OSI模型中的位置。
  • 了解模型的每一层，以及每一层为什么重要。
  • 了解OSI模型中特定类型的硬件在哪里运行(交换机、路由器等)。

IP寻址和子网划分

关于 IP寻址和子网划分的进一步阅读。

  • 知道IPv4和 IPv6 地址代表什么。
  • 了解地址和网络的 CIDR 符号(例如:/24)。
  • 理解私有和公有IP地址的区别。
  • 了解IP地址的子网掩码的用途以及它存在的原因。
  • 研究静态分配的IP与动态分配的IP(DHCP)。

TCP和UDP

关于 TCP和UDP 的进一步阅读。

  • 了解TCP和UDP协议的主要区别。
  • 理解一些使用TCP和UDP的服务以及原因。

域名服务器(Domain Name Server)

关于 DNS 的进一步阅读。

  • 了解将域名转换为IP地址的高级流程。
  • 知道当你把电脑的DNS解析器换成谷歌的8.8.8.8会发生什么。

精灵

关于 NAT 的进一步阅读。

  • 调查PAT ( 端口地址转换)。
  • 理解为什么端口转发存在，为什么你需要这样做。

硬件的基本类型

阅读更多关于基本硬件类型的信息。

  • 路由器
  • 第2/3层交换机
  • 接入点
  • 防火墙/安全设备
  • 调制解调器
  • 网卡

情景测验

我发现衡量你的知识和测试差距的最好问题之一是“情景”问题。

场景问题是一个宽泛的开放式场景，你必须尽可能详细地解释为完成一项任务而在两点之间发生的每一层网络行为。

这是一个很好的例子，我要求你回答并用它来测验你自己:

“你在办公桌前坐下，打开笔记本电脑。你打开浏览器，在地址栏输入www.microsoft.com。详细描述当你按下回车键时发生的一切

不要一概而论，要明确！

  • 首先，您的计算机是如何连接到互联网的？
  • 数据包如何知道如何离开您的家庭网络？
  • 你怎么知道微软在哪里？
  • 底层数据是如何传输到浏览器中显示的页面的？

继续下去，你会暴露出一些让你思考的问题:

“嗯…我不知道这是怎么回事…”

这里的目标并不是真的得出一个最终的结论或一个具体的答案，而是揭示你可能没有完全掌握概念的领域。例如，如果你很难填写有关DNS解析的细节，那么在你的研究中更多地关注DNS，直到它有意义为止。

更多教育资源

在上一节中，我提供了每个网络元素的快速概述和解释，以及它们为什么重要。我强烈建议你深入研究这个领域，了解你感兴趣的每个领域的更多细节。

在过去，我花了无数的时间学习和参加认证考试，虽然我不提倡他们，但学习材料通常是一流的。

我不主张开发者跑出去获得他们的CCNA或Network+的原因是，我认为价值已经不存在了。认证让你为非常狭窄的环境做好准备。作为一名开发人员，您需要广泛的、通用的、独立于供应商的网络知识。

如果你喜欢更有条理的学习，我建议你参考以下资源:

  • CBT掘金网络基础(精彩视频课程)。
  • ITPRO。电视(更多精彩视频课程)。
  • CCNA官方证书指南(不是一本小书，但即使你不参加考试，也是一本很好的指南)。
  • 思科学习网(思科提供大量免费学习资料)。
  • Reddit (没错，/r/networking子编辑相当活跃有趣)。

有了这些优秀的资源和一点谷歌的魔力，你可以很容易地为自己建立一个坚如磐石的网络基础。记住不断挑战自己，将概念与现实世界的场景联系起来。

为了将事情与您当前的开发工作联系起来，您可以调查您的应用程序需要哪些端口来进行通信。您还可以查看您的应用程序是否使用TCP或UDP进行通信。

请记住将这些现实世界的联系与你的工作联系起来，以强化网络知识，并使其更加真实。

把所有的放在一起

你不需要花哨的网络设备或正规的教育来提高你的技能。你所需要的是花时间和精力学习基础知识，其中大部分都有免费教育。

您学习的概念越多，然后将它们应用到实际应用程序开发中，您就会变得越熟练。

理解制作数据包并将其从源传输到目的地的过程，发掘出一个充满好奇和阴谋的全新世界。让你的应用程序在网络上说话是一回事，理解它如何做到这一点是完全不同的。
