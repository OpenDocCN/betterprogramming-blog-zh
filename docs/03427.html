看看它们最有用的特性

前几天，我和我的同事再次讨论我们选择的集成开发环境。

自从我大学毕业后的第一份工作中那位了不起的高级开发人员向我介绍了 IntelliJ IDEA (向朱利安·伍德大喊)以来，我已经把自己算作 JetBrains 的粉丝好几年了。

令我惊讶的是，我目前办公室的大多数开发人员都在使用非JetBrains IDEs。有些人考虑过转换并决定不这么做，有些人正在权衡这种差异，有些人拒绝给除了他们最喜欢的IDE之外的任何IDE一个机会。

剩下的几个人，我自己正好在中间，面对完全处于劣势时，通常会努力捍卫我们的选择。你看，我不会假装知道其他ide的所有特性。见鬼，我甚至不会假装知道JetBrains IDEs的所有特性(姑且称之为 JIDEs 吧？).

然而，我可以说的是，我已经尝试了一些不同的方法，没有一种方法像JIDEs那样让我的许多不同项目的开发变得如此容易。“但是Jetbrains生产这么多不同的产品，”我听到你抗议。

的确如此，但是他们中的许多人都有很多我想在这里指出的惊人的特点。忍耐一下，这是值得的。

因此，让这成为一个免责声明，我a) 与Jetbrains没有任何关系——我只是喜欢他们的工作，b) 不在这里比较 IDEs。不要打开罐头。

其他非jid可能有类似的特性集，或者实现得更好或更差，或者以类似的方式实现——我会让您自己做决定。

相反，我想强调一些使我最喜欢的ide如此强大的东西，以便当讨论第无数次出现时(毫无疑问会出现)，并且我再次被提示解释我的少数观点时，我可以简单地展示这篇文章，以及所有喜欢它的聪明人，让它自己说话。

好了，聊够了。我们开始吧。

插件

有什么比可选功能更好的开始呢？JIDEs中有各种各样的插件。它们中的许多增加了额外的功能，一些增加了便利性或集成性，但所有这些都增加了一些宏伟的东西。

令人惊奇的是，虽然JIDEs已经有了太多优秀的特性，但是任何可能缺少的东西都可以通过这种方式添加进来。可能性是无限的！以下是我一直以来最喜欢的一些。

有没有想过开始制作，比如说，一个颤振项目？不要害怕，IntelliJ IDEA可以让你安装插件。它增加了语言和框架支持、语法突出显示、代码完成、检查、快捷方式等等，来帮助您开始并很好地使用这项技术。

这只是众多例子中的一个，还有更多插件支持大多数主流和许多非主流语言和框架。

CodeGlance 为活动文件添加一个重要且熟悉的小地图。当你已经用坏了你的鼠标滚轮时的理想选择。

你喜欢Docker吗？因为我当然知道。我不太擅长启动它，但没关系。你知道什么能帮上忙吗？你猜到一个 Docker 插件了吗？嗯，你猜对了。

不再需要记住那些终端命令和步骤来配置和运行您的容器——有了它，IDE会为您处理所有这些事情。

如果你正在做一个云项目，也有一个插件。无论你使用的是 AWS 、 Azure 还是 GCP 、、——它们各自的插件都允许你与每个提供商提供的各种服务进行整合。

如果你不需要因为各种原因离开你的IDE，你可以更快更不间断地开发，这些插件可以改变这种效果。

几乎与IDE的选择和主题的选择一样重要，甚至更重要。你最喜欢哪个特定的暗模式调色板？

JIDEs有一些默认的主题，编辑器允许你微调编辑器的所有颜色。但是如果这太费力的话，很有可能你会找到一个你喜欢的主题作为插件。

如果到目前为止，我已经说服你给JetBrains一个机会，那么你很幸运，因为也有几个插件是为新用户准备的。 IDE特性训练器显示了关于IDE特性和功能的有用提示，你猜对了，这是入门的最佳方式。

Key Promoter X 是另一个很棒的学习工具，当你用鼠标做的事情可以通过快捷方式完成时，它会通知你。

最后，如果没有我最喜欢的两个插件，这个列表是不完整的。你们中熟悉JetBrains的人可能想知道我什么时候会提到他们。

为什么，他们不是别人，正是年轻化进度条和力量模式二，当然！

前者自然地将所有的装载栏变成微小的猫一样的生物，留下彩虹的轨迹。当有几个后台进程运行时，这是特别必要的，这导致许多nyan生物为了你的享受而相互竞争。这个插件就像它听起来一样有趣。

后者将编写代码的过程游戏化了——在编辑器中键入会导致地震以越来越大的幅度震动屏幕，应用代码更改会显示漫画般的气泡，惊呼“BAM”或“POW ”,它会在IDE中嵌入音乐控件，等等；你明白了。

这听起来让人分心(有时确实如此)，但在大多数情况下，它实际上非常有效地让你进入状态，或者说心流状态，如果你愿意的话。作为一个终身游戏玩家，这个愚蠢的插件让我有宾至如归的感觉。

此外，当同事们看着你的屏幕时，他们总会感觉有点超负荷。

版本控制

没错，jide带有集成的版本控制工具，消除了人们可能需要离开IDE的另一个原因。这里有适合每个人口味的东西。

直接从众所周知的盒子里，JIDEs给你所有你习惯的VCS功能。活动分支显示在右下角。单击它允许您查看和切换到其他本地，以及检查远程分支。

菜单中的专用VCS选项卡提供了核心命令，如推、拉、取等。和其他功能，如查看活动文件的历史记录，包括每行代码的作者。

对于视觉倾向者，文件根据其状态进行颜色编码:

  • 蓝色代表编辑过的文件。
  • 新增文件为绿色。
  • 橄榄(芥末？)用于gitignored文件。
  • 红色表示需要解决合并冲突的文件(是的，还提供了冲突解决功能)。
  • 白色表示未更改的文件。

Repo视图是一个编辑器选项卡，令人惊讶的是，它显示了存储库的历史，包括打开的分支、合并和提交。当你需要回到过去或者检查你当前工作的分支的最新状态时，这是一个方便的工具。

我并不经常使用这个窗口，但是当我确实需要检查我的回购中发生了什么时，这个方便的窗口使这项任务变得快速而简单，只需几秒钟。

同样，Commit editor选项卡允许您查看本地更改列表，并使用自定义提交消息提交您选择的更改。换句话说，这是一个运行git add . ; git commit -m “Lint”命令的奇特视图，以防 Git Bash 不适合您。

如果你像我一样喜欢使用普通的Git Bash，这些ide可以让你用自己选择的终端来替换默认的终端。自然，这也适用于Windows。

除非完全安装Bash是绝对必要的(在这种情况下，您可能会使用Linux的Windows子系统)，否则我发现Git Bash足以满足我的日常终端需求。

要将您的集成终端 更改为Git Bash，只需进入设置>工具>终端，并根据Git在您机器上的安装位置，将Shell path中的内容替换为类似于“C:\Program Files\Git\bin\sh.exe” — login -i的内容。

你可以将终端标签重新命名为类似于Git或ThisIsNotTheTerminalYouAreLookingFor的名字，但这只是美学，也就是软件开发中最重要的部分。

上下文操作

Alt + Enter允许您调用上下文动作。也就是说，您可以在某一点或选定的代码块上运行一组相关的操作。

比如说，如果你选择了一个变量的类型，调用这个快捷键会给你提供改变它的类型的选项，如果语言允许的话，用一些东西(大括号，if/else块，等等)包围住选择。)，格式化选择，等等。

或者，如果您在方法之间点击快捷方式，上下文操作可能会包含一个选项，根据还没有getterss和/或setterss的类成员生成getter和/或setter。

让我们再做一个例子:你可以在一个大方法中选择一段代码，并将其提取到它自己的方法中。另一方面，如果你想内联一个小方法，你也可以通过这个快捷方式。

通过上下文操作可以找到许多其他的重构选项——这种快捷方式节省时间的潜力是巨大的！

代码生成

Alt + Insert是代码生成的快捷方式。如上所述，这给了你从阴间召唤整块代码的能力。

常见的选项包括生成构造函数、解构函数、属性和赋值函数。如果你正在使用 Rider 处理一个 Unity3D 项目，你也可以生成单行为事件函数，比如Update和OnDestroy。

就我个人而言，我发现Insert是一个键，它的位置我的肌肉记忆并不熟悉，所以我反弹了快捷键Alt + G。

Ctrl +单击

也许是这个简短列表中最重要的快捷键，因为你在操作鼠标的时候会用到它。如果在类/成员/方法定义上使用，它会找到它们的用法，并让您导航到那里。

反过来，如果用在其中一个的用法上，它会直接跳到定义上。将它与Ctrl + Alt + LeftArrow结合起来，回到你来的地方，获得非常流畅的文件导航。

显示文档

Ctrl + Q显示所选类/成员/方法的文档。仅此一点就使得编写适当的Javadocs、XML风格或其他正式注释变得值得一试。

这是一个很好的捷径，尤其是在团队环境中，你可能不是100%熟悉你正在使用的每一段代码。

参数信息

Ctrl + P专门用于显示方法的参数信息。这包括显示使用您已经输入的参数组合仍然可能出现的所有重载。

搜索

Shift + Shift是搜索 事物的便捷方式。你有没有遇到过及时找到任何东西的困难？试试这条捷径可以更快到达那里。

几行上的相同操作

Ctrl + Ctrl (hold)是你的朋友，如果你需要在几行上做同样的事情。通过单击CTRL键一次，然后按住它，可以启用多行编辑。按住CTRL键的同时，只需按向上或向下箭头，即可在上方或下方添加一个光标。

将这与常用的Ctrl + Left/RightArrow快捷键结合起来，将每个光标移动到下一个单词的开头或上一个单词的结尾，您就拥有了一个非常强大的编辑工具。

重新命名

到目前为止，所有开发人员最常做的事情是什么？为什么，当然是改名 的事情了。但是简单地重命名一个变量并不容易，如果你还需要重命名该变量的所有实例的话。

输入Shift + F6，它不仅重构你所在的行，还重构你正在重命名的类/方法/成员的所有实例。

当在Unity中工作时，这甚至会维护inspector引用，所以在重构之后不需要重新分配脚本或inspector值。

实时模板

有人说捷径了吗？实际上还有更多，尽管种类不同。实时模板缩短了编写通用的预定义代码的时间。

考虑一下sout，它在Java中插入了System.out.println(“Message”);。这样打字就少了很多！打印变量和错误信息也有类似的变体(soutv变为System.out.println(“SomeVar: ” + SomeVar);，serr变为System.out.printerr(“Message”);)。

模板的另一个好的用例是循环。如果你想写一个for循环，输入fori会产生类似于for (int i = 0; i < UpperBound; i++) {}的东西。

类似的变体还有数组迭代(itar)、列表迭代(itli)和集合迭代(itco)。您还可以编辑现有的实时模板，或者为您需要反复编写的代码添加自己的模板。

高级格式

如果你自己或者在一个小团队中做一个小的副业项目，保持一个常规的风格和执行格式规则可能不是非常重要。然而，随着团队和项目范围的扩大，对一致性的需求也在增加。

JIDEs会支持你的。格式规则可以在系统>编辑器>代码样式下找到并调整。

另一个简洁的工具是 Convert Indents ，顾名思义，它可以让你将缩进转换成制表符或空格。您可以对活动文件、整个项目或脚本子集执行此操作，方法是选择您想要的文件，右键单击并在上下文菜单中选择此选项。

您可以使用的更多格式化功能包括:

  • 代码清理(根据您的风格准则自动重新格式化您的代码)。
  • 自动缩进(对齐选择的缩进)。
  • 优化导入(对导入或using语句进行重新排序，对它们进行分组，并删除未使用的导入)。
  • 能够在右下角随心所欲地改变行尾、文件编码和间距 。

检查

我再怎么强调这一点也不为过:JIDEs不仅仅帮助你编码并为你提供漂亮的工具；它们真的会让你成为更好的程序员。

每个IDE都有一组默认警告 和建议 ，范围从简单代码优化的格式规则违规到潜在的缺陷或错误以及严重的性能障碍。

光是这一点就价值连城，但除此之外，JetBrains还会进一步解释为什么会给出建议或警告。仅从这一点上，我就学到了大量关于代码设计、语言能力、编程模式等方面的知识。

更重要的是，如果建议或警告不是您所关心的，您可以编辑检查的严重性(即，将警告变成建议或错误)，或者完全忽略它。

数据库和HTTP请求工具

开发人员通常需要离开IDE去做的一些其他事情是与数据库和API交互。嗯，跟小伙伴们打个招呼:数据库Tab和HTTP请求划痕。

对于前者，您可以连接到大多数(几乎任何主流)数据库系统，并在编辑器中编写和运行SQL查询。

对于后者，编写和发送GET和POST请求变得简单。只需右击项目选项卡底部的 Scratches 目录，并在上下文菜单中选择New > HTTP Request。

自己写请求，添加请求模板，或者粘贴您的一个卷发。然后，通过左侧的绿色小箭头发送请求，响应将显示在运行选项卡中。

您还可以查看已发送请求的历史记录，甚至查看一些示例。

待办事项视图

如果你像我一样，你不会第一次就把所有东西都编码好。也许你正在处理的问题的解决方案取决于你同事的公关被合并。

在这种情况下，一个合适的待办事项会提醒你未来的自己(这个故事的真正英雄)。但是很容易忘记要做的事情，并且不得不去寻找它们是很乏味的。

不要害怕，我的同伴，JetBrains在这里。不仅包含待办事项的注释行被特别突出显示以便于查看，还有一个方便的专用视图列出了项目中所有剩余的待办事项。

图表

一些JIDEs，比如IDEA，可以自动生成你的项目的类层次图(右击一个脚本，然后从上下文菜单中选择Diagrams > Show Diagrams)。

你可以调整你想在里面看到多少信息。这是退一步做一个高层次的双摄的完美特征。另外，你可以用它向你的同事展示你所做的事情。

测试跑步者

为了工作一定要写单元测试吗？你喜欢证明你的代码是有效的吗？你有兴趣让你的同事、领导和你自己相信你的代码不会引入回归吗？

你现在正在培养你灵魂中与某些与传送门相关的恶棍产生共鸣的那部分吗？或者测试只是你的事情？无论如何，JIDEs使编写、运行、评估和调试测试变得轻而易举。

结论

这只是JIDEs拥有的所有特性中的一小部分。我鼓励每一个阅读这篇文章的人至少检查一下它们——在它们的设置、菜单和(写得非常好的)文档中可以找到更多的工具。

JetBrains在他们的产品中投入了大量的工作和思想，这是显而易见的。

除了它们的特性集，jide(IMHO)看起来也非常优雅和漂亮。最棒的是，其中一些是完全免费的。那么，你还在等什么？去提高你的生产力吧！
