当你听到一个编程的新热点时，你会怎么做？

如果你和我一样，你可能会读一点，然后很快觉得这听起来很酷。你可以将一些网站加入书签，或许可以将它们添加到学习任务列表中。

有时你甚至可以学习——看一些教程，浏览文档，玩玩它。

通常，在你能掌握它之前，新的闪亮的东西出现了，你的注意力转移了。

结论很简单——要学的东西太多了；你怎么能掌握这一切呢？

你不能。

你没有无限的时间。你没有无限的关注。你不可能什么都学会。其实你学的东西应该比你想象的要少。

# 你为什么学习

今年你想学习一门新的编程语言吗？

也许你宁愿掌握一个新的框架？图书馆？

为什么？

也许你有成长的冲动。也许你认为你必须学习新的东西来保持相关性。也许这是下一件大事，你想保持领先地位。

这些都是原因。但是它们是好的理由吗？不一定。

你可能喜欢学习新的东西，即使它们没有用。我喜欢。但是如果你没有很多空闲时间(谁有很多空闲时间？)并想进一步发展自己的事业，为了乐趣而学习是不行的。

**学习一项新技能只有一个正当的理由——使用它。**

为什么你要花几百个小时学习一门新的编程语言而不去使用它呢？一两年后，你会忘记一切，回到起点。

时间是你最宝贵的资产；不要轻易扔掉。学习错误的东西就像看网飞一样没用。也少了娱乐性。所以，小心不要浪费时间，不要自欺欺人地认为自己在进步。

高效学习的唯一解决办法是专注于对你重要的事情，忽略其他的。要实现它，你必须自律和无情。社交媒体的炒作很容易冲昏我们的头脑；我们倾向于相信每一个闪亮的新事物都值得我们花费时间。不是的。大多数新事物会在几年甚至几个月后被遗忘。

# 什么值得你花时间？

![](img/f978547dcc1bc6a8058e1bd537ed029e.png)

你需要学会如何识别哪些技能值得你花费时间。不同的技能对不同的人很重要，所以你的情况是独特的。幸运的是，在选择基本技能时，我有一些指导方针可以遵循。

## 你使用的技能

我们从最明显的开始。无论你的领域是什么，你都有一些最新的技术。也许你用 Laravel 框架做后端，或者用 React 做前端。当你在寻找新的东西来学习时，你应该从这里开始——成为你所使用的工具的主人。

各大语言和框架都在不断更新，你需要迎头赶上。许多新功能使您的工作更容易或更快，所以您不想错过它。

掌握你已经使用的工具提供了两个主要的好处——你可以立即使用新知识，并且你的技能会得到回报(或者可能得到回报)。这是显而易见的。

## 与你当前技能相近的技能

当你是一个领域的大师时，是时候扩张了。而最好的拓展地就是和你专业相邻的区域。

假设您是一名前端开发人员，专注于在 React 中创建出色的用户界面。你可能想探索哪些邻近地区？也许它是无服务器的——你将能够自己构建成熟的应用程序，而不需要与后端开发人员合作。或者设计——如果你能在没有设计师帮助的情况下，自己设计应用程序，那就好了。

对于使用 Django 框架的专业后端开发人员来说，相邻的技能可能是 DevOps 或前端基础。通过扩展到这些领域，您将开始成为一名全能的全栈开发人员，能够独立构建和部署整个应用程序。

## 您将使用的技能

有时候你不想深究你目前的技能，相邻的领域也没什么令人兴奋的。现在该怎么办？我打赌你已经有了一些应用程序的想法。

学习构建这个应用程序所需的技能。

小心点，我们正进入危险地带。许多程序员有一些模糊的应用程序想法，并在选择堆栈时受到炒作的驱动。他们大肆宣传一种新的数据库或语言，但两个月后却发现一无所获。为了避免这种情况，确保建立在你现有的技能上，只在必要的时候学习新的东西。

您的应用程序不应该成为学习全新编程堆栈的借口，而应该一步一步地扩展您的知识。

## 基本原则

你可能会感到迷茫，不知道该把注意力放在哪里。在这种情况下，遵循一个简单的规则:

当有疑问时，关注基本面。

掌握算法、数据结构、设计模式等。，总是一个很好的赌注。即使你不去想它，你也在不断地使用基础知识。如果你能有意做到这一点，你的编程技能将会突飞猛进。

更重要的是，基础是编程元技能——无论你做什么都会用到它们，所以它们在语言和框架之间是可移植的。

# 什么不值得你浪费时间

因为有挑选你需要学习的技能的指导方针，所以在你投入时间学习之前，有一些危险信号需要警惕。

## 新的热门语言(框架/库)

不要学习每一件新事物。如果制造商声称这是自切片面包以来最好的东西，这并不重要。你所钦佩的人对此感到兴奋并不重要。你的同事对此赞不绝口也没关系。

大多数新事物会很快消失。编程工具市场是无情的；即使是巨人也经常失败。没有人会把你花在学习过时工具上的时间还给你。

这是否意味着你应该放弃一切新事物？号码

如果新事物有很大的成功机会**并且**符合你当前的技能组合，那么学习新事物是有意义的。当新的语言或库是你当前技能的延伸或临近时，考虑在它上面下注。如果你幸运的话，你可能会从早期采用者中受益。

专攻新事物是一种高风险高回报的策略，因为你可以很快成为专家，但这也会浪费你的时间。

## 你可能只会用一次的东西

有时候你不打算多次使用一个技能；然而，你需要把事情做好。这表明你不应该把时间花在学习上，而是应该找到一条阻力最小的道路。

假设您是一名前端开发人员，需要构建一些复杂的图表。您可能想学习 D3 . js——一个奇妙的数据可视化库。问题是，这不值得。D3 是一个庞大而复杂的解决方案，而你的问题相当有限。你只需要图表，不需要整个图书馆。

有哪些替代方案？你怎样才能得到你的图表，而不用去学习一个复杂的图书馆？

你可以在 D3 上使用更简单的抽象；他们中的一些人可能有办法解决你所有的问题，并完成工作。

一个更简单的替代方法是使用允许创建图表的无代码工具。检查一下——也许有一个合适的工具，并且只需要几个小时就可以理解大多数无代码工具。

最终，你可以花钱请人帮你制作图表。对于有限的问题，雇用外部开发人员通常比花时间学习一项将来可能不会用到的技能更便宜。

## 与你的技能组合无关的技能

学习远离你专业领域的技能通常很有诱惑力。我多次落入这个陷阱。

我是一名前端开发人员，专注于后端的一些专业知识，试图学习各种各样的东西，如机器学习和创造性编码。我还花了一些时间学习 Vue，Svelte 和 Angular。

你可以想象，上面提到的事情我几乎都不记得了。它们离我的专业太远了，或者说我没有机会经常使用它们。因此，随着时间的推移，我的技能逐渐衰退，直到我需要从头开始才能赶上。

不要重复我的错误；学习那些补充你技能的东西，而不仅仅是那些听起来很酷的东西。

如果你确信你想学习一项新的不相关的技能，我只有一条建议给你——确保你尽可能快和尽可能经常地使用它。

## 没落的语言(框架/库)

没落的语言和框架是在浪费时间。每项技术都有它的截止日期。可能很难注意到这项技术正在消亡，但如果你看到它，尽快远离它是一个不错的选择。

当我开始的时候，jQuery 库还是一个东西，但是它的受欢迎程度正在下降。为了找到第一份工作，我不得不学习它，但一年后当我放弃它时，我并不后悔。更谨慎的做法是专注于新兴的前端框架之一——React。

你可能会被迫学习没落的技术。那就做好了。但是尽可能地关注现代科技。否则，你现在学的很多东西很快就会过时。

你准备好专注于必需品了吗？可能很痛苦，但是值得。
