我的Git类比时刻

原文：https://betterprogramming.pub/my-git-analogy-moment-a4aaab0caa0e?source=collection_archive---------10-----------------------#2022-08-10

一个更好的现代参考，包含了我们讨论过的命令的快速列表。

我刚刚在辅导一个中学生编程的时候有了一次教Git的极好经历。虽然我的学生已经用Python编写代码好几年了，并且有一个Github帐户，但是他还没有开始使用Git。

在介绍了版本控制的一般目的之后，我分享了我的屏幕，并让他浏览了我每天使用的各种命令。他似乎对我使用命令行来做任何事情感到惊讶，尽管我向他展示了您可以通过图形界面来访问文件。然后，我向他展示了我的Github帐户，创建了一个新的存储库，推进了我的主分支，并向他展示了结果。

然后我介绍了Git分支的主题。我创建了一个新的，进行了更改，提交了更改，并将其推送到回购。为了让这个概念更容易理解，我将分支解释为可以从主分支分叉的“交替现实”。对于我的同事，我在解释这一点时使用了《回到未来II》中的1984年作为类比，但这次我忍住了。考虑到这个学生还在上中学，他获得推荐信的几率是3720比1。

在向孩子解释了“交替现实”的想法后，他说，“哦，这就像多元宇宙！”这是多么伟大的时刻。然后我们把它和蜘蛛侠联系起来，谈论不同的彼得·帕克是如何生活在不同的“树枝”上的我预计我现在会用这个新的类比来修改我的方法。是的，我说，一个 Github repo可以认为是一个小的多元宇宙！

下面是我们介绍的命令的快速列表，如果您发现它对您自己的工作有帮助的话。

  • git init启动存储库。
  • git add <file>为提交准备文件
  • git commit -m <message>做出承诺
  • git status打印文件的状态
  • git diff告诉你自上次提交以来文件中发生了什么变化
  • git branch，打印您的本地分支机构并突出显示当前分支机构
  • git checkout <branch>将检出一个已存在的分支
  • 显示你所有提交的日志
  • 推动你的改变
  • git remote add origin <remotepath>添加一个远程原点
  • git clone <remotepath>允许您根据权限克隆现有的回购协议
