## 初级工程师入职对年轻公司来说不可持续的原因

当我告诉人们我的公司不面试初级工程师时，我与之交谈的人往往会感到惊讶(而且往往会被冒犯)。这是一种可以理解的反应；不雇佣初级员工的全面政策意味着某些负面的东西。它可能意味着傲慢:我认为我非常聪明和有才华，初级候选人根本跟不上。或者它可以暗示糟糕的价值评估:我认为大三学生不值得培训。或者可以暗示自私:我觉得大三值得培养，但是很贵；我会让其他一些笨蛋吃掉训练他们的成本，然后挖走他们。

一方面，我不能决定公司的招聘策略，所以冷静点！然而，另一方面，我必须承认，我认为这是一个明智的政策，值得证明。为此，我需要补充几个限定条件:我们目前不面试初级工程师**，因为我们是一家专注于增长的小公司，我认为专注于增长的小公司不应该雇佣初级员工。**

**(注意，这篇文章试图回答的主要问题是“雇佣初级工程师对初创公司有益吗？”它并没有试图回答相关的问题“雇佣初级工程师对初级工程师有益吗？”或者“即使对个人没有好处，雇佣初级工程师是正确的吗？”)**

# **风险和时间**

**注重增长的小公司正在与时间进行激烈的竞争。该公司尚未盈利，正在 R&D 烧钱和争取客户。这通常意味着尽可能快地构建，拜倒在客户脚下，并且将你的产品路线图旋转得像一张草图。在这样的环境下，我看不出雇佣一个初级工程师并投入几个月的资源来实现零生产率是如何有效利用资源的。**

**问题不在于费用，甚至也不在于回报率。初级工程师比高级工程师雇佣起来更便宜，从培训中受益也更多。雇佣和培训初级员工的回报率*高得令人难以置信。更确切地说，这个问题是一个*风险*和一个*时间返回*的问题。初级员工天生差异较大，因为他们的工作经历或可评估的证明资料较少，而年轻的公司已经淹没在风险中。此外，年轻公司的生命线是以月来衡量的，而不是以年来衡量。对于这些公司来说，每一天每一美元都是为了获得下一个更长的生命线(风险投资，曲棍球棒增长等。)在大家都破产搬回家和父母住之前。经过三个月的训练和学习，一个初级工程师可能已经准备好成为一名明星球员，但那时可能已经没有球队可以继续比赛了。***

**在扑克游戏中，筹码短缺的玩家别无选择，只能缩小范围，玩低方差牌。初创公司也是如此:不管雇佣初级工程师的回报有多高(或者有多无私)，这些公司就是缺乏资源，无法成为一个有吸引力的选择。**

# **快速雇佣，快速解雇**

**如果问题是招聘初级员工需要比初创公司更多的资源，解决办法可能是减少所需资源的数量。“快速雇佣和快速解雇”策略是一种流行的方法。在这种策略中，初级员工被给予所需的最低限度的资源(可能是入职指南、一些代码实验室和分散的在线资源列表),并被告知要找出答案——如果他们不知道，他们就走人。**

**这与大型公司在脸书、谷歌等公司招聘初级员工的方式类似:新员工要经过数周甚至数月的入职培训和入职训练营，并准备好立即投入工作。我们的战略基本上与超级公司相同，只是没有时间、金钱和资源，气氛不太像大学欢迎周，更像饥饿游戏(除非超级公司是亚马逊，在这种情况下，它们基本上是一样的)。**

**我不得不承认，这似乎是小公司评估初级候选人的最具成本效益的方式:风险水平受到限制，雇佣不成功的负面影响被降至最低。唯一的症结在于它是邪恶的，邪恶的名声往往不利于招聘。虽然这种策略对公司有利，但从新员工的角度来看，这是一种掠夺。**

**(有些人可能会认为，掠夺性关系总比没有关系好，这是那些提供无偿工作或通过“曝光”获得报酬的公司所使用的观点。我确信雇佣/解雇快速策略可以调整得更加公平，但我还没有找到任何足够令人信服的实施方案。糟糕的早期职业经历可能会造成严重后果，除非我确定这种经历对初级工程师来说很棒，否则我宁愿远离它。有足够多的公司雇佣任何有脉搏的人，并无情地淘汰群体。)**

# **增长潜力**

**有一种观点认为，大公司的工程师往往过于专业化，而小公司的工程师则更加独立。我发现这种信念与我的经历完全不一致。多年来，我有机会与大公司的中层工程师和小公司的中层工程师一起工作。一般来说，来自大公司的人感觉在经验上领先好几年:他们有大量的模式和架构可以借鉴，他们知道如何有效地跨职能工作，他们可以扩展数据库，他们可以有效地进行领域建模。**

**即使你愿意赌上你的导师能力，并投入资源来培养低年级学生，这样做可能不会给低年级学生带来最好的长期发展。模式识别和分析现有系统是初级工程师学习的两种重要方式。工程师在大公司比在小公司见证更多的成功和更多的失败；这种证明什么在规模上行得通，什么行不通的证据，是对抗货物崇拜心态的有力屏障。说到系统，初创公司通常不以最佳实践和稳定架构的闪亮堡垒而闻名；在追求产品与市场匹配的过程中，事情发生了很大的变化，产品经常被制造成或多或少的一次性产品。**

**初级员工在小公司发展壮大并非不可能，但在大公司的增长率可能会高得多。我相信这篇文章标题的反过来也是正确的:如果可以选择，大三学生不应该加入初创公司。(当然，初级工程师的选择本来就有限。对于未来的低年级学生来说，把他们的长期增长潜力看得比他们支付租金的短期能力更低可能是明智的。)**

# **公司应该什么时候开始雇佣初级员工？**

**对我来说，雇佣低年级学生的问题不是“如果”，而是“什么时候”我认为，早期公司不应该雇用初级员工；同时，我认为从长远来看，雇佣初级员工的公司注定要失败。在一定规模下，拥有稳定的初级人才供应对公司的持续生存至关重要:根本就没有足够的高级工程师。(像网飞这样的公司是个例外，它们用天价薪酬和引人注目的工程品牌来继续吸引高级人才。)无论何时采取一种立场，确定立场将会改变的边界条件是有帮助的。**

**那么拐点在哪里呢？没有一个通用的答案。早期创业公司通常不会长期思考，但有一些成功的指标(或者更保守地说，“合理预期继续存在的信号”)可能会影响是否雇佣初级员工的决定。里程碑事件，如达到 100 人，或达到某个估值目标，或筹集到一定数量的资金，都可能是这样的转折点，围绕这些转折点，招聘政策应该重新审视。**
