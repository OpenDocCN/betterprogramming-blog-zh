促进开发团队之间知识共享的9项行动

原文：https://betterprogramming.pub/9-actions-to-boost-knowledge-sharing-among-development-teams-ad5f7acd2048?source=collection_archive---------4-----------------------#2020-01-24

1.不要谈论代码

作为一名程序员，你可能已经观察到我们最有价值的技能之一是我们的学习能力。你越擅长学习新事物，你就越擅长编程。

当您开始您的程序员之旅时，互联网上有大量的新信息可供您使用——而且很有可能，您偶然发现的所有文章都会教会您一些东西。

可悲的是，一旦你读了几遍文档，开发了一个待办事项应用程序，并在网上搜索了几个星期，学习新东西的速度就会大大减慢。

幸运的是，在大多数公司，你周围都是有才华的人，他们知道很多可以教你的东西。你可以反过来教他们一些东西。

在这篇文章中，我想分享一些可以帮助你建立知识共享文化的想法，让你在工作中享受更多的时间。

建立社区

首先，我们需要召集对某个领域感兴趣的人——不管他们是专家还是新手。越多越开心。

如果你幸运的话，你会在你的公司里已经有了某种分组:也许你有一个Slack频道，一个领域专家团队，或者一个分会。

创造激励

第二，我们需要激励人们参与活动。开始一项学习活动并不总是一帆风顺的，因为人类天生懒惰，尤其是程序员。

铭记这一行为是公司培养学习文化的关键一步。这些激励可以是任何东西:一轮掌声，给予认可，酷t恤，美味的食物，礼品卡，甚至是团队旅行。

1.不要谈论代码

格式:聚会

工作原理:聚会是一种很好的联系方式。吃点披萨，喝点啤酒，介绍一下自己，了解一下你的同事。

好处:分享经验和人脉比分享知识更重要。我们一天中很大一部分时间都在工作——和你的同事成为朋友让工作变得非常有趣，并且在最后期限变得太紧的时候有所帮助。

2.你不知道的事情…还不知道！

格式:游戏

工作原理:每一轮， 一个开发者分享三个有用的，认为其他开发者不知道的技巧/窍门/hacks。

演示者为每个不知道诀窍的参与者获得一分，并且一个排行榜保持高分。

示例:

a.VS代码中的多个光标:Cmd + D
b. let value = obj && obj.value || defaultValue
 c. ('b'+'a'+ + ‘a'+'a').toLowerCase() // banana

好处:游戏的简单性让人们很容易参与并开始知识分享。尽管这是一个快速的游戏，但是一些小技巧最终可以节省大量的时间。

3.协作通讯

格式:邮件/帖子

运作方式:会员在抓取互联网的同时，将自己喜欢的文章发送给组织者。

组织者合并、过滤并创建时事通讯。理想情况下，这些文章将关注公司使用的技术、经常遇到的挑战以及公司正在走向的技术愿景。

好处:时事通讯通过阅读与开发人员水平相匹配的高质量文章，促进了共同意识并节省了时间。当意见被分享时，它作为开发人员之间的对话的开始。最后，它提供了一种简单的方法来确保每个人都是最新的，这样团队就充满了强大的领域专家。

4.跨团队结对编程

形式:会议

工作原理:来自不同部门的开发人员被配对在一起，他们被分配一个简单的任务(因为其中一个开发人员对代码库的背景了解很少)。

好处:结对有这么多好处。跨团队结对编程有助于在公司中创建整体代码所有权，并促进跨团队的更多代码重用，因为开发人员将了解公司中的其他项目和挑战。

5.吹牛时间

格式:演示

工作原理:开发人员或团队分享他们引以为豪的最新技术实现——展示解决方案的优雅、问题的复杂性，甚至是实现的持续时间。

好处:能够表达和解释技术解决方案是开发人员需要掌握的一项重要技能，尤其是当您的团队中有初级人员时。

传播关于公司最佳工作的知识有助于保持高质量，并为人们展示他们漂亮的代码和想法创造了空间。

6.速成班

形式:长时间演示

工作原理:长时间的演示是指开发者分享他们最近学到的东西，深入并简化它。最好这是一个新的或者流行的话题，比如GraphQL或者hooks。

好处:教学是最好的学习方式之一。提供速成课程的开发人员将在准备课程时成为该领域的专家，而其他人将在他们的学习曲线上获得提升——#双赢。

7.玩代码

会议形式:

工作原理:一群开发人员聚在一起，在像 CodinGame 或 Code Combat 这样的平台上竞争完成编程难题。你也可以在 HackerRank 上进行算法挑战。

好处:有些开发者喜欢写代码取乐。有时间编写无目的的软件会让开发人员面临新类型的脑筋急转弯。加入一点友好的竞争，你就有了一个令人惊讶的快乐时光的食谱。

8.举办聚会

会议形式:会议

工作原理:找出你的组织一直在做的、你引以为豪并希望与公众分享的事情，或者谈论已经在组织中深入使用的技术。

好处:举办聚会可以让你把外部领域的专家带到公司，允许外部知识交流。开发者展示将有助于回馈社区，并发展其他技能，如公开演讲。

而且，最重要的是，见到开发伙伴总是令人惊奇的！

9.组织一次回顾展

会议形式:会议

工作原理:像所有的团队回顾一样，创建一个安全的环境来回顾和增强过程，找出问题，并确定即将到来的挑战。

好处:促进协作和沟通对收集反馈很重要，这让我们能够从陷阱中学习并认可成就。
