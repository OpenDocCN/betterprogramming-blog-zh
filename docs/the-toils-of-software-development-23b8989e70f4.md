在您软件开发生涯的不同阶段，您至少会遇到这里列出的一个问题，如果不是全部的话。我的目标是在你遇到阻碍你成为开发人员的事情之前，告诉你这些事情。这会让你做好处理这些问题的准备，用一个你头脑清晰的计划。

为什么？因为发展是艰难的，它*需要*坚持才能变得更好。

# 失败

编程的第一个真理是你会失败。我一直都这样。编译器(或解释器)善于提醒我的失败率。

它不擅长的是告诉我“干得好！”当我修复那些讨厌的 bug 时，我必须自己去做。学会欣赏大大小小的失败。把失败的编译和程序崩溃看作是让你做得更好的学习经历。如果你想成为棚子里最锋利的工具，你必须找到并去除毛刺！

# 不耐烦

你会在某个时候失去耐心。当您碰到代码的一个乏味部分时，这种情况尤其可能发生。我第一次意识到这一点是在我试图为一个著名的第一人称射击游戏建立声望跟踪器的时候。

在手工输入了几个水准点要求后，我感到沮丧，因为编写软件既慢又乏味。后来，我会利用这种不耐烦，通过寻找更好的做事方法(例如从 Web APIs 解析 JSON)来为自己谋利。

虽然使用别人制造的轮子很棒，但有时你必须自己做繁琐的部分。当你这样做的时候，确保你用一种可重用的方式来写它。你未来的自己会感谢你。

# 乏力

精神疲劳是我最大的敌人，可能比任何其他发展的辛苦都要多。当你面临疲劳时，很难开始新的项目或完成现有的项目，因为你已经为自己创造了技术债务，现在你需要修复。

这种疲劳也来自于在许多不同的项目中工作，在同一项目中用许多不同的语言工作，或者只是在一个项目中用你不知道的语言工作。你过度使用你的思维能力，发展缓慢到近乎停滞。

处理疲劳的最好方法是做一些与发展无关的事情，更好的是，电脑时期。出去和别人一起绕着大楼走 15 分钟。这有助于让你分心一段时间，如果他们是开发人员，你们可以就如何解决你们其中一个可能遇到的问题交换意见。

当我在家的时候，我可能会玩一个我不用想太多的游戏，玩 30 分钟。

# 冒名顶替综合症

冒名顶替综合症是一种心理游戏，当你作为开发人员获得第一份工作时就会出现。大多数时候，当我们想到开发人员时，我们会想到取得巨大成就的计算机科学家(图灵、里奇、托沃兹、盖茨、沃兹尼亚克)。正因为如此，我们会觉得自己和其他开发人员不属于同一个群体。

最好的解决方法是每天学习新的东西。如果事情看起来很复杂，很有可能你错过了所需的信息——一个先决主题。过一段时间，事情会开始变得有意义，你会从感觉像一个骗子变成感觉像一个开发者。

在这个阶段，你要把自己看得太重，这一点很重要。不要让别人影响你，把所有的批评都当成改进的机会。

# 分析瘫痪

分析瘫痪说起来很有趣，但它可能是一种真正的痛苦。你曾经在提交到分支之前重构过你的代码吗？在你写代码之前，重构你的代码怎么样？这本身是正常的，但是你曾经花了超过一个工作日的时间来做这件事吗？如果是这样，你可能经历过分析瘫痪。

只有一个方法可以解决这个问题:让它工作起来，即使它缺少一些功能或者可能不如你希望的那样有效。把所有这些想法都变成明天的食物，带着你所拥有的继续前进。

# 教育界限

一位经理兼朋友曾经告诉我，他们曾经有一位员工，无论他们被教导多少次，都无法完全理解。他说他们已经发挥了他们的才能。我认为这是部分正确的，并没有完全理解任务所需的先决知识。

编程对我来说从来都不容易。但凭借毅力，我能够通过学习其他东西来突破我认为的能力上限。如果你在教育上走得足够远，遇到了这些其他的问题，能力上限不是你的问题。

如果你遇到这些问题中的任何一个，记住你不是唯一一个陷入这些陷阱的人，总会有出路的。祝你好运！
