## 寻求帮助是可以的

现在是下午 4 点，在工作中，你的代码仍然不能工作，你不知道为什么。你的大脑尖叫着让你寻求帮助，但你不想打扰别人。事实是，你只是讨厌被拒绝的想法。不过，截止日期并不是很宽松；它越来越近了。真是进退两难。

因为破碎的代码或看不到解决方案的讨厌的 bug 而碰壁的事情时有发生。这是必然的。在作为一名开发人员工作了五年多之后，我想反思一下到目前为止我学到了什么。

我最终意识到，知道何时以及如何寻求帮助，对任何工作来说都是一项非常有价值的技能。这是我一生中学到的最艰难的一课，我感谢在这么多层面上帮助过我的每一个人。

## TL；速度三角形定位法(dead reckoning)

*   只是问问
*   不要等太久才问
*   总是采取主动
*   请询问方向
*   分享你的代码或一小段录音

# 只是问问

我们都想独立。我们都想完美无瑕。我们想要不寻求帮助而解决问题所带来的自豪感。我不知道，这种骄傲伴随着高昂的代价，导致我不必要的精神疲惫。

最终，现实来了。有时候，我生存的唯一选择就是寻求帮助。我意识到有时在尝试后接受失败对我们来说同样重要。

我要说的是——脆弱和寻求帮助是可以的。不要陷入认为你应该知道所有事情的答案的陷阱，即使是作为一个高年级学生。

## 如果我被忽视了该怎么办

人们变得忙碌。如果他们没有回复你，礼貌地“打扰”他们。很多时候，我意识到向一个特定的人寻求帮助比在空闲的频道或群聊中寻求帮助更有效。

不要让事情就这么放在那里，然后被遗忘。最糟糕的事情就是保持沉默，根本不寻求帮助。

# 不要等太久才问

当你面临一个难题时，尽你所能去做。然而，不要在上面花费几天或几周的时间。有时我们过于深入细节而看不到大局，这是完全正常的。

不要在某件事情上停留太久，尤其是当你知道某件事情有人会在 5-10 分钟内知道答案的时候。

我喜欢使用的技术之一是[时间拳击](https://en.wikipedia.org/wiki/Timeboxing)。时间框仅仅意味着分配一个固定的时间段(例如，30 分钟、4 小时等。)到一个计划好的任务。当计时器结束时，如果你仍然受阻，那么开始寻求帮助。

# 主动

那么，我们如何寻求帮助呢？当你向别人寻求帮助时，永远不要“空着手”去找他们。换句话说，千万不要去找某人，然后简单地给他们一个含糊的陈述或问题，例如，“聊天机器人没有响应”、“下载按钮不起作用”或“为什么仪表板不加载？”。

取而代之的是:

*   “这是我做过的事情”
*   “以下是我尝试过但失败了的事情”以及你认为它们失败的原因
*   "这是我提出的解决方案，你觉得怎么样？"
*   “以下是我的计划，但我担心的是…”
*   “这是我对其工作原理的理解”

通过给他们以上所有的信息，你向他们展示了你的主动性。最重要的是，你会避免“你试过了吗？你试过 B 吗？”这是一个巨大的时间浪费。让人们从你这里获取所有这些信息是人们在给予帮助时变得非常愤怒和沮丧的原因。

最重要的是，通过给他们这些信息，你也可以给其他人一张你对这个问题的理解的地图。如果他们不知道你需要帮助的事情的广度，没有人能有效地帮助你。

## 它帮助你组织你的想法

当我这样做的时候，经常发生的事情是，在收集这些信息的过程中，我偶尔能够自己解决问题。在寻求帮助之前把信息放在一起，迫使我有逻辑地表达我的想法，允许我从另一个角度看问题。

# 请询问方向

如果你完全空白，不要明确地向他们寻求帮助，请他们给你指明方向。使用类似“如果你能给我指出下一个方向，我会很感激，”而不是“你能帮我吗？”。

虽然大多数人可能没有意识到，这样问和不这样问之间的情感差异是巨大的。

# 分享您的代码或捕捉一个简短的记录

对于想帮忙的人来说，确切地知道自己在说什么是非常有帮助的。通过一段代码或一小段录音，他们可以更有效地理解问题。不要让他们开口。

如果您正在调试代码，请发送包含行号的代码的确切路径，如下所示:

```
[https://github.com/ngshiheng/burplist/blob/production/burplist/utils/parsers.py**#L89-L96**](https://github.com/ngshiheng/burplist/blob/production/burplist/utils/parsers.py#L89-L96).
```

如果系统以一种不可预料的方式运行，捕获问题的简短记录并发送出去。以下是我用过的一些 GIF 记录器:

*   Linux — [Peek](https://github.com/phw/peek)
*   Mac 和 Windows — [LICEcap](https://github.com/justinfrankel/licecap)

不要给他们发垃圾邮件，给他们一堆截图或长录音，这些会让他们非常困惑和粗鲁。这里的最后一个警告是永远记住敏感信息。

# 最后的想法

希望到最后，你已经设法解决了你的问题。在您继续之前，确保您了解实际上哪里出错了，或者修复实际上在做什么。写下来！

不要一遍又一遍地重复同样的问题。对于提供帮助的人来说，没有什么比不得不重复一遍更令人沮丧的了。

最后，向其他需要帮助的人伸出援手。让他们知道脆弱和寻求帮助是可以的。告诉他们不知道所有事情也没关系。
