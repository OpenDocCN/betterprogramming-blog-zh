在一个软件开发人员的职业生涯中，有一些重复出现的需要解决的问题，例如计数、声明是否满足某个条件、选择适合特定情况的最佳数据结构等等。

最常见的误解之一是，当编译成功结束时，将问题标记为已解决。无论如何，当一个项目、一个应用程序、一个框架或一个库正确构建时，就是一个重要的里程碑。

# 什么是问题解决？

对于软件开发人员来说，解决问题是一项永无止境的技能。

它是一个多学科的化合物，不仅包括编码技能，还包括数学和逻辑。最重要的是，它与计算机科学没有密切关系，但它可以完美地适应我们日常生活的几个方面。

简而言之，我们可以说解决问题指的是采用一些方法，加上我们目前拥有的工具和能力，可以使我们实现不同的目标。

它适合许多任务，例如，修理电脑，解决数学方程，下棋。

当涉及到计算机科学时，两种常见的解决模式大量地涉及到问题的解决。而且，好像这还不够，它们也是 it 领域面试中很大程度上需要的模式。

更具体地说，它们是“双指针模式”和“频率计数器模式”

事不宜迟，让我们开始发现它们吧。
