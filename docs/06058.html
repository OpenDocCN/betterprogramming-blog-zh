成功的研究生软件工程师的标志

原文：https://betterprogramming.pub/the-hallmarks-of-successful-graduate-software-engineers-e1e771d034c1?source=collection_archive---------1-----------------------#2020-08-30

科技公司对软件工程师毕业生的要求是什么

上周，我与一些毕业的软件工程师坐下来，讨论了我们在2021年要做什么，我计划如何宣传和审查我们的申请人。

它始于我寻找的一些特征和标志——我见过的过去导致成功毕业生的东西——以及他们在人生的每一步应该注意什么。

在一口气说出我的几个观点后，我问他们是否认为我的方法和期望是公平的和被理解的。如果您正在考虑从事软件工程，那么这些要点构成了本文的基础，值得分享、考虑和实施。

应用程序

附信

我相信你已经一遍又一遍地听到过这句话，“确保你有一封求职信”，但是让我详细解释一下。首先，当谈到申请职位时，有很多我认为是垃圾邮件发送者。很明显，这些人追求的是数量而不是质量。这些人的典型例子是:

  • 没有求职信
  • 寄给另一家公司的求职信
  • 一封充满“数据和事实”的普通求职信例如，“我的销售增长了300%”，当工作不是销售的时候！

求职信中应该注意什么

你一开始就把自己介绍成一个人。“我叫X，我对Y充满热情”。我现在把你的名字和一个人联系起来了。Y是什么也不重要。但是现在我知道你关心一些事情。如果你的热情与你申请的公司有某种联系，那么在我看你的简历之前，你就已经领先了。

求职信应该基于他们感兴趣的东西和公司所能提供的东西来表明他们为什么想为公司工作。从那里开始，触及个人兴趣领域(即使在技术之外)，附带项目和大学论文经历——“我在大学写了一篇数据科学101论文，并爱上了它”。

基本上，你的求职信不应该把你的技能作为商品出售，而是应该展示你的个性，展示你的为人。

简历

简历可以很有趣。作为申请人，有一点要明白，如果你只加上学历，你的简历很可能会和其他人的一样。

大学的问题在于，它们最终会千篇一律地复制自己的校友。当然，不同的分数，不同的论文或专业可能会有所不同，但事实是，它们几乎是一样的。

在简历中寻找什么

介绍。这个应该和求职信的不一样。它更具一般性，主要用简短的句子给出关于你自己的一般性事实:

“在大学期间，我越来越渴望在计算机科学领域工作，我在大学的时光教会了我时间管理的重要性，以及在遇到问题之前讨论和考虑解决方案的必要性。”

适当的技能

不要列出按成绩排名的论文。如果你申请的公司对数据科学非常感兴趣，写几篇相关的论文或项目，但不要太多。

项目，项目，项目

这是一个大的。这可能是我寻找的最重要的东西，也可能是成功的软件毕业生的最大标志。如果你想立刻让自己超越其他候选人，做质量项目。在 GitHub 上开一个免费账户并开始活动。如果你可以在一个“活跃的”GitHub账户下展示多个项目，这将允许企业探索你所做的领域，你可能甚至没有意识到这些领域是重要的。

什么是“优质”项目？

一个高质量的项目显然不是一个教程。我看到的React Todo列表的数量令人难以置信。虽然这可能表明你愿意学习一门语言和做教程，但事实是，这只是表明你遵循了指示。

扩展教程很好。去越野，用那种技巧去做一些事情，去尝试一些新的有趣的东西，才是金子。

我遇到的质量最好的项目是一个软件工程师毕业生申请人，他对Te Reo毛利语(新西兰奥特罗亚的土著语言)充满热情。他们使用Google Maps API将新西兰所有的街道名称重新命名为Te Reo毛利名称。事实上，这并不一定是开创性的工作，但仅从这个项目中，我就获得了比他们提供的任何其他材料都多的关于候选人的信息。

我来解释一下。

首先，我来自新西兰。Te Reo是我们三种官方认可的语言之一，尽管没有翻译到谷歌地图上。事实上，有人花时间做这样的事情，并使用新学到的技能来进一步提高这样做的能力，并提高对这一事实的认识，这真的很特别。它比我见过的任何其他项目都更能说明他们是什么样的人。

带回家的测试

很有可能，你会得到一个带回家的测试。并不是所有的公司都这样做，但它们非常普遍。

当谈到带回家的测试时，除非另有说明，用你最熟悉的语言来做。不要试图炫耀——仔细阅读问题和要求，并满足这些需求。

检查一下他们是否在问题中给了你一些提示，这些问题是关于他们作为一个企业在代码方面认为什么是重要的。可读性，可扩展性，简单性，测试。这些单词中的一些可能会在他们关于他们想要看到的代码的演讲中出现。

确保在代码中出现重要逻辑的地方包含注释，并确保使用良好的函数命名，将逻辑充分抽象到函数中——但不要过度。如果你发现自己在主要函数中使用了大量的if语句，考虑一下是否可以使用一些validation函数。

测试时，确保在代码中使用了validation函数。如果您发现自己需要在测试中复制逻辑，这是一个好迹象，您的逻辑应该被抽象成一个function。

你对代码的整洁和自信通常是显而易见的。

面试

如果有的话，这应该是容易的部分。事实是，你已经预先装载了他们对你作为一个人的所有期望。如果你没有撒谎，现在只需要备份就行了。

面对面的面试可能会有两个部分。一部分是关于您的问题，另一部分是技术部分，提供动手编程或技术问题。

面试中要注意什么

诚实。现实是你不应该知道所有的事情，你也不应该知道。如果你假装什么都知道，或者认为你知道，你可能得不到这份工作。

你能做的最好的事情就是诚实地面对你的能力。如果有人问你不知道怎么做，或者不知道问题的答案是什么，就直说。此外，如果你真的想知道答案是什么，请他们告诉你，并密切关注。

衡量一个毕业生是否有可能成功的一个很好的指标是他们消除自我的能力。他们明白自己知道什么，不知道什么，但他们有超越自己的欲望。

一名毕业生申请者最近的成功故事是，从理论上讲，他们并不是最好的人——他们的课后测试绝不是最好的。然而，当谈到他们的测试和扩展时，他们能够接受建议和批评，以这样一种方式更新他们的代码，使其完美地符合描述的内容。

发挥你的优势

一定要依靠自己的优势。你可能知道一些。当你开始涉及这些领域的对话时，确保你详细阐述了它们，给出一个例子，说明你已经将这种优势运用到了一个适用的目的上，你可以作为一名毕业的软件工程师这样做。

用你最熟悉的语言进行技术测试的一个关键原因是，在面对面的面试中，你可能会被问及为什么使用特定的语言功能，或者你可能会被要求当场扩展逻辑。如果你对自己写的语言有信心，那么在一群人面前做这件事时，你会更有信心。

企业想要什么

企业从面试候选人的过程中能得到的最好的东西，是对一个人是什么样的人的完美描述，他们在软技能和技术技能方面的优势和劣势，以及对个人激情所在的深刻理解。这使得一家公司能够将合适的人投入到合适的领域，知道他们的努力会得到很好的回报，并使员工的成功最大化。如果企业明白这一点，而你又符合他们的要求。你被录取了。

你的第一年

你在一家公司的第一年可能是最重要的。
它设定了你的意图，为你提供了前进的舞台。我给每个毕业的软件工程师的第一天的建议是这样的。

  • 成为一名毕业的软件工程师是一种超能力。
  • 你应该问问题。任何问题。
  • 当你寻求帮助时，人们会感激你的。
  • 当你做一些你后来会觉得无聊的事情时，你会爱上它们。
  • 享受你的时间，知道你是一个研究生工程师，这太棒了！
  • 做一块海绵，尽你所能学习一切。
  • 不要对机会说不，让你尊敬的人和你想学习的人围绕在你身边。

成功的一年级毕业生应该具备的素质

现实是，尽管许多企业都有研究生课程，可能会雇佣数百人，但并非所有毕业生都是平等的。

人们采纳上述建议并真正应用它的能力，已经在将优秀毕业生从大部分人中分离出来方面显示出显著的成功。这个东西，很像一所大学。人们可以遵守规则。遵循一个程序，按照他们被告知的去做。但是要把自己从大堆中分离出来，还要进一步应用。不要依靠一个研究生项目来告诉你去哪里和什么时候去那里。与和你一起工作的员工建立关系网。确定人们倾向于向哪些高级职员提问——和他们交朋友。

这是一种自我激励去追随你的激情的能力，它将使你获得巨大的成功。

在成功的工程师中，经常会有这样的故事:毕业的软件工程师自愿展示他们在技术方面的知识，或者在午餐时为他们的同事演示他们感兴趣并经过优化的一段代码。

自愿参加敏捷会议并不少见，只是为了让他们学习，了解如何管理团队。典型地，这些职责和期望被排除在经典的研究生课程路线图之外，但正是这些事情显示了你在那些你希望成长的领域推动和应用的愿望。

综上

这篇文章的目的是提供一些成功的研究生软件工程师的真实价值和真实世界的标志。

问题是，越来越多的技术人员不再希望和机器人一起工作。我们想和人类一起工作，和那些好奇的人一起工作，和那些有长处和短处的人一起工作，并发挥他们的长处和短处。各个年龄段和职业生涯各个阶段充满激情和兴奋的人现在都变得正常了。你以诚实的方式展示这一点的能力将为你的成功奠定基础。

最后一点

这些特征永远不会改变。所有这些特质和标志应该伴随你的整个职业生涯。这些不仅仅是成功的研究生软件工程师的标志——它们也是任何成功工程师的标志。
