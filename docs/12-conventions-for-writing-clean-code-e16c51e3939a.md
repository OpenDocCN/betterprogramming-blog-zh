编码约定是编程的风格指南。其中包括编程最佳实践和原则。这里我们将讨论一些编码约定。

# 遵循惯例的好处

*   干净的代码
*   代码质量
*   代码的可读性
*   使代码维护更容易

> “干净的代码简单而直接。干净的代码读起来像写得很好的散文。干净的代码永远不会掩盖设计者的意图，而是充满了清晰的抽象和直截了当的控制线。”罗伯特·马丁

# 3.评论

这个是对某人的一种人身攻击。注释有助于人们以后理解，也有助于其他程序员在同一个项目中工作。代码中的注释意味着你的代码可能不是自解释的。这是杰夫·阿特伍德关于写评论的一句名言。

> “虽然评论本身没有好坏之分，但它们经常被用作拐杖。你应该总是写你的代码，就像注释不存在一样。这迫使你用你能想到的最简单、最明白、最自我记录的方式来编写代码。”——**杰夫·阿特伍德**

**注释应该是好的，但是你的代码需要不言自明。**

# **4.避免大型函数**

**当一个函数或者一个类大很多的时候，那么建议把它分成倍数。这将使我们的代码更容易，干净，易于理解，并且可重用。**

**假设我们需要加减两个数。我们可以用一个函数来完成。但是好的做法是把它们一分为二。当有单独的功能时，这将在整个应用程序中重用。**

# **5.代码重复**

***重复代码*是指在你的代码中重复不止一次的代码块。这意味着您的代码部分需要提取到一个函数中。**

**这是我们之前在第二点中使用的一个例子。，深嵌套，上图。请看第一部分:同样的事情我们已经重复了三次。让单个函数做同样的事情的解决方案是更好的解决方案；而且，这是可重复使用的。**

# **6.变量命名**

**Camel case 是变量和函数以及其他标识符的命名标准。这意味着一个名字应该以一个小写字母开头，下一个单词的每个首字母都是大写的。**

**函数和变量都必须遵循规则。**

```
let camelCase = ''const thisIsCamelCase = () => {
    //so something
}
```

# **7.有意义的名字**

**有意义的名字是最重要的惯例之一。始终为变量、函数和其他变量使用有意义的名称。选择一个能表达你的目的意义的名字。**

**如果我们需要一个函数来获取用户的银行信息，那么我们一定不能使用像`getUserInfo`这样的名字。我们应该用`getUserBankInfo`更具体一点。**

# **8.偏好描述性而非简洁**

**任何命名都要尽量使用细节。假设我们需要一个功能来找到一个有手机的用户。这里我们可以使用有意义的名字，但是如果有其他类似的函数，就很有可能出错。**

**我们必须使用一个详细的、有意义的名字，用简单的话表达意思。**

```
//We want a function for search user against phone no//Bad practice
const searchUser = (phone) => {
//Do something
}//Good practice
const searchUserByPhoneNo = (phone) => {
//Do something
}
```

# **9.根据概念使用一致的动词**

**这是重要的命名惯例之一。如果我们需要一个 CRUD 函数，我们用`create`、`get`或`update`来命名。**

**如果我们需要从数据库中获取用户信息，那么函数的名称可以是`userInfo`、`user`或`fetchUser`，但这不是惯例。我们应该用`getUser`。**

```
//Good prancticefunction getUser(){
//do something
}
```

# **10.使用名词作为类名&使用 Pascal 大小写**

**上课不拿东西；他们就是那个东西。类主要是某个东西的蓝图。不要在类名中使用动词。**

**同样，一个类应该包含 Pascal 大小写。Camel case 用于对象，所以如果您使用 camel case 用于类，这不会很清楚。**

```
//bad practice
class MakeCar = {
    //...
}//Good practice
class Car = {
    //...
}
```

# **11.常量值大写(蛇形大写)**

**这是我们需要遵循的另一个惯例。始终使用完全大写的常量名称。**

***蛇形大写*表示所有的字母都将是大写的，下划线将分隔所有的单词。**

```
const DAYS_IN_A_YEAR = 365;
```

# **12.避免单字母变量名**

**一个字母的变量是一个非常非常不好用的东西。不要用这个作为变量名。**

**但是在一个循环中，我们用一个字母来表示一些变量，用起来是可以的。**

```
//bad practice
const q = () => {
    //....
}//good practice
const query = () => {
    //....
}//this is also okay
for(let i = 0;i < 10; i++){
    //...
}
```
