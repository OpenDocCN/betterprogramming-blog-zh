# 7 个未被发现的 YouTube 频道来学习 React

> 原文：<https://betterprogramming.pub/7-underdiscovered-youtube-channels-to-learn-react-b1b3b6ebf788>

React 初学者的有用资源

![](img/d57b6633ef0c5a42f951357ce28abaf3.png)

乔治·波特拍摄的照片

在这篇关于[在 YouTube](/where-to-start-learning-to-code-on-youtube-c1e0a9853851) 上从哪里开始学习编码的后续文章中，我将告诉你一些具有良好知识库的频道，它们的关注者数量低于它们应有的数量。您将要阅读的列表由主要关注 React 前端开发的优秀教师组成。

如果您刚刚开始使用 React 作为框架，建议您先学习基础知识，然后再随意浏览其他资源。

*   你可以从 Mozilla 的[React](https://developer.mozilla.org/en-US/docs/Learn/Tools_and_testing/Client-side_JavaScript_frameworks/React_getting_started)入门指南开始，它向你展示了构建一个待办应用的基础。
*   之后，在'['入门](https://reactjs.org/docs/getting-started.html)'指南中阅读 React 官方文档，通过文档了解更多理论方面的内容。
*   务必注意“[挂钩](https://reactjs.org/docs/hooks-intro.html)部分。

我将提供的一些资源提供了结构化的课程，指导您从头开始构建应用程序的整个过程，而其他资源提供了更广泛的教程供您挑选。

事不宜迟，让我们进入列表。

# Leerob

[Lee Robinson](https://www.youtube.com/user/MaStaleee) 是一名前端开发人员，擅长清晰完整地传达编程概念。他几年前开始创作 YouTube 视频，主要关注 React 和 Next.js，目前是 Next.js 背后的公司 Vercel 的开发者关系主管

也可以看看他的网站，里面有很多很酷的功能，还有关于如何在你自己的网站上实现这些功能的博客文章。

*我最喜欢的东西*

*   他的初学者系列上 [Next.js](https://www.youtube.com/watch?v=uQeidE2LA1s&list=PL6bwFJ82M6FXjyBTVi6WSCWin8q_g_8RR)
*   他与 Next.js 合作建造 SaaS 的新系列， [React 2025](https://www.youtube.com/watch?v=JCOPVq2AYXc&list=PL6bwFJ82M6FXgctyoWXqj7H0GK8_YIeF1)

# 错误的阿克兰

我真的很喜欢 Akram Khalid 的频道，因为他给了 YouTube 一些它所缺乏的东西，一些我很喜欢的东西，即向我们这些外行人展示你如何能够创建与 Awwwards.com 的 T2 不相上下的互动网站，使用动画库，如 T4 的 GSAP 和 T6 的 Framer Motion，并用 React 开发它们。

Akram 有关于使用 Figma 准备您的设计、使用 React 和 Framer Motion 的基础知识的教程，甚至还有获奖网站的完整再现。

他发帖子的频率比以前少了一点，但你仍然可以在他以前的视频中找到很多精华。

*我最喜欢的东西*

*   [Awwwards 重建第 1 集|使用 React 重建 Awwwards 网站& GSAP |第 1 部分](https://www.youtube.com/watch?v=ig7ZPRRqMz0&t=2s)
*   [React 和 Three.js (React 三纤)入门](https://www.youtube.com/watch?v=fdtqqyeKRJk)

# 阿里·斯皮特尔

Ali 是 AWS 的高级开发专家，也是 web 开发社区的长期发言人。她有自己的 [YouTube 频道](https://www.youtube.com/c/AliSpittelDev/videos)，在那里她教初学者 React、Next.js、GraphQL，还有 AWS Amplify。

她在 [alispit.tel](https://alispit.tel/) 也有自己的网站，并在[welearncode.com](https://welearncode.com/)教授代码，在那里她分享关于上述主题的完整初学者指南。

*我最喜欢的*

*   她在 Next.js 上的初学者视频，比如[这个短片](https://www.youtube.com/watch?v=nBkRxwHMrto)
*   [在 AWS | Amplify Storage、Auth、Hosting 和 Data 上创建 Fullstack Next.js 应用](https://www.youtube.com/watch?v=GsObT64SRhA)

她在她的频道上也有一个“我在别处”的播放列表，在那里她和其他讲座一起在我们的下一个主题频道上出现。

# 杰森·伦斯托夫

Jason Lengstorf 目前在 Netlify 担任首席开发人员体验工程师。Jason 的 YouTube 频道主要是他未经编辑的常规直播流的集合，其中他与一位知识渊博的嘉宾讨论了各种编程主题，主要是与 [JAMStack](https://jamstack.org/) 相关的现代前端技术，简而言之就是 JavaScript、API 和标记，这是一种前端与后端分离的开发理念，允许灵活的开发人员体验。

虽然在他的信息流中有相对大量有趣的闲聊，但也有硬技术质量编码。对于每一个可以想象的与现代前端开发相关的话题，Jason 都有涉及。如果你想了解更多的现代网络技术，Jason 有一个专门的视频。

还有，看看他俏皮的[个人网站](https://www.jason.af/)；他也在 FrontendMasters.com 大学教书(没有从属关系)。

*我最喜欢的*

有太多的直播流充满了可行的建议，但我只选择几个:

*   [Contentful，GraphQL，&付费内容(与 Stefan Judis 一起)——与 Jason 一起学习](https://www.youtube.com/watch?v=uZI1wWevtPM)
*   [react-three-fiber 中的动画和 3D(与 Paul Henschel 一起)——与 Jason 一起学习](https://www.youtube.com/watch?v=1rP3nNY2hTo)

# 威廉·坎迪隆

William 为我们提供了来自“美丽的瑞士苏黎世”的源源不断的 React Native 教程，重点不仅仅是代码的功能方面，还有开发有吸引力的应用程序的视觉方面。

他的系列*能在 React Native 中完成吗？*很好地介绍了 React Native 在设计和功能上模仿原生应用的可能性。非常透彻，非常容易理解。

*我最喜欢的东西*

*   [带 React Native 的 SVG 动画](https://www.youtube.com/watch?v=oZHIwKJHrq0&t=1s)
*   [比特币基地专业烛台擦洗器——“它能在 React Native 中完成吗？”](https://www.youtube.com/watch?v=r5XXSb4yQes)

# 利·哈利迪

Leigh 的频道充满了有价值的中等长度的视频，直接进入许多有趣的 React 技术和库的肉和肉汤，您可以立即将其合并到您的项目中。

利是一个伟大的老师，他以平易近人的方式讨论这个主题。

*我最喜欢的*

*   [Next.js with GraphCMS:静态站点生成&使用无头 CMS 的 API 路由](https://www.youtube.com/watch?v=LJhWOUNkNwc&t=1s)
*   [React 中的谷歌地图——构建互动地图](https://www.youtube.com/watch?v=Pf7g32CwX_s)

# 肯特·多兹

如果你在 React 世界有任何经验，你一定会在某个地方碰到 Kent。他是 React 世界的知名人士，提供课程，讲授 React，并编写了 [react-testing-library](https://kentcdodds.com/blog/introducing-the-react-testing-library') 。

他的 YouTube 上充斥着直播和其他与节目相关的内容。他已经记录了许多与自己项目相关的材料，每次会议都要处理几个与项目相关的任务，例如[将 Tailwind 添加到他的书架应用程序](https://www.youtube.com/watch?v=4SDPofcAte8)。

他的*办公时间*被现场直播，他回答观众的问题，深入具体 React 技术的本质，因为它们发生在现实生活的项目中。

*我最喜欢的*

*   他的会议讲话，[只是简单地作出反应](https://www.youtube.com/watch?v=AiJ8tRRH0f8)
*   最近一次[办公时间](https://www.youtube.com/watch?v=0yrRgvTroLM)直播
