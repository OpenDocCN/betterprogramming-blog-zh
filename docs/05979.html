我是如何在亚马逊卢森堡找到工作的

我的流程是什么样的，你也可以怎么做

最近，我收到了大量的信息，询问我一个来自新学院的学生如何在几乎没有指导的情况下在亚马逊卢森堡找到工作。我向每个人承诺，我会在博客上分享我的故事。这就是我从T4斯里城到卢森堡的旅程。

关于我

我是Krishna Kumar Dey，亚马逊卢森堡的SDE 1。我于2020年从IIIT斯里城毕业。我也是2019年Chapel Language的GSoC学生开发者，目前是同一组织的导师。

那么，一个非常新的学院的计算机科学学生是如何在亚马逊卢森堡找到工作的呢？它是一个校园提议吗？我被推荐了吗？

最后两个问题的答案是“否”。 亚马逊没有到我的校园，我也没有得到推荐。

申请阶段

我通过亚马逊工作门户申请。具体日期不记得了，是2019年9月。这个过程很简单，包括像简历提交、填表、点击提交按钮这样的步骤。

在线评估

一个月后，我收到了亚马逊招聘团队关于下一步计划的电子邮件。我的许多朋友已经做过类似的评估，但从未收到回复，所以我也有同样的期待。尽管如此，我还是决定接受评估。它分为三个部分。

第一部分，调试回合(20分钟):在线评估的这一部分，我被要求在七段代码中找出bug。

第2部分，编码(70分钟)和工作风格评估(30分钟):在线评估的这一部分包括使用Java、Python、C#或C++的编码部分。除了编码部分，还有一个围绕亚马逊领导原则建立的工作风格评估。

第三部分，工作模拟(150分钟):评估包括工作模拟和逻辑能力测试。在线评估的工作模拟部分是亚马逊SDE一天生活的互动视频模拟。我遇到了各种各样的情况，并被要求选择如何应对。

一个月后，我收到一封电子邮件，称我入围了亚马逊卢森堡办公室的现场面试。我的面试被安排在一月的第二周，但是由于没有时间，被转移到了二月的第一周。

入围原因

我对此不太确定。在所有三轮中表现良好是一个重要因素。我的开源贡献和实习也有所帮助。在我的简历中有好的项目和领导角色是一个额外的优势。

现场

我在面试前一天到达卢森堡。面试那天，我第一次见到了我的招聘人员。她带我参观了整个办公室，并告诉我流程和可能会发生的事情。15分钟后，我在会见室。面试分三轮进行，每轮45分钟。其中两个主要关注数据结构和算法，以及在时间和空间复杂性方面优化最初提出的解决方案的方法。最后一个问题更像是面向对象的设计问题。

所有轮次都有与亚马逊领导原则相关的行为问题。面试是我和面试官之间的一次极好的讨论。最初，我被要求在白板上解决一个问题，然后讨论转移到如何优化解决方案并改善时间和空间复杂性。

这些问题是容易和中等水平，可以在LeetCode上练习。

提供

在最后一轮面试后，我在两个工作日内收到了招聘人员的回复，说我已经进入了该职位的候选名单。又过了两天，她发来了录取通知书。

准备

我花了很多时间在破解编码面试书和 LeetCode 易中题上。还有很多其他网站可以练习做题。我在两周内解决了大约160个问题(95个简单，65个中等，10个困难)，涵盖了最常见的主题，如两个指针、二分搜索法、链接列表、树、图、DP等。

我利用了很多网上资源，制作了这张表来准备。你也可以检查 14个模式来Ace任何编码面试问题。在最后一轮之前，我觉得自己对数据结构和算法有了很好的理解，并专注于亚马逊过去问过的问题。

最后，我让我的一些CS同行模拟了一个技术环境。他们会问我一个LeetCode问题，我会像对待一次真正的面试一样对待它。它帮助我摆脱了IDE，改进了我的面试编码。然后，我们交换，我可以采访他们。双方的角色扮演帮助我了解作为一名受访者我可以改进什么。

我还写下了每个项目并随身携带，这样我就可以在任何面试前进行修改。这节省了很多时间。我还建议准备好一份简短的自我介绍，介绍给面试官。

对于行为部分，我按照亚马逊关于领导原则的行为问题，根据我的经历准备了背景故事。我根据明星原则(情境、任务、行动、结果)准备了我所有的故事。

个人注释和提示

大多数编码问题都有多种方法。在现场采访中，通常希望您提出多种解决方案，并讨论每种方法的利弊，主要是在时间和空间复杂性方面。首先想出一个强力的方法，然后尝试用不同的方法优化它。

对于使用何种编码语言，通常没有限制。你可以使用任何你觉得舒服的编码语言。此外，与在计算机上自动完成的标准IDE相比，您需要在白板或纸上进行编码。除了内部工作和复杂性之外，精通语法和所有库也很重要。培养这种能力的一个好方法是首先在笔记本上解决一个问题，然后在练习的时候打出来提交。

沟通是另一个需要关注的重点。试着向面试官解释你所有的想法和步骤，而不是只专注于静静地想出正确的解决方案。在编码面试中，大声思考是一种强烈推荐的做法。你的思考过程比直接提出正确的解决方案更重要，只有当你擅长沟通时，这才有可能。

试着用样本输入来测试你写的代码。这会给面试官留下你有能力发现和排除错误的印象。此外，永远不要忘记检查和正确处理所有的角落情况。

从众多求职者中脱颖而出应该做的关键事情

  • 有好的项目——两到三个强有力的项目就足够了
  • 开源贡献
  • 在大学或公司担任一些职位。领导素质有助于获得更多的关注。
  • 做实习。
  • 写博客，并在简历中加入博客的链接。
  • 有时候，从你申请的公司的现有员工那里获得推荐是很好的。

简历上的提示

  • 有一个体面的基于文本的单页模板。
  • 使用黑色文本，保持字体和大小的整洁、简单和一致。
  • 千万不要有任何错别字。
  • 使用项目符号。
  • 不要包含很长的描述性文字。把你自己放在一个审查简历的人的位置上，他可能会同时阅读几十份简历。
  • 确保你包括你的联系信息。是的，这似乎是显而易见的——但是要确保你的名字和电子邮件地址是显著的。在顶部附上您的Github链接或类似链接，并附上您的联系信息。
  • 如果你是学生或相对较新的毕业生，先受教育再有经验；如果你已经工作了不止一两个职位，先有经验再受教育(按时间倒序)。
  • 对于应届毕业生，公司坚持学校、学位、专业、GPA、毕业年月。你离大学越远，你期望包含的大学信息就越少。
