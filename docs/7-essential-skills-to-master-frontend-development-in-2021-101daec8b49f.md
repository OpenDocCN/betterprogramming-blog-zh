# 为什么是 Web 开发？

在这个数字时代，每天都有越来越多的公司上线。这意味着每个公司都需要一个需要开发的网站。这些网站通常是由 web 开发者开发的。即使是现在，你也可能在由一个 web 开发团队开发的 web 浏览器上滚动浏览媒体。在这个不断发展的数字时代，网站对企业乃至一个国家的发展起着重要作用。大多数社交媒体网站，如 Instagram、Twitter、脸书，甚至 YouTube，都是由 web 开发者创建的。

# 公共福利协会的兴起

现在，随着智能手机的兴起，2021 年[将有 38 亿智能手机用户](https://www.statista.com/statistics/330695/number-of-smartphone-users-worldwide/#:~:text=The%20number%20of%20smartphone%20users,the%20100%20million%20user%20mark.)，对网站有着巨大的需求。

当智能手机在 2007 年开始流行时，网站大多由 internet explorer 驱动，浏览网页并不是一种很好的体验。在那段时间里，智能手机具有 GPS、摄像头和其他只有本地应用程序才能使用的功能。本机应用程序是专为特定设备构建的应用程序，可以直接安装。

现在，随着渐进式网络应用程序的兴起，如果应用程序在 Play Store 或 App Store 上，这些 pwa 能够提供与原生应用程序相同的功能，而无需遵循谷歌或苹果的切割。因此，随着 PWAs 的兴起，对网络开发人员的需求只会增加。如果你开发一个 web 应用，它将是跨平台的，不像创建一个原生应用。

# 1.HTML、CSS 和 JavaScript

当我们谈论 web 开发时，我们需要了解的主要基础知识是 HTML、CSS 和 JavaScript。

我们来比较一下网站的要素和人体的要素。HTML 定义了网页的结构，你可以把它想象成你身体的骨架。JavaScript 就像大脑一样，使用它可以执行复杂的操作，如播放视频、显示交互式内容等。CSS 用于给你的网页添加样式。你可以把它看作是你穿的不同类型的衣服。这是 web 开发入门的必备技能。

# 2.反应

根据 [Stack Overflow](https://stackoverflow.com/) 的 [2020 开发者调查](https://insights.stackoverflow.com/survey/2020)，React 是最受欢迎的 web 框架。

![](img/e4f008ace4fd06d5837474548aaa9b45.png)

来源:[堆栈溢出](https://insights.stackoverflow.com/survey/2020)

这是一个非常强大的 JavaScript 库，用于构建交互式用户界面。它使用基于组件的设计，因此一切都被分成组件。我们有父组件，它呈现子组件。区别于其他前端框架的特性是虚拟 DOM。

真正的 DOM(文档对象模型)基本上就是你在浏览器上看到的 HTML 结构。虚拟 DOM 和真实 DOM 是一样的。我们基本上可以假设虚拟 DOM 是真实 DOM 的副本。它具有与 DOM 对象相同的属性和方法，但是它不能控制屏幕上显示的内容。虚拟 DOM 中的操作非常快。

当代码发生变化时，虚拟 DOM 中的每个组件都会被重新呈现，然后与先前虚拟 DOM 状态的快照进行比较。只有被改变的元素会在真正的 DOM 中重新呈现。这个过程也被称为*对账*。因此，我们可以使用 React 构建快速高效的用户界面。

它非常简单和容易，因此它有一个指数学习曲线。

如果您学习 React，您也可以通过学习 React Native 进入本机应用程序开发，React Native 与 React 非常相似，并使用与 React 相同的基于组件的设计。如果你知道反应，学习母语是很容易的。

# 3.饭桶

无论是前端还是后端，Git 对每个开发人员来说都是必不可少的。Git 是一个分布式版本控制系统。所以你可能会问*版本控制系统*到底是什么。举个例子，假设你在手机上下载了一个应用(我们姑且认为它的版本是 1.0)。一周或一个月后，根据应用程序的不同，您会在名为版本 1.1 的更新中获得新的额外功能。这不过是版本控制。

假设你正在做自己的项目。您构建了一个具有一些特性的项目，并且您的项目运行良好。然后可以使用 Git 保存这段代码。稍后，您可以继续添加更多功能。然后，您意识到您添加的新特性正在导致您的应用程序崩溃。Git 允许您保存项目的不同版本，所以如果您愿意，您可以随时返回到您之前的代码。因此，它被称为版本控制系统。

# 4.Redux

在 React 中，我们有一个基于组件的设计，我们使用父组件来呈现子组件。我们将数据(也称为*道具*)从父组件发送到子组件。当我们需要将这些数据从父母发送给孙子或曾孙或更远的时候，我们会遇到一个被称为*道具钻探*的问题。这意味着我们总是需要添加所有中间子组件的数据。为了解决这个问题，我们有一个名为 Redux 的解决方案。

![](img/3664c00a3264431047f44a78f845ef2f.png)

[Redux](https://redux.js.org/) 用于前端框架中的应用状态管理。我们可以通过使用 Redux *store* 来解决这个问题，Redux*store*拥有可以从 UI 的任何部分访问的所有所需数据，因此可以处理 prop drilling。在有大量数据的应用程序中，我们需要一个真实的来源，我们可以使用 Redux。

随着 React 中 Redux 挂钩(`useDispatch( )`和`useSelector( )`)的引入，在项目中实现 Redux 变得前所未有的简单。

我们还可以使用 JavaScript 的新特性，比如可选链接和 nullish 合并，而不用担心兼容性，因为它将被编译成 ES5 JavaScript。因为 TypeScript 帮助我们更快地发现问题和错误，所以它是最受欢迎的语言之一。

# 6.Rest API

应用程序编程接口(API)主要帮助不同的应用程序相互通信。在这个时代，我们几乎在所有事情上都使用 API，因为我们的产品使用了很多服务和功能。因此，API 使我们能够使用其他应用程序开发的特性和功能，并将它们集成到我们的应用程序中。

Rest 代表*表象状态转移*。它帮助我们使用 HTTP 方法来创建、读取、更新和删除(CRUD 操作)所需的数据。我们每天都用这些。您正在阅读这篇文章，因此您发出了一个 GET 请求，从服务器获取这篇文章，以便您能够阅读它。您可以假设我使用了 POST 请求来添加和发布这篇文章。如果需要，我可以使用 PUT 请求来更新本文。最后，我可以使用删除请求来删除这篇文章。因此，使用 Rest API 确实让我们的生活变得更加轻松，这无疑是一项重要的技能。
