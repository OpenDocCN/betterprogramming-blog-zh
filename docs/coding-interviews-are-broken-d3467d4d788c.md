# 这些算法到底意味着什么？

IT 行业有一种信念，为了让软件开发人员证明他们的价值，必须通过一项测试。测试逐渐变得更加关注算法问题，这些问题…证明不了什么，也没有实际用途。

好了，我说了。算法问题毫无意义。而你最终将不可避免地失去优秀的候选人，因为他们无法通过这个测试。

我不知道是谁决定这些问题或练习证明了一个开发人员的价值，但是那个人错了。简单明了。

我怎么知道这个？嗯，在创作和推广音乐的间隙，我还作为一名技术采访者与一家自由职业公司合作。他们使用一个平台，候选人在这个平台上解决不同的任务，我想我已经见过大约 10 个没有通过算法测试的开发人员。但是当他们不得不解决一个现实生活中的任务时(你知道，他们实际上得到报酬的东西)，他们出色地通过了。

我非常明确地告诉他们，我会超越分数，如果有人有潜力，我会想和那个人一对一地谈谈，看看我是否错了。

甚至不要对我说“但是我们想看看开发者是怎么想的！”真的吗？这真的是你最好的吗？让某人接受一到两个小时的测试，测试中他们可能被允许也可能不被允许使用互联网？在测试中，他们需要使用大学里教的算法来解决一些随机的无用问题，而一些开发人员已经多年甚至几十年没有使用这些算法了。从中可以获得什么信息？这能带来什么好处呢？

# 关于疯狂冗长的采访…

然而，这个故事还有更多。我也遇到过这样的工作，作为编码挑战的一部分，我被要求解决一些事情…并被告知我应该分配六个小时来做这件事。

听好了，所有有抱负的开发人员:如果你曾经遇到过这样的场景，很有可能该公司正试图为他们的项目解决任务，而没有付钱给任何人去做。所以，他们把这作为编码挑战的一部分，你解决了它，失败了，他们有了代码。

对此要警惕。和公司，请停止这样做。我不是说你们都这样做。我是说，当我不得不基本上休息一天来解决你的编码挑战时，这里面有猫腻。

我认为在这样的挑战中发生在我身上的最有趣的事情是当我正确地解决了需求 1-4 而我失败了，因为我没有正确地解决秘密需求 5 和 6。专业提示:永远不要开发你没有得到报酬或者客户没有要求的东西。

很有可能客户不会喜欢它，我很有信心说客户不会欣赏你在他们没有要求的东西上浪费他们的时间和金钱。

# 做什么

那么，所有的希望都破灭了吗？我们没有办法解决这个难题吗？当然有。

简单地把所有的算法问题丢给那些对这些问题毫无用处的工作。前端开发人员主要负责建立一个网站，不需要做算法来制作 Angular，React 或 Vue.js 应用程序。

同样，如果你正在寻找一个后端开发，但该开发主要负责建立网站的后端部分，放弃算法。它们没有任何用处，即使有一点点可能他们会需要它们，也总会有互联网来寻找解决方案。在这些面试中，不要把注意力集中在 1%的日常任务上。99%的时间，后端开发人员都忙于连接数据库和检索数据。

然后，也许，仅仅是也许，开发人员可能会倾向于给你的公司一个机会，你将不再错过好的候选人。
