我想看看造就一个伟大的开发者的三个要素。现在，这些不是*只有*三样东西。他们也不局限于已经在这个领域工作了一段时间的人。它们适用于任何阶段的任何人，无论是刚开始还是更有经验。我不认为成为一名优秀的开发人员意味着不断地生产代码。当然，您可以编写漂亮的代码，整天提交，并运行成功的测试。但是我认为还有其他的因素，其中三个是我今天要讲的。

# 职业化

我认为专业精神有两个方面:人和为人服务的准则。对人，就是你如何对待别人。可能是初级开发人员，也可能是高级开发人员，也可能是客户。如果你从初级开发人员的角度来看，你会意识到，作为一个高级开发人员，他们经常会带着问题或不同的想法来找你。倾听这一点很重要，因为它们通常会带来不同的观点。你可能被困在一个项目中——如果你不听下级的话，你可能意识不到有更好的方法来解决你的问题。

人们也可能向你寻求支持。帮助和指导是一个一举两得的好方法。你要帮助他们成为更好的开发者，你自己也要成为更好的开发者。当你这样做的时候，你也提高了你记住你所教内容的能力。如果你从高级开发人员的角度来看，倾听你的技术主管，作为一个初级开发人员，可能提出的建议或委派是一个好主意。我并不是说你应该盲从——如果你有任何问题，提出问题并分享你的观点是个好主意。

就客户而言，客户都认为他们的业务是他们最重要的资产。他们把所有的骄傲和努力投入到每一件事情中。这意味着，当他们来完成一项工作时，他们通常会希望立即完成。如果你同意，他们的工作应该是优先的。大多数开发人员不会根据预算在不同的项目上投入相同的精力。我认为一个好的开发者应该不管项目如何都投入相同质量的工作。这是因为，在一天结束的时候，你正在努力实现企业主的目标。

谈到代码，有几件事需要考虑。尽管如此，我还是会坚持基本原则。一般来说，编写可读性强并且有内联注释的代码通常可以做到这一点，因为我认为一个好的开发人员可以考虑未来。我这么说是什么意思？一个好的开发人员会考虑他/她希望他们的代码在未来如何为他们服务。他们能在几个月或一年后阅读并理解它吗？如果需要的话，他们能够通读它，制作一个模型并重构它吗？大多数(如果不是全部的话)开发人员讨厌回头看旧代码，甚至讨厌看别人的代码。你应该确保你的代码是可重用的，并且有一个有意义的语法。

另一件事是测试。如果你的代码写得很好，你就不仅仅是在执行一个简单的测试，比如检查一个元素是否呈现，而是要考虑客户，考虑在他们破坏系统的情况下有效地检查你的代码的方法。这种方法的诀窍是确保你已经完全覆盖了你的申请。

# 责任

你能兑现你的承诺吗？不管是一段代码还是一个应用程序，没有什么比从企业主那里听到上一个开发人员要求更多的钱、更多的时间、更多的细节，但项目从未交付更糟糕的了。一个好的开发者应该了解他们的技术栈。一个好的开发人员应该有能力估计任何项目将花费多少时间和金钱。他们还应该有能力处理过程中可能出现的任何问题——这些问题总是会出现。很多时候，你在这里看不到。

负责任的另一部分是当你犯错误时承担责任。没有什么比拥有一段有缺陷的代码更糟糕的了，尤其是当你受到时间限制的时候。我敢肯定，在我们的职业生涯中，我们都犯过这样或那样的问题。能够承认错误真的很重要。有些人害怕向经理承认他们没有写出完美的代码。有时候，我们被期望那样工作，但是在你第一次写代码的时候，没有干净的代码这样的事情。我有一篇关于编写干净代码的神话以及实现它的方法的文章——我建议你读一读:

理解每个人都是人，都会在某些时候犯错误，这在发展中很重要。如果你不承认你的错误，并且生产中有一个错误，它将影响业务和客户，没有人希望这样。没有比开源社区更好的地方来了解承担错误的责任，在那里你可能有一个库或一段代码，然后被很多人审查。没有人有完美的答案，但是你可以从别人的建议中学到很多。他们会有不同的做事方式。

# 提高的能力

这可以采取几种不同的形式，因为你需要在不同的方面进行改进。一个可以自强。这可能意味着您确保了解您的技术堆栈以及它每年是如何变化的，无论是 PHP 还是 JavaScript 等等。您需要理解，这些技术不会一成不变，在特定的技术中会添加新的功能——比如 JavaScript 中的箭头函数或分解。如果你不改进，你可能在接下来的一两年里看代码，甚至不理解它。自我提高也可能意味着你正在检查新的图书馆。例如，当 SASS 问世时，它使编写 CSS 的速度几乎提高了一倍，甚至更多。那些没有走在这条曲线前面的人在从事一个项目时会花费更多的钱。

改进也意味着你的代码是最新的。在项目中的很多时候，我们可能会让我们的代码有点陈旧，因为我们没有更新所有的模块，我们只是停留在相同的版本库中。这不会破坏应用程序，但我们需要更新我们的代码库，因为可能有安全漏洞和其他问题需要修复。如果您让代码过时太久，以后的重构可能会是一个大麻烦，以至于您宁愿重写整个应用程序。我认为一个好的程序员应该愿意更频繁地重构代码，以保持代码符合最新的变化。

我认为这是成为一名优秀开发者的三个要素。显然，事情远不止如此。我可能是对的，甚至是错的。如果您对如何成为一名优秀的开发人员有一些反馈或自己的想法，我期待着阅读您的建议。
