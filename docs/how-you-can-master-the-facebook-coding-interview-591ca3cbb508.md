随着禁闭的继续，我意识到是时候改变我的职业生涯了。虽然我已经离开就业市场几年了，但我决定探索我对编码的热爱，并申请在[亚马逊](https://medium.com/swlh/5-things-you-need-to-know-to-ace-the-amazon-coding-interview-ab7e8453848e)和脸书的软件工程师职位。通过这个过程，我了解到获得这份工作不在于经验，而在于实践。一旦你的脚迈进了门，被叫去参加面试，权力就在你手中，你可以通过为这次会面做准备来增加对你有利的筹码。

我的一个朋友目前是脸书办公室的软件工程师。一旦我申请了这份工作，我就向他寻求建议，告诉他我该如何准备。根据他的建议和我的经历，在面试过程中，我了解到有几个关键的要素需要给人留下深刻印象。

面试过程的结构不同于标准的问答形式。面试官很小心，不要用一般性的问题浪费你的时间，而是用密集的面试来真正确定你对软件工程的掌握。

一旦你申请了这个职位，你将有两次电话面试来审查你的资历，并确定你是否是一个合适的候选人。你会被安排一个招聘人员，他是你最好的联系人。招聘人员会帮助确保你为面试做好准备，并引导你完成面试过程。招聘人员将为您提供一套信息，供您查看，并询问您何时愿意安排全天现场白板面试。给自己至少两周的时间准备白板面试。这是过程开始的时候。

如果您已经参加了白板面试，这就是您需要准备的地方。现场面试包括六个部分:两个使用白板的编码问题、三个系统设计问题和一个行为管理部分。面试很累人，但它是你和软件工程师职位之间的最后一步。如果你能正确准备，你就能成功获得这份工作。

我把我的经历归结为五个关键部分。

# 1.依靠你的招聘人员

随着你在面试过程中的进展，你会有一个招聘人员分配给你，他将是一个不可思议的资源。他们想确保你成功，并会给你提供一包关于你在面试当天会遇到什么的信息。通读信息包——不仅仅是一两次，而是尽可能多的通读，直到你从记忆中了解信息。

当他们要求面试的时候，试着给自己额外的时间来准备面试，并要求至少提前两周。一定要问招聘人员任何关于准备和面试的问题。

# 2.为编码问题做准备

编码面试将在现场面试时进行，由两部分组成。每节课都有一名员工要求你在白板上尽可能准确地解决 1-2 个问题。他们并不追求完全的准确性和代码的运行，但是您希望确保您的编码技能达到代码看起来能够运行的水平。

为了做好充分的准备，您需要尽可能多地理解编码问题。你每天至少要花三个小时练习编码问题，以提高你的速度。为了准备编码问题， [LeetCode](https://leetcode.com/) 是一个很棒的免费在线资源。在面试日之前，练习中等难度和难度的问题，并尽可能多地完成。

面试当天你会收到各种各样的问题，面试官会确保你了解这些基本知识。在他们提出问题后，您希望能够通过您的过程和不同的方法来讨论编码问题。请务必复习排序问题、矩阵问题，并理解您在编码时所做的假设。面试官可能会用新的标准来跟进原来的问题，这将使问题变得更加复杂。

最好的准备方式是尽可能多地练习高难度的编码问题，因为 45 分钟的课程似乎没有时间。

# 3.准备系统设计问题

由于脸书在编程领域的广泛影响，程序设计问题可以在它们提供的不同服务之间的适用性范围内。他们可能会问与脸书、WhatsApp、Instagram 或 Spotify 相关的问题，并提供他们试图在其中一个领域解决的问题的背景。

在现场面试当天，你将有三次 45 分钟的程序设计会议。为了准备这些问题，你需要熟悉脸书母公司的服务。要解决这些问题，您需要了解问题的组成部分以及如何扩展系统。 [YouTube](https://www.youtube.com/) 是寻找系统设计面试问题范例的绝佳资源。理解创建系统、扩大系统规模的方法，以及完成任务所需的资源。

# 4.准备心理面试

现场面试将包括一小时的高层管理会议，其中包括心理社会面试。在面试中，管理层会询问你为什么想要这份工作，你在寻找什么。他们也会问一些更普通的问题。需要准备的问题有:

1.  说说你自己吧。
2.  你最引以为豪的成就是什么？
3.  告诉我一个事情没有按计划进行的时候。你的角色是什么？你是如何应对这种情况的？
4.  你的强项是什么？
5.  你的弱点是什么？

为他们可能提出的问题准备一些经验。确保你的经历回答了这个问题，但也表明你能够解决问题并很好地应对困难局面。你要用一种能显示个人成长的方式来描述每一个困难。诚实地面对你所面临的挑战，但也要表现出你从中吸取了教训。

以我的经验，面试官问了很多后续问题，一定要非常了解你回答背后的背景。提前和你的家人或朋友练习面试问题，以获得对你声音的反馈。你想要自信地去参加面试，并且能够展示出你最好的一面。

# 5.失败不是失败

最后，我通过了面试，但没有得到那个职位。这次面试并不是游戏的结束，总会有更多的机会。脸书面试极其困难，但很有收获。最终，我知道如果我想得到脸书的工作，我还有很多要学，但是准备过程让我成为了一个更好的程序员。

生活是一系列的失败，中间穿插着成功的时刻。

希望你能用我的经验去学习。我发现我倾向于从失败中学到比事情按计划进行时更多的东西。如果你发现自己处于类似的情况，要明白失败是没关系的。我们的老茧是在崎岖的地形上形成的。一次面试并不能说明你是一名程序员。花一天时间，感受悲伤，然后是时候再试一次了。
