我在去谷歌工作的路上学到了什么

用事实、数据和经验讲述我的求职故事

我生命中的最后八个月都在面试和寻找下一份工作。一路上，我都做了详细的记录。我梳理了一下资料，发现了一些有趣的信息。如果你正准备开始自己的求职之路，你应该看看这些数据，了解你将进入什么样的领域。我甚至有一些建议，希望能有所帮助。

当我开始找工作时，我在卡耐基梅隆大学的软件工程硕士项目已经完成了一半。当时是2020年5月，所以新冠肺炎正处于高峰期。一些公司已经进入招聘冻结期，看起来很难找到工作。我12月份就要毕业了，所以我没有时间去等待更好的就业市场。

我的背景

为了给本文的其余部分设置背景，我将简要地解释一下我的背景。好奇的话可以上 LinkedIn 查我。我保留了详细的资料。

在我找工作的时候，我是卡内基梅隆大学的硕士研究生。在读研究生之前，我在马萨诸塞州和新罕布什尔州的自由互助保险公司工作了六年多。我主要在负责API管理平台的DevOps团队工作。我专攻RESTful APIs。我是一个多面手，但我主要从事Node.js微服务。当我离开公司时，我已经是一名高级软件工程师了。

除了在Liberty Mutual的工作之外，我还有共同创办一家初创公司的经历。这家初创公司诞生于一次成功的黑客马拉松。它在市场上没有受到欢迎，所以我们在三年后关闭了商店。

在加入Liberty Mutual之前，我毕业于本特利大学，获得了计算机信息系统学位。

我的时间线

我的求职从5月份开始，一直持续到12月底。八个月比平均时间要长，但是我有时间，因为我要到12月份才能毕业。我想在不到四个月的时间内完成整个过程是非常困难的。如果你开始这个过程，你应该预计需要4-6个月。

我通过弄清楚我将如何准备来开始这个过程。我把大部分准备时间都用来组织面试学习小组帮助同学了。小组成员解决问题，讨论解决方案，并互相采访。我们提高了解决问题的能力，并适应了面试形式。

我还用 Algo Monster 和 LeetCode 练习技术面试题。到了八月中旬，我已经可以连续进行三次模拟面试了。然后就该开始申请了。

我开始通过搜索LinkedIn的招聘信息和公司的职业页面来申请。这是一个乏味的过程。我通过每天花一点时间来申请，使它变得更容易管理。

申请的头几周令人沮丧，因为有很多人立即被拒绝了。我不知道为什么我的简历被忽略了。最终，招聘人员开始联系我。一旦我预约了几次面试，每天处理1-3次拒绝就容易多了。

从10月到12月初，我正忙得不可开交。我申请并面试了。我收到了拒绝和提议。我的第一份工作是在11月初。在整个12月中旬，他们继续陆陆续续地来到这里。总而言之，我最终从185份申请中得到了6份工作。

面试结束后，我又花了两周时间与协商我的薪酬。然后过程就结束了。当我在谷歌的报价上签字时，是香槟和微笑。

以下是每项活动的内容:

  • 准备——我花了这段时间收集资源，想出如何练习面试。最耗时的活动是和同学一起组织面试学习小组。
  • 练习——我花了额外的时间练习编码面试，因为我从未上过数据结构或算法课。我通过解决问题和模拟面试来练习。
  • 申请——我阅读了LinkedIn和职业页面上的招聘信息。我也联系了业内的朋友，看他们公司有没有招人。我在不同的公司申请了各种各样的职位。感觉我花了超过43个小时申请职位，所以很可能我遗漏了一些数据。
  • 面试——这是花在电话屏幕、在线评估、技术电话面试、虚拟现场面试和报价谈判上的时间。这也包括我花在协调这些活动上的时间。我很惊讶我花了多少时间安排面试。如果你开始面试，你发邮件的时间会比你想象的多。

我申请了什么角色？

我想找一份高级软件工程的工作。我喜欢我的人工智能软件工程课，所以我想从事人工智能系统方面的工作。如果可能的话，我还想通过构建可伸缩的系统来应用我的分布式系统知识。

我注意到中型公司最有可能有独特的面试方法。我将中型企业定义为150-500名员工或10-50亿美元的估值。大多数公司使用传统的技术面试流程。这包括人力资源电话筛选、算法电话面试和最后一轮技术、设计和文化面试。

然而，中型公司更有可能偏离常规。我看到的最常见的推导是带有一组需求的带回家的问题。通常，问题与业务或角色直接相关。有时，需求故意含糊不清。老实说，我更喜欢这类问题。我觉得我的解决方案展示了实际的工程技能，不像传统的算法问题。

面试有多难？

在我找工作的过程中，我申请了185个职位。正如人们所料，这一过程中最困难的一步是获得初步回应。只有五分之一的公司在收到我的简历后表示有兴趣。很难知道我的简历是被人工智能拒绝了，还是因为公司处于招聘冻结期。很多公司一直没有回应。

从那以后，我获得了更多的成功。在面试过程的每个阶段，我都有超过50%的通过率。这意味着，对于任何给定的申请，我有3.3%的机会获得录用。

为了收集这些数据，我对面试过程的各个阶段进行了分类:

  • 已申请—提交了我的简历。
  • 招聘人员——人力资源部的人或招聘经理安排了30分钟的电话面试。
  • 电话面试—一小时的技术电话面试和/或带回家的在线评估。
  • 现场—一系列2-6小时的技术、设计和文化适应面试。因为我在新冠肺炎采访，这些采访是虚拟的。
  • 提议——我通过了！

让我惊讶的是，我的通过率在整个后期都是恒定的。我预计每一轮面试都会比上一轮更难。通常情况并非如此。除了少数例外，电话面试和现场面试的难度并没有增加。我认为现场采访的目的是确保电话采访不是异常现象。

经验教训

1.练习，练习，练习

我找工作最重要的部分是我花时间练习技术面试。我超过了100小时的练习。根据一项interviewing.io调查，这种准备量是常见的。

要想擅长面试，你需要擅长解决技术问题。你需要对数据结构和算法有一个基本的理解，还要有很多练习题。我认为 Algo Monster 是学习算法的最佳资源，而 LeetCode 是寻找练习题的最佳地方。

面试的另一个方面是适应面试的形式。面试更多的是表演而不是编程。在面试中，你必须一边写代码一边讨论，这对大多数工程师来说并不容易。我做的最好的准备就是创建一个学习小组。我强烈建议联系工程师朋友，创建自己的小组。

像许多表演一样，你应该在大型演出前做一次低风险的练习。人们不会从洗澡时唱歌变成在阿波罗剧院表演。他们通常从几个开放的麦克风之夜开始。如果你和你的同事一起练习面试，那么面试的表现将成为你的第二天性。如果你找不到可以一起练习的人，网上有提供模拟面试的资源，比如面试。

2.不要气馁

我知道你已经知道了。我在找工作的时候就知道这一点，但是到了九月中旬，我还是泄气了。记住，公司更关心误报(收到工作机会的坏候选人)而不是误报(被拒绝的好候选人)。你会被你最适合的工作拒绝。他们的损失。继续申请，继续面试。

3.适用于许多机会

既然你会因为你应得的职位而被拒绝，你需要广撒网。这不是一件坏事。职位描述并不完美。你可能会发现一个角色比你原来想象的更有趣。你也可能会发现你喜欢的工作没有它描述的那么令人兴奋。

申请很多机会的另一个原因是为了练习。早期，你想要一些你不介意轰炸的采访。这将帮助你为那些让你兴奋的公司做好准备。我也遇到过这样的情况，在面试过程的最后，我需要一个实习公司。我有两周的时间没有任何技术面试。我已经有点生疏了，所以我用一家公司来为另一场面试热身。

4.计划什么时候申请大公司

大公司的一个优势是他们有明确的招聘流程。他们有如此多的申请人通过系统，很容易知道这个过程需要多长时间。不利的一面是，参加面试的候选人数量增加了面试过程的灵活性。

例如，如果你有一个即将到期的工作机会，需要加快这个过程，这在一个大公司会困难得多。小公司在加快流程或延长截止日期方面更加灵活。

我遇到过这样一种情况，我在申请FAANG(脸书、亚马逊、苹果、网飞、谷歌)公司的过程中遇到了麻烦，不得不要求加快面试。他们尽了最大努力，但还是花了一个月。你应该预计FAANG公司的面试过程需要4-6周。申请的最佳时间是在你收到第一份工作邀请前的2-3周。

5.使用你的联系人

浏览LinkedIn，看看你是否认识这家公司的人。我承认我不擅长做这件事。请求帮助可能会让人不舒服，但会有回报。

在我找工作的过程中，我接触了从密友到朋友的朋友的每一个人。没有人拒绝帮助我。推荐的最大好处是你更容易被注意到。我推荐的人中有57% (4/7)的人会接到招聘人员的电话。这比没有推荐的18%的回调有了很大的改善。我甚至让一个朋友的朋友多次跟踪人力资源部，查看我的申请状态。我肯定人力资源部会忘记我的申请，但我最终还是参加了面试。

我想推荐对这个过程的后期影响较小。但如果他们只是在第一阶段提高了几率，我从任何一份申请中获得录用的几率仍然从3%跃升至10%。实际上，我从推荐人那里看到了28% (2/7)的出价。

如果你对要求推荐感到不舒服，记住大多数公司都有慷慨的推荐政策。因此，你在帮助一个熟人获得丰厚的奖金，他们也在帮助你获得一份工作。你们都赢了。

结论

由于每次求职都是独一无二的，我希望我的经历会和你的不同。无论你是谁，这个过程都将充满挑战。我在前进的道路上挣扎过，但我最终找到了一份新工作。

我希望我提供的信息和提示能在您的旅途中帮助您。狩猎愉快！
