开始使用这些库或提高您的技能

Python几乎总是数据科学家的最佳选择。这是因为它的多功能性和简单性，但最重要的是，这要归功于社区和重要公司分发的开源包。

作为一种通用编程语言，Python被用于web开发(使用Django和Flask)、数据科学、机器学习、网络安全等等。

今天，我们将讨论每个数据科学家都必须知道并应该使用的13个数据科学和机器学习库。

数据科学基础库

这些基础库使Python成为数据科学和机器学习的有利语言。以下软件包将允许我们分析和可视化数据:

  1. NumPy 是用Python进行科学计算的基础包。其中，它包含一个强大的N维数组对象、复杂的(广播)函数、集成C/C++的工具和Fortran代码。它在线性代数、傅立叶变换和随机数功能中非常有用。除了其明显的科学用途，NumPy还可以用作通用数据的高效多维容器。可以定义任意数据类型。这使得NumPy可以无缝、快速地与各种数据库集成。
  2. SciPy 在NumPy的基础上增加了一组算法和高级命令来操作和可视化数据。这个软件包包括数值计算积分，解微分方程，优化等功能。
  3. 熊猫实际上是可视化、读取和写入数据的最佳工具。我发现自己经常使用它，尤其是在使用。csv文件。
  4. Matplotlib 是用于创建2D图和图形的标准Python库。它使用起来非常灵活，但有点低级，所以绘制更复杂的图表或图形有点棘手。然而，它是我经常使用的库——尤其是在处理不需要可视化的数据集时。为了给我的模特打分。

机器学习库

机器学习处在人工智能和统计分析的交叉地带。以下库为Python提供了应用许多机器学习活动的能力，从运行基本回归到形成复杂的神经网络。

  1. scikit-learn 建立在NumPy和SciPy的基础上，增加了一套用于常见机器学习和数据挖掘任务的算法，包括聚类、回归和分类。它包含大量预先训练好的机器学习模型，数据科学家使用这些模型，而不是创建自己的模型。显然，这取决于你需要使用什么样的ML模型。如果你正在为你的意图寻找一些非常具体的东西，也许创建你自己的模型更好。
  2. Theano 使用NumPy的语法来优化和评估数学表达式。它使用GPU来加速其进程。Theano的速度使得它对于深度学习和其他计算复杂的任务尤其有价值。我发现与TensorFlow和Keras合作非常有用。
  3. TensorFlow 由谷歌开发，作为dist faith的开源继任者，dist faith是他们之前训练神经网络的框架。TensorFlow使用多层节点系统，允许您快速设置、训练和部署具有大型数据集的人工神经网络。很实用，很好用。它的创建者Google也使用它，并且有大量的文章和教程提到TensorFlow。
  4. pickle 是一个开源包，允许我们序列化我们的ML模型。我选择pickle而不是许多其他的模型序列化器，因为我发现它使用起来非常简单而且高效。这是共享您的模型或在另一个程序中使用它的最有效的方法之一。

数据挖掘和自然语言处理库

“数据挖掘是在大型数据集中发现模式的过程，涉及机器学习、统计学和数据库系统的交叉方法。数据挖掘是计算机科学和统计学的一个跨学科子领域，其总体目标是从数据集中提取信息(用智能方法)并将信息转换成可理解的结构以供进一步使用。”— 维基百科

自然语言处理(NLP)是语言学、计算机科学、信息工程和人工智能的一个分支，涉及计算机和人类(自然)语言之间的相互作用，特别是如何对计算机编程以处理和分析大量自然语言数据— 维基百科

  1. Scrapy 是一个快速的高级网页抓取和网页抓取框架，用于抓取网站并从其页面中提取结构化数据。它可以用于广泛的目的，从数据挖掘到监控和自动化测试。
  2. NLTK 是一套为自然语言处理设计的库。它经常用于文本分类和分析，从情感分析到聊天机器人。
  3. 模式是Python编程语言的web挖掘模块。它拥有用于数据挖掘的工具(谷歌、Twitter和维基百科API、网络爬虫、HTML DOM解析器)、自然语言处理(词性标签、n-gram搜索、情感分析、WordNet)、机器学习(向量空间模型、聚类、SVM)、网络分析和<画布>可视化。
  4. seaborn 是一个流行的可视化库，建立在Matplotlib的基础上。一旦你在使用Matplotlib之后尝试了一下，就会像在伊甸园一样。它非常复杂，与Matplotilib不同，它是一个高级包。这意味着我们可以轻松地绘制更复杂类型的图，如热图等。

奖金

Flask 是一个强大的基于Python的web开发框架。但是为什么它会出现在数据科学家需要了解的工具列表中呢？还有，Django不是更适合web开发吗？嗯，有时你可能需要将你的ML模型嵌入到一个web应用程序中，因为这意味着任何人都可以很容易地从互联网上访问你的分类模型。你甚至可以创建一个在线分类服务！要回答第二个问题，是的，Django实际上更适合web开发，它也很容易使用，但不像Flask那么简单。

总的来说，我肯定会使用Django来建立一个正常的网站。但是如果你只是想让你的模型嵌入一个网站，Flask其实更简单，更直观。

本文中列出的所有库都是可以在网上找到的开源包的一小部分。这些只是每个数据科学家都必须知道的基础数据科学和机器学习库。
