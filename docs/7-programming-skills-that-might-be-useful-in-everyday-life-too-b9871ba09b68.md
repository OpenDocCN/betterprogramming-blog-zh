## 程序员不仅在职场上有竞争力，在生活中也是如此

每个职业都有自己独特的优势。但是作为一名程序员，你可以做任何事情——从重烙铁和写脚本到修理笔记本电脑和在飞行员突然死亡的情况下让飞机着陆。我只需要随身携带一套螺丝刀和几本厚厚的手册。

开发人员首先是工程师，所以你不应该对我们被期望解决大多数技术问题感到惊讶。好消息是，从程序员这个职业中，你不仅可以挤出一个有趣的消遣或聪明同事的聚会，还可以挤出一些在现实生活中会派上用场的技能。

# 1.做出诊断

问题的迅速解决和病人的有效治疗直接取决于诊断的准确性。然而，发展的特殊性使得有必要一天多次做出这样的结论。最好是尽可能做到平衡和客观。否则，患者只会因为错误选择的药物而病情恶化。

程序员从事高精度诊断的频率甚至超过了其他一些职业的代表，这使得在日常生活中有效地使用这一技能成为可能。

# 2.控制情绪

只有冷血的交易者才有能力在股票市场上赚大钱。开发人员很清楚情感决定的价值:对于一个技术人员来说，这是一种负担不起的奢侈。

职业克制，转移到生活中，会让你用冷静的头脑解决日常问题，达到更好的效果，不会屈服于一时的情绪波动。

# 3.直接说

程序员的这种技能是双向的——对计算机和人。一方面，程序员的工作是合成可理解的有生命和无生命的实体——人和机器。但是，另一方面，这种用通俗易懂的逻辑语言解释一切的能力在生活中很有帮助——无论是在交流还是在通信中。

另一方面，开发者也必须理解人类和机器的语言。这种技能使他成为一个很好的倾听者和读者，他寻求深入对话和文本的本质，同时删除一切不必要的东西。

沟通技巧也包括提问的能力。适时问“谁？”或者“为什么？”在任何业务中都可以节省十几个工时。

# 4.能够辩论

程序员被他们发达的逻辑装置惯坏了，所以他们很乐意卷入各种纠纷。

当你需要捍卫你的观点和你在工作中解决问题的方式时，这项技能很重要。一个好的程序员知道如何在与同事和高层老板的谈话中找到正确的论点，而不用担心因傲慢而被打脸。如果他不这样做，那么很有可能，他将不得不清理开发中的不一致，这些不一致在任务的初始阶段和过期的最后期限时没有被识别出来。

虽然在“平民生活”中，辩论的艺术有助于解决日常问题，如与 Shvonders 和税务当局的争吵，但科学家还没有发明出一种与对话者辩论的方法，对话者使用“哦，好吧，一切！”作为论据。

# **5。集中注意力**

如果不聚在一起仔细考虑每一件事情，一个开发人员将很难解决哪怕是一个普通的任务。因此，为了以某种方式将自己与办公室的刺激物隔离开来，他们使用带音乐的耳机，一张严厉的扑克脸警告潜在的超时空噬菌体:“不要来，杀人！”，甚至一支笔在一张纸上摸索着画重要的流程图。

好消息是:脱离外部刺激并专注于主要问题的技能不仅在工作中有帮助，在日常生活中也有帮助，不管你做什么——甚至是拳击，甚至是足球。

# 6.先思考，再做

我想你们中的许多人在生活中的某个时候都尝试过牛仔编码——当你像一个键盘牛仔一样编码时，没有花很多时间去考虑架构和可能的长期后果，随着游戏一起铆钉原型，在 X 分钟/小时内发现它必须以不同的方式完成。

这一点可以与关于沉着的第三点联系起来，因为它恰恰允许你不屈服于情绪，不急于归档。即使在紧急恐慌、焚烧合同、撒谎的服务器和即将停电的情况下，程序员也会从桌子上拿起一杯茶，看着窗外，带着睿智的哲学面孔喝一两口，坐下来，叹口气，呼气，只有到那时，事情才会开始解决。在日常生活中，这种方法非常有效。

任何问题的测试版解决方案:不要抽搐，去睡觉，早上解决方案就好了。类似这样的。

# 7.说实话

如果你没有时间完成任务，你告诉项目经理，现在这是他们的问题。如果你不知道如何做某事，你告诉你的朋友——然后你们一起想出一个主意。但是软件开发不能原谅的是保留和半真半假，就像建造桥梁一样。

迟早，所有的秘密堵塞和缺陷会变得明显，桥梁将在服务器之后倒塌。因此，你的团队和上级应该提前通知你所有的事情。是的，说实话，就好像你被注射了一种真实的奶酪制品，扮演了《怪物史莱克》中的一只猫。

例如:“明年，我计划去纳米比亚，那里有很高的感染疟疾和把鱼鳍粘在一起的风险，所以对于每个消防员，计划没有我的第八次迭代。”

一旦这种说真话的技巧从办公室转移到日常生活中，生活就开始变得多姿多彩。没必要欺骗任何人，没必要装成任何人，没必要被不好笑的笑话逗笑，每个人都会喜欢。生活变得轻松了一百万倍。

# 最后的想法

在列出的要点中没有“学习”,因为在我看来，我们获得学习技能的时间远远早于我们开始编程的时间——还在学校。

在我看来，程序员的学习方法似乎有些情境化和非系统化。此外，由于缺乏在现实生活中使用的书面证据，我没有提到阅读文档的技巧。也许这是要补充的？
