1.变量是类型化的吗？

如果您来自不同的编程语言(例如Swift)，您可能知道变量在声明时有特定的类型。例如，如下所示，如果您在Swift中从一个整数变量开始，当您试图为该变量设置一个字符串值时，您将会遇到一个错误。

var number = 5
number = "five"error: cannot assign value of type 'String' to type 'Int'

原因是这些语言是强类型的，这意味着每个变量在其生命周期中都有特定的类型。只有当新值具有兼容的数据类型时，才能更改变量值。然而，Python却不是这样。下面的代码向您展示了Python中运行起来没有任何问题的类似代码。

>>> number = 5
>>> number = "five"

这一特性(即改变变量的类型)被称为动态类型化，这意味着给变量赋予任何数据值都是合法的，即使它与开始时的类型不同。上面的代码可以通过一个额外的检查来扩展。

所以这个问题的答案是肯定的，也是否定的。 Python变量有类型，这是由在检查它们的类型时它们绑定到的值决定的。然而，与强类型语言不同，Python允许用其他类型的数据设置现有变量。

2.变量和对象是一样的吗？

我们可以显式地检查在上面的代码片段中创建的四个变量的数据类型。我们可能都知道，Python中的一切都是对象，包括这些内置的数据类型实例、函数、类和模块。通过创建这四个变量，我们在内存中为这些数据类型创建了四个相应的实例。但这就意味着变量和对象是一样的吗？

>>> type(number)
<class 'int'>
>>> type(text)
<class 'str'>
>>> type(api_response)
<class 'tuple'>
>>> type(numbers)
<class 'list'>

简单的回答是否定的。知道区别的最好方法是更好地理解它们的概念和密切相关的知识，如下所述。

  • 变量可以被视为与定义的特定对象相关联的标签。我们可以使用多个标签来引用同一个对象，这被称为别名，但是一个标签只能引用一个对象。
  • 对象是创建时保存在内存中的数据。另外，当我们创建对象时，它们在内存中都有特定的类型(如tuple、int)。有时我们可以把对象看作变量的值。

  • 后来，这个变量被更新为一个字符串值，这导致变量a具有不同的内存地址。

3.元组是绝对不可变的吗？

我们常说有些Python数据类型是不可变的，有些其他数据类型是可变的。例如，字符串、数字和元组是不可变的常见Python数据类型。在讨论元组的不变性之前，我们先简单回顾一下这个概念。

在上一节中，我们已经了解了变量和对象之间的区别。与这一区别相关的是，数据可变性指的是内存中的底层对象。不可变对象(例如字符串)是那些在内存中创建后就不能更改的对象。可变对象(例如列表)是那些在内存中创建后可以改变的对象。

  • 在上面的代码中，我们能够改变一个元组对象的值。元组对象不是不可变的吗？怎么可能改变它的值呢？
  • 原因是元组对象是数据容器，这意味着它们通过使用对内存的特定引用来保存其他对象。同样，理解变量和对象之间的区别对于理解这个概念也是必要的。
  • 创建的元组包含两个对象——一个整数对象和一个字典对象。当我们更新字典的数据时，对象本身仍然是元组保存的同一个对象。因此，我们并没有真正改变tuple对象，它仍然拥有同样的两个对象。

因此，对于这一节的问题，我们大概应该说元组是相对不可变的。如果它们拥有可变的对象(例如，字典和列表)，我们可以改变元组的值。然而，应该注意的是，我们永远不能改变元组的底层对象，因为元组对象在Python中是不可变的——它们总是包含相同的对象，不管是否有可能改变所包含的数据。
