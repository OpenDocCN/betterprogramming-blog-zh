## 类型别名和接口都很相似，那我该用哪个呢？

类型别名和接口非常相似，在许多情况下都可以自由使用，但它们仍然有一定的差异和限制。接下来我会分不同情况来解释。

# 原始类型

可以看到类型别名可以定义基本类型(符号、布尔、字符串、数字、bigint 等。)，而接口不能。

注意，它被称为类型别名，因为它不创建新类型，但是接口总是创建新类型。

# 工会类型

联合类型也只能使用类型别名来定义。

# 元组类型

元组类型也只能使用类型别名来定义。

# 对象/函数类型

接口和类型都可以声明对象类型或函数类型。但是接口可以多次声明同一个接口，它们会自动合并，而类型别名不会合并，并且必须唯一。

另一个要点是，类型别名使用交集，接口使用继承。

在某些情况下，接口的自动合并和可扩展性会很有用。想象一下，如果你做了一个第三方库，公开了公共 API，用户可以通过接口机制来扩展它，那就太棒了！

所以这里的建议是，如果要声明一个对象类型，请先使用接口，需要的时候使用类型别名。

# 映射的对象类型

您可以看到映射的对象类型只能使用类型别名来定义，包括关键字`in`和关键字`keyof`。

# 未知类型

当我们处理未知类型时，我们可以使用`typeof`进行类型捕获，但这样我们只能使用类型别名而不能使用接口。

综上所述，类型别名几乎涵盖了接口的所有特性，但是接口总是可扩展的，而类型别名不是。所以你可能需要根据具体情况来选择它们。
