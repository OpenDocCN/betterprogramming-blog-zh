如何为你的亚马逊网络服务找到合适的区域

Amazon Web Services是一项全球性服务，覆盖25个地理区域，有80多个可用区域。计划在不久的将来再推出五个地区和超过15个新的可用性区域。

根据您希望启动的服务，您可能希望或者必须选择一个特定的AWS区域。例如，如果您有义务将数据保存在特定的国家或洲，您可能会希望选择适当的AWS区域。此外，您可能希望启动分布在区域内多个可用性区域的高可用性服务。

在今天的文章中，我们将讨论什么是AWS区域和可用性区域。此外，我们还将介绍您在选择地区时必须考虑的五个不同因素。

AWS中的区域和可用性区域是什么？

AWS区域是分布在世界各地的数据中心集群，例如北美、南美、欧洲、中东、非洲和亚太地区。每个地区都有其唯一的标识符。例如，俄亥俄州的美国东部数据中心被命名为us-east-2。AWS上提供的大多数服务都是区域性的。这意味着，如果你在两个不同的地区使用相同的服务，你将被收取两次费用。

每个区域都是完全隔离的，由多个可用区域组成。每个可用性区域至少是一个独立的数据中心，在AWS区域有一个冗余电源和连接。像区域一样，可用性区域也是相互隔离的。

可用性区域使我们能够在生产环境中启动高可用性和容错应用程序。跨多个可用性区域分布节点可以帮助您应对断电或地震等灾难。

选择您的地区和AZ时需要考虑的5个因素

如前所述，AWS中的大多数服务都是区域性的。这意味着在推出新服务时，您必须选择一个地区。

根据项目的要求，您可能希望在特定区域运行服务。下面，我们将讨论在为新服务选择AWS区域和可用性区时必须考虑的五个重要因素。

1.可用服务

并非所有地区都支持AWS上提供的新服务。通常，AWS需要一段时间才能在所有地区推出新功能。因此，有时您必须使用特定的区域，因为它们是唯一支持您希望启动的服务的区域。

2.地理位置

如果您正在启动一个面向特定地理位置的人的应用程序，选择一个远离它的地区没有太大意义，因为这将增加延迟。因此，您应该选择一个靠近绝大多数用户所在地的地区。显然，对于大规模的应用程序，您需要在多个地区启动您的服务，以保持较低的延迟。

3.合规性和法规

通常，公司必须满足某些法律要求。例如，您可能希望将数据保存在部署应用程序的国家。有时，这是选择AWS区域时考虑的唯一因素。如果根据法律，您有义务将您的数据或应用程序托管在某个特定的国家或大陆，那么没有任何其他因素可以超过这个因素。

4.可用性和容错性

除了选择AWS区域之外，您可能还必须考虑选择特定的可用性区域来托管您的应用程序。AZs允许您运行高度可用和可伸缩的服务。

例如，让我们假设我们想要推出一个Kafka服务。如果代理的Kafka集群分布在我们选择的AWS区域的可用性区域中，那么服务将能够经受住诸如停电和自然灾害之类的灾难。

5.定价

最后，同一项服务在不同地区可能会有不同的定价。有时候选择最便宜的地区是有意义的——特别是如果它离你喜欢的更贵的地区足够近的话。

您可以在服务的定价范围内找到每个地区的价格。

结论

在今天的文章中，我们讨论了在选择Amazon Web Services的区域时必须考虑的五个重要因素。此外，我们讨论了可用性区域的重要性，以及它们如何帮助我们在生产环境中启动高可用性、可伸缩性和容错的应用程序。

选择正确的AWS区域非常重要，因为它有助于您降低成本，遵守特定的法规，并减少用户使用您的服务或应用程序时的延迟。

最后，请务必访问官方的 AWS全球基础设施地图，了解AWS地区在全球的分布情况。
