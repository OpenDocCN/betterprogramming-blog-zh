# 获得第一份(或下一份)软件开发工作可以做的 3 件事

## 让自己脱颖而出

![](img/ef385674c031837470d80e146ed409ca.png)

戴维·伊斯坎德尔在 [Unsplash](https://unsplash.com/t/business-work?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

获得软件开发的第一份(或下一份)工作可能会很艰难。即使是初级职位，要求通常也包括 *x* 年的专业经验，如果你刚刚毕业或即将毕业，这可能是个大问题。

幸运的是，有一种方法可以让你向大公司和小公司的潜在雇主展示，尽管你在软件工程领域的活跃时间不长，但你可以成为一名有价值的团队成员。

通过做下面的任何一项或全部，你会大大增加被雇佣的机会。

# 1.擅长不止一件事

意思是你应该*主动* 炫耀你所有的技能。对于许多雇主来说，仅仅拥有一个学位、理论知识或一篇发表的作品是不够的。

通过展示你精通多个领域，你向潜在雇主强调你比简历上显示的更灵活、更好奇、更有经验，并且你可以将学习从软件开发的一个领域转移到另一个领域。

如果你在考虑一个项目时有困难，不要强调它——这些展示你技能的展示不需要太多，只需要让招聘人员看到就行了。

想想如何把自己的兴趣爱好转化为 dev 活动，以显示自己对事物的热情。

这里有几个例子，你可以挑选出来放在简历上:

*   参与[游戏卡顿](https://ldjam.com/)。(自己或[与他人合作](https://crowdforge.io/jams/ludumdare#!)。)
*   做一个小的安卓或者 iOS 的 app。(比你想象的要便宜容易。)
*   使用最新游戏的公共 API 为你最喜欢的浏览器构建一个插件。
*   在 GitHub 上发布源代码和作业(并确保每个 repo 的自述文件都是最新的)。

你选择从事的副业类型取决于你的偏好和职业目标，所以花点时间想想你想成为哪种类型的开发人员，以及你想在简历上展示的技能类型。

# 2.擅长交际

你认识的人越多(也有越多的人认识你)，你就越容易听到机会，分享消息，并表明你是一名软件开发人员。

被推荐一份工作，仅仅通过认识某人就能让你在竞争中领先。雇主在考虑雇用你时会找你，所以你要控制自己的信息流动:

*   确保你的 LinkedIn 个人资料看起来很专业，是最新的，并且列出了你的兼职项目。
*   作为一名开发人员，创建一个开发博客，写一些科技文章，或者开始投资一个社交媒体。
*   去当地的聚会或大学俱乐部，认识一些人，如果你有信心的话，做一些演讲。(俗话说，最好的学习方法是教。)

# 3.准备狩猎

这是一个显而易见但极其重要的观点。一个人不能简单地在面试中胜出——要成功需要大量的准备。

要留下良好的第一(第二和第三)印象，正确回答技术问题至关重要，但这还不够:

## 给你的简历增添情趣

给你的简历增添情趣！当然，不要包括明显的谎言，但是要包括你所有的成就。

展示你的成就的最好方式是通过关键词(对应于职位招聘要求的资格)和数字。简历的艺术是用可量化的方式简洁地写下你所做过的所有相关工作的简短段落。

此外，如果你以前的雇主是小公司或者不太出名，用一两句话描述一下这些公司做了什么。你想让招聘人员尽可能容易地阅读你的简历，知道你是否适合这个职位。

## 补上基础知识

补上基础知识。你不可能涵盖面试中可能被问到的所有问题，但这并不意味着你不应该尝试。你会被问到技术性和非技术性的问题，所以准备好简洁地回答常见的问题。理想情况下，预先编写解决方案。

解释你在做什么，你的决策过程是什么，而编码是不自然的，但很多面试官要求你这样做。

为了以防万一，您可能还想在纯文本文档中练习编码。有很多很多资源列出了所有技能水平的所有发展领域的可能面试问题，所以去找到它们并使用它们。

## 有疑问

有自己的疑问。如果你在申请一家初创公司，询问他们是如何融资的几乎总是一个好主意。

检查是否有成长的机会，以及团队是如何组织的。诸如此类的问题表明你考虑周到，并且很有可能突出任何危险信号。

## 研究公司

这对大公司来说可能更重要，但所有招聘人员都想知道你是否只是为了这份工作而来(这也是一个完全有效的观点，尽管这不会给你带来额外的印象分)，或者你是否对该主题有更深层次的投入。

## 练习软技能

你要展示你能融入这个团队，让人们喜欢你。风度翩翩，但要专业。炫耀你的技能，但要谦虚。要有礼貌，但要问一些深入表面的问题。

使用适当的技术术语，但只谈论你真正理解的东西。当有疑问时，穿得过多>过少。

想想坐在桌子对面的人面试你是什么感觉，他们在公司的职位是什么，他们想从你这里听到什么，然后采取相应的行动。

如果你有问题，在适当的时候提出来(宜早不宜迟)。

## 保持最新状态

跟上潮流、发展和创新。软件开发的每个领域和相关领域都有很多很好的新闻来源。一定要经常阅读它们。

另一个很好的技术见解来源是流行的播客——非常适合在早上通勤时听。保持对最新消息的了解会给你更多的视角，并使你能够评论你不一定熟悉的事情。

这反过来会让你看起来更有经验，让你能够进行高水平的比较，如果你没有招聘人员希望你拥有的那么多经验，这可能会让他们对你产生好感。

这也表明你致力于实现目标，并且总是尽力而为。

## 保持决心

保持坚定！你可以做好每一件事，但你的大部分申请仍然会被拒绝或无人回应。事情就是这样。

坚持下去，最终会有办法的。面试本身就是一项需要练习的技能，所以如果事情没有马上解决，不要气馁。

但是，如果你发出了大量的申请，一定要为每一份申请量身定制简历。
