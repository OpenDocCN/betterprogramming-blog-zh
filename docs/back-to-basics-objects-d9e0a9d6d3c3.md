# 物体的兴起,(简要)历史

在计算时代，编程是绝对必要的。没有 ide，没有 GUI，什么都没有。在编程的初期，任务更容易概念化。程序由一系列发送给计算机执行的特定指令组成。然而，随着计算需求的增长，研究人员试图创造更好的工具。

Simula 67 被认为是第一种面向对象语言，于 1962 年首次出现，用于构建离散事件模拟。Simula 的关键特征是“可停止程序”的概念。这些可停止的过程有属性，许多内部过程，后来被称为类。此外，他们将类的实现称为**对象**。我们今天仍然使用这个术语。

那些密切关注 Simula 的人喜欢他们所看到的，并在 1972 年使用其概念创建了一种语言，旨在构建面向图形的应用程序 Smalltalk。通过使用对象建模，Smalltalk 被认为是“人机共生”的“新世界”的基础。这是物体真正开始起飞的时候。

# 什么是对象？

在其核心，一个对象是一个概念性的代码片段，旨在模拟一个现实世界的实体。随着业务需求变得越来越苛刻，程序员需要一种方法来对真实世界的数据和交互进行建模，而对象完美地填补了这一空白。

我最喜欢的例子是一辆经典的汽车。当将车辆建模为数字对象时，我们需要问自己:“是什么让车辆…成为车辆？”第一遍可能是这样的:

*   品牌、型号、颜色、年份、类型、车牌

好的，我们有一辆“红色 1998 丰田凯美瑞轿车”，太棒了！我们可以为具有这些属性的车辆类定义一个**接口**，以及设置和检索每个字段的方法。最后，我们需要一种方法来创建我们的车辆，这就是所谓的“构造器”:

`Make`、`model`、`color`、`year`和`type`被称为**实例变量**，因为它们属于车辆对象的任何给定实例。方法`setMake`和`getMake`被称为“getters 和 setters ”,因为出于安全原因，实例变量最好是私有的，否则任何进程都可能直接改变我们的对象，而不是通过我们提供的接口。

这个车辆对象本身很棒，但是它只是真实世界模型的一小部分。车辆有各种形状和大小——轿车、SUV、跨界车、摩托车、卡车、小型货车和拖拉机拖车。随着车辆类型数量的增加，我们模型的需求也随之改变。我们可以为每个支持类型创建一个 Vehicle 的子类**,而不是创建一个支持许多不同类型的冗长、讨厌的 Vehicle 对象。这个概念是面向对象编程的核心，被称为继承。**

## 遗产

在 OO 程序中，对象可以分层配置，从一组**超类**中继承。在我们的示例中，所有列出的汽车都可以从一个公共的 Vehicle 类继承，自动从 Vehicle 获取字段和方法，并在需要时定义新的方法或字段。例如，摩托车可以有两个或三个轮子，而其他车辆有四个轮子。

继承是对象真正力量背后的驱动力之一，另一个是它们的执行模型。对象可以作为由其他对象组成的更大的生态系统的一部分而存在。在对象之前，在任何给定时间只能执行一个命令。对象出现后，许多对象可以同时执行命令，将控制委托给操作系统来管理任务的执行。Smalltalk 和 Java 以其“一切都是对象”范式而闻名，允许分布式责任的高级模型——这是我们现代的一个普遍需求。

请注意，轿车被声明为车辆

这个简单的例子介绍了对象和继承的强大功能。作为开发人员，当我们对在什么情况下可以使用什么类型的对象施加限制时，这种力量会变得更加强大。轿车是一种交通工具，但并不是所有的交通工具都是轿车。

如果我们要求一个对象的类型是 Vehicle，这就允许开发人员或用户使用我们创建的任何车辆、轿车、摩托车或卡车类。这种灵活性和可扩展性造就了今天的现代应用程序。
