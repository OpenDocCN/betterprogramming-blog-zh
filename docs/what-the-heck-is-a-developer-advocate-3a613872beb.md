# 谁是开发者拥护者？

开发人员倡导者是一个纯粹的技术人员，他有编码经验，或者对整个软件开发生命周期和范例有足够的了解。一个开发人员倡导者为软件开发人员和产品团队的利益而工作，这样两者就能同步地为组织的成长而工作。开发倡导者的角色是一个非常有创造性的领域，因为一个人必须戴上演说家和沟通者的多重帽子，并且他要通过产品和开发团队的思维方式进行思考。

显然，这个领域有它的优点和缺点，这使得它与特定的人群相关。我们将讨论谁是这样一个角色的合适人选，以及如何在这个领域取得进展。

如果还是很困惑，不用担心。让我们用简单明了的英语来讨论开发人员倡议。开发者倡导者是一个角色，在这个角色中，一个人必须与社区联系，并帮助他们理解该人所倡导的公司的相应产品。通过这样做，他们帮助人们了解他们的产品相对于类似产品的缺点和亮点。接下来，他们可以向产品和软件开发团队提出社区报告的新特性或问题。

现在让我们深入讨论一下开发人员倡导者的角色和职责。

# 开发者倡导者的角色和职责

如果我们谈论你作为一个开发者倡导者的角色和责任，那么我们必须说他们围绕着与社区的交流，将一个组织的产品公之于众。像其他角色一样，即使是开发人员也主张职责根据组织而变化。但是，我们还是会试着把它们分成这三大类。

## 内容

开发者倡导者的主要角色是通过发布能够帮助开发者使用他们的产品、API 等的内容来帮助开发者社区。以公司所能提供的最好的方式。开发者倡导者在制作新内容方面不受任何限制。内容可以是 YouTube 视频、书面博客、播客——任何能把事情做好的形式。

他们中的许多人甚至沉迷于为各自组织的产品创建一个开源项目，在那里他们可以得到业内其他开发者的反馈。开源控制源(如 GitHub 和 Bitbucket)中的 issue 部分展示了产品的受欢迎程度，也可以通过向内部开发团队提供反馈来改进产品。

## 社区

社区是开发者倡导者每天都要迎合的东西。开发者倡导者主持现场会议，在那里他们可以与其他开发者交流。议程主要是让开发者社区了解你的产品。这为组织提供了免费的宣传，这反过来又有助于组织在他们的产品公开时扩大规模。

他们可以多次协调平台，如 Slack 和 Discord 频道，以便通过加入志同道合的个人来建立网络。一个开发人员倡导者拥有丰富的技术经验，因此回答关于他正在开发的产品的堆栈溢出问题也成为工作的一部分。

## 密码

这是有争议的，基于不同的组织要求。他们不直接参与产品的开发，但是使用开源产品并回答堆栈溢出问题，除非需要编码。当来自 Google 和 IBM 的倡导者被询问时，有一个编码的部分需求。但是不能一概而论，因为一个人可以在多个领域成为开发者拥护者。

# 知道成为一名开发者拥护者是否适合你

让我们来讨论一个人应该具备的一些社会和技术特征，以便更好地适应开发人员的角色。同时，在这个过程中，你也可以发现这个域名是否符合你的要求。

## 旅行

作为开发者代言人，你需要经常出差，因为你必须参加在全球不同地方举行的许多会议。很多时候，一个人甚至不得不组织一些会议。一个开发者拥护者会有很多机会去探索新的地方，但这是以牺牲一些周末去旅行为代价的。如果一个人可以经常出差，可以牺牲几个周末，那么这个角色就是一个完美的组合，我们可以说，是天造地设的一对。但是很明显，工作旅行有时完全不同于休闲旅行，因为一个人在内心深处认为他们是在工作。但这完全取决于个人，他们处理这种情况的方式。

## 机会

开发者倡导者有很多机会了解新技术的发展趋势，因为他们必须在互联网上提供内容。他们遇到了许多不同领域的行业专家，这些专家可以帮助他们建立一个强大的志同道合的人的网络，他们可以在需要的时候提供帮助。

## 个人品牌

作为开发者的拥护者，你必须在互联网上发布大量的内容。这可以是你的博客帖子，YouTube 频道，播客等。，反过来也有助于创建个人品牌。一个开发者倡导者有机会在很多会议上发言，所以从参加会议走向舞台。观众开始认出他们。但是很多开发者不喜欢探索这么多，也不喜欢分享知识。如果一个人喜欢建立个人品牌并与普通观众分享知识，那么这个领域对他们来说是完美的。

## 社会人格

不言而喻，一个人需要社交才能在开发者倡导者的位置上表现得更好。但是一个开发者拥护者不应该只有外向的个性。内向的人也能表现得更好，因为对许多人来说，他们在工作中的行为会发生变化。一个人应该活跃在不同的社交平台上，在那里他们可以增加人脉。因此，想要成为观众并与其他开发人员分享他们的知识的人可以去担任开发人员倡导者的角色。

最后，我要说的是，开发者拥护者不仅仅是一个开发者，更是一个社会人。另外，拥有编码开发的知识是成为一个好的开发者拥护者的锦上添花。议程应该是帮助其他开发者社区，他们可能会使用你所宣传的产品。此外，将来自社区的反馈带回给你的内部团队，以便可以增强产品并添加新功能，尽管你没有编写任何新功能的特权，而只是给出关于可以实现的新功能的建议。

所以这是一个开发者倡导者角色的样子。我希望你对一个开发者倡导者的角色和责任有所了解，并且你可以决定这个领域是否适合你。
