# 3 有害的开发团队行为

> 原文：<https://betterprogramming.pub/3-toxic-dev-team-behaviors-5e039695860d>

## **以及如何解决它们**

在我作为开发人员的职业生涯中，我和团队中许多不同的人一起工作过，团队规模从 2 人到 10 人不等。

这些年来，由于某些行为导致了我们工作环境中的有毒物质，我学到了很多东西。

在这篇文章中，我想分享最糟糕的行为——那些会彻底破坏团队工作环境的行为。

# **1。做出假设**

这适用于在许多领域工作的团队，而不仅仅是开发。

## **问题**

大家都知道，沟通是团队成败的最大因素之一。如果你想实现大目标，你必须一起工作，并有明确的沟通准则。

当一些指导方针被忽略或者不清晰时，团队就会受到影响。

如果团队中有人从一个假设开始，这可能会导致不可预测的结果、讨论甚至应用程序中的错误。

## **解决方案**

当团队成员或经理让你做某件事时，你需要确保你正确理解了他们。你可以这样做，告诉你的团队成员，“那么，当你说这个的时候，你是指 x…或者在不明白需要做什么的时候简单提问。

我绝不是一个完美的团队成员，但当我问那些额外的问题或用我自己的话重复别人告诉我的话时，我看到了积极的结果。

我希望这对你的团队也有积极的影响。

# **2。缺乏沟通**

作为开发人员，我们经常被视为独自坐在黑暗中，整夜编码的人。对我们大多数人来说，这并不完全正确——但也有一定的道理。

## **问题**

我们知道当开发一个应用程序(或者它的一部分)时，有一个清晰和连续的交流流是很重要的——没有这个，错误、错误和无结果的讨论是常见的。

我经历过很多次了。通常，不是故意的。我们大多数人都陷入沉思，为我们需要解决的问题寻找解决方案。

忘记沟通在我们的工作中很常见。但是，由于团队中的大多数问题都是由于缺乏沟通造成的，我们需要解决这个问题，并提出一个系统。

## **解决方案**

当你遇到这样的问题时，是时候集思广益了。我们需要定义我们何时交流，交流什么，以及如何相互交流。

把这些事情写下来很重要。尤其是对开发人员来说，因为我们在工作中大量使用大脑。系统不需要很难！一个简单的解决沟通问题的系统就足够好了。

我的解决方案可能受到敏捷框架 Scrum 的高度启发。但是在我去年的发展中，我发现它们非常有效。

## **每日站立会议**

我发现站立会议对于有计划的交流很有用——当然，如果团队中的每个人都参加的话。如果你的团队是远程的，使用 slack 和 Geekbot 是有用的。

我和我的团队每天早上 11:00 通过视频通话站起来，描述我们昨天做了什么，今天计划做什么，以及我们是否有事情阻碍我们的工作。这样便于彼此帮助和沟通。

## **使用 scrum 或看板的形式**

许多团队使用敏捷宣言并使用一个框架，比如 Scrum 或看板，来实现。这些框架的便利之处在于，它们已经定义了一些您可以应用的东西。

两个框架都使用了一块板子，你可以看到团队中的每个人在哪里工作。如果你的球队没有预算使用像吉拉这样的球员，你可以很容易地使用特雷罗。或者，如果您使用 Github 编写代码，请使用这些板。

在这里你可以分配工作给人们，把它放在一个状态栏里，看什么时候事情完成了。很简单，但是对沟通真的很有帮助！

## **回顾会**

每隔几周(你和你的团队应该决定多久一次)讨论一下事情的进展是很好的。

我喜欢每个人都能描述这几周的三次胜利和三次进步。我喜欢听到队友的胜利和进步。

如果是胜利，花和进步一样多的时间来谈论它。庆祝我们的胜利和从错误中学习一样重要。我相信当你把错误变成教训时，没有不犯错误的胜利。

所有这些解决方案都有助于团队的沟通。如果你的团队中有什么事情不顺利，改进并调整它，直到每个人都满意为止。这就是我喜欢敏捷思维的原因！

# **3。不承担责任**

老实说，在团队中工作既有趣又有挑战性。

## **问题**

当引入错误和 bug，甚至更糟时，服务器会因为错误而停机，人们的真实本性就会浮出水面。通常合作愉快的人开始互相指责。这对团队氛围不好！

问题是大多数人害怕承认自己犯了错误。也许他们害怕被解雇，或者担心会失去团队中的一些尊重。人们可以有很多理由去责怪别人。但那对谁都没有帮助！

## **解决方案**

这完全是由于缺乏责任感。如果没有人承担责任，团队中的氛围会越来越差。

每个人都应该坦然承认自己犯了错误。之后，与其说是团队中一个人的问题。如果你是一个团队，一个失败，所有人都失败。如果一方赢了，所有人都赢了。

我相信承担集体责任——一起解决彼此的问题！

我相信一起跑马拉松*而不是每个人都为自己。因为，如果我们诚实，我们需要彼此。*

# ***结论***

*我希望如果这些行为发生在你的团队中，没有人开始指责别人。但是我真的希望有人能承认*我们*作为一个团队有问题，并且你们能一起找出解决方案！*

*如果你在你的团队中发现了这些事情，并找到了其他解决方案，请在评论中告诉我们，这样我们可以相互学习，相互启发。*
