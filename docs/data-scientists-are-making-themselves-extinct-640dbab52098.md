多年来，数据科学一直是数据领域的热门话题，但从 2019 年开始，[数据科学家的工资同比下降了 1.2%](https://insights.dice.com/2020/05/26/data-scientist-salary-starting-average-which-states-pay-most/) ，主要原因是市场过饱和。

它变得太受欢迎了。现在工具和技术已经足够抽象，非数据科学家可以在没有大量数据科学背景的情况下完成大部分工作。

相反，[“数据工程师”仍然是最热门的技术职位，在过去的 12 个月里增加了 88.3%](https://insights.dice.com/2019/06/04/data-engineer-remains-top-demand-job/)(再次在 2019 年报告)。这种增长是因为企业已经认识到，数据科学家的产出质量取决于他们使用的数据质量。

# 为什么数据工程师越来越多？

数据工程正在崛起对抗数据科学，就像软件工程已经超越计算机科学一样(这种趋势更难看到——大多数软件工程师确实学习计算机科学，但后来成为软件工程师，很少使用正确的计算机科学)。

随着我们将计算机的复杂性从二进制和逻辑门抽象到图形操作系统和现代编程语言，软件工程变得比计算机科学更受欢迎。这是因为这种抽象要求人们能够将不同的构建模块组合在一起创建一个应用程序，而不是从头开始对所有的构建模块进行编程。

企业希望快速构建软件，因此重用预先存在的库和包并将它们放在一起比自己花几周时间构建它们更可取。

软件工程师很少重新发明轮子。他们是建筑工人，使用所有可用的工具。

同样的模式正在数据科学和数据工程领域出现。一开始，你需要一些专家，他们懂得如何编写构建推荐引擎和神经网络所需的复杂代码和数学模型。

许多这些东西现在已经被抽象到这样一个程度，你可以用你最喜欢的编程语言导入一个库(或者甚至使用一个云服务)，放入一些数据，并得到答案，而不必理解中间发生了什么。你所需要的是一个能正确格式化和清理数据的人，以便产生最好的输出。

这就是为什么对数据工程师的需求越来越大。

像他们的软件同行一样，人们需要能够获得预先构建的数据科学构建块，并以产生类似结果但更短时间的方式将它们拼凑在一起，而不是从头开始构建每个块。

数据科学的难点不再是数据科学，而是*数据工程*。获取、处理和存储用于构建这些模型的数据，然后将它们投入生产。

# 数据科学真的是一个垂死的领域吗？

当然不是！这只被高估的股票已经被市场修正了。

这是一个广为流传的流行语，声称可以用机器学习和人工智能解决所有的商业挑战。然而，在大多数情况下，许多组织都没有数据基础设施。这减缓了数据科学家的速度，并阻止他们实现“全是歌唱和舞蹈的未来”

现在大家都知道，数据科学本身并不是灵丹妙药。你需要数据工程师和数据科学家的良好组合来产生最好的结果。但是，在大多数情况下，如果作为一名数据科学家，你的日常工作看起来和感觉起来像是数据工程工作，不要感到惊讶。

然而，仍然有一些领域和领域需要铁杆数据科学家。像真正的计算机科学家一样，他们将推动数据科学的创新，并在特斯拉等公司工作，帮助为自动驾驶汽车技术铺平道路。

### 这是他们将要做的

总会有企业追求非常高的模型准确性，这是数据科学家最有价值的地方。专家们可以挖掘杂草，从模型中挤出额外的 0.01%。

然而，大多数公司只是想要一个推荐引擎，也许是一些预测用户流失的东西；这两者都可以使用云提供商工具来完成，如[亚马逊个性化](https://aws.amazon.com/personalize/)或者将一些库导入到 R 或 Python 应用程序中。

幸运的是，其他数据科学家已经在这些琐碎的问题上做了艰苦的工作，并将它们打包成易于使用的模块，任何工程师都可以从中受益。

*这并不是说数据科学要灭绝了，它比以往任何时候都更容易获取和传播。*

由于流行技术的抽象，数据科学曾经需要的专业技能正在减少。

能够通过时间考验的数据科学家是该领域真正的专家。那些能够挤出额外的性能或发表下一篇大论文来更快或更简单地解决以前的难题的人。

随着就业市场的调整，我们将开始看到数据科学和数据工程职位的招聘变得更加平衡。

如果你目前担任其中一个角色，我建议你多了解另一个角色。在我看来，这两方面的知识会让你在任何申请中脱颖而出。
