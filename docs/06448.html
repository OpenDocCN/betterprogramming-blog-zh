使用云解析合同、文章和其他文档

解析和处理文档可以为公司的几乎每个部门提供大量价值。这是自然语言处理(NLP)可以派上用场的众多用例之一。

NLP不仅仅是聊天机器人和Gmail预测你要在邮件中写什么。NLP还可以用来帮助自动分解、分类和分析文档。例如，也许你的公司正在通过你所有的合同寻找关系，或者你正试图对博客文章或电影剧本进行分类。

这就是使用某种形式的NLP处理非常方便的地方。它可以帮助分解文档中的主题、重复主题、代词等等。

现在的问题是，你怎么做？

你应该从头开始开发一个定制的神经网络来分解句子、单词、意思、情感等吗？

这可能不是最好的解决方案——至少对你最初的MVP来说不是。相反，有很多库和云服务可以用来帮助分解文档。

在本文中，我们将探讨三个选项，以及如何使用Python实现这些工具来分析文档。我们将研究AWS理解、GCP自然语言和TextBlob。

AWS理解

AWS understand是AWS提供的众多云服务之一，它允许您的团队利用神经网络和其他模型，而无需构建自己的复杂模型。

在这种情况下，AWS understand是一个NLP API，可以使处理文本变得非常容易。

AWS understand的伟大之处在于它会自动分解文档中涉及的概念，如实体、短语和语法。如果您试图分解文档中引用的事件、组织、人员或产品，实体尤其有用。

有很多Python库可以很容易地分解名词、动词和其他词类。然而，这些库并不是为了准确标注这些名词的类别而构建的。

TextBlob和Python

除了使用云服务提供商，还有一些库也可以从文档中提取信息。特别是Python中的 TextBlob库非常有用。就我个人而言，这是我学习开发NLP管道的第一个库。

它远非完美。然而，它在解析文档方面做得很好。

它提供词类解析，如AWS理解和GCP自然语言以及情感分析。然而，就其本身而言，它不会对存在的实体进行分类。

它仍然是分解基本单词类型的一个很好的工具。

使用这个库，开发人员可以分解动词、名词或其他词类，然后寻找模式。有哪些常用词？哪些具体的短语或单词在吸引读者？哪些词和其他名词通用？

根据你的最终目标，你仍然可以回答很多问题，开发很多产品。

自然语言处理并不一定很难——在某种程度上

NLP是一个很好的工具，可以帮助处理文档，对它们进行分类，并寻找它们之间的关系。多亏了AWS和GCP，许多不太懂技术的开发人员可以利用一些NLP特性。

也就是说，NLP有很多困难的方面。例如，开发擅长跟踪对话和上下文的聊天机器人并不是一件容易的事情。事实上，在Medium上有一个很棒的系列节目，亚当·盖特吉(T1)就讲述了这一点。您可以在文章中了解更多自然语言处理的乐趣。
