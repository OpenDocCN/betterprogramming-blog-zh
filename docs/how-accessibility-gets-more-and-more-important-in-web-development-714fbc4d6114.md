对于希望为所有人创建高质量网站的企业和组织来说，可访问性至关重要。对于这些企业和组织来说，不排斥任何人使用他们的产品或服务是很重要的。

但是到底什么是无障碍网站呢？无障碍网站是残疾人可以完美使用的网站。这可能是视力受损、失明或失聪的人，等等。

据 [W3](https://www.w3.org/) 报道，全球残疾人市场超过 10 亿人。随着预期寿命的增加，人们一生中大约有 11%的时间生活在残疾中。尽管如此，大多数网站都没有考虑可访问性。至少还不够。可访问性的伟大之处在于，它不仅改善了残疾人的在线体验。可访问性通常会改善所有用户的在线体验。

你如何知道你的网站是否完全无障碍？一般来说，如果所有用户都能平等地访问信息和功能，那么网站就是完全可访问的。

# 易访问性的商业案例

除了你的网站对每个人都有用之外，还有一个强大的商业案例就是可访问性。案例研究表明，可访问的网站具有更好的搜索结果(SEO)，降低了维护成本，增加了受众范围。一旦网站变得越来越容易访问，整体客户体验就会得到改善。

我之前说的 10 亿残疾人，他们的消费能力超过 6 万亿美元。所以当你建立你的网上商店。你也许应该花些时间让每个人都可以访问它。如果你想最大化你的销售额，你应该从让你的网上商店面向更广泛的受众开始。

但是也可能有一个完全不同的原因让你的网站易于访问。许多国家都有要求数字无障碍的法律。最小化法律风险有时是你希望你的网站可访问的原因。

> 许多组织开始意识到接受可访问性会带来多种好处——降低法律风险、加强品牌形象、改善客户体验和同事生产力。
> 
> Paul Smyth，巴克莱银行数字可访问性主管

# 快速清单

随着可访问性变得越来越重要，作为开发人员，这是一个非常好的工具。这份清单上的许多要点在技术上并不难实现，但却极大地提高了网站的可访问性。

你可以为你的网站做一些快速的胜利:

## 图像的替代文本

HTML 中使用 alt 标签来描述图像。如果你曾经接触过 HTML，你可能对 alt 标签及其用法很熟悉。

Web 内容可访问性指南(WCAG) 2.1 对文本替代项做了如下说明:

> 为任何非文本内容提供文本替代，以便将其转换为人们需要的其他形式，如大号字体、盲文、语音、符号或更简单的语言。

alt 属性需要对屏幕阅读器有帮助的描述。假设你是盲人，你需要知道关于图像的什么？“孩子”没什么帮助，但是“在花园里玩玩具的孩子”可能会给使用屏幕阅读器的人提供更多关于那个图像的信息。

注意，如果图像纯粹是装饰性的，可以将 alt 标记留空。

## 对比

具有低对比敏感度的人，主要是老年人，以及不能区分某些颜色的色盲的人受益于前景色和背景色之间的足够的颜色对比。

足够的色彩对比使你的内容可读性更好，即使对没有残疾的人也是如此。而这并不需要花很大力气就能搞定。

WCAG 2.1 对网站上的对比度做了如下陈述:

> *文本和文本图像的视觉呈现对比率至少为 4.5:1，大文本、附带和标识除外。*

在本文底部的工具列表中，我包含了一个对比度检查器，您可以使用它来查看您是否有足够的对比度。

## 键盘输入

虽然这可能是最难实现的，但从技术角度来看，键盘输入对可访问性极其重要。许多有运动障碍的人依赖键盘。

WCAG 2.1 对键盘输入做了如下陈述:

> *内容的所有功能都可以通过键盘界面操作，而不需要特定的单次击键时间(…)。*

## 音频抄本

耳聋或重听的人在处理听觉信息时有困难。为了在你的网站上为这些人提供视频和音频支持，需要文字记录。

## WCAG

上面的清单包括了对提高网站可访问性很重要的速赢。当然，您可以做的改善可访问性的事情几乎是无止境的。如果你想要一个完整的列表，列出你可以做的事情来提高可访问性，我建议你去 W3.org 网站上的 [WCAG 2.1 部分](https://www.w3.org/TR/WCAG21/)。

# 想知道你的网站站在哪里？

所以你想知道你的网站在可访问性方面处于什么位置。太好了！幸运的是，有很多工具可以用来评估你的网站。

以下是您可以使用的工具列表:

1.  WebAIM 的[对比度检查器](https://webaim.org/resources/contrastchecker/)
2.  [谷歌无障碍开发工具](https://chrome.google.com/webstore/detail/accessibility-developer-t/fpkknkljclfencbdbgkenhalefipecmb?hl=en)
3.  [A11Y 合规平台](http://www.boia.org/?wc3)互联网无障碍事务局
4.  AccessLint ，一个 GitHub 应用程序，检查你的拉取请求的可访问性
5.  WebAIM 的 [WAVE](http://wave.webaim.org/)

如果你想要一个更大的工具列表，你应该查看一下[网页可访问性评估工具列表](https://www.w3.org/WAI/ER/tools/)。
