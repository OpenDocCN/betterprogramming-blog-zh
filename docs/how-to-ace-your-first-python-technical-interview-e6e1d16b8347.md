# 应对白板面试指南

白板面试是一种面试形式，要求你不用电脑，用手解决一个编码程序。

白板面试可能会令人生畏，但是在这些建议的帮助下，你将会在面试中大获全胜。假设面试官让你解下面这段代码。

> 编写一个 Python 程序，打印 1 到 100 之间的所有数字

你会怎么做？

## 第一步:理解问题

在着手解决问题之前先了解问题，这样可以确保不会出现沟通错误，并且你完全理解面试官对你的期望。

你可能会因为解决方案看起来对你来说很简单而想当然地认为是可交付的，但是准确理解面试官想要什么是很重要的。

问题中可能会用到你不懂的术语，这是询问的好时机。例如，上面的问题使用了术语 disarium **。**如果您不知道该术语的含义，请随时要求澄清。

## 步骤 2:输入和预期结果

就像你解决任何其他代码或程序一样，知道你的输入以及预期的结果是一个好主意。永远记住这一点:

```
Given X find Y
```

其中`x`是输入，`y`是预期结果

例如，您为我们的代码问题输入的可能是一系列数字，而结果将是满足您的程序条件的数字。

## 第三步:你打算如何解决这个问题

一旦你定义了你的输入和预期的结果，这就是你向面试官解释解决问题的步骤。重要的是要记住，这不是一个“你”的问题，而是一个合作的努力。

制定大纲时，与面试官接触，并认可任何建议。如果你发现自己卡住了或者思维冻结了，寻求指导是没问题的。在这一步中，你可能会发现你的信心和清晰度增加了，并且你在开发高级解决方案时处于更有利的位置

## 第三步:编写代码

此时，您已经有了输入、预期结果和编写代码的步骤。现在开始写代码。记住和面试官一起讨论你代码的每一步，而不是沉默地写代码。完成后，检查代码并确保没有错误。

虽然您不会使用编辑器运行代码解决方案，但请确保您的代码没有语法和任何其他类型的错误。

## 步骤 4:测试和分析代码

测试您的代码是确保它按预期工作的唯一方法。例如，测试打印 1 到 100 之间的所有 disarium 数字的代码需要将实际数字插入到函数中，并确保它返回预期的结果。

代码分析包括优化代码以获得最大效率，如何改进代码，以及任何可能影响性能的安全问题。

## 第五步:熟能生巧

知道你有信心应对白板面试；你如何使用上面的步骤来解决下面的代码问题？

> 在 Python 中创建一个函数，检查一个字符串是否包含重复的单词。如果句子中的任何单词有相同的字母，该函数应该返回 True，否则返回 False。

## Q1。编写一个 python 程序来计算一系列数字的平均值

```
def average(lst): return sum(lst)/len(lst)print(average([4,4,4]))
```

## Q2。列表理解的一个例子

```
my_list = [i for i in range(1, 11)]
```

## Q3。如何用 Python 写注释

```
#This is a comment
```

## Q4。PEP 8 是什么？

Pep 8 是一个为编写 Python 代码提供指南和最佳实践的文档

## Q5。列表的特征是什么

*   列表是可变的，即元素可以改变
*   列表由方括号创建
*   列表中的每个元素由逗号分隔
*   列表允许重复元素，即不需要唯一

## Q6。什么是元组，以及如何从列表中创建元组。

元组是不可变的(不能被改变),并且由圆括号而不是方括号创建，每个元素由逗号分隔。

```
mytuple = ('max,'john')#Create a tuple from a list
tuple(["max","steve"])
```

## Q7。定义词典

字典由键值对组成，由大括号创建，每个键值用逗号分隔

```
mydict = {'name':'steve', 'age':27}
```

## Q8。什么是集合？

集合是项目的无序集合，用大括号创建，由逗号分隔的单个元素组成。

```
myset= {2,4,5}
```

集合不允许重复，即集合中的每个元素都是唯一的。例如，如果您从包含重复项的列表中创建一个集合，它将返回一个删除了所有重复项的集合。

```
lst = [1,2,2,2,5,8,9,6,6]set1 = set(lst)print(set1)
```

输出将是:

```
{1, 2, 5, 6, 8, 9}
```

## Q9。用字符串创建一个集合

```
myset = set("hello")print(myset)
```
