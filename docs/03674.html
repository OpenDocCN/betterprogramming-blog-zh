你不需要JavaScript框架

原文：https://betterprogramming.pub/you-dont-need-a-javascript-framework-df2a36c2dd0a?source=collection_archive---------2-----------------------#2020-02-26

有时反应，角，或Vue.js可能太多了

我们遭受JavaScript框架疲劳了吗？还是我们忽略了用户的性能和数据捆绑？

让我们回到jQuery时代

你还记得那些大家什么都用 jQuery 的日子吗！之前是jQuery，之后是jQuery，所有东西闻起来都是jQuery的味道。每个网站或网络应用都包含jQuery。

为什么是？

开发人员发现普通的JavaScript太难或太复杂而难以处理。浏览器之间有很多很大的差异。

感谢上帝，那些日子已经过去了！jQuery修复了JavaScript社区的一个大问题。但这让我们大多数人变得懒惰，因为我们不明白引擎盖下发生了什么。

对我来说，当这个网站推出的时候，一切都结束了。

JavaScript疲劳时代

眼下，2020年才刚刚开始，我们却在加码网站和网络应用。我们可以选择Angular，React，Vue.js，svelite，或者 Polymer 。不好意思，我就不一一列举了…

也许你在想:“好吧，你自己去判断吧。”这也是我写这篇文章的目的。我对自己的JavaScript疲劳非常不满。

但是说实话。用一个简单的命令从头开始创建一个网站或web应用程序从未如此简单。对于角度，我们使用ng new project name，对于反应，我们使用…，等等。

大多数时候，我们选择最适合自己的。但是，我们必须进一步降低JavaScript的权重，并对加载到应用程序中的内容更加深思熟虑。

因为可能HTML，CSS，JavaScript就够了。

当我谈到JavaScript框架时，我也指JavaScript库。棱角分明的、反应灵敏的、 Vue.js 、苗条的对我来说就是这一类。(是的，还有很多……)

使用JavaScript框架的利与弊

使用JavaScript框架有一些优点和缺点。因为这是关于善与恶的。更像是一种情况比另一种情况更好。

我在日常工作中使用Angular，并喜欢尝试React或Vue.js，或者尝试一些小型的JavaScript微库，它们只能很好地做一件事。

当你不应该使用JavaScript框架时

在一些情况下，我不推荐使用JavaScript框架，而是使用JavaScript微库或普通JavaScript。

  1. 您的应用程序很简单或很小。如果你正在构建一个小项目，比如试验一个新的JavaScript API。
  2. 对性能要求严格。当您的应用程序需要非常高的性能时，即使是在低带宽连接上。

线路上发送的每个字节都很重要。然后，用大框架就不聪明了。因为JavaScript框架的开销是有代价的。

有很多工具可以让这变得更容易管理。预防问题比解决问题更好。

什么时候JavaScript框架是明智的选择

有些情况下使用JavaScript框架是明智的。但是，我们仍然需要小心谨慎。

1。您的应用程序很大

当你在构建一个大的应用程序时，使用JavaScript框架可能是明智的。一个框架，大多数时候，来自于社区的巨大支持。

大多数时候，一个社区会提供大量的学习材料，帮助构建一个长期支持的应用程序。

2。你或公司看重开源

开源最大的好处是每个人都可以随心所欲地使用它(如果许可允许的话)。

这个框架的很多元素都是由那些花时间在上面的人构建的。这不会花费开发者或使用它的公司任何钱。

如果一个公司/开发者喜欢为JavaScript框架做贡献，其他人也可以从中受益。

3。新功能的高速开发

因为大多数框架提供了大量的工具，它们使得开发人员更容易在更短的时间内构建新的特性，他们可以依靠为他们做了大量测试的聪明工程师。

结论

在我的故事中，当我们谈论JavaScript框架时，我认为利大于弊。(如有补充，请在评论中补充！)

但是即使使用框架听起来很好，我们也必须小心加载到我们的网站和应用程序上的内容。

总是问这样一个问题:“我为什么需要这个框架？我们自己构建或者使用已经存在的东西会有什么好处？”

“这对用户有什么影响？低端手机是否也能够使用与高端手机相同的用户体验？”
