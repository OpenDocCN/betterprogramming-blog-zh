我们都听说过，但没人能就定义达成一致

我喜欢流行语。区块链。艾。云。物联网。它们被一次又一次地使用，直到它变得像一个电话游戏——这个词现在有了和我们第一次听到它时完全不同的含义。就拿无服务器这个词来说吧。它曾经有一个清晰简洁的定义，现在它意味着从“云”到神秘的“解决我们所有问题”的任何事物。

那么到底什么是无服务器？事实上，这是解决我们所有问题的方法吗？大概不会。它能解决我们的一些问题吗？大概吧。让我们探索一些关于无服务器的神话，并讨论它到底是什么与它不是什么。

它不是什么

实际上无服务器

与这个词让你相信的相反，你的代码仍然在服务器上运行。服务器从你身边消失了。这意味着你可以整天写代码，把它推给master，它就会到达它需要去的地方。您的代码将在某种类型的容器中运行。无服务器的意义在于你不关心服务器在哪里。作为开发人员，您关心软件的业务能力。

抛弃你的巨石的快速通道

你在那块巨石上投入了多年的心血。您已经紧密耦合了您的域，您构建了一个高功能的状态管理系统——您的应用程序需要在本地运行。无服务器应用程序不像你的monolith那样工作——它们是松散耦合的、无状态的、分段的代码。将您现在拥有的东西一个接一个地塞进一个无服务器功能的集合中并不简单——您需要进行一次大的返工。在某些情况下，从你的独石中吸取经验教训并从头开始可能会更快。

无限扩展

经常被吹捧为无限可扩展的无服务器应用程序往往被寄予不切实际的期望。归根结底，代码是在物理硬件上运行的，因此在吞吐量、CPU能力和内存方面存在不可避免的限制。然而，这些因素有着难以置信的上限，你不太可能将它们发挥到极致。与本地安装相比，无服务器应用程序似乎具有无限的可扩展性，因为它可以根据需求增加运行的服务器数量。

唯一的云选项

您可以通过多种方式整合云。可能性包括在托管的虚拟机中运行您的应用程序，部署一组容器，甚至只利用一两个服务，同时在本地托管您的应用程序的其余部分。一个完整的、无服务器的应用程序正在将云推向极致，它绝不是您云之旅的唯一选择。

事实真相

无忧基础设施

无服务器的主要好处是消除了硬件的责任。您的云提供商将从您那里抽象出这一层，这样您就可以专注于重要的东西:软件。通常，提供商将提供自动缩放功能，直到函数级别，帮助您实现最佳性能。

波动但可预测的定价模式

无服务器就是为你使用的东西付费。具体来说，它为计算时间和吞吐量付费。因为你的使用量不会每个月都完全一样，所以你的账单会变化。一些计算繁重的功能可能会运行一个月以上，或者您带来了一个新租户，该租户比其他人更多地使用一些新服务。幸运的是，大多数主要的云提供商都提供类似于 Azure 和 AWS 的计算器来帮助估算你的账单将会是多少。

心态转变

如果你习惯于构建单一的应用程序，或者是微服务的新手，进入无服务器的思维模式会很困难。将代码分解成小的可消化的函数，这是对一个有3000行代码的类文件的一个很大的改变。此外，由于无服务器的自动伸缩性、速度和灵活性，开发人员倾向于NoSQL数据库，而不是关系数据库。这种类型的数据库更改可能会改变应用程序的游戏规则，但会让开发人员绞尽脑汁。从关系存储迁移到文档存储会带来一系列挑战，需要彻底转变思维方式。

快速、分段开发

使用像无服务器功能这样的服务迫使你将代码分割成离散的操作块。这些块遵循单一责任原则:它们很简单，它们很好地结合在一起，它们是松散耦合的。这里无服务器的好处是，你可以像说“API层”一样快地突破REST端点拥有小的操作代码块不仅可以帮助你更快地完成工作，还可以简化潜在的复杂业务流程。将代码分成几个部分可以让开发人员对一个领域有深入的了解。不再有意大利面条式的代码，这里的一个变化会破坏那里的三个东西。

语言不可知

无服务器函数可以用多种语言编写。因为它们彼此独立运行，所以每个函数都可以用不同的编程语言编写。在构建无服务器应用程序时，语言是一个需要考虑的重要因素，原因有很多，从函数的冷启动时间到哪种语言最适合某项任务。所以要注意你为时间敏感函数选择的语言。

最后的想法

现在有很多关于无服务器的传言。每个人都开玩笑说，在无服务器开发中仍然有服务器参与，但这是大多数偶然的无服务器对话的范围。

无服务器应用程序仍处于起步阶段，但有望被广泛采用。从架构的角度来看，将代码分解成单个责任块是有意义的。从基础设施的角度来看，谁不想将硬件的责任转移给服务提供商呢？从商业成本的角度来看，没有比付费使用更好的了。没有前期成本或配置成本，只是一个可预测的模型，可随您扩展。

如果你的公司正在探索无服务器的世界，那就去吧。构建概念验证或快速原型。你不需要花费大量的时间来判断这是否是你的正确道路。无服务器开发的好处在于它很快，所以把你的想法写下来并测试它们。几乎所有的无服务器供应商都为他们的服务提供免费层，唯一的成本就是你的时间。所以，开始行动，进入无服务器的世界吧！
