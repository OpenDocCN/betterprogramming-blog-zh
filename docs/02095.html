一个初级程序员到底需要知道什么？

这是对初级程序员第一份工作的期望

如果你刚刚开始进入软件工程领域，并且已经开始申请初级工程职位，你可能会遇到阻碍。每个公司似乎对工作职责的定义都很松散，也没有明确的期望。

这尤其棘手，因为每个公司都有自己的软件工程师规则和标准。很有可能你已经读过一长串的需求清单，你可能已经开始想知道什么时候才是开始申请初级工程师工作的正确时间？

本文将打开潘多拉的盒子，并解释什么，什么时候，以及如何打开。当你作为初级开发人员开始你的第一份工作时，软件即服务公司可能会对你有这样的期望 ( SaaS ) 。

注: 我写过没有计算机科学学位的你如何在180天内成为一名web开发人员。这不容易，但如果你愿意投入工作，肯定是可行的。

对一门编程语言有一个像样的掌握意味着什么？

每个公司都希望你能很好地掌握一门编程语言。但是“得体的把握”到底是什么意思呢？

因为我主要是一名JavaScript工程师，所以让我们从JavaScript的角度来考虑这个问题。如果你对JavaScript有很好的理解，那么你可以很容易地学会新的框架。或者，至少，你不会迷失在学习中。

如果你很好地掌握了一门语言，那么你应该能够学习公司特有的新概念，或者使用专门为这门语言设计的技术。

大多数地方不会指望你成为语言本身的专家，除非你这样推销自己。一些面试官可能会非常认真地探究你的知识，但这通常只是为了看看你知道些什么。

总有成长的空间，面试官会打破界限，看看你哪里不足。不要害怕告诉别人你不知道答案。这是一种观察人们如何处理未知的技巧。我更愿意雇佣这样的人，他会回答“我不知道，但我会这么做来找出答案”，而不是试图掩盖自己缺乏知识的人。诚实大有帮助。

你需要能够建立一些东西并展示你的潜力

当谈到在编程领域就业时，最重要的技能是能够构建一些东西。程序员是制造者，他们生产代码，构建产品。需要特定的心态才能不断生产。你需要接受脱离自己的舒适区，你必须做好学习的准备，并渴望推动事情向前发展，即使是在逆官僚主义和有时无能的管理潮流而动的时候。

今天就开始建设，不要浪费时间，毕竟这是我们拥有的最宝贵的资源。我们总是可以获得更多的金钱和物质，但我们不能获得时间。对每个人来说都是有限的资源。

你可以建造任何东西，只要你在建造。旋转PostgreSQL数据库，用动物填充它，获取数据，并在浏览器中显示它。如果你不知道该做什么，这里有六个前端挑战，你可以从今天开始着手。

我总是说成为一名真正优秀的开发者的秘诀是构建东西。把它比作健美或慢跑；你必须每天去健身房练举重才能达到效果。编码也是如此。你必须打开你的代码编辑器，开始构建东西来获取知识。不要花太多时间只看书。80%的时间花在代码编辑器上。

将大多数工作“要求”视为“最好的要求”

当你在工作岗位上看到一长串要求时，不要惊慌。这些都是“最好的”，因为理想的候选人并不存在。通常，招聘人员对技术问题一无所知，会在一张纸上背诵一堆单词。我的建议是研究公司的背景和使命。如果你相信这家公司正在做的事情和它的使命，那就值得申请。

对于初级程序员来说，有两种类型的雇主:问题解决者和即将成为专家的。

问题解决者(如谷歌、苹果、微软)希望雇佣问题解决者。数学专家或算法奇才；能够发明解决新问题的方法的人。他们还希望你能快速掌握新技术。几乎没有手把手，在第二或第三个月的时候，你就会被期望成为一个高效的问题解决者。拥有扎实的算法教育和出色的问题解决智能的人会喜欢这类公司。

专家类型(例如政府、应用工厂、大多数银行、内部IT部门)需要在特定框架或技术方面有丰富经验的工程师。他们在一个很好理解或高度专业化的技术领域中运作。他们对技术创新不感兴趣——他们在寻找熟知技术的快速执行。他们要么对现状感到满意，要么可能通过专注于产品的创新来推动他们的业务。这些也被认为是“维护”工作，因为你需要保持轮子和系统运行。这些类型的工作更轻松，要求更少，通常工资更低，也没有那么有趣和挑战性。

弄清楚你适合哪种工作类型。如果你认为你不能(或不想)成为一个杰出的问题解决者，那么就专注于专家角色。在这种情况下，我会用他们的需求开始一个电子表格；把最常见的加起来，花三到六个月的时间构建并发布一个端到端的项目。确保公开让每个人都能看到。

我应该什么时候开始申请初级工程职位？

有句谚语是这样说的:“种一棵树的最好时间是20年前。第二好的时机就是现在。”在申请工作的背景下，开始申请的最佳时间已经是昨天了。你在工作的时候学到的东西最多，因为你经常被现实世界的问题困扰。

每天花一个小时浏览工作，看看你对什么感兴趣。如果你看到一个有趣的职位，研究一下这家公司，看看他们做什么，谁在那里工作，他们使用什么样的技术。如果你真的想成为一名谷歌人，那就去学围棋或C++。如果你想为创业公司工作，学习Node和React。弄清楚你想要什么样的工作，并开始为要求做准备。没有好事落到你头上。你必须付出努力并为之奋斗。

结论

从你走进公司的那一天起，你就应该清楚地知道公司对你的期望。如果你运气好，被邀请参加面试，这里有八个你应该在接受工作前问雇主的问题。

值得做的好事都来之不易——如果容易，每个人都会去做。当你刚刚起步时，这很有挑战性，但我保证这是一条有趣且令人兴奋的道路。

感谢你的阅读，保持好奇心，在前进的时候保持高昂的头。祝你冒险好运！
