# 我合作过的 13 种软件开发人员

> 原文：<https://betterprogramming.pub/13-types-of-developers-i-have-worked-with-565b74b66475>

## 爱他们或恨他们，你不能忽视他们

![](img/86a858a5d4afd6311f70b0bfc9798ad8.png)

卢卡斯·安德拉德在[的照片](https://www.pexels.com/es-es/foto/casco-en-el-suelo-2862718/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels)。

在我的职业生涯中，我和许多不同类型的开发人员一起工作过。我喜欢和他们中的一些人一起工作，但是对于其他人，我只希望完成这个项目，不再一起工作。

在本文中，我将列出其中的一些类型。

# 1.卖烟的

这种开发人员总是向每个人承诺事情——客户、你的老板或其他开发人员。但是当一些事情真的需要做的时候，如果可以的话，他们会跳到另一个项目或者试图把所有的工作委托给其他开发人员。这是非常危险的，因为它会给你带来严重的麻烦。他们销售一种他们不会用从未使用过的技术来制造的产品。

如果你不了解他们，你会认为他们知道他们在谈论的一切，他们可以帮助你，但很快你就会意识到这是一个谎言，你不能指望他们。

# 2.多任务开发人员

他们不是一个糟糕的开发者。他们通常有很好的技术技能，但是他们太忙了，以至于他们从不专注于任何事情，你永远也不能指望他们做一个中等规模的项目。很难引起他们的注意，尽管他们有时有很棒的想法，但要让他们完成某件事或致力于某件事还是需要付出很多努力。

# 3.有证书的专家

对于这种把所有优点都建立在证书上的人，我没有什么好的经验。我认为对你将要从事的工作有一个扎实的知识基础是必要的。我不反对证书。但在大多数情况下，如果没有真实的经验，有证书并没有太大的作用。

几年前，我曾经为其开发软件的一个客户雇佣了一名软件架构师。他的角色是组织和领导所有与客户合作的外部开发人员。

起初，我对此并不介意，但我很快意识到，他所做的只是听别人的解决方案，并假装这些方案是他自己的。

就我而言，我在他试图改变我们工作方式的会议上浪费了很多时间，但这些会议只是基于他读到或听到的想法，而不是付诸实践。

当然，有证书或学位是好事，但前提是要有真实的经历。否则，你将成为一个“理论开发者”。

# 4.混乱的开发者

他们的代码一团糟，没有遵循任何好的实践，混合了所有的设计(如果有的话)。

我和这样的人一起工作过。在计划会议结束时，我通常会问他们是否对我们讨论的内容有任何疑问，因为我只看到他们在一张纸上做一些乱七八糟的笔记。他们总是回答我他们没有。几天后，他们开发了一些与我们在会议上同意的完全不同的东西。

有很多线索可以识别一个混乱的开发者:

*   他们有条理地记笔记吗？
*   他们会把所有事情都写下来以便不会忘记任何事情吗？
*   他们的桌子整洁吗？

# 5.理论开发者

理论很多，实践很少。他们总是告诉其余的开发人员如何做事情，但他们在大多数情况下从未应用他们所说的话。当老板在的时候，他们会吹嘘他们“知道”的一切，但他们会想方设法逃避实际的工作。

在我的一份工作中，有这样一个人。他到处都在说我们可以如何做事，但我很少看到他开发任何东西。当我和他一起工作时，我几乎不得不告诉他怎么做每件事。

# 6.普通的开发者

这是最常见的开发者类型。他们倾向于做一般的事情，他们通常希望待办事项列表中有可预测的和舒适的任务。

# 7.安全开发者

他们从不主动。他们总是等待你告诉他们如何做每一件事，使用什么技术，或者提醒他们在会议中关于他们必须开发的部分说了什么。然而，如果出了问题，责任会落在你身上。

他们花更多的精力和时间来确保如果事情出了差错，他们不会受到责备，而不是真正考虑做一项工作的最佳方式。

他们总是选择最舒适的方式来完成工作，即使有更好的选择。

# 8.通用开发人员

大多数项目的优秀开发人员。通常没有什么超级技能，但可以开发任何项目，从后端到前端。

# 9.自恋者

通常，他们非常自负，团队合作能力很差。

他们希望被认为是团队中最好的开发人员。他们倾向于过度设计简单的东西，并使用“聪明的代码”,这对于其他人来说是难以维护的。如果他们审查你的代码，他们总是试图重构它，并添加一个注释，即使它是不合理的。

其他时候，他们会告诉你有问题，他们会用一些超级巧妙的方法来解决。但是最后他们不会去做，当老板问你为什么一直失败的时候，他们会说已经警告过你了。

简而言之，这是一个通常拥有良好的技术知识，但倾向于将事情复杂化，以使自己脱颖而出，并将自己置于他人之上的人。他们不容易共事。

# 10.痴迷于一切的人

这种开发人员痴迷于所有的标准和方法。虽然这在原则上可能是好的，但如果走极端，灵活性就会丧失，一切都会变慢。

# 11.独角兽

在我的公司，有一个我认为是“独角兽”的开发人员。他拥有一切:很好的技术知识，很好的沟通技巧，以及持续不断的帮助意愿。

如果有问题，他总是愿意解决。如果他说他会做什么，他就会做。

总的来说，他们什么都擅长，但在同一份工作中往往不会持续很久，因为他们知道自己的能力而不自恋，并且有好的工作机会。

# 12.快速开发者

他们倾向于很快完成所有事情，但继续下一个任务，而不是 100%完成上一个任务。

他们寻求快速完成事情并使其工作，尽管在许多情况下，没有测试任何东西或应用好的实践。他们不记录或遵循记录。

问题是一切都必须事后重构或修复以避免错误。

# 13.那个总是乐于助人的人

这是我最喜欢的类型之一。他们通常什么都懂，知道怎么做好。他们总是乐于助人，不盛气凌人，也不摆架子。他们通常是经验丰富而谦逊的人。

不幸的是，有些人会像回答孩子一样回答你的问题，如果你反驳什么，他们永远不会接受任何批评。

# 结论

最后，最重要的是寻找一个平衡的团队，团队成员容易共事，可靠，并且愿意学习。技术知识和经验是日积月累学来的，但一个人的存在方式更难改变。

由于这仅仅是我与其他开发人员一起工作的个人经验，我相信您已经遇到了许多其他不同类型的软件开发人员。我希望你能在下面的评论区分享你的经历。
