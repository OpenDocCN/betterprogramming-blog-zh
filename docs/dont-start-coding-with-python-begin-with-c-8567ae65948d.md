## 不要被围绕 Python 的炒作所迷惑。你以后可能会后悔的

Python 推动编程的能力令人惊叹。它微妙地支持了谚语“如果你懂英语，你应该知道如何编码”的观点。凭借其类似英语的语法、缩进范式和丰富的库，Python 让其他编程语言看起来毫无用处。

Python 是“21 世纪最性感的工作”的默认编程语言。是的，数据流行语仍然让数据科学家变得“性感”，尽管目前的情况并不完全支持这种说法。令人印象深刻的是，Python 恰好回答了大多数(如果不是全部)数据科学问题。

Python 在区块链、DevOps 和网络安全领域也很流行。

围绕 Python 的宣传呈指数级增长。这种语言被用作无数在线课程和学术项目的编码媒介。

也就是说，尽管 Python 做出了种种承诺，但这种语言并不是测试编程水域的可行选择。c 是更好的选择。

在本文中，我将向您解释为什么您应该使用 C 而不是 Python 作为起点。

# 强势开始

我的论文导师曾经告诉我，“永远要坚强地开始。第一拳决定战斗的命运。”

Python 是一种高级编程语言。这意味着 Python 享有高度的抽象性，允许用户和语言之间的平滑连接。

换句话说，Pythonic 解释机制倾向于容忍其他编程语言可能标记的错误。没有变量声明，没有列跟踪，自动转换。所有这些都增强了编程体验，并使 Python 对用户友好。

有趣的是，Python 的好处也是它的弊端。这种“即食”方法培养了脆弱的编程心态，助长了急躁和懒惰。

另一方面，c 是一种中级编程语言。它介于低级编程语言(汇编、机器语言)和高级语言(Python、Ruby 等)之间。).实际上，这使得 C 比 Python 更难写。然而，这是值得的。

c 展示了编程的真实面貌。c 程序员忍受艰难来让他们的代码运行起来，这灌输了良好的编程习惯。

众所周知，c 开发人员会一丝不苟地编写相对有组织、干净的代码。因为建立这种技能的过程发生得过早，它们不仅会影响 C 编码，还会影响未来的编程努力。

简单地说，C 是一个很好的起点，因为它为有抱负的开发人员奠定了坚实的编程基础。诸如动态内存分配和数据结构之类的计算机科学基础知识使得编程成为一个艰难的旅程，但是开发人员将会收获他们努力的回报。

# 不切实际的期望

让我心碎的是 Python 带来的不切实际的期望和虚假的希望。许多人相信 Python 是他们职业抱负的黄金门票，因此越来越多的人希望掌握这门语言。

例如，就在几天前，在脸书的一个小组中，我无意中看到一个 7 岁的孩子在寻找数据科学自由职业的帖子。他附上了一个 Python 在线课程证书，作为他编程熟练程度的证明。

现在，我决不会低估这个男孩，因为许多年轻的开发人员编写的代码表明他们精通编程。然而，让我畏缩的是在这么年轻的时候就找工作的想法。

看到这一点，我意识到技术领域已经发生了根本性的变化。现在每个人都想掌握 Python 来获得承诺的高薪工作。

我不是说这是 Python 的错。这是我们的错。

这是我们的错，因为我们往往把标准定得太高，好像 Python 就是一切。实际上，在当今这个不可预测的世界里，想要成功，需要的不仅仅是意愿和编程语言。

以疫情引起的经济衰退为例。这场危机把就业市场变成了战场。这反过来削弱了计算机科学毕业生，他们曾经没有进入求职飓风就获得了就业。因此，除非你超越了一个愚蠢的证书，并努力工作，你不会有机会。

就这么简单！

> "现实一点:为奇迹做准备."— [奥修](https://www.goodreads.com/quotes/121347-be-realistic-plan-for-a-miracle)

c 是一个很好的选择，可以训练你的手指，也可以让你的大脑运转起来。最后，至少你会对前方等待你的挑战有所了解。

# Python 是用 C 煮的

Python 无非就是一堆规则。

管理 Python 的规则可以用一些语言实现。通常，我们将使用的语言和 Python 的缩写连接起来表示实现。

例如，我们有 C 语言的 [CPython](https://github.com/python/cpython) 和 Java 语言的 [Jython](https://www.jython.org/) 等等。

最受欢迎的 Python 实现是用 C 语言实现的。因此，在破译 Python 代码时，大量的 C 语言知识肯定会派上用场。

最重要的是，相当数量的 Python 包(如 NumPy)是用 C 和它的弟弟 C++编写的，以解决效率问题，因为 Python 相比之下非常慢。

确实，当一个人习惯了快如闪电的运行时，他们会对执行缓慢的程序更加敏感。这导致了产生高效代码的倾向——不管是用 Python 还是另一种编程语言。

# 外卖食品

我一生中接触过超过五种编程语言，C 是第一种，Python 是最后一种。

虽然我很感激老式 C 语言赋予我的编程基础，但我肯定不会偏向 C 语言。在与“普通的 Python”开发人员一起工作时，我对首先学习 Python 的疑虑被消除了。

我注意到，我接触过的大多数(不是全部)开发人员都缺乏重构技能。他们很少认真对待运行时问题，他们所关心的只是拥有一个功能代码。实际上，编码不仅仅是这些。

那一刻，我意识到在蟒蛇妆的背后，还有一张丑陋的脸。如果从一开始就精心护理的话，这张脸会更漂亮。

总之，如果有 C 思维的支持，掌握 Python 的道路就不那么陡峭了。

感谢阅读。我希望我没有冒犯任何人。
