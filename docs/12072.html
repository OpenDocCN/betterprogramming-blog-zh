问题# 20——查看一些Git建议

欢迎来到第20版咖啡字节，更好的编程每日文摘。

GitHub最近宣布，将要求所有在GitHub.com上贡献代码的用户在2023年底前启用2FA。查看他们官方博客上的完整公告。

在这一期中，我们收集了一些详细的git指南和建议来帮助你马上开始git。

  • 你的Git提交历史应该读起来像一本历史书。以下是方法。 —由 Mirco 提供。Mirco的第一篇文章《更好的编程》分享了一种通过git挂钩强制执行常规提交消息的方法，以使您的提交历史可读。这个故事在Reddit社区引起了很多反响。请查看并告诉我们您的想法。
  • 为什么在Git中存储秘密和密码是个坏主意——作者 Alessandro Segala 。将秘密与源代码存储在一起可能不是最好的方法。亚历山德罗的作品围绕着采用更好的技术。我喜欢用钥匙库的主意。
  • 使用Git-Crypt——作者迈克尔·博根，安全地将你的应用程序秘密保存在Repos中，即使它们是公开的。如果你仍然想在公开回购中保守你的应用程序的秘密，迈克尔通过采用git-crypt分享了一个伟大的技术。
  • 用Git钩子 自动化手动提交任务。另一个有用的强大Git挂钩列表，用于自动化日常提交任务。对我来说是新的。
  • 写出更好的Git提交消息来提高你的工作效率——Paul Knulst。总是那些微小的变化会带来巨大的不同。Paul的指南讨论了一些组织提交信息的方法，以提高团队的生产力。
  • Github模板:在开发团队中形式化拉取请求的更聪明的方法——作者 Paige Niedringhaus 。Paige's展示了Github的pull_request_templates，一种有序管理团队拉取请求的方式。
  • 抛开饭桶结账。考虑一下Git Switch的变化——作者 Dmytro Khmelenko 。git checkout是在分支之间导航的实际方式。但是你知道git switch是一个很好的选择吗？Dmytro认为对于完全不熟悉Git的人来说，理解switch更容易。
  • 为什么以及如何签署你所有的Git承诺——理查德·纳吉。您可能不总是需要签署每个Git提交。然而，如果你正在管理一个任何人都可以贡献内容的库，你可能想要查看Richard的指南。

这一期到此结束。一定要看看我们以前的版本——更多有趣的故事，比如 #19 和 #18 。
