# 成为软件工匠的 5 个理由

## 找出是什么驱使一些人每天用电脑工作

![](img/c236b1804f46f1a025e5730f3c529c78.png)

[亚历山大·奈特](https://unsplash.com/@agkdesign?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照。

人们经常问我以什么为生。回答这个问题并不总是容易的。基本答案是“我和计算机打交道。”

当有人对你做的事情感兴趣时，谈论软件开发所提供的伟大的东西是很好的。因为软件开发很棒。

在这篇文章中，你会发现为什么软件工匠热爱他们的职业。如果你在以下五个原因中认识到自己，并且你还没有在这个行业中活跃，那么开发软件可能是你的事情。

术语“软件工艺”来自于[敏捷宣言](https://agilemanifesto.org/)。如果你对软件工艺不熟悉，Sandro Mancuso 的书*The Software craftman:professionality，practices，Pride* 是一个很好的起点。如果作为一名软件专业人员，你在工作中找不到太多的价值，这是非常值得推荐的，因为这将改善你的心态。

# 成为一名创造者

作为一个孩子，我们经常被问到我们梦想成为什么样的人。消防员、飞行员、律师都是不错的答案。这些工作广为人知，目的也很明确。我从来没有听到一个孩子说他们想成为一名 IT 专业人员。

也就是说，我们与极富创造力的人一起工作。对我来说，创意非常重要。一个好的软件工匠可以成为他们世界的神。他们可以自己创建网站、应用程序，甚至是一个像《我的世界》一样的世界。十几岁的时候，我就已经很好奇一个网站是怎么做出来的，或者一个游戏是怎么拼起来的。如今，人们被虚拟现实惊呆了。所有这些都是由软件驱动的，没错。

承担责任并成为你开发的世界或应用程序的所有者之一会非常有成就感。思考你或你同事的行为对系统的其他部分意味着什么，这对你的公司来说是至关重要的。

# 永远不会无聊

软件开发充满了挑战。大多数时候，你甚至不知道接下来会发生什么。我们记录未来三个月范围(瀑布)的时间已经过去很久了。开发团队不知道他们长期会遇到什么。他们认为在所谓的 2-4 周的冲刺阶段。工匠也要记住，他们正在跑一场由许多短跑组成的马拉松。

软件开发并不容易。需求来来去去。顾客不知道他们想要什么，但他们希望他们的想法今天就变成现实，如果不是昨天的话。这些要求可能是由公司本身推动的，也可能是由法律、疾病和迁移等因素推动的。

这个行业充满了惊喜。这是你应该热爱的事情。如果你只想上班，不想想太多，那么软件开发不适合你。

这是不是意味着你必须工作到很晚，长时间工作？不，不是这个意思。我们努力以可持续的速度做到最好。这是你和你的雇主都应该明白的事情。很难把你的注意力从一个你在接近下班时间努力解决的问题上移开。不过，你应该休息。软件工匠是思想工作者，没有充电的电池，他们的工作质量会降低。

# 解决问题

开发人员是真正的问题解决者。如果你轻易放弃，软件开发可能不适合你。作为一名软件开发人员，你被卡住的次数多得令人难以置信。有时，我们认为某件事相当简单，但是开发有时感觉就像试图说一门外语。让别人理解你很难。让计算机理解你有时是极其困难的。

不是所有的东西都可以通过简单的谷歌搜索找到，但它确实有助于成为一个好的搜索者。

一个真正的工匠意识到他们在这个世界上并不孤独。他们在正确的地点搜索，问正确的人，节省时间。他们知道在哪里可以快速找到信息，并确保他们不需要多次敲同一个门。他们不怕问问题，但是他们不想因为多次问同一个问题而打扰别人。

> "我从未发现有人在我向他们求助时不想帮我。"— [史蒂夫·乔布斯](https://www.inc.com/peter-economy/steve-jobs-on-the-remarkable-power-of-asking-for-what-you-want.html)

# 与人们一起工作/为人们工作

![](img/42e1b588e2f51d2e2fda98014138c8ac.png)

由[活动创作者](https://unsplash.com/@campaign_creators?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片。

当我说最好的软件设计师是非常社会化的时候，人们可能会说我疯了。软件工匠关心与他们一起或为他们写代码的同事——以至于他们整天都在关注他们做代码的方式。

你所做的工作必须容易被你的同事理解。不这样做是极其自私的。你可以把它比作非法倾倒。这种情况时有发生，居住在该地区的社区深受其害。在专业软件开发中，几乎所有的事情都是在一个团队中完成的。

社区在软件开发领域非常重要。正是因为这些开源社区、会议、作家和知识共享，我们的世界才取得了如此大的进步。

虽然一个软件开发人员和其他人交往不多可能是真的，但我可以说他们中的大多数人在一起都很有社交能力。有些例外喜欢独自工作，不帮助别人，但是他们永远不会被称为“软件工匠”

# 保持相关性

如果我们环顾四周，我们看到许多人害怕一个正在迅速变化的世界。老一代人很难跟上技术的发展。如今，不能发送电子邮件或理解 Wi-Fi 是一个严重的限制。

如果你不努力跟上，一切都很复杂。我的职业迫使我保持与时俱进。一个软件专业人士是不会害怕终生学习的。他们知道这是跟上的唯一方法。

但是，不要误会。拥有多年软件开发经验的人仍然领先于没有这种背景的人。年轻的软件专业人士经常低估这一点。对我们周围的软件工匠表现出足够的尊重是很重要的。对于每一个努力保持相关性的人，软件工匠都会表现出尊重。

软件工匠明白需要跟上。尽管出现了许多新技术，但基本原则和方法仍然存在。除了在技术领域保持相关性，学习软技能也非常重要。

# 结论

你在这五个理由中认可自己吗？那么软件开发绝对适合你。在软件开发中，不缺乏挑战。相反，成为你所在领域的大师才是真正的挑战。在一个变化如此之快的世界里，这是一个美好的人生目标。

一个真正的软件工匠永远不会称自己为大师。没有人比工匠更明白，他们可以从别人那里学到无限的东西。你不会在一天之内或者通过学习一门课程成为大师。这并不意味着你不能称某人为大师或者像大师一样付钱给他们。

我们为我们正在做的工作感到非常自豪。一个真正的工匠不仅会提高自己，还会教育他人。他们带着尊重做这件事。我最尊敬的是我职业中最卑微的人。
