## 而且我还不到 40 岁…

先说清楚:我不老。38 岁时，我觉得自己像生命中任何时候一样年轻、有能力——无论是身体上、精神上还是其他方面。尽管每天午饭前我的孩子们会说我老了十几次，但我仍然觉得自己像曾经的那个 23 岁的开发人员。我仍然对新技术和整个网络的发展感到兴奋，但在我的领域，我现在被视为一个老人。

我很幸运，我在一家重视我的公司有一份高级开发人员的好工作，我对我们的代码库了如指掌。我能够快速添加功能、诊断问题，并快速响应新的请求。然而，我们的代码基础是基于 PHP/MySQL/JavaScript 的过时世界。

对我们公司来说，这不是问题。事实是，我们不需要一个超现代的前端框架连接到 NoSQL 文档数据库来完成我们的工作。PHP/MySQL 做得很好，让我们每天都可以完成所有我们需要的事情，并且它可以很好地与我之前的所有遗留应用程序兼容。我一直是功能性解决方案而不是新潮技术的支持者，我从来不会为了现代化而感到“现代化”任何东西的压力。

我在一家工业制造公司工作。这被视为一项重要的业务，所以幸运的是，在新冠肺炎的这段时间里，我们没有破产，我的工作也没有问题，但没有人能幸免。我是四个孩子的父亲，还背负着抵押贷款，我突然想到，我至少需要四处看看有哪些工作列表，以防我的情况发生变化。老实说，前景有点可怕。以下是我作为一名“老程序员”学到的一些东西

# 我需要花更多的时间更新我的技能

我绝不忽视新技术。我曾经尝试过，并且真的非常喜欢很多新的框架。Vue 和 React 都是特别优秀的 web 解决方案，我完全理解它们的受欢迎程度。对我来说，问题是我根本不需要它们，而且我不再有时间像以前那样熬夜做激情项目来学习新工具。

这大概是大多数 30 岁以上有家室的开发者能体会到的。然而，事实是，我们忽视了新技术，后果自负。我们必须优先考虑保持更新，因为如果不可思议的事情发生，你发现自己失业了，这些新技术将是市场所寻找的。

# 现在也是扩展这些技能的好时机

开发人员的角色仅仅是编写代码的日子正在迅速消逝——如果他们还没有死去的话。部署、构建环境工具、版本控制等等现在都属于开发人员的工作范围。

仅仅熟悉这些技术可能已经不够了。我个人看了很多关于 Docker，AWS，Azure，Netlify 等。，但我对它们的实际体验非常少，因为我还不需要它们。然而，如果我再次在就业市场上找到自己，我已经可以听到一些 20 多岁的能人坚持对我说 Docker 是所有网络技术的顶峰，我不知道这一点是不可原谅的。

扩展不仅仅是你的编码技能。作为一名开发人员，让自己变得更加全面，会让你在未来对公司变得无限有价值，并让你不会被视为一件遗物。

# 不要太绝望——遗留代码是存在的

在另一种生活中，我是少数自豪的 ColdFusion 程序员之一。老实说，即使在我开始写它的时候，ColdFusion 也已经死了，但它是我当时从事的许多技术的核心。因此，我不得不匆忙地学习。

我加入了许多 ColdFusion 论坛、小组和电子邮件列表，直到今天，我仍然会定期收到电子邮件，询问 ColdFusion 开发人员，因为许多公司都有用该语言编写的现有代码库。在某种程度上，他们无疑会重写一些更现代的东西，但是对于大型企业安装来说，这个任务可能会非常庞大，他们会尽可能地推迟。这意味着一个有才华的 ColdFusion 开发人员仍然可以作为一名职业杀手赚一些不错的钱。我在本土也听说过类似的关于 COBOL 甚至 PASCAL 的故事。

值得记住的是，PHP 现在比 ColdFusion 更加根深蒂固。我想说的是，可能总会有许多遗留代码库，需要人们去维护甚至扩展它们。这些机会会更难找到，但它们会存在。

# 结论

成为“代码界的元老”(我更喜欢这个头衔，而不是“老程序员”)可能有点可怕。事实是，随着年龄的增长，一些智慧*也随之而来，老程序员可能不了解最新的技术，但他们解决实际问题的经验和记录仍然是非常有价值的。*

*永远努力跟上时代，但也不要让它把你送进恐惧引发的昏迷。你不必一夜之间学会所有的东西，你已经掌握的技能仍然是有价值的。找到你自己的平衡，并为作为一名开发人员经历了这么多而感到自豪。*
