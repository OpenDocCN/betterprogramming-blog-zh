# 1.变量是类型化的吗？

如果您来自不同的编程语言(例如 Swift)，您可能知道变量在声明时有特定的类型。例如，如下所示，如果您在 Swift 中从一个整数变量开始，当您试图为该变量设置一个字符串值时，您将会遇到一个错误。

```
var number = 5
number = "five"*error: cannot assign value of type 'String' to type 'Int'*
```

原因是这些语言是强类型的，这意味着每个变量在其生命周期中都有特定的类型。只有当新值具有兼容的数据类型时，才能更改变量值。然而，Python 却不是这样。下面的代码向您展示了 Python 中运行起来没有任何问题的类似代码。

```
>>> number = 5
>>> number = "five"
```

这一特性(即改变变量的类型)被称为*动态类型化*，这意味着给变量赋予任何数据值都是合法的，即使它与开始时的类型不同。上面的代码可以通过一个额外的检查来扩展。

# 3.元组是绝对不可变的吗？

我们常说有些 Python 数据类型是不可变的，有些其他数据类型是可变的。例如，字符串、数字和元组是不可变的常见 Python 数据类型。在讨论元组的不变性之前，我们先简单回顾一下这个概念。

在上一节中，我们已经了解了变量和对象之间的区别。与这一区别相关的是，数据可变性指的是内存中的底层对象。不可变对象(例如字符串)是那些在内存中创建后就不能更改的对象。可变对象(例如列表)是那些在内存中创建后可以改变的对象。
