在我的旅程中，我不断地犯下巨大的错误，导致看似不可能的障碍。在最初的几个月里，我甚至不能理解我所学的东西。我很想说，“你知道吗？我跑得很好。这不适合我。”但我没有。

# 将错误视为机会，而不是惊慌失措

仅仅因为你遵循了所有的指导，你的代码看起来和你正在看的教程中的一模一样，并不意味着你不会出错。当您遇到这些错误时，不要认为编写本教程的人需要为您修复它。

错误是职业程序员的一部分。程序员是链条中的最后一个人。你不能打电话给任何人帮你解决。你可以借鉴其他程序员的想法，但并不总是有灵丹妙药。你可能是第一个收到这个错误的人。你也会是第一个修好它的人。

使用 Google 或 Stack Overflow 通常可以很快找到答案。你至少会发现有人遇到过类似的问题。然后，你只要不断挖掘和尝试，直到你最终找到解决方案。有时候是有道理的。有时你不知道你实际上做了什么来修复它。

教程中出现的错误通常与过时的库或软件有关，并且很容易解决。因此，将该错误消息视为教程的一部分:您需要克服的挑战。

# **为变量、函数和类使用自定义名称**

我记得浏览过在多个领域使用相同名称的教程。他们将一个变量声明为`variable`，然后将它传递给一个函数`variable`，在那里他们将把内容传递给另一个变量，这个变量也被称为`variable`。然后他们会把`variable`送回`variable`。

其他时候，他们会突然使用一个关键字，但并不真正解释这是一个保留字。这一点现在看来很明显，但当时却远非如此。有时候这让我很困惑。

我开始对任何变量、函数、类等使用自己的名字。这样我就能看到我能控制什么。我也觉得我更加关注了，因为这个程序感觉像是我的程序，而不仅仅是我从一本书或一个视频中复制的东西。

# **YouTube 教程不是官方的——阅读文档**

刚开始在 YouTube 上看教程的时候，我对他们说的任何话都没有质疑。我想如果有人在制作一个关于编程的视频，那么他们一定很在行。

我开始注意到人们做事情的方式各不相同——用 Python——一种以严格要求做事方式而闻名的语言。

我意识到我从这些人中的一些人那里染上了坏习惯。我也意识到他们掩盖了很多真正能帮我理清头绪的话题。

我决定开始阅读原始文档，这为我打开了一个全新的世界。是的，这可能令人畏惧和困惑。我不是说你应该只阅读文档作为学习的来源。但是，当您想要了解某个主题的更多信息，或者如果您想要确保您遵循了最佳实践，这应该是最终的决定和选择。

# **JavaScript 有包袱**

出于几个不同的原因，JavaScript 是一种很难理解的语言。主要是因为随之而来的还有很多其他的东西。当你刚刚起步时，想要准确地穿越这个世界几乎是不可能的。

到处都可以看到两种不同版本的 JavaScript:ES5 和 ES6。ES6 有很多重大变化，知道这些真的很棒。但是 ES5 仍然存在，所以很多时候，你将学习两种不同的方法来做任何事情，以防你遇到 ES5。

你需要熟悉 HTML5 和 CSS 基础知识。即使是更好的框架也仍然使用其中的一些。

接下来是 Node.js。这是一个运行时环境，允许您在浏览器之外运行 JavaScript。它可以和你的电脑进行互动，这在以前用 JavaScript 是不可能的。

Node.js 还提供了一系列模块。例如，Babel 为不支持 ES6 的浏览器将 ES6 代码转换为 ES5 代码。您需要学习 Node.js 的基础知识。

然后，您需要弄清楚您想要使用哪个框架。框架允许您通过 API 从用户界面与服务器通信。为了理解整个过程，您需要学习几个库和模块。

我已经走上了 React 和 React Native 的道路，因为这是我的公司所使用的。React 不是一个框架，但是 React Native 是。我知道你在想什么。这太有趣了。

这里的要点是 JavaScript 不仅仅是另一种语言。它带来了一个全新的世界，这对一个新的程序员来说是非常可怕的。一开始我并没有意识到这一点。这使得理解 JavaScript 变得非常困难，尽管代码本身非常简单。

# **使用、学习、掌握 Git**

我不记得有哪一个教程强调在编码时使用 Git。Git 教程除外。

在`github.com`注册，开始推码。GitHub 有一些非常好的指南，可以帮助你设置好一切，让你开始进入分布式版本控制的奇妙世界。

# **了解你的终端**

在我开始学习如何编码之前，我已经在使用 Linux 了，所以我对 Bash 很熟悉，但并不擅长。我错误地认为，随着我的深入，我可以学到更多关于 Bash 的知识。这使得事情变得比原本应该的更加困难。

最终，我花时间阅读了适当的 Linux 文档、指南和教程。结果我不知道很多基本的命令。我尤其不知道现在让我的生活变得更容易的所有酷的技巧和提示。

无论你使用的是 Linux、OS X 还是 Windows，你都将使用终端。试着开始学习你在那里能做的一切。

# **总之**

自学编程是我做过的最难的事情。每次遇到路障，我都会质疑自己所做的一切。这真的值得我浪费时间吗？我真的能成功吗？

每天都有新的东西要学，有时你必须修正一些别人从未见过的东西。

有一件事我一直做得很好，那就是不放弃。即使感觉自己什么都不知道，我也坚持这个信念。

如果你刚刚开始你的旅程，我祝你好运。对于那些坚持下去并经历艰难困苦的人，我表示理解。坚持下去。
