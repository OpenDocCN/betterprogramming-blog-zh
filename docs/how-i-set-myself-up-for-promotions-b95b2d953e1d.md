我相信聪明工作而不是努力工作。有计划的聪明工作比没有计划的努力工作回报更多。

晋升的路线图并不总是很明确。我工作过的每家公司都有不同的晋升标准。

以我的经验来看，我使用的方法似乎增加了我获得晋升的机会。它对我有效，但不一定对每个人都有效。当然，没有什么是万无一失的，所以要有所保留。

# 让重要的人看到我

在我之前的每一个职位中，我都必须向某人汇报。这意味着我的工作并不总是得到认可。我不在乎获得荣誉。

我关心的是我的报酬，以反映我的努力，因为我们在谈论晋升。

我开始在一家中型公司工作，做初级工程师。在我工作的头两个月，我尽可能地了解了公司的一切，以及我的团队一整年的交付路线图。

这让我能够积极主动地了解我需要提高或完全发展哪些技能。在接下来的四个月里，我开始得到实际的工作，并确保我的表现在最坏的情况下也令人满意。

我的努力工作没有被我的经理忽视。我知道这一点，因为在我们偶尔的闲聊中，我的经理的经理不止一次问我在做什么。

我需要他对我的工作有更多的了解，因为他有最终决定权。我确信他有工具可以让他看到每个人的团队表现。但是我发现当我能把一张脸和一个名字联系起来的时候会更容易。

我把我的计划告诉了我的经理，在接下来的几个月里，我用任何我认为相关的电子邮件抄送给他和其他几位高层。我的经理对此完全没有异议。

当我需要一个快速的回答时，他会在我的经理有机会之前回复我，如果他有空的话。我的失败、成功和伤痛都被直接传播给了重要的人。

# 实现我的承诺

我开始从事一些有趣的前端和后端短期项目。我甚至学到了我们正在开发的产品的商业方面。

我注意到，有时候，工程师会过度承诺，因为他们希望被认为是勤奋的。我也看到一些工程师解释为什么他们不能兑现他们的承诺。

当我加入我的团队时，我试图找出我的技能与我的队友相比如何。我通过平均最资深和初级工程师完成一项任务所需的努力量来实现这一点，然后确保我的技能组合更接近资深工程师。

如果我的技能与高年级学生相差甚远，我会努力提高自己，让自己更接近高绩效的一端。这种方法让我能够更准确地估计交付任务所涉及的工作量。

从来没有，也永远不会有防止人们因不可预见的情况而整体落后的灵丹妙药。

然而，我确实尽了最大努力来防止它。采取这些方法让我变得更加可靠，我的经理知道，到最后关头，他可以依靠我。

# 在需要帮助的地方伸出援助之手

十一个月后，我们努力工作的产品准备好了。我们举行了盛大的发布会，一切进展顺利。接下来的几个星期，我在各处实现了一些小的改进。

我认为这是我了解产品核心功能的好时机。我和我的经理讨论了将我 25%的时间分配给另一个需要我这种技能的团队的可能性。

我的经理非常重视团队合作，所以这对我很有利。三个月后，接下来的四个月，我的时间 100%分配给了团队。我学到了很多关于后端产品的知识。

# 如果有机会，带头发起一项倡议

在我与另一个团队共事期间，项目经理(PM)向我提出了一个关于工具的想法。我自愿承担这个项目。我很幸运地得到了另一个团队成员的加入。

我领导了从设计到完成的整个过程。这个工具非常成功，项目经理和我的经理都对我所做的非常满意。按照约定，四个月后我回到了我的团队。

回到团队大约三个月后，我的经理把我叫到他的办公室，告诉我他一直在关注我，我的努力工作和奉献精神没有被忽视。

我不得不再等一个月，但最终，我被提升为中层。我很高兴我的努力得到了回报。

每个公司都有自己的发展誓言或路线图。找出你工作的公司的核心价值观，然后举例说明这些价值观。
