# 5 个你可以马上使用的代码审查技巧

> 原文：<https://betterprogramming.pub/5-code-review-tips-that-you-can-use-right-away-5e553d608a0d>

## 让我们为评审者和被评审者丰富这个过程

![](img/608ce25eb18edd67bd36f75550d27ad4.png)

蒂姆·莫斯霍尔德在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片。

代码审查是任何开发生命周期的一个重要部分，因为它给你的团队一个机会来确保需求已经被拉请求(PR)所满足，并且代码是高质量的。

这也是每个人学习和提高的绝佳机会，无论是涉及语言中的新方法还是理解新概念。在今天的文章中，我们将关注代码审查的这一方面。

这些年来，我从同事那里学到了很多，其中很多来自于代码审查。我个人喜欢审查代码，多年来，我积累了一些审查他人代码时使用的技术。我将在这篇文章中分享它们。

# 1.在你的解释中要详细

这听起来像是一个显而易见的问题，但是我已经记不清有多少次我看到了类似“我们可以在这里使用 DI 吗？”或者“这应该是一个值对象。”

你不应该假设阅读你评论的人熟悉某个术语或概念。这并不意味着你必须为你留下的每一条评论写三段。通常，一个解释你所谈论的资源的链接是一个很好的起点。

请记住，不仅仅是你和提出公关的人会阅读你的评论。最有可能的是，你团队的其他成员，也许将来其他团队的人会读到你的评论。通过留下详细的解释，你可以帮助将来的任何人阅读你的评论并学到新的东西。

# 2.阅读描述

我花了很长时间养成在深入研究代码之前阅读 PR 描述的习惯，但这是重要的第一步，因为它让你在开始审查之前获得尽可能多的背景信息。同样，如果描述中有相关的故事(例如 JIRA 事件)，如果你不熟悉正在发生的变化，也可以阅读这些故事。

虽然留下一个由描述回答的评论并不是世界末日，但它最终会浪费你和公关负责人的时间。

# 3.解释你的推理

当你写下每一条评论时，试着思考你是否能回答以下问题:“为什么做出这样的改变是有价值的？”如果你不能回答这个问题，很可能你没有一个有效的理由来提出这个建议。

例如，如果你的建议是使用一个[单例](https://sourcemaking.com/design_patterns/singleton)，那么合理化这个建议。如果你，作为一个组织，认为[蛇案](https://en.wikipedia.org/wiki/Snake_case)比[骆驼案](https://en.wikipedia.org/wiki/Camel_case#:~:text=Camel%20case%20(sometimes%20stylized%20as,word%20starting%20with%20either%20case.)更好，你应该链接到做出这个决定的地方，提供做出这个决定的背景。

# 4.提问而不是陈述

代码审查过程应该感觉像一次对话，而不是一次考试。通过把你的评论作为问题提出来，这开启了对话的能力。

不要说“我们应该在这里使用循环”，试着把它表述为“使用循环会更好吗？”

这种措辞给工程师一个思考问题的机会。也许他们已经有了一个很好的理由决定不使用它！如果你把它表述为一种陈述，他们更有可能只是离开并做出调整，而不是真正思考它，这使得他们不太可能真正学到东西。

它也不那么强势，更具协作性。就影响力而言，是推而不是拉。当涉及到你的团队时，你通常希望把他们拉向你，而不是把他们推开。

> “问题是大多数人都把推送作为他们的默认风格，不管这是不是正确的选择。就像北风一样，他们的劝说可能会无效。更糟糕的是，你可能会因为独裁、不愿意听取他人意见，或者更糟糕的是，有点恃强凌弱而声名狼藉。”— [中小型企业杂志 Ally Yates](https://www.smeweb.com/2017/08/03/broadening-influence-difference-push-pull/?cli_action=1613581604.642)

推动有它的位置，但是在代码审查的环境中——通常代码是非常主观的——很少有一种正确的方式来看待事情。

# 5.给予赞美和肯定

在我的经历中，在简历上除了问题或建议之外什么也看不到，这是非常罕见的。注入一些积极性是目前为止你能做的最好的事情之一！有几种不同的方法可以做到这一点:

*   它可以简单到只有一行字“太棒了！”或者你选择的庆祝表情符号(我个人最喜欢的是🎉)来认可 PR 中写得好的部分。
*   如果你在公关上签字，“干得好”或“出色的公关”可能会让任何人微笑！
*   当你打开你的一份简历时，看到五到十条评论会让任何人感到沮丧，所以用类似“这看起来真的很好——只是一些小建议”这样的话来鼓励他们当你留下很多评论的时候。
