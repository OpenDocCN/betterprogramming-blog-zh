现在的趋势似乎是让大学生和那些试图进入软件行业的人先涉足 web 开发。这个方向没有错——网络开发人员有很多工作和机会。大多数新的工作列表都在寻找网络开发人员。

说了这么多，我并不认为先学 web 开发一定更好。我的观点是，使用移动开发作为你的主要知识库，而不是全栈 web 是一个更好的途径。

# 移动开发有了更清晰的学习路径

如果你所学的只是编程入门课程，或者你刚刚进入这个领域，那么你从哪里开始 web 开发呢？

“传统的”起点是 LAMP(如果您是 web 开发新手，您必须先学习所有这些栈首字母缩写词)。目前，最流行的堆栈是平均或 MERN。但也有 LEMP，Django，Ruby on Rails，以及许多其他的。学习中的部分困难是弄清楚从哪里开始——这里有很多地方可以开始。

即使当你选择了一个学习路径，堆栈的每个部分，在高层次上，都与其他部分非常不同。一旦你学会了第一部分，接下来你会学到一些不同的东西。虽然这种语言可以翻译(这也是为什么使用 MERN 堆栈很受欢迎的部分原因)，但学习其他部分仍然是一个艰难的过程。

我们来对比一下 iOS 和 Android 的开发。

如果你想学 iOS，可以去学 Swift(一种相对简单的基础级别的语言)和 UIKit 或者 SwiftUI。虽然学习 SwiftUI 和 Swift 并不一定比学习 MERN 更容易，但路径更清晰，我觉得学习更简单。

如果你想学习 Android，你只需要下载 Android Studio，打开你选择的 Kotlin 或者 Java，开始学习 Android 框架。作为一个喜欢 iOS 开发胜过 Android 的人，我认为学习 Android 比学习 iOS 更容易。因此，学习 Android 将是一条非常简单的道路，如果你以前有过 Java 的经验，这将是一条相对容易的道路。

# 找工作会更容易

虽然全栈、后端和前端工作肯定有更多的空缺，但有理由认为移动开发求职“更容易”

移动角色的应用明显较少。Web 开发最近变得特别受欢迎，越来越多的训练营教授全栈开发，大学里的 IT 课程将现代 web 开发作为一门学科来教授。如果你看看 2019 年[栈溢出开发者调查](https://insights.stackoverflow.com/survey/2019)，你可以看到 18%的受访者认为自己是移动开发者，相比之下，全栈和后端的比例为 50%，前端为 33%。是的，从事 web 开发的人比从事移动开发的人多，但是如果你想进入这个行业或者刚刚起步，移动开发人员仍然有很多空缺的职位。

在申请移动角色时，满足基本要求要容易得多。对于 web 开发角色，应用程序可能会因堆栈和语言的不同而有不同的要求——因此，如果你知道一个不同的应用程序，你可能会觉得自己是一个不太合格的申请人，他们可能会忽略你。对于移动，大多数角色只需要了解 iOS 角色的 Swift/Objective-C 和 UIKit，以及 Android 角色的 Java/Kotlin 和 AndroidX。

# 技术堆栈的变化不那么频繁

Objective-C 是在 1984 年为史蒂夫·乔布斯的中介公司 NeXT 创建的，至今仍在使用。`NSString`，`NS`代表 NeXTStep，仍然是 iOS 操作系统和 iOS 作业的基础。Google 和大多数其他大公司仍然主要使用 Objective-C。因此，如果您十年前学习了 Objective-C，该技能仍然有效(尽管从那以后添加了一些像 ARC 这样的附加功能)。

类似地，Android 是建立在 Java 之上的，大多数大公司仍然在寻找具有这种技能的开发人员，因为向 Kotlin 的过渡——尽管是无痛的——才刚刚开始。因此，就像 Objective-C 一样，如果你十年前在 CS 课上学习了 Java，那项技能仍然是 Android 开发人员技能集的重要组成部分。

这与 web 开发形成了鲜明的对比，web 开发是一个快速变化的行业。web 开发技术正在不断改进，以创建更快、更容易开发的更好的框架。虽然这对用户(因为他们的网站响应更快、交互性更强)和了解最新堆栈的人来说很好，但如果您必须每隔几年就不断学习新的堆栈，那就不太好了。对于一些天生的学习者来说，这听起来像是一场梦。但对于那些更喜欢停留在某一技能上并加以提炼的人来说，这可能是一场噩梦。

# 从 Web 开发入手不好吗？

绝对不行。如果你觉得 web 开发更有趣或更有回报，那么从 web 开发开始是一个非常好的途径。正如我提到的，web 开发人员有大量的工作机会，而且这个领域似乎还在继续增长。

但如果你想打入 CS 领域，我认为移动可能是一条更受欢迎的路线。我就是这样开始的。

我从高中的 iOS 开发开始，自己开发和发布应用程序。它为我和我认识的所有人创造了奇迹。大一的时候，我去了一所相对低调的学校，在谷歌实习。我能够以学者的身份参加苹果的 WWDC。我可以创业，并和我在 WWDC 认识的朋友一起销售。

移动开发对我来说总是特别有意义和有趣。有了更浅的学习曲线，更少的应用程序竞争，和更休眠的堆栈，我相信这是一个伟大的方式来推动自己进入 CS 的职业生涯。
