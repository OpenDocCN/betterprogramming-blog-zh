## 三个月后，我才刚刚开始热身

![](img/9e7afb0cb198f2cb6dea93b44eb7bb16.png)

彼得·约翰·玛丽韦尔在 [Unsplash](https://unsplash.com/?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

作为一名移动开发人员，今年错过 WWDC 让我感觉像是站着不动，而其他人都拿到了下一步的路线图并带着它出发了。当然，休陪产假是我能想到的最好的借口。

然而，当我回到工作岗位时，我意识到会议已经过去一个月了，我仍然没有赶上进度。

在观察我儿子的过程中，我开始观察会话，并注意到关键的变化、游戏规则的改变和感兴趣的事情(它们并不总是同一件事)。

有了我整理好的清单，下一步就是尝试其中的一些。但是我不只是想通过一个教程，“学习”，然后当我在一个遥远的未来某个时候实际使用它时，不得不再次重复这些步骤。

如果我想学习，我需要改变它，看到它，并通过可见性来让自己负责。我想如果我能写下我所学到的东西，它会更有吸引力，也可以作为回去的参考。

开始一个新的博客似乎比它的价值更大，而且会分散我的注意力。我开始思考在哪里可以放下我的作品。

在[栈溢出](https://stackoverflow.com/)之后，当我使用我的开发者 Google-fu 时，Medium 是下一个最常弹出的站点。我觉得那是个张贴的好地方！实际上，我不知道这个网站是什么。而且，仔细想想，我不知道为什么会有这么多技术文章出现在这里。

经过一些研究，谷歌搜索，和更多的研究，它似乎会是一个很好的适合。另外，合伙人计划提到了钱。为什么不从我的训练中赚点钱呢？我可以到处多花 1.5 美元。

# 停不下来，不会停下来

三个月后，我就停不下来写文章了。更重要的是，我不能停止写代码。

这是我第 20 篇关于媒介的文章。是的，不是所有的都有代码在里面，包括这个。偏见和粉丝的意见也没有被保留。然而，每一篇文章，每一次鼓掌，以及(最重要的)每一个粉丝/追随者都产生了一个重大且不可预见的后果。

虽然我最初开始写文章是作为一种责任的形式，但在媒体上写作也已经成为一种令人难以置信的动力、灵感和创造力的来源。

是的，作为千禧一代，荣誉和成就是受欢迎的。被出版商选中，一篇文章出现在两个以上的精选名单上，成为这些名单上的特色作家，看到故事在世界的另一端被转发，以及出现在开发者文摘上，这些都是极大的(自我)自信提升。

它极大地拓展了我的网络，扩大了我的影响力，增强了我对周围虚拟开发者社区的认识。

# 发现自己在下一次投球中

然而，驱使我写下一篇文章的是寻找下一个要写的主题。

迄今为止，SwiftUI 是我最热门的话题。但是在我需要一个调色盘清洁剂之前，我只能嚼这么久的骨头(支持保罗·哈德森和他的 *100 天的 SwiftUI* ，你是一个骑兵，我的朋友)。

我认为最好的方法是在一个[特雷罗](https://trello.com)板上列出我的兴趣，然后把这个列表放在手边，这样当我遇到事情的时候就可以添加进去。

结果对我来说有了全新的意义。它成为我的软件开发人员个性/身份的最清晰的反映。对于在一个有着难以置信的广泛和复杂的关注领域和学科的行业工作的人来说，这是一个深刻的定义和启示。

这并不是说我的兴趣很狭隘，恰恰相反，我的兴趣非常广泛。看到它们被摆放出来，并在一定程度上被优先排序，提醒了我对学习和扩展我所知道的东西的渴望。

# 关于思维的思考

我原本希望写作能让事情留在我的脑海里，结果比我预期的要好。

在过去，我会看视频或阅读文章，只是为了继续下一件事，而文章迫使我思考，消化材料，甚至从中产生一些东西。

事实证明，额外投入少量时间和精力会更有回报。

没错，有一种挥之不去的恐惧，那就是如果我试图在我的一篇文章中造假，你们会在评论中指责我。我知道互联网是无情的(但不是你，你太棒了！).

然而，这并没有阻止我去努力，我认为这是另一层责任。要么花点心思，要么就别费心了。

因此，思考也帮助我理清了我感兴趣的东西和我真正喜欢的东西。如果我发现我开始厌倦我正在研究的话题，也许它不适合我。

兼职写文章让我挑挑拣拣，实际上要求我设置一些条件。我没有太多的时间来写这些文章，所以为什么要把时间浪费在对我来说很无聊的事情上呢？

# 重燃我的代码

我是一名软件工程师，这意味着我已经以编码为生。我热爱我的工作，并且很幸运能在我所在的领域里。

然而，自从我大学毕业后，我的编码时间变得越来越局限于办公时间。我担心，如果我在业余时间写太多代码，我会精疲力尽并厌倦它，我在其他人身上看到过这种情况。

同时，我也想以自己的方式做自己喜欢的事情。我有一种强烈的愿望，那就是不断学习、构建和跟上最新的东西。总之，我想玩得开心。

现在，我已经组织了我的兴趣，研究了它们，思考了它们，并找到了一个为它们工作的“奖励”系统，这重新点燃了编码作为一种休闲的热情。

从字面上看，我的 GitHub 回购增加了一倍，本地项目增加了近十倍。事实上，我在那些项目中有许多分支，在那里我测试了不同的模式、实现和我很想测试的用例。

结果是我现在知道的更多了。更多的技术、更多的模式、更多的尝试经验，以及我的开发人员工具箱中的更多技巧和窍门。

这不仅体现在我兼职做的项目中，也体现在我的工作中。我的信心增加了，我能够提出更多的建议和想法。

我也变得更加(建设性地)批评代码的整洁和过程。写文章对写代码有很大的影响，而且是越来越好。

# 展望未来

在媒介上写作的好处是不可预见的，但肯定是受欢迎的。我的计划是继续前进，不管最终会把我带到哪里。

我写这些不一定是为了让你知道，而是因为我认为反思这个“第一季度”很重要。如果你熟悉敏捷/Scrum，可以把它看作是一次回顾。

但是当我这么做的时候，如果你是一个寻求成长的开发人员，我会鼓励你考虑尝试写作，无论是在 Medium 还是(策展人，请原谅)其他更适合你的地方。

不，它可能不适合所有人。不过，除非你尝试一下，否则你不会确定。
