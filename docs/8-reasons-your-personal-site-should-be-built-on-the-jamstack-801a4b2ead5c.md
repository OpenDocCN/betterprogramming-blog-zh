## 离开 WordPress，拥抱未来

WordPress 占互联网网站的 30%以上。

我们都用过它，不管你是 WordPress.com 的会员还是自己安装。这实际上是一种通行权。

但我相信 2020 年不一定是正确的选择。在我看来，JAMstack 静态站点生成器已经取代了它，成为大多数拥有个人站点的人的最佳选择。

JAM 代表 JavaScript、API 和标记。基本上，您用 JavaScript 为生成器构建规则，使用 API 获取数据(远程或本地)，并编写标记来设计美学。

JAMstack 不是每个字母都是一种特定技术的传统技术堆栈，而是设计架构的转变。现在，您的站点是在部署之前生成的，创建了一个静态站点，而不是每次都必须为每个查看者构建一个静态站点。

这里有八个理由说明为什么你需要搭上 JAM train，把 WordPress 或者你使用的其他网站服务留在后面。

# 1.它是免费的

是的，你没看错——它是免费的。除了注册域名，我没有为我的个人网站支付一分钱。

在 GitHub 上托管我的网站，并通过 Netlify 进行部署，我不再向 WordPress.com、像 GoDaddy 这样的网络主机或像 DigitalOcean 这样的副总裁付费。

# 2.很容易维护

一旦你的网站建立起来，维护通常就像添加、更新或删除降价文件一样简单。此外，通过一些小的配置，您可以使用自动发布更新的工作流。

# 3.它很快

由于你的页面是预先呈现的，这意味着它们是在部署时而不是在用户请求时创建的，你的网站将会非常快——比 WordPress 和任何其他依赖于向服务器请求动态内容的服务要快得多。

# 4.它是搜索引擎友好的

预呈现还带来了一个好处，那就是你的所有内容都可以被机器人和爬虫获取。这会让你在谷歌搜索结果中上升一个或 10 个位置，让你获得更多曝光。

# 5.很安全

服务器、数据库和客户机之间的来回越少越好。每次传输都代表一种安全威胁。预先呈现的特性限制了潜在安全风险的暴露。

# 6.这是一项新兴技术

就像每个人在 WordPress 的青年和青春期都蜂拥而至一样，在 JAMstack 上工作是很酷的。您将了解新的网络设计架构，并成为更有眼光的内容消费者。

# 7.你有选择

因为 JAM 是一种设计理念，而不是一组“复仇者联盟”式的特定技术，所以您可以选择如何实现。无论您使用基于 React 的 GatsbyJS、基于 Vue 的 Gridsome 还是其他，您都可以浏览文档、指南和入门指南来找到合适的匹配。

# 8.社区支持

当你不可避免地遇到挑战时，这些社区非常棒。人们有一个共同的愿望，那就是合作和鼓励采用，这导致了强大的社区。我还要补充一点，因为 JAMstack 是部署文档的理想选择，所以大多数框架都有真正的典范文献。

# 结论

我强烈鼓励任何正在使用 WordPress、Squarespace、Wix 等的人。试用一个基于 JAM 的静态站点生成器。你不需要成为一名开发人员来使用它们——尽管你可能会发现修改工作是如此令人愉快，以至于你最终学到的东西比你预期的要多。

我一直在重新设计我的个人网站，[j-hsu.com](http://www.j-hsu.com)，结合使用 Gridsome、Tailwind CSS 和 Markdown。我能够在一天之内重建我的网站，而不是在 WordPress 中需要一整个星期。另外，它可以免费运行——从 GitHub 部署到 Netlify。
