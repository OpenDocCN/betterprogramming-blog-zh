# 我们为什么不多谈谈学习呢？

我们这个行业不太讲学习。这很奇怪，因为学习是我们最重要的技能。没有它我们不可能成功。科技是巨大的。我们需要知道的事情很多，而且都在不断变化。每天都有新的框架、新的基础设施、新的构建工具、新的架构热点。突然，每个人的名字都落在某项技术上，感觉就像“我错过了一个公共服务公告吗？那东西哪来的？”

与软件保持同步意味着我们在不断地学习。没有它我们无法工作。但是我们不怎么谈论它。

# **我们的 Instagram 生活**

这是个问题。当我们不解释我们是如何知道所有我们知道的事情时，我们并不承认是*用* *工作*才到达那里的。对于业内的新人来说，这会产生不切实际的期望。如果其他人似乎都神奇地知道一堆东西，而你却不知道，这会让人觉得你不够好。感觉你不应该在这里。

这是 Instagram 的问题，每个人的生活看起来都很完美。我们看不到隐藏在视线之外的凌乱，20 张照片在正确出来之前就被丢弃了。我们看到人们使用他们的知识，但我们没有展示正在获得的知识。我们看不到令人难以置信的不直观的文档，错误的路径，糟糕的第一个原型，花了几个星期才实现的想法。

我们不会告诉新人学习有时会很慢、令人沮丧、很困难，那只是…正常的。如果学习是第一技能，坚持可能是第二。有些事情只是需要时间。

# 所以我们应该大声学习

在我去年的一次演讲中，我说过我们有高级工程师的原因是有人会问一些愚蠢的问题:

*   “等等，那个词是什么意思？”
*   “你能把这个解释成我五岁吗？”
*   “我们为什么要这样做？”
*   “我们在努力解决什么问题？”
*   “这样的东西不是已经存在了吗？”

如果我确信大多数人不知道答案，我有时会在已经知道答案的情况下在会议上提问。学长这么问就没那么吓人了。

出于同样的原因，我尝试公开学习:我在工作日历中划出学习时间，并清楚自己想从中获得什么。“到这两个小时结束时，我会知道足够多的 websockets 来制作一个可怕的聊天应用程序。”我做到了。太可怕了。这是最好的。

# 学会学习

学习本身就是一种技能——当你做的时候，当你发现什么对你有用的时候，你会变得更好。

例如，我曾经花时间试图找到“最好”的起点，试图选择我正在学习的这个新主题的最重要的方面。我已经得出结论(对我来说！)从阅读任意一篇文章开始和阅读维基百科上关于这个主题的页面一样有效，所以我只是选择一些我不理解的东西去阅读。重复一遍。如果我持续这样做几个小时，我会对重要的事情有一个很好的印象。

如果这是一个我觉得无聊或困难的话题，我知道我可以通过立即使用我刚刚读过的任何东西来使它更有吸引力(例如，通过写一篇博客帖子或一个总结我刚刚学到的东西的内部文档)。我的大脑有无限的耐心阅读我想写的或想教给别人的东西。编码练习和小项目有同样的效果:我可以永远学习理论，但是没有什么比在项目中使用它更巩固知识了。

# 这都是可以学习的

我认为科技领域最重要的认识是一切都是可以学习的。绝对是一切。有些东西很难，需要知道的东西很多，但没有一样是神奇的。最坏的情况，只是需要时间。

如果你是一个大三的人，如果感觉有很多东西你不知道，不要觉得不好。这很正常。有很多东西要知道！熟悉感会随着时间而来…直到你进入一个新的领域，然后你会再次回到不知道自己在做什么的状态。不断学习如何学习，新概念会一次比一次来得快。

如果你是一个高年级的人，请表明你也在学习。分享资源，提出问题，告诉人们你刚刚发现的任何有趣的事情。如果我们承认头号技能是学习——而且这需要大量时间，我们会让我们的行业变得更好。
