## 利用这些技巧来处理倦怠和保持生产力

软件开发总是被不健康的习惯所玷污——倦怠就是其中之一。大多数软件开发人员都经历过精疲力竭。

它有一些不健康的影响，如果不加以控制，会让你偏离你想要的目标。

# **什么是倦怠？**

倦怠主要是由于长时间暴露于某事而导致的疲惫。有没有过这样的情况，无论你想做什么，对你来说都没有任何意义？当键盘和电脑屏幕变得毫无意义的时候？当你觉得一切都糟透了，你的代码对你来说就像垃圾一样的时候？

嗯，很可能这就是精疲力竭的感觉。这清楚地表明你需要离开电脑屏幕一段时间。大多数时候，倦怠会伴随着消极情绪而来。

在这种情况下，效率低下可能会导致其他不健康的习惯，比如拖延症。把这想象成一个坏水果，它最终会破坏所有其他看起来新鲜的水果。倦怠之后可能会伴随着其他不健康的习惯。

# **如何应对倦怠**

我们不能完全避免倦怠的发生，但是我们可以利用一些技巧来处理这种不健康的开发习惯。当我们没有实现健康的开发习惯时，我们可能会将我们的生产力置于危险之中。

## **找一个编程以外的爱好**

爱好基本上是驱动你对某一特定事物感兴趣的东西。大多数程序员总是对机器和学习周围新事物的好奇心感到兴奋。这是非常迷人的，直到你发现自己一路上不知所措。

在编程之外有一个爱好并尝试做一些其他事情被认为是一种健康的习惯。这不一定是能让你赚钱的事情，但它至少应该有助于将你的注意力从编程上移开，并打破你的正常循环。同样有益的是，它将你的注意力转移到其他事情上，而不会让你陷入一个可能滋生不健康习惯的想法中。

爱好可能从足球、曲棍球和橄榄球等游戏到其他基于社区的项目。从事编程之外的爱好可能是一个巨大的优势，因为它可以把你介绍给其他人，并让你在这个过程中获得其他技能。为了让它变得更容易，只要找到你喜欢做的事情，并且大部分不涉及在电脑上。

## **考虑休息一下**

休息可以是对抗疲劳的一种好方法。考虑一个例子，你的大脑长时间专注于一个特定的任务。因此，你很可能会感到无聊。

无聊之余，倦怠总会悄悄袭来。在做非常重要的事情时，休息一会儿——尤其是当你想长时间保持高效的时候。全天候工作不利于健康的生活方式或成功的编程环境。

短暂的休息会让你的注意力从一个特定的任务上移开，让你的注意力从你正在执行的任务上移开一点。这是其他生产力机制用来帮助你保持生产力并长时间完成更多工作的方法。番茄工作法利用短暂的休息来确保你在更长的时间里保持高效。这个机制已经被很多人证明是非常有效的。

这是一个健康的习惯，不仅可以防止精疲力竭，还可以确保我们保持高效。

# **最终想法**

考虑实现这些基本的指导方针，试图阻止这种不健康的开发习惯。短暂的休息和在编程之外寻找一个爱好是你可以采纳和实施的一些更好的习惯，以避免与精疲力尽相关的风险。
