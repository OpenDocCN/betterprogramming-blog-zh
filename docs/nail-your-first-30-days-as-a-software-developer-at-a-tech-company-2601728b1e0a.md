## 如何通过了解和明确你的角色期望，成为一名优秀的软件开发人员

![](img/fa43495d6e899de4ece2eca13f0679ea.png)

照片由[西格蒙德](https://unsplash.com/@sigmund?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

最近，微软对 31 个国家的 3 万名工人进行了关于工作的调查，根据调查结果，全球有 41%的工人正在考虑辞去目前的工作。不需要天才就能看出科技公司将是受此影响最大的公司，因为科技行业是全球疫情中增长最快的行业之一。因此，如果你是一名[软件开发人员](https://eisabainyo.net/weblog/category/articles/software-engineer-career-growth/)，你可能会在未来 12-18 个月内加入一家新公司。

换一份新工作可能会很有压力，但这也是一件好事。它让你走出舒适区，伸展你的身体。它让你看到并思考你在以前的工作中通常不会考虑两次的事情，例如对公司的回购做出承诺，看到你的工作在生产中投入使用，加入你的团队，并请人审查你的拉动请求。当你在工作场所过于舒适，并且已经习惯了流程和工具时，你倾向于想当然，并以自动驾驶模式运行。改变你的环境，无论是在同一家公司的一个新团队还是在另一家公司的一份新工作，都会让你更加了解你自己以及你作为一名开发人员每天都在做什么。

除了一般的新员工问题，比如*你应该什么时候开始工作*？*第一周你应该有多友好？作为一名新的开发人员，本文旨在为您提供所有与软件开发相关的问题的答案，这些问题您想问其他开发人员或您的开发经理，但又不敢问。*

![](img/b2be2ab563039536292bed52e1fa9ab4.png)

[Bonnie Kittle](https://unsplash.com/@bonniekdesign?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

我指的是这样的问题:

> 何时应该将代码添加到公司回购协议中？
> 
> 你什么时候能看到你的作品投入生产？
> 
> 什么时候应该分享你对特定设计模式的看法？

作为一名雇佣了许多开发人员和技术领导者的工程经理，让我与您分享一些最常想到的(有时会被问到的)问题以及我对这些问题的回答。我希望它们能帮助你，作为一个新的开发人员，在一份新的工作中，留下最好的第一印象，同时，让每个人都知道真实的你和你能提供什么。

我将为每个问题提供两个答案:一个针对初创公司和小公司，一个针对大中型企业。这是因为，从我作为一名技术领导者在科技行业的初创公司和财富 500 强科技公司工作了近 20 年的经历来看，我知道公司的规模和环境在预期中起着很大的作用。此外，请注意，我的答案只是指导方针，因为您可能会遇到一些细微差别。无论如何，我希望它们有助于给你一个视角，让你知道作为一家公司新聘用的软件工程师，你需要以什么为目标。

# 第一部分:这都是关于代码的

![](img/c35cbd08092ea231b2991b501be00682.png)

詹姆斯·哈里逊在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

**你应该什么时候在你的机器上设置你的开发实例？**

小型创业公司:第一天

大型企业:前两周内

你应该在什么时候做出第一次承诺？

小型创业:前三天内

大型企业:前两周

你预计什么时候能看到你的作品投入生产？

小型创业:前十天内

大企业:看情况。第一个月之后的下一个发布周期。

**您应该何时了解最佳实践和编码标准？**

小型创业:第一周内

大型企业:前两周内

**什么时候你应该理解你将要贡献代码的系统的架构？**

小型创业:第一周

大型企业:第一个月

**什么时候你应该独立地从团队的待办事项中挑选并完成一项任务？**

小型创业:在你的第一次冲刺中。

大型企业:第一个月内。

不管你是初级软件开发人员还是高级软件开发人员，你的技术主管或工程经理应该有一些任务，你应该能够根据你的资历独立完成。确保你与他们交流这一点。

**你应该什么时候开始参与故事评估？**

小型创业公司:在你第一次冲刺之后。

大型企业:经过两次冲刺*。

*   一次冲刺=两周。

# 第二部分:关于过程

![](img/ef4ef63e12623c4eab028f66fed86dde.png)

**你应该什么时候开始成为团队服务的随叫随到人员(尤其是如果公司实行 YBIYRI 的话)？**

小创业:第一个月内。

大型企业:在您跟踪了随叫随到的轮班之后。通常是几个月后。

**什么时候你应该举手推动团队活动，例如举办复古活动？**

小型创业:在你至少看过两次它是怎么做的之后。

大型企业:同上。

**在什么情况下，作为他人拉动式请求的批准者，你应该感到自在？**

小型创业:15 天内

大型企业:60 天内

**您应该何时了解构建和发布流程？**

小型创业:第一周内

大型企业:三周内

# 保留您自己的入职清单

![](img/11f1f3e068945e4c075b6b9bca8aaff9.png)

照片由 [J. Kelly Brito](https://unsplash.com/@heykellybrito?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

您的经理可能会为您提供入职文档，但保留您自己的文档也很重要，以确保您跟踪自己的进展，并在最初几周达到软件开发人员角色的最低期望。

除了编码和技术任务，您还应该将以下重要项目添加到您的清单中:

*   加入必要的 Slack，微软团队，或任何其他沟通渠道
*   要求被列入必要的邮件列表和日历邀请
*   与你的经理安排一次常规的一对一会谈
*   了解社会团体，与同事建立联系
*   了解对你的角色的期望，以及从你的经理的角度来看成功是什么样的

这里有一份清单，可以帮助你在新公司做软件开发的头 30 天里站稳脚跟。

```
⬜️ Dev instance set up
⬜️ First commit made
⬜️ Code live in production
⬜️ Drove a team activity
⬜ ️Actively contributed to estimation
⬜ ️Have been on-call
⬜ ️Independently completed a task from the backlog
⬜ ️Shared your career aspirations with manager
⬜ ️Approved a Pull Request
⬜ ️Understood build and release process
⬜ ️Participated in a Guild
⬜ ️Contributed to a technical discussion
⬜ ️Understood the architecture of the system(s)
⬜ ️Learned about the product
⬜ ️Learned about best practices and coding standards
⬜ ️Gained operational knowledge
--------
⬜ ️Joined important communication channels 
⬜ ️Included on necessary mailing lists and calendar invites ️
⬜ ️Set up a regular 1:1 with manager 
⬜ ️Made social connections with likeminded groups
⬜ ️Understood the expectation of your role
```

# 留下良好的第一印象

![](img/857a3fc22bdb08f0dee671df84e827f0.png)

[品牌&人](https://unsplash.com/@brandsandpeople?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照

我们都知道这个:**第一印象很重要**。你对工程经理和其他开发人员的第一印象始于你加入公司的那一刻，这种印象会持续很长时间，有时会持续你在公司的整个时间。因此，通过了解作为一名软件开发人员对你的期望并跟踪你的进展，把标准定得高一些。
