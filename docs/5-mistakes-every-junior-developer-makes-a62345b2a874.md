# 每个初级开发人员都会犯的 5 个错误

> 原文：<https://betterprogramming.pub/5-mistakes-every-junior-developer-makes-a62345b2a874>

## 以及如何避免它们

作为一名软件开发人员，走出去找到你的第一份工作可能会令人望而生畏。根据我的经验，我想分享一些初级开发人员常犯的错误，让你不那么焦虑，更有准备。

# 1.问错误的问题

![](img/28916fbd65546be5fec6ef17979ddbe3.png)

图片由 [Arek Socha](https://pixabay.com/users/qimono-1962238/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=2492009) 从 [Pixabay](https://pixabay.com/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=2492009) 拍摄。

这不一定意味着你问了错误的问题，而是你没有问正确的问题。我再怎么强调在解决问题之前能够明确你的问题是多么的重要。一个好问题已经有了 50%的答案，当你掌握了这个技巧，没有什么能阻止你。甚至还有一种基于提问的调试策略叫做[橡皮鸭调试](https://en.wikipedia.org/wiki/Rubber_duck_debugging)。

问题的另一个问题是初级软件开发人员倾向于问很多问题。这听起来不是一件坏事，但是一个开发者应该有一定程度的独立性和责任感。如果这是一个小问题，比如命名一个变量或决定使用哪个 CSS 属性，最好做出判断并承担责任，因为这是真正学习的唯一方法。

# 2.糟糕的时间管理

![](img/a700e67a74cb94268812576c0424ef2d.png)

图片由[皮特·林弗斯](https://pixabay.com/users/TheDigitalArtist-202249/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=4099971)从[皮克斯拜](https://pixabay.com/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=4099971)拍摄。

初级开发人员的另一个问题是糟糕的时间管理技能。在一个时间紧迫的项目中，决定不做什么比做什么更重要。有些错误比其他的更重要，如果你花太多时间在一个小的 UI 问题上，你就不会花时间让你的应用程序实际工作。

当然，每个项目都有不同的需求和目标。一个好的开发人员应该能够发现它们，并优化他的时间以获得最大的产出。面对 bug 或重要架构决策的一个好方法是，当你注意到你在一个问题上花了太多时间，就停止工作，转而做别的事情。然后第二天再来看这个问题，一个解决方案就会显现出来。这被称为“睡上一觉”，通常在解决难题时会有所帮助。

# 3.学习不够

![](img/80173e5a0b7dc289b8f104de1fcfa1ab.png)

图片来自[皮克斯拜](https://pixabay.com/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=1052010)的[达留什·桑科夫斯基](https://pixabay.com/users/DariuszSankowski-1441456/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=1052010)。

这是过度自信的开发人员最常遇到的问题，但不仅限于他们。虽然您可以从 HTML 和 React 的基本知识开始工作，但是只要您是软件开发人员，您就应该期望学习新的东西。现在不仅有每个开发人员都应该有的设计模式、算法和库，而且每天都有新的模式、算法和库出现。

如果你想在劳动力市场上保持你的价值，你应该不断地自我教育，包括但不限于:

*   阅读书籍/文章/博客/新闻。
*   用副业尝试新事物。
*   为 OSS 做贡献，或者至少通读你最喜欢的库的源代码。
*   参加会议，与你的开发伙伴交流。

# 4.学习太多

![](img/3d0fa7d287c46dfd4b9fa2e8b32b43db.png)

图片来自 [Pixabay](https://pixabay.com/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=1655783) 的 [Nino Carè](https://pixabay.com/users/ninocare-3266770/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=1655783) 。

这似乎与我最后的论点相矛盾，但请听我说完。如我所说，每天都有新的东西出现。但这并不意味着你必须学习所有的东西。虽然您可以掌握 React、Vue 或 Angular，但很难掌握所有这些技术并成为专业人员。选择不学什么和选择学什么一样重要。

就你的市场价值而言，专业化是强大的。你的专业越窄，竞争越少，需求越多。我明白在职业生涯的早期选择你的专业是困难和可怕的，但这是不可避免的。

# 5.逃避责任

![](img/be8c251540122a8db4c7615b30141840.png)

来自 [Pixabay](https://pixabay.com/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=690084) 的[免费图片](https://pixabay.com/photos/?utm_source=link-attribution&amp;utm_medium=referral&amp;utm_campaign=image&amp;utm_content=690084)图片。

作为一个初级开发人员，没有人对你有太大的期望。你的主管明白你需要时间来成长和学习，他们不会(希望)给你关键的任务，如果你失败了，这些任务会毁了产品。但是你应该开始对小问题和代码所有权负责。

当高级开发人员和项目经理不需要微观管理初级开发人员，并且这些初级开发人员有一定程度的独立性时，他们会喜欢这样。除非你愿意承担风险并全身心地投入，否则你永远也不会学到东西并变得有责任感。
