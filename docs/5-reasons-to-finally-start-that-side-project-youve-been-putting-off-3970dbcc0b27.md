他们说如果你热爱你所做的事情，你这辈子一天也不会工作。我倾向于同意那种说法。作为一名软件工程师，我“工作”的日子并不多。

我想这就是我的天性。我喜欢解决问题。我热爱学习。弄脏我的手。创造。

写代码对我的生活产生了如此深远的影响，以至于我下班后还在家里继续写。我早起，写一点代码，去上班，写很多代码，回家，阅读关于编码的书籍。

最重要的是，我下班后的编码让我白天的工作更加愉快。这是为什么呢？

副业激发了我的创作热情。它们增强了我的学习欲望。如果你是任何类型的工程师，我知道你知道这种感觉。

那么，有什么特别的副业是如此…*撩人*？

# 难以置信的学习机会

工程师天生喜欢学习。我不知道谁会跟你说不同的话。弄清楚*为什么*和*事情如何*工作只是我们做的事情。兼职项目给你机会学习新的模式、编程语言和概念，否则你会忽略它们。理论只是理论，直到你付诸实践。

我的观点是，直到你做了一件事，你才会真正理解它。你的兼职项目给你提供了一个机会来增强你对自己领域的理解。你可以直接接触新事物，并亲自解决所有问题。

# 你的创造力得到发挥

副业的好处就在于没有约束！你可以随心所欲地建造任何东西。你想学习一门新的编程语言吗？继续构建 POC。你想不想兼职赚点外快？无论如何，试着建立一个电子商务网站。

把已知的问题拿出来，看看你是否能想出解决它们的方法。例如，我在一个项目中需要在地图上存储大头针来显示兴趣点。每当你滚动和用任何新的大头针更新地图时，地图会做地理搜索。

这个问题的难点在于我使用的是 NoSQL 的数据库 DynamoDB。如果你对 NoSQL 很熟悉，你可能会知道它不适合搜索。这是一个为查找而设计的数据库。所以我花了一些时间，想出了一个[的方案，解决了我的问题](https://medium.com/better-programming/how-i-built-a-serverless-geo-search-app-with-dynamodb-491879233754)。

有必要这样建吗？没有。但是我想想出一个创造性的方法来解决问题。

# 你有机会承担额外的责任

在日常工作中，你很有可能在一个开发团队中工作。团队的成员有他们自己的一套责任来使团队尽可能强大。

你的副业，你得到所有的责任。你负责设计。你得想出架构。你可以写所有的代码。你猜怎么着？当你发现缺陷时，你就可以维护它。

通过承担大量的责任，你为自己提供了一个机会，看看你想如何推进你的职业生涯。你喜欢这个设计和建筑吗？也许你应该努力成为一名软件架构师。

也许编码更适合你？没问题，首席工程师符合要求。

你最喜欢的部分是不是提出了功能列表，并为先做的事情建立了时间表？也许你天生就是做产品的料。

让自己承担众多的责任会帮助你缩小你真正的热情所在。如果你对每天的工作充满热情，那你就不是在工作，对吗？

# 给你的日常工作带来额外的经验

建立兼职项目组合的一个好处是，你可以把它们作为日常工作的参考资料。他们说，大多数开发人员一生中从未写过一行独特的代码。一切都是复制/粘贴/替换。

虽然这可能有点夸张，但也有一定的道理。想一想——如果你有一些有用的东西，你为什么还要再写一遍呢？重新编写只会增加犯人为错误的机会。

在你的项目中，你可以把代码片段作为例子来展示你的团队正在努力完成的事情。你有帮助你的团队前进的工作代码。或者也许你只是拥有一种新模式的知识和经验。

它甚至可以像越过错误消息的诀窍一样简单。如果你以前遇到过错误并花时间研究和排除故障，与你的同事分享这些知识。他们可以从你的所作所为中获益。你也会从他们从事的副业项目中受益。

# 你可以为开源社区做贡献

如果我感到沮丧，需要恢复对人性的信心，我可以去 GitHub。我知道这听起来很蹩脚，但这是人们帮助他人的完美例子。

成千上万的开发人员花费大量的时间仅仅为了帮助他人而开发软件。他们提供工作软件、参考资料和功能示例来互相帮助。

你的副业是一个回馈社区的机会。你有机会让某人克服他们一直存在的问题。你可以让成百上千的人生活得更轻松。

为开源社区做贡献也有助于激发你工作中的主人翁意识。知道有开发人员在使用你写的代码，你会有一种自豪感。它鼓励你去学习一些东西，并把它做好。

# 结论

我不知道你，但写这篇文章让我想开始另一个兼职项目。有太多的东西需要学习，想法需要实现，经验需要积累。

如果你认为自己太忙了，没有时间做兼职项目，请记住*你有时间做真正重要的事情*。也许今天，你不用花一个小时玩动物穿越，而是花时间浏览一些 AWS 教程或建立一个新的存储库。

你只能受限于你的创造力所能构建的东西，所以走出去，写一些代码，并使之开源！
