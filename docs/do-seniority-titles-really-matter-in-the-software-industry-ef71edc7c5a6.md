## 本叔叔曾经说过，权力越大，责任越大

我记得那一天，我向我在旧金山梅西科技公司的经理递交了两周的辞呈。几分钟后，我们部门的主任打电话给我，约我一对一。

> “你为什么要离开？”他问。“是因为钱吗？”

部分原因是因为赔偿。这也是一个奇怪的现象，许多人已经辞职，并转移到他们的下一个职业生涯。我预感到一些事情将要发生，一些大事件，比如旧金山办公室的裁员。但是，我刚刚大学毕业，想找一份工作，挑战自己的日常。

“这不是钱的问题。是那次经历让我做出了这个决定，”我紧张地说

他告诉我，如果我想加薪，他可以提供或支付对方公司的工资。在那次一对一的谈话中，他给了我一些职业建议，这让我思考为什么大多数人会离开一家公司。

他说如果他发现自己的头衔或薪水有所提升，他会跳槽。然而，在这两者中，头衔的提升胜过薪水的提升。

这让我思考——标题重要吗？其他软件工程师说这个头衔并不重要。重要的是你的责任，对方公司会把你的责任转化为他们的水平。

# 对降级的恐惧

在科技行业跳槽要么带来经济上的好处，要么带来头衔上的好处。然而，这两者并不同时存在。许多初创公司的工程师在被 FAANG 等更大的科技公司聘用时会被降级。例如，一个高级软件工程师可能成为 SWE 2，或者一个非技术型公司的副总裁成为高级工程经理。在大多数情况下，他们不太可能接受这个职位，继续在其他地方面试。

另一方面，他们会获得与前任经理相同的加薪。也许在他们当前公司的晋升不能超过他们在工作中得到的级别。

# 头衔在公司意味着什么

在你考虑头衔是否重要之前，我们必须看看头衔在公司里意味着什么。为什么我们在公司有职称而不是一个级别的都有？

## 责任

该公司利用这一头衔来确定其项目管道中每一层的责任。例如，一个高级软件工程师将有不同于软件工程师的职责。

高级软件工程师是*手艺*的大师。他们知道他们不知道的。他们可以根据涉众的需求独立处理中等复杂问题的所有方面。他们试图利用并与其他工程师合作来推动项目向前发展。

他们的职责可能是在一家大公司中从头到尾设计功能，并精通该公司的技术堆栈。这意味着过程、架构、设计、实现、测试和可持续工程。他们所做项目的一个例子可能是在其系统中加入一个新的支付处理流程。在代码审查期间，他们通常是指导其他工程师，并就如何正确导航项目给予他们指导的人。他们也知道如何处理棘手的情况，即他们的工作受到其他团队的阻碍。

软件工程师的责任是交付正确的特性。通常，他们在交付项目时可能需要一些指导。这可能是一个技术问题，另一个团队的阻止者可能会减慢他们的项目完成速度。他们负责调查分配给他们的问题，并找出如何设计解决方案。然而，有人负责组件之间的 API 或在它们实现之前签字。

## 工资范围

责任越大，回报越高。每个职位都有自己的薪酬范围，可能与不同的级别挂钩。脸书软件工程师的平均年薪为 267，000 美元，而高级软件工程师的平均年薪为 390，000 美元。

## 长辈

在一些公司，资历开始发挥作用。从非技术的角度来看，头衔与资历相关。假设你看了软件工程师和高级软件工程师的工作描述。一个软件工程师需要 0-3 年，一个典型的高级软件工程师至少需要 5 年。一些在短时间内成长起来的高增长创业公司有时会将他们最初的工程师放在比后来加入的工程师更高的职位上。

虽然总的经验年数不一定意味着资历，但平均来说，一个软件工程师通常需要几年的经验才能达到工艺大师的水平。

一个软件工程师在一家公司工作的时间越长，也就越了解这家公司的领域和技术。因此，随着时间的推移，公司会相信他们会履行更高层次的责任，这与他们的头衔增加是一致的。

# 什么时候标题很重要？

各个公司的头衔并不统一。A 公司的高级职称不一定转化为 B 公司的高级职称，因为 B 公司的高级职称与 A 公司的高级职称有不同的职责。

然而，在一些情况下，标题确实很重要。

## 当你想在你的公司获得更多的影响力和发言权时

头衔代表在公司的资历。因此，它也投射了你在当前项目范围内的可信度。

在公司有资历就有被尊重的感觉。其他工程师和利益相关者可能会认真对待你，他们不太可能因为你的头衔而挑战你。

> “对于那些代表性不足的群体来说，头衔在建立可信度方面有着天壤之别。一位在一家中型科技公司工作的首席工程师来自一个代表性不足的群体，他说:“一旦我被提升为高级主管，然后是主管，情况一次比一次好。尽管如此，即使我有首席工程师的头衔，我还是经常被质疑我是否是技术人员。想象一下，如果我没有这个头衔和它所承载的信号，会发生什么。"
> 
> —务实的工程师

拥有一个更大的标题会影响项目当前的范围、预算分配等。例如，一位高级主管可以说服高层关注技术创新，而不是产品创新。

## 当你想换工作的时候

哦，是的，非工程师有时更倾向于看头衔来联系你，因为他们认为你的职责与公司里的相同头衔有些相似。

如果你说了你的软件工程师头衔，招聘人员不会主动向你寻求经理职位。他们会向拥有技术领导或经理头衔的人伸出援手，以证明他们已经获得了这份责任。

有句话是这么说的——跳槽的最佳时机是你刚升职的时候。其中一个原因是头衔的改变相当于责任的改变。如果工程师还没有表现出这种责任感，大多数公司是不会提升他们的。

所以，如果你已经晋升到下一级，你很可能已经“准备好了”，并且一直在做下一级的角色。然而，公司内部的晋升可能不会像去下一个公司的下一个级别那样加薪。其他公司希望在同一级别上聘用工程师，因为他们已经在前一家公司“履行了这一职责”。

# 外卖食品

有两类人主张题目重要不重要。俗话说，权力越大，责任越大。公司会创建一个头衔来区分组织内部的职责。责任更高的工程师在公司眼里更有公信力。在一些公司，头衔往往反映了你的资历。

当你想建立自己的信誉并想换工作时，头衔确实很重要。这个头衔决定了你在公司中的影响力。

为了避免在转换域时被降级，当你有机会的时候，试着在你的公司内转换轨道。除此之外，理解责任和期望可以帮助你走向成功。超额完成任务总是好的。因此，从长远来看，承担太多的责任可能会导致压力和较少的成功。尽管如此，太少的责任可能会让你感到无聊和沮丧。
