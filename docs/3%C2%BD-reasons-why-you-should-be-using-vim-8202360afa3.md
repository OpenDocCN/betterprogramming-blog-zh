是否使用 Vim 是软件工程师之间激烈的争论。这是一个类似于音频行业模拟与数字之争的争论，早在你们中的一些人(和我)之前就有了。

在大多数工程师看来，Vim 是一个旧的、过时的文本编辑器，缺乏先进的功能和现代 IDE 的舒适性。实际上，Vim 不仅仅是一个旧的文本编辑器。

Vim 就像一辆手动挡汽车。

你必须学会如何驾驶它。齿轮必须手动换档，但一旦你习惯了，这种体验会变得更加有益和愉快。你正在*积极地*驾驶:在蜿蜒的道路上行驶，非常巧妙地换挡，全神贯注。你和车连在了一起，它成了你自己的延伸。

这就是 Vim 的样子，这也是它使用起来如此值得的原因。

# 1.不使用鼠标

使用 Vim 时，鼠标毫无用处。你被迫用键盘做一切事情，这将使你成为一个更强大、更高效的开发者。

你的手离开键盘和触摸鼠标的次数越多，你工作的效率就越低。

每一次鼠标移动实际上是一个小型的上下文切换，慢慢地破坏你的整体速度。学会使用键盘快捷键。

Vim 的一个好处是它完全是键盘快捷键。你不能用鼠标。

习惯就好。

# 2.自然发展

默认情况下，Vim 或 vi 在大多数 Unix 系统上可用。这些编辑器完全在命令行上运行，不需要 GUI，这使得它们更加通用和轻量级。

当您通过 SSH 连接到一个远程系统，并且需要编辑一个文件或者甚至进行动态开发时，完全由命令行驱动会非常方便。

当使用像 Ubuntu Server 这样的操作系统时，这一点也很重要，Ubuntu Server 默认不包含 GUI。

您不再需要管理在系统之间来回移动文件。您不必使用 SSHFS 来挂载远程系统。你所要做的就是打开 vim(也许带上你的配置)，编辑，和`:wq`。

# 3.不要分心

您真的需要自动完成和集成代码分析吗？你真的需要一个花哨臃肿的电子 UI(看着你 Atom)？

学会从简单的角度思考。Vim 消除了大而笨重的 IDE 的所有干扰，将您带回真正重要的东西:代码。

使用 Vim 时，您会变得更加有条理和精确。你更清楚你正在编辑什么文件，你在什么目录，东西在哪里——即使它们没有显示在你面前。

由于编辑器的简单性，编辑、剪切、粘贴和在文件之间移动变得更有触觉和目的性。

当然，有时你可能需要额外的功能。有大量的 Vim 插件在您的指尖，用于各种目的。这些插件大多是按菜单定制的，本来就应该如此。你选择你需要的，留下你不需要的。

使用[病原体](https://github.com/tpope/vim-pathogen)和一点耐心，管理插件轻而易举。要获得最好的 Vim 插件列表，请查看这篇[文章](https://medium.com/@huntie/10-essential-vim-plugins-for-2018-39957190b7a9)。

# 3 街信用

其他不使用 Vim 的开发人员可能会认为你使用它不是疯了就是精英。将你同事的这种判断用于你的优势，并让它提高你的生产力。

请记住，您可以比他们开发得更快，因为 Vim 使用的内存比他们庞大的现代 IDE 少得多。

无论您是否正在犹豫是否要切换到 Vim，或者由于其他因素而被迫使用它，您都可以学会喜欢它。你可以让它成为你自己的延伸。

Vim 是最锋利的厨师刀，赢得了最大的尊重。

你用它切割或者被它切割。
