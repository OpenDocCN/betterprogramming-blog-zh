## 为什么编程比学校教给我的更多

对于像我这样的青少年来说，学校就是你所知道的一切。从你记事起，你就一直醒来去一个机构，在那里你坐在教室里学习信息，这些信息可能与你以后的生活有关，也可能无关。虽然我确实看到了学校的目的，但不可否认的是，它仍未能真正让孩子们为现实世界和他们将面临的挑战做好准备。

对我来说，这就是编码的由来。虽然它不能替代教育机构的所有失败，但编程确实教会了我一些学校应该一直教我的东西。

# 1.自我激励

在学校提供不同的学习和鞭策自己的机会的同时，还有外在动力的压力。像成绩、毕业和大学这样的事情是驱使人们以某种方式表现的原因。虽然外部动机是好的，但学校让这成为表现的驱动因素，随着时间的推移，教育孩子他们需要其他东西来让他们做一些事情。虽然有些人天生就有上进心，但大多数人不是，我就是其中之一。

虽然我总是试图在学校表现良好，但这从来不是为了简单地尽力而为，而是为了在一次作业中获得一定的分数。这适用于教育系统。但是一旦我纯粹出于兴趣开始学习编程，很快就发现我在学校使用的策略行不通。每次我开始一个项目，我都会无法完成它，因为没有什么能确保我做到。我可以放弃找工作，但没有人会知道。只有我知道我失败了。正因为如此，我让这种事情一次又一次地发生。

然而，这些失败开始在我的脑海中积累。久而久之，这发展成了它自己的激励形式。我开始完成编码项目，向自己证明我可以。现在我是在为自己工作，而不是为某种外力。尝试做一些事情来证明我能做到，这让我成为项目中的激励因素，而不是依赖分数。

明确地说，我不认为编码本身造成了这种差异——但它仍然发挥了很大的作用。因为有了互联网，任何人都可以学习编码。这使得编程比我可能尝试的许多爱好更容易实现。但同时，这也使退出变得更容易，给代码这种独特的平衡，这是很多活动所没有的。

# 2.毅力

无可否认，总的来说，我是一个固执的人，但没有什么能迫使我像编码一样坚持下去。虽然学校的作业迫使我不知疲倦地工作来完成一个产品，但编码将毅力带到了一个全新的水平。

学校作业有截止日期；个人编码项目不会。正因为如此，我可以想做什么就做什么。一旦我培养了开始一个项目的自我激励，我很快就知道如果我想完成它，我需要能够工作几个小时。帮助我发展这种能力的是 bug。

在开始时编写代码有些容易。但是，正如程序员所知，错误很快就开始出现，使得程序很难正常运行。我遇到的错误，加上我新发现的自我激励，让我花了几个小时试图解决问题，直到我累死。为了完成我的项目，我必须了解即使看起来没有解决方案，也要不断寻找解决方案意味着什么。有时我不得不工作几个星期，试图让一个功能正常工作。

多亏了随着时间的推移信念的建立，我才能够继续做这些项目，直到我睡着。虽然学校可以在某种程度上教授这一点，但学校的截止日期和期望使我们很难将某件事情做到完美。有时候，更有意义的做法是继续做某件事，直到它让你得到你想要的分数，然后放弃它。

# 3.思考

这是我在尝试编码时学到的最重要的东西。虽然其他一切都很好，但没有什么比知道如何思考问题更重要。编码教给我的是如何真正以创造解决方案的方式思考，而不仅仅是想法。

在我看来，思考一个问题过去只是创造一个想法，而不是不断完善它。编码教会我思考不仅仅是这些。思考实际上是一遍又一遍地创造新的想法。它是关于新旧思想的融合。它是关于以新的和创造性的方式使用别人的建议，以得出完全不同的东西。

编码是学习这一点的最佳方式，因为解决问题有各种不同的方法。我可以编写五个独立的打印函数，或者放入一个 for 循环；可能性实际上是无限的。对我来说，编码就像一个沙箱，允许我尝试不同的可能性而没有什么后果，真的允许我以各种可能的方式思考问题。

学校阻止这样做，因为学校的目标不是创造一个工作产品，而是制造一些符合某人标准的东西。这就限制了可以创造的东西。它变成了如何建造而不是建造什么。编码是思想的游乐场，这是学校所缺乏的。

我并不反对学校，但我很感激生活在一个可以免费学习编码这样有价值的东西的时代。我希望其他人也能利用这个机会学习我所学到的技能。
