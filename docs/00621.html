如何成为一名优秀的编程导师

原文：https://betterprogramming.pub/how-to-be-a-great-programming-mentor-86a3c8c43b82?source=collection_archive---------10-----------------------#2019-06-19

由经验丰富的学员为导师提供参考

无论你是导师还是被辅导者，每个人都可以也应该同意，编程中的辅导是很难擅长的。因为很难识别阻碍新学习者的问题，教学往往是一项令人厌倦的工作。幸运的是，导师和教学有一些非常重要的差异。

编程导师很难

在像软件开发这样不断变化的环境中，很难确定哪些信息有足够的价值保留或传递给其他人。我将稍微解释一下为什么这种方法比仅仅向他们展示如何做事要复杂一些。

程序员固执己见

每个程序员，不管他们是否知道，都有强烈的观点。这里只是开发人员几乎每天都在进行的无数无意义斗争的几个例子。这并不重要，但是我已经把我自己的观点用粗体字写了出来:

  • 两个空格的制表符与四个空格的制表符(两个空格的制表符)
  • 硬标签与软标签(软标签)
  • 完美语言与其他完美语言(第一种)
  • Git Rebase与Git Merge (Git Merge)
  • Vim vs. Emacs (Emacs，但我还是用Vim)

基本上，如果能站在哪一边，程序员就会站在哪一边。这使得在向新手展示诀窍时很难做到客观。

新开发人员容易受影响

人们想象的程序员的典型形象是对数字世界有可靠知识的人。当人们发现我是一名开发人员时，我通常会被问到的第一个问题是，“那么，你知道如何破解东西吗？”

虽然不是每个新程序员都有黑五角大楼的梦想，但他们都有自己想成为什么样的程序员的想法。通常是游戏开发商。这些想法会促使人们走捷径，走最短的路到达终点。

那么，导师的工作是什么？

这是我最好的简洁描述，

导师的工作是 而不是 传授材料和通知。它只是向新开发人员展示如何学习，鼓励良好的习惯，并作为一个支持者。

向新开发人员展示如何学习

这需要强烈的同理心，当然，程序员通常不具备这种感觉。也就是说，你所需要做的就是回忆起与抛给你的每一个新概念斗争的感觉。对于新开发人员来说，事情不仅仅是有意义的。学习一门编程语言就像…嗯，学习一门新的语言。你烂熟于心并认为显而易见的规则对一个新人来说是完全陌生的。

可惜你学不到给他们看。直到发明了某种技术可以将知识从一个人传递到另一个人，你必须弄清楚他们是如何学习的。很多编程概念客观上很难掌握。在一天结束的时候，要由学习者来解决这些问题。解释可以被抛来抛去，但是你应该寻找的是让它发出声音的东西。

鼓励好习惯

我知道这听起来有点老套。即便如此，我也数不清自己说过多少次，被告知多少次，“忘了文档吧。”当你获得足够的经验知道什么时候是错的时候，嘲笑像文档和好的变量/方法名这样的习惯是很有趣的。然而，对于一个新开发人员来说，像“文档是愚蠢的”这样的话不会被理解为一个笑话。它被理解为一个教训。

导师是一个从一开始就培养这些好习惯的机会。即使你个人没有遵循其中的很多(尽管你应该)，帮助学习者尽早养成这些习惯也是很重要的。这里有几个好习惯，我希望在我第一次学习编码的时候就把它们强加给我:

  • 记录你的代码(有意义地)
  • 花更多时间规划/思考，花更少时间编码
  • 编写代码的方式要一致。不要有一个名为badApple的变量和另一个名为good_apple的变量
  • 如果你有一个问题，制定一个精确的问题。与其说“它不起作用”，不如问“为什么我的方法是改变原始数组而不是创建一个副本？”很多时候，问题可以通过问正确的问题来解决。

扮演支持者的角色

这条建议来自我沮丧、疏离、过去的自己。我遇到的一些导师也是这样认为的。归根结底，技术社区是由数量惊人的支持者组成的。你应该努力成为他们中的一员。

导师和学员的关系不是一种权力关系。你们是同行。尽管你可能比你指导的人知道得更多，但是没有一个开发人员应该学习完。和你的学生在一起是最重要的，以确保他们明白犯错误是可以的。我们都会犯错。越早采取这种态度越好。

结论——成为你希望拥有的导师

对于我们这些足够幸运，在成为开发人员的过程中有一个或多个导师的人来说，汇编一份从他们那里学到的最有帮助的经验并加上你自己的经验是很有帮助的。如果你从未有过导师(我非常尊敬你)，想象一下你想和什么样的人一起学习并成为那样的人。

如果你真的想帮助新开发人员，就和他们一起学习。作为一名新开发人员，见证一名经验丰富的老手如何学习新事物，这真的是一次不可思议的经历。

你应该成为一名导师，而不是讲师。与人交谈是教授概念的有效方式。毕竟，大学就是这样的。但你不是一个人的大学，它不会帮助任何人学会适应和成长。

如果你知道有人正在努力学习成为一名开发人员，而你有一定的经验，或者至少知道一些他们不知道的东西，那么考虑成为一名自称的导师，加入他们吧！我真的很支持导师制度。
