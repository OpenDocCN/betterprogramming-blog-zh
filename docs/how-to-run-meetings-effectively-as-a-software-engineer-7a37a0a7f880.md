## 对晋升到更高职位至关重要的领导技能

在你成为高级软件工程师，或者最终成为人事经理的道路上，你需要发展的最重要的技能之一，就是主持顺利而富有成效的会议的技能。

在我个人看来，一个好的会议:

1.  有明确的目的—无论是讨论最新的项目更新，还是解决一系列具体的未决问题。
2.  是必要的——也就是说，手头的主题需要面对面的讨论，而不是电子邮件或聊天室。
3.  包括合适的人。
4.  抓住重点，充分利用时间。
5.  以清晰的结论和行动项目结束。

以下是我开好会议的最佳建议。

# 1.不要害怕取得所有权。

根据您的工作场所，如果您是工程项目的首席工程师，您可能需要主持工程项目会议。在我工作过的地方，情况确实如此。如果你不确定，和你的经理谈谈，了解你是否应该主持会议。

如果这是你的会议，拥有它。主持工程项目会议可能会令人生畏，因为他们通常涉及不同学科的参与者，从工程到 UX 团队，再到产品负责人。经理们，包括你的直接经理，经常参加这些会议也是很常见的。

不管参与者是谁，都不要害怕承担责任。主持一个会议不是关于优越感，甚至是资历。通过带头，你没有冒犯任何人，而是促进了对话。你要确保讨论是富有成效的，并以每个人都在同一页上结束。

# 2.花点心思安排会议。

作为会议的所有者，首先决定是否有必要是你的责任。

对于周期性会议来说尤其如此。如果你主持一个会议，比如说，每周一次，那么事先花点时间评估一下本周是否有足够的内容可以讨论，或者你是否应该取消。通过聊天或电子邮件联系其他参与者，了解他们的想法。

> “大家好，本周二我们有什么要讨论的吗？我这边还在做【某某】，所以没有什么特别有意思的分享。我希望在[某个日期]完成那件事。
> 如果没人有什么要讨论的，我们是不是取消这个星期二，改在下周见面？”

# 3.建立一个议程清晰的会议记录文档。

在会议开始之前，确保有一份清晰的议程，所有与会者都可以阅读并参与其中。

例如，您可以在会议邀请中附加一个文档，其中包含您能想到的任何议程项目。至少在会议前几个小时发送电子邮件或群聊，提醒与会者查看议程，并添加他们想提出的任何其他事项。议程最终可能会是这样的。

*   *Noran:发布时间表的一些更新*
*   *约翰:建议改进动作按钮中使用的语言*
*   *阿玛亚:我们能不能为了赶上发布截止日期而降低[某某功能]的优先级？*

对于定期会议，您甚至可以自动发送这封提醒邮件，它也会提醒您查看议程。

> 大家好，
> 
> 在今天的会议之前，请查看今天的议程，并添加您想要分享或讨论的任何内容。
> 
> 在那里见！

# 4.为会议的进行制定一个计划。

对于以项目为基础的会议，其主要目的是让团队成员分享他们的更新，那么检查一下您选择的项目管理工具并让每个人谈论他们分配的任务可能是一个好主意。

否则，使用会议记录文档中的议程作为指导。您可以按顺序浏览议程要点，但您应该在认为合适的情况下，随意将一些要点排在其他要点之前。

> “Javier，我看到你这里有一个关于[某某]的项目，但如果你同意的话，我们先讨论一下[某某]，把它解决掉，这样我们就可以畅通工程工作了。”

# 5.确保有人在做笔记。

在会议期间，应该有人在讨论时更新议程项目，添加想法、决定或行动项目。

如果你不能在会议期间做到这一点，那么问问另一个参与者是否愿意自愿。

# 6.知道什么时候插话。

如果你觉得谈话偏离了主题，或者讨论变得太激烈，或者占用了太多时间，你应该插话分散这种情况，让事情回到正轨。

> “好吧，我认为这很有成效。我知道我们时间不多了，所以让我们再考虑一下，改天再讨论这个话题。与此同时，让我们继续下一点。”

# 7.处理好遗留问题。

开好会议的一部分是确保没有任何疏漏。注意时间。如果你已经设法在剩余时间内讨论了所有议程要点，请确保询问每个人是否有其他未列入议程的自发要点要讨论。另一方面，如果你已经没有时间了，而你还有重要的事情要讨论，那么就提议安排一次后续会议。

# 8.会后让每个人都了解情况。

会议结束后，确保执行以下任何适用的操作。润色会议期间匆忙写下的会议笔记。根据需要分配行动项目。发送一封电子邮件，附上讨论的笔记或摘要，以及后续步骤。

> 大家好，
> 
> 这是今天的会议记录。正如所讨论的，我们将在本周晚些时候与 UX 团队安排一次会议，讨论有关最新模拟的问题。下周我一定会把结果告诉大家。
