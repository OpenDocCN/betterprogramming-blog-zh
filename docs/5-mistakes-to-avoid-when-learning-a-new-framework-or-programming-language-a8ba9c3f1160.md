# 学习新的框架或编程语言时要避免的 5 个错误

> 原文：<https://betterprogramming.pub/5-mistakes-to-avoid-when-learning-a-new-framework-or-programming-language-a8ba9c3f1160>

## 通过理解关键的学习错误保持警惕

![](img/5c53d01b80b2737759814f27e36de8c9.png)

安娜·格鲁在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

学习、提高技能和接触最新的技术和趋势是任何软件开发人员生活的一部分。他们不是可有可无的附加物，而是成功实现[一个开发人员职业目标](https://eisabainyo.net/weblog/2021/03/10/10-strategic-career-goals-for-web-developers/)的关键。然而，无论开发人员目前处于职业生涯的哪个阶段，他们都会犯一些常见的学习错误。

常见的学习错误有:

1.  没有学习计划。
2.  没有设定明确的最终目标。
3.  选择太宽泛的话题。
4.  消费过量。
5.  没有跟踪进展。

# 1.没有学习计划

![](img/3c6ea709ad4d2bf5cefda92cf858c092.png)

照片由 [Avel Chuklanov](https://unsplash.com/@chuklanov?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

有句话说，如果你没有计划，你就计划失败，这也适用于学习。有时，开发人员会不耐烦，毫无准备地直接投入到主动学习中。例如，一个开发人员可能会说他们想学习一个 JavaScript 框架，React。他们可能开始阅读关于 React 的教程，却不知道他们想关注什么领域，或者如何达到他们的最终目标:能够编写 React 应用程序。

在这种情况下，更好的学习方法是查看官方文档，从主要概念到高级指南和 API 参考资料一步一步地学习，并在 React 中创建一个示例应用程序。在这之后，看看其他的例子并向他们学习，为他们学习的每个阶段设定 SMART 目标将会很有帮助。S.M.A.R.T .目标是好的目标，因为它们是具体的、可衡量的、可实现的、现实的、有时间限制的。

# 2.没有明确的最终目标

![](img/2916dba3d3c1a97564c1ae34d90b3918.png)

[engin akyurt](https://unsplash.com/@enginakyurt?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

在[软件开发](https://hackernoon.com/tagged/software-development)行业中有如此多的新技术和新的做事方式。不要误会我；我相信不断学习是一件好事。如果你乐于学习，你就能增长你的技能和知识。然而，如果一个开发人员对学习他们听到的每一个新事物有太多的 FOMO(害怕错过)——那么就没有时间来提高效率或很好地利用她的知识和技能。

在你学习某样东西之前，我鼓励你思考一下你为什么要学习它——以及你将在哪里以及如何使用你新学到的知识或技能。例如，如果你是一名后端开发人员，你正在学习 JavaScript，你这样做是因为你的目标是转向全栈开发。或者，如果你是 DevOps 并且正在学习 AWS，你可能这样做是为了增加你在使用 AWS 的公司获得新的就业机会的机会。

# 3.选择太宽泛的主题

![](img/a23b0f214487c00c9cfc8aa7c89e96eb.png)

马修·梅布尔在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

听说过这样一句话吗——除了厨房的水槽，什么都可以？有时，开发人员过于贪婪地想要了解某个主题的所有内容，他们无法缩小范围，而是设定了一个非常广泛的学习目标。我听到你问，多宽才算太宽。在我看来，如果你不能用几句话向另一个开发者清楚地表达你的学习成果，那么这个主题就太宽泛了。

学习成果是关于开发人员在特定主题的学习旅程结束时应该获得的知识或技能的陈述。当然，你可以永远学习某个特定的主题，但是必须有一个时间点，让你自己决定这就足够了。足够让你对努力实现最终目标充满信心。

例如，下面是学习 React Javascript 框架的一些学习成果。

在我的学习计划结束时，我应该能够从头开始创建一个新的 React 应用程序，它可以:

*   在 REST API 中使用 JSON 负载
*   每隔`x`分钟刷新一次应用程序的内容
*   导航到不同的屏幕
*   还记得我之前访问的最后一个屏幕吗(如果有的话)
*   接受单元测试

请注意，上面的学习结果并不太具体或详细(例如，像用 JSX 语法创建一个组件)，因为您还不知道 React 框架的来龙去脉。它们也不太宽泛；每个语句都有一个清晰的结果，而不仅仅是一个宽泛的、通用的结果，比如“创建一个 React 应用程序”

# 4.消费超载

![](img/12d01b73a27e0c89bbc4c40d5647e715.png)

[迈克·刘易斯 HeadSmart 媒体](https://unsplash.com/@mikeanywhere?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

辅导疲劳——这是真实的。如果你认为通过阅读无止境的书籍、视频和教程可以学到东西，那么你只是在浪费时间。选择几个选项，可以是书籍、视频或任何其他形式，并坚持下去。你在视频中学到的东西很可能与你正在阅读的同一主题的教程非常相似。

再给你一个例子，我在亚马逊上快速搜索了 React Javascript framework，大约有 700 本关于这个主题的书——其中几本甚至有完全相同的书名，*学习 React* 。你不可能看完所有的书，即使你看完了，也不能保证你会成为 React 方面的专家，或者这是对你时间的一个很好的利用。

# 5.没有跟踪进展

![](img/6a033841ad02342907994410954b097a.png)

照片由[迈克·范·登博斯](https://unsplash.com/@mike_van_den_bos?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

有一个清晰的计划和明确的目标是好的，但是如果你没有跟踪你的进展，或者如果你正在达到你的目标，它们就没有什么用了。每两周，或者至少每个月回顾一下你的进步，问问你自己你是否在朝着正确的方向前进。

我建议在你的日历中设置一个提醒，在学习之旅开始时定期回顾你的进步——这样你就不会忘记了。如果你没有及时实现你的目标，这是一个机会，让你明白你可能在哪里挣扎，并修改你的计划。另一方面，如果你正在实现你的目标，庆祝它们来保持你前进的动力和动机。

# 6.做一个高效的学习者

![](img/4ea76c51ef1f3b9ffc07bae5dbe58f12.png)

照片由[元素 5 数码](https://unsplash.com/@element5digital?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

在这个时代，成为一个高效的学习者会给你带来优势，并为你的职业成功奠定基础。我坚信，如果你花时间去理解这个过程，并且愿意改进，你可以在生活中做得更好。学习也不例外。这是一种可以通过练习来提高的技能。当谈到学习时，知道什么不该做和知道什么该做一样重要。正如中国社会哲学家孔子所说:

> “学而不思则罔；思而不学是危险的。”——孔子
