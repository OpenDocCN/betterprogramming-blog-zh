## 软技能至关重要，你需要继续努力

当人们描述他们认为一个软件开发人员是如何工作的时候，他们经常说些陈词滥调。他们认为我们在电脑前独自工作几个小时，不和任何人说话。

当然，这不是真的。有时候我们独自工作，试图找到“心流”，这样我们很有效率。但通常情况下，与我们的同事讨论一些技术决策或参加与客户的会议，以确保我们真正了解他们想要什么，这是非常重要的。由于我们中的许多人*都是内向的人，这就不那么自然了。*

我们怎样才能对此感到更舒服呢？还有，如何利用这段时间更有效率？软技能真的很有帮助。

# 什么是软技能？

软技能是以人际关系或自我管理为中心的技能。以下是一些在我们日常工作中非常重要的软技能:

*   沟通
*   解决问题
*   协力
*   时间管理。

当然，还有其他的，但我认为这些是最重要的。

它们是绝对必要的，因为它们帮助我们开发客户真正想要的东西。此外，我们会过得更轻松，因为当我们善待他人时，他们也会善待我们。

# 怎样才能进一步发展自己的软技能？

## 沟通

根据我的经验，最重要的软技能之一是有效沟通的能力(无论是口头还是书面形式)。这是我个人仍然纠结的一点。

当我试图向我的妻子解释一些技术性的东西时，我经常得到的反馈是，她不能理解我使用的行话，或者她错过了一些真正抓住要点的上下文。我该如何改进这一点？

首先，我试着用更长的时间来思考我想要解释这个问题的方式。然后，当我在说话的时候，我真的会试着专注于对话。我注意到，当我这样做时，我妻子理解要点的速度更快。

在与客户/项目团队开会时，我现在倾向于在说话前多思考一点，在倾听时也更积极。这有助于快速找到一个共同点，我可以在那里建立一些解释，并真正理解一个特性应该解决的问题。

沟通的另一个重要部分是倾听。对我来说，这比实际交谈要容易得多，但即使是在倾听的时候，给对方反馈也是非常重要的，这样你们才能保持一致。我注意到，给你的对话伙伴一个关于你到目前为止所理解的内容的简短总结是非常有帮助的，因为通常在对话中，知识会逐渐积累。如果你尽早并持续地给出反馈，你就能确保你能跟上谈话。

所以，为了在一对一的谈话中不断提高，我真的试着利用以上几点。我注意到我的沟通技巧正在提高。此外，为了进一步提高我的技能，我正试图了解更多关于这些话题，并慢慢地将它们不仅应用到我的职业生活中，也应用到我的私人生活中。我慢慢地吸取教训，因为你只能做这么多，直到筋疲力尽。

## 协力

第二个非常重要的技能是团队合作。作为公司的开发人员，我们通常在团队中工作以实现我们的目标。你曾经在一个成员之间竞争激烈的团队中吗？或者团队个人相处不好的地方？如果是这样的话，你肯定已经观察到团队的生产力会显著下降，直到问题被解决。如果项目负责人和开发人员之间存在一些问题，这一点尤其明显。

我们怎样才能在第一时间阻止这样的事情发生呢？以我的经验，项目负责人在这里起着最重要的作用。如果他们试着与团队同理心，不夸大错误，每个人都可以茁壮成长。但如果事实并非如此呢？

如果团队试图相互理解，即使项目领导是有毒的，他们也能保持很高的生产力。在我工作的一个团队中，我们有时会在下班后喝一杯啤酒，并以这种方式进行社交。在另一个团队，我们一起度过午餐时间，在那里聊天。

社交有什么帮助？如果团队成员互相了解，互相喜欢，就更容易接受对你的代码的建设性批评，并给出好的建设性批评。这当然对改进代码库和提高自己的技能有很大帮助。此外，如果你和朋友一起工作，结对编程会有趣得多。

# 结论

当然，对于软件开发人员来说，还有其他重要的软技能。如果我们继续在这些方面投资，我们不仅会更有效率，而且会更加享受我们的时间。即使你独自投资这些技能，很快其他人很可能会跟随，因为人们倾向于对对他们友好的人友好——即使是复杂的人。
