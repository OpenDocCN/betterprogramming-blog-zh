# DevOps — MLOps — AIOps

MLOps、DataOps、AIOps——前缀词太多，越来越难以管理。

顾名思义，DevOps 的概念有一些联系，devo PS 是一组旨在自动化任何业务流程的概念。主要的思想是有效和准确地构建、管理、维护、测试和发布软件。

# 什么是 MLOps？

机器学习运营(MLOps)有助于简化运营团队和机器学习研究人员之间的机器学习模型的管理、后勤和部署。

基本上是 DevOps 应用到机器学习领域。显然，由于机器学习的工作方式，存在一些差异。还有一个方面就是写代码，测试 bug。但是，在机器学习领域，你是用数据训练模型，用新数据再训练。

MLOps 不仅简化了集成并使协作更容易，而且还允许[数据科学家和机器学习研究人员](https://towardsdatascience.com/4-must-have-skills-every-data-scientist-should-learn-8ab3f23bc325)承担更多项目并开发更多模型。借助 MLOps，数据培训、测试和部署通常会实现自动化，因此整个工作流程变得更加高效。

现在， [15%的组织已经是高级 ML 用户](https://financesonline.com/machine-learning-statistics/)，大约 [65%的公司正计划采用机器学习](https://financesonline.com/machine-learning-statistics/)，因为它有助于企业决策。这就是您的组织需要 MLOps 的原因。它遵循类似于 DevOps 的模式，因此在您的操作流程和开发周期之间有一个无缝的集成。

在将 MLOps 引入您的组织之前，您必须考虑几件事情:

1.  **采用:**对于任何新的模式，都需要做大量的工作来帮助员工采用新的部署模式。当您开始自动化测试和部署时，需要很多信任。您需要让数据科学家和运营 IT 团队都参与进来。这通常是通过培训、获得认同和新技术的良好计划发布来实现的。
2.  **关键绩效指标:**从数据科学团队到利益相关者，您的 KPI 必须清晰，并且可以被每个人衡量，以便他们能够理解如何利用这些见解并向前迈进。
3.  **监控:**由于机器学习使用非直观的数学函数，因此需要持续监控，以确保您在法规范围内操作，并且程序返回高质量的信息。
4.  合规:最重要的一点是确保 GDPR 不会影响你的项目。因此，您的 MLOps 需要以这样一种方式进行规划，使您的程序变得可审计。
5.  **内置偏差:**自动化机器学习模型部署和测试并不能免受人类偏差的影响。因此，重要的是要确保你的团队考虑如何开发一个不会无意/有意偏向特定人群的系统。定期审核您的机器学习模型是评估输出的一种方式。

# 什么是 AIOps？

IT 运营人工智能(AIOps)是另一个范式转变，它允许机器自己解决 IT 问题，而不是需要人类的帮助。它是一种多层技术，可以增强 It 的运营。AIOps 使用机器学习和分析来分析通过不同工具获得的大数据。这使得平台能够实时自动发现 IT 问题并做出反应。它是机器学习和大数据中核心技术功能的持续集成和部署。

现在的问题是为什么您的组织需要 AIOps。统计表明 [72%的 IT 组织](https://docs.broadcom.com/doc/top-five-reasons-you-need-aiops)依赖不同的 IT 监控工具来支持现代应用程序！显然，企业需要这些工具来管理日常 IT 运营。

但是还有更多！

尤其是 AIOps，它不仅有助于解决复杂的 IT 挑战，还能让您迎合数据的指数级增长。有效的 AIOps 可跨混合环境自动执行整个操作流程，并为机器创建准确的清单，以独立关联数据点。它应用于机器学习来检测模式，从而降低噪声。

# AIOps 在哪些方面有帮助？

## 噪声降低

如果您曾经在开发团队中随叫随到，那么您可能非常清楚会收到多少警告和通知。这些通常是善意的，因为它们旨在帮助您的团队发现问题。但是，有时问题是暂时的，会自行解决。其他任务和操作可能只需要手动重新运行。AIOps systems 提供了一个自动化的人工智能系统，可以帮助减少运营团队遇到的问题数量。它通过分析过去的问题和解决方案来做到这一点，并试图减少交给运营团队的任务和通知。

如果问题可以自动解决或忽略，系统会自动解决。像 [Resolve.io](https://resolve.io/aiops/noise-reduction) 这样的公司提供这种类型的服务，可以帮助您的 IT 运营团队更好地管理他们的系统和用户。

## 异常检测

如前所述，开发团队将创建检测特定信号的通知。但这需要人工干预。就像，有人经常需要创建特定的模型来管理特定的问题。

然而，更不可预测的行为呢？这就是机器学习和 AIOps 可以再次介入的地方。同样，像 Resolve.io 这样的公司已经开发了一些工具来帮助更好地检测您的系统可能从未见过的异常和错误。

## 系统映射和依赖性管理

第三方云提供商简化了许多开发，但这也带来了一系列全新的问题。

例如，无服务器使得部署基础设施在某些方面更容易、更便宜。你可以用很少的代码创建一个 API。随着无服务器的推进，出现了一系列新问题，如可观察性和系统映射。

对于所有这些不同的技术部分，内部、云和第三方服务，您如何管理依赖性和测试？当您部署代码时，很难看到所有更改的影响。这是 AIOps 的另一个用例。公司正在开发工具来帮助规划系统，以便您可以看到代码更改和更新的影响。

像 Epsagon 这样的工具可以帮助自动检测系统依赖关系，使您的软件团队清楚地了解他们的系统的发展方向，以及在迁移和更新相互交互的系统时需要监控哪些系统。

这是 AIOps 适合您的 IT 战略的三个很好的使用案例。总的来说，MLOps 和 AIOps 之间有一个非常明显的区别。

# MLOps 与 AIOps

术语“AIOps”经常与“MLOps”互换使用，这是非常不正确的。

AIOps 旨在实时支持和应对问题，并为您的运营团队提供分析。这些功能包括性能监控、事件分析、关联和 IT 自动化。根据 Gartner 的说法，AIOps 结合了大数据和机器学习来自动化 IT 运营流程。因此，AIOps 的最终目标是自动发现日常 IT 运营中的问题，并使用人工智能主动做出反应。事实上，研究表明 [21%的组织计划在一年内采用 AIOps](https://thenewstack.io/the-current-state-of-aiops/) ！

另一方面，MLOps 专注于管理有效创建机器学习模型所需的训练和测试数据。它完全是关于监视和管理 ML 模型的。它专注于机器学习操作化管道。AIOps 是关于应用认知计算技术来改善 IT 运营的，但不要与 MLOps 混淆。

# 是的，AIOps 和 MLOps 非常不同

再加上现代应用程序架构的日益复杂性，数字经济的需求使得 IT 运营的角色更具挑战性。因此，ML 和 AI 已经出现，以自动化一些人工业务流程并提供效率。

MLOps 或 AIOps 都旨在服务于相同的最终目标；即业务自动化。虽然 MLOps 弥补了模型构建和部署之间的差距，但 AIOps 侧重于实时确定和应对 IT 运营中的问题，以便独立管理风险。

我们目前正处于人工智能的黄金时代。然而，这些行动的成功实施需要培训和投资。因此，如果您希望在当今行业的竞争中保持相关的*和*领先，了解并采用这些技术是至关重要的。
