## 这不是一次性的事情。这种折磨是真实的

我们都经历过疫情的负面影响。工作已经失去，生活水平也没有下降。大多数有现场工作的人都失业了。在这个时期，最有利可图的市场是技术市场，因为我们可以在任何地方工作。我们只需要一台能上网的电脑。也就是说，有很多人希望进入编程领域。这是好事。只是我认为他们的期望并不像他们想象的那样。

在这篇文章中，我想触及一些大多数人在教授或建议人们进入科技行业时没有涉及到的东西。大多数人不会谈论编程的严酷或黑暗面。我们谈论了很多关于编码、构建软件和一般技术的积极方面，但是我真的认为强调一些严酷的事实是很重要的，当你正在学习编程或者是一名程序员时，这些事实可能不太被提及。

我想这么做的原因是，当你学习编程时，或者如果你是一名程序员，你可以准备好学习时可能会遇到的问题。这并不是要打击你或者让你不想继续你的编程之路，实际上恰恰相反。这是为了让你具备必要的技能或洞察力，如果你面临这些挑战，你会知道这是完全正常的，你会度过难关。

# 1.不，你不可能在一个月内学会编程

我相信你已经在网上看到了无数的人或课程，他们承诺让你在 30 天内成为一名全栈开发人员，否则你将在 30 天内获得 6 位数的报酬。这是非常不可能的！我不会说不可能。像任何技能一样，学习如何编程需要时间。这需要持续不断的重复，这不会一夜之间发生。对于非技术背景的人来说，这个时间甚至更长。学会程序化思考也需要时间。你需要学习的不仅仅是一些命令、框架或者不同的语言。它改变了你的整个思维模式，让你用特定的方式思考，用特定的方式解决问题。

你知道谷歌搜索本身也是一种需要学习的技能吗？我的意思是学习如何用谷歌搜索问题，这些问题实际上会给你你正在做的编程所需要的答案。所以，如果你看到一门课程承诺让你在一个月内成为一名专业程序员，那可能比他们透露的更多。这使我想到另一点。我想谈谈编码训练营。这些通常只有三个月。

那么，之后能不能成为一个精通或者厉害的程序员呢？嗯……这真的取决于你在进入新兵训练营之前有什么样的背景，另外，即使从新兵训练营毕业，你也永远不会觉得自己完全准备好进入这个行业，这没关系。你只需要意识到不断学习和成长的心态，即使你已经毕业了。这需要时间，而且几乎就像肌肉一样。你练习的越多，你继续学习的就越多，你会变得更强更快。

# 2.教程和课程不会让你找到工作

你可以参加世界上所有的教程和课程，但它们不会成为你找到工作的主要原因。是的，当然，参加教程和课程是你学习和成长的一个重要部分，当然还有你自己的项目，但不仅仅如此。你需要建立关系网。你需要走出你的舒适区，开始与人交往。如果你现在几乎什么都在做，那就开始在 LinkedIn 和其他社交平台上寻求帮助。是的，即使你是一个内向的人，想整天呆在你的小房间里看着你的电脑屏幕编码，如果你想得到那份工作，人际关系网是必须的。

新手通常不会被录用，因为他们有经验。哪种体验？他们刚刚涉足科技领域。公司看的东西不一样。你渴望学习吗？你饿了吗？你想继续学习吗？他们想了解你这个人，并认为你很适合他们的团队。

教程和课程无法帮你找到(或找不到)第一份工作的另一个原因是:尽管它们在你的学习过程中至关重要，但公司希望看到你的投资组合。这应该有你已经建立的项目。您可以在教程项目的基础上进行构建并使之成为您自己的项目，也可以完全从头构建项目。试着[避免使用教程](/3-reasons-you-should-avoid-udemy-projects-in-your-portfolio-fac72b0e5a8b)中的相同项目作为你文件夹中的一部分。

你在找工作的时候，需要表现出你自己从零开始做过一些项目。对于一个初级职位来说，这些不需要是一些宏大的项目。它可以像一个有趣的修改过的待办事项应用程序一样简单。显然，你的投资组合需要更多，但不要试图打造下一个 Snapchat 或脸书。公司只是想看到你掌握了基础知识，不需要教程也能做到。[逃离教程地狱](/escaping-the-vicious-tutorial-cycle-7760d1294dca)。

# 3.你会放弃不止一次

相信我。总有一天你会讨厌编程。你会发现自己在质疑为什么要花时间编程。冒名顶替综合症将会发作，就好像它在等待第一次失败的尝试。戒烟的冲动会达到顶峰！你猜怎么着，它发生在我们最好的人身上。这是完全正常的，有这种感觉是正常的。不要试图推开这些感觉，开始拥抱它们，并意识到对编码感到沮丧是正常的。这将有助于更快地理清这些感觉，并帮助[克服冒名顶替综合症](https://medium.com/geekculture/overcoming-impostor-syndrome-1c909c1ec4c8)。

我想指出的最后一点是，当你开始学习如何编码时，放弃是没关系的。诀窍是认识并承认这是正常的，并提醒自己为什么要开始学习如何编码。

请记住，这些建议并不是真的能帮助你保持动力。它们是为了帮助你保持自律，因为，老实说，你不会总是对编码有很高的积极性。随着时间的推移，它会到来。

# 4.你需要学习不止一种编程语言

是的，我说了。然而，在我开始之前，我想澄清一些事情。如果你刚刚开始学习你的第一门语言或框架，我坚信在进入另一件事情之前要先了解一件事情。不要偏离你目前的道路，开始学习其他的东西。然而，在你的职业生涯中，作为一名程序员，你很可能需要使用不同的编程语言。因此，如果你一开始是一名 JavaScript 开发人员，可能会有需要使用 Python 的时候，你需要选择不同的框架。你会不断地学习。

尽管这是一个残酷的事实，但我认为这是一个非常好的事实，因为这意味着你作为一个人在不断地学习和成长，随着技术的不断变化和发展，你需要跟上它。对于[的问题，你需要学习多少种语言？](https://javascript.plainenglish.io/how-many-programming-languages-do-you-need-to-know-88a0bb259f6f)这要看你当时在做什么，以及你作为开发者的最终目标。

# 5.你的背景很重要

我谈了很多关于任何人如何在没有任何经验的情况下进入技术或编程领域。是的，这是真的。为什么这是一个严酷的事实？嗯……这样想吧，如果你会演奏一种乐器或者了解音乐理论，那么对你来说唱歌会比那些一点经验都没有的人更容易。这同样适用于技术。与完全来自另一个领域的人相比，以某种方式与代码有过一些互动的人的学习曲线会小一些。这并不是科技领域独有的现象。

不要烦恼，因为还有希望。对我来说，科技最大的好处就是没人关心你的论文。忘记那些要求你有计算机科学学位和一堆初级职位语言的工作岗位，那些只是一个笑话。

在技术方面，他们更关心你可以展示的技能，你可以谈论的技能，你的知识。这真的很棒，因为它真的打破了任何想进入科技行业的人的门槛。我的意思是，你不必回到学校四年或类似的事情。最重要的是坚持。如果你不是技术出身，那你就得持之以恒，不断向前推进。你会成功的，但你不能放弃。

说到永不放弃，还有一件事我想我应该提一下。我不是说你一直把自己的精神健康推向危险的边缘。我知道许多人一次又一次地努力，但几乎没有成功。你应该明白的是，人们成长和学习的方式是不同的。仅仅因为它对别人比对你更有效，并不意味着你有问题。这只是意味着你需要给自己留点时间，当你精神饱满准备好重新投入的时候再去做。

记住，只要你做得好，他们永远不会拒绝你的机会。
