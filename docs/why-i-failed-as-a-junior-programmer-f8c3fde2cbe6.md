## 我从得意洋洋到最终失败的旅程

![](img/1c45eae0891451639c83f9e87b753bcc.png)

安妮·斯普拉特在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片。

经过三年的自学编程，我最终决定走出去，寻找初级 Python 职位。我从招聘人员开始。尽管我可以通过评估测试，在面试中表现出色，但他们都表示，由于我缺乏专业经验，很难找到工作。

我最终通过 discord 联系到了另一个开发者，他给了我一个他们公司申请表的链接。在几次电子邮件交流后，他们安排了我四次面试中的第一次。我所有的面试都非常顺利，包括与他们的一名员工进行的面对面编程会议。我还是没想到我真的会得到这份工作。我把这看作是我可以带到下一份工作中去的经验。

最后一次面试是与我可能共事的整个团队的视频会议。当他们问我的个人生活时，我提到我是一名音乐家。原来他们都是音乐家。在接下来的采访中，我们谈论了吉他和录音设备。太棒了。事情结束后，经理说他们会商量一下，然后给我回电话。

我生命中最长的十分钟后，经理打电话来了。他们说，“嗯，你觉得怎么样？”在整个面试过程中，我一直很诚实，所以我说，“这比我想象的要让人紧张得多。但我觉得进行得非常顺利。”他们等了一会儿，然后说，“埃文，我真为你开始这项事业感到兴奋。”

三年来始终如一的坚定决心导致了这一时刻。我会说这感觉就像是一场梦，但这一刻似乎太遥远了，以至于我从来没有梦到过。这正在发生。我想成为一名职业程序员。

作为初级软件工程师的那几个月，我想了很多，虽然我失败的原因有很多，但我觉得这些是最大的原因。

# **不求助**

当你被一份高薪工作录用，而这份工作需要具备多种技能，在技术和心理上都具有挑战性时，你会对自己有很高的期望。你想表现出公司雇佣你时没有浪费他们的时间和金钱。你觉得你需要尽快证明些什么。因此，很难向他人伸出援手，承认你一开始不知道自己到底在做什么。这是一个错误。公司希望你提出问题。他们期望你学习。

# **假设每个人都知道的比我多**

被贴上初级软件工程师的标签会对你的心理产生影响。尽管你仍然保留了到目前为止你所积累的所有知识，但是每当你和另一个程序员交谈时，你会立刻忘记你所知道的关于编程的一切。

有一个非常令人讨厌的神话延续了这种遭遇:高级软件工程师什么都知道。他们没有。他们和其他人一样使用谷歌。

# **选择在家工作**

我知道这是目前的标准，但是当我被雇佣的时候，我可以选择在家或者在办公室工作。在家工作意味着他们会给我装上惊人的装备，我也不用每天花一个小时去上班。我后来发现，如果我坐在另一个程序员旁边，而不是浏览 Slack 或考虑我是否应该请求与某人进行 Zoom 会议，我应该问的许多问题会更容易提出。

# **一次承担太多**

我的团队由前端、后端和全栈开发人员组成。我被聘为后端 Python 开发人员，但我确实有过构建应用程序的经验，所以我对全栈编程略知一二。

在完成了一些涉及数据库、API、HTTP 请求和功能测试的任务后，我感觉非常好，想在前端测试一下。他们用了 React Native，这是我当时不太熟悉的东西。我问我是否可以买一张前台票，结果花了三个工作日和我的周末学习 React Native 和一切随之而来的东西。

最后，我还是不太明白需要做什么。我觉得我在这一点上花了太多时间去问一些基本的问题，比如“我到底要做什么？”

# **不再从事自己的项目**

在我自学编程的学习过程中，一个巨大的驱动因素是做我自己的项目。我工作的公司甚至在周五也留出时间来做这件事。因为我仍在努力学习该公司软件的各个方面，并试图证明自己是一名合法雇员，我觉得让我把精力放在自己的代码上是不合适的。这让我失去了被录用前的那种兴奋感。我没有大胆探索新的想法，而是觉得自己在拼命追赶。

# 结论

我的第一份工作是初级程序员，这让我可以和了不起的人一起工作，做不可思议的事情。我在这六个月里学到的东西比我之前自学的三年还多。不幸的是，我与那家公司分道扬镳了，但就像我的编程之旅中的任何其他失败一样，我试图从我做错的和做对的地方学到尽可能多的东西。

希望我能让其他初级程序员为承担这个看似不可能的角色的陷阱做好准备。这实际上是我想分享的最后一个想法:这看起来确实不可能，但如果你能找到一份工作，相信公司确实在你身上看到了一些东西——相信你自己能胜任这个角色。

非常感谢你的阅读。我希望我所有的程序员同事在这个不可能的旅程中好运。如果任何人对我的旅程有任何问题或评论，请随时回复。
