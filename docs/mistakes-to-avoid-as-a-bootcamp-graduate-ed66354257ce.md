你刚刚完成了一个编码训练营，你即将开始你的技术职业生涯。

你可能对此感到兴奋，这是理所当然的——我毫不怀疑你会喜欢这个旅程。你刚刚在很短的时间内学会了非常有价值的技能，这需要很长时间来消化。很难看出[从这一点上该做什么，而且很容易被你的热情冲昏头脑，陷入以下陷阱之一。](https://medium.com/le-wagon/what-to-do-after-a-coding-bootcamp-cad2a5f25bd4)

# 抛开你的背景

为你以前所做的感到骄傲。所有的经历都有价值。无论你是生物学博士还是多年餐饮从业者，你之前的生活都为你的新职业提供了宝贵的经验。一个让你在工作中变得非常严谨，有耐心。另一个肯定迫使你学会如何在压力下工作。

我的个人背景包括学术界和许多学生工作。我曾经惊讶地听到一位 CTO 说，他雇用我的部分原因是因为作为一名前神经科学学生，我必须是那种通读文档的人。

在进入编码领域之前，你可能已经做过的大多数工作也可能比计算机科学研究或工程学校教给你更多的软技能。

在科技领域，能够恰当地交流是非常重要的。许多经理和首席开发人员厌倦了与只关心技术方面的程序员打交道。向利益相关者或非技术同事正确解释你的选择的能力不是每个人都有的，因此，这是一项有价值的技能。

现在，即使你认为你以前的经历不会给你在技术领域带来任何东西(我非常怀疑)，你也会很高兴听到许多团队正在寻求改变他们开发人员的背景，只是为了在员工中创造一种多样化的文化。

一个完全由“书呆子”组成的团队不是每个公司都在寻找的，也不总是最适合面对他们会遇到的每一种问题。

# 推销自己只能胜任你刚刚接受的培训

这个有两个方面:你用来描述自己的头衔和你说你知道的技术。

我看到许多训练营的毕业生用和他们学校完全一样的措辞来介绍自己。这在一定程度上是好的。但我认为你可以很快变得比你刚刚训练的更好，你应该以此为目标。如果你的训练营教你成为一名全栈开发者，那很好，但是要小心:一家公司或招聘人员眼中的全栈开发者可能是不同的。

最负盛名的科技公司(走极端)通常会列出一大堆需要掌握的技术来申请初级开发人员的职位。其次，如果你在 LinkedIn 的个人资料标题中使用了和你的训练营相同的措辞(举例来说)，你就没有把自己和其他进入就业市场的校友区分开来。

我建议你以初级开发人员的身份出现。如果你喜欢，你可以申请全栈职位，但你也可以申请前端或后端职位。这个想法很简单，就是不要马上把自己归入这三个类别中的任何一个，但是要表现出你对其中任何一个都是开放的。如果你愿意，你会有时间专攻。

例如，如果你刚刚完成了 JS 训练营，这并不意味着你不应该学习 Python。如果一门不同的语言或技术让你感兴趣，或者如果你所在地区的就业市场更青睐与你所学不同的东西，那就去学吧。

说到编程语言或(某些)框架，你可以很快达到和你 bootcamp 的堆栈中相同的水平。你的培训时间很短，这一事实不应该被解释为“我只是这样，没有别的”，而应该被解释为“这就是我在短时间内能学到的东西。”

# 认为一切都可以在几个月内学会

就像我说的，我相信你应该一直保持学习，你可以很快使用许多技术。但是全面掌握事物的运作方式需要时间。计算机科学，处理服务器，设计模式，以及许多其他围绕软件开发的主题需要更多的时间来深入理解。

当然，最好是尽快开始学习那些东西，但不要在两周的在线课程后就自称计算机科学知识渊博。你可能会失去你的可信度，让人们认为你在简历上列出的所有技能都只是浅尝辄止。

许多教程为你提供了很多指导，当你不知道从哪里开始时，这是一件好事，但它们会给你一种对所涵盖主题掌握的错觉。很容易混淆熟悉程度和实际熟练程度。

记住，你往往不知道自己不知道的事。只有在真正的项目中工作，你才会发现缺失的部分，并通过填补这些空白和将你已经知道的东西付诸实践，在你正在使用的任何东西上发展出真正的熟练程度。

# 抢购最新的酷炫科技

所以在训练营之后学习新东西很酷，但是你应该明智地选择。当然，获得一些最新技术的技能可以帮助你找到工作，但你需要在拥有坚实的发展核心概念基础和你选择的语言之间找到正确的平衡，并在简历中添加一两个趋势科技。

重要的是你按正确的顺序做事。因为当时急着找工作，所以在认真学 JS 之前先学了 React，在掌握 HTTP 请求基础和 REST APIs 之前先学了 GraphQL。如果我能更早地为核心概念打下更坚实的基础，我会浪费更少的时间，更快地对自己的技能有信心。

基础变化不大，但是框架一直在变化。当你对它们背后的一切有了清晰的了解，至少对之前有什么有了概念，你会学得更快。

# 结论

不要让这些建议挫伤你的热情。如果你渴望在训练后继续学习，你会做得很好。如果你能赢得一些时间，让你的职业生涯有一个平稳的开端，那就太好了——但你会通过这样或那样的经历学到这些东西。保持好奇，提出问题，保持自信。
