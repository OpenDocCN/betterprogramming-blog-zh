## 在这个层面上，软技能往往胜过硬技能

作为一家大型成长型科技公司的高级软件工程师，我有幸帮助面试了许多其他申请与我一起工作的软件工程师。在过去的一年里，我参加了大约 50 次面试，职位范围从中级软件工程师到高级软件工程师再到工程经理。

这一经历让我有时间反思我所看重的其他高级软件工程师的品质和技能，因此，我试图以这样一种方式来设计我的面试问题，希望我能在与每个候选人相处的短暂时间内对这些品质有所了解。

所以，事不宜迟，下面是我在招聘高级软件工程师时需要考虑的一些事情。

# 1.能够与技术人员和非技术人员清晰地交流

这很重要，原因有二。首先，作为一名高级软件工程师，你将作为初级和中级工程师的导师。其他工程师会经常带着问题来找你，你需要能够用简单的术语向他们解释技术概念。当您给出代码评审反馈时，您需要清楚地解释为什么一段代码的当前编写方式不理想，以及如何改进。

第二，作为一名高级软件工程师，你通常会通过间接影响来领导。你并不直接管理任何人，但是你确实需要能够在整个组织中传播你的想法，并团结人们加入到你为干净的代码、更高的工程标准或其他任何东西而奋斗的事业中来。如果你不能有效地交流你的想法，你将很难说服别人你的想法值得他们花时间。

# 2.注意细节

这种品质有多种形式。你会经常在工程师自己的代码中看到它，也会在他们查看别人的代码时看到它。

非常注重细节的工程师在将他们的工作提交给其他人审阅之前，一定会反复检查。他们编写有意义的提交消息。他们清楚地命名他们的变量和函数。他们热爱干净的代码。

需要注意的是，高级软件工程师并不是不会犯错的。他们仍然会犯错。关键是，大多数时候，他们自己发现了这些简单的错误，这样别人就不必替他们做了。

这些实践和习惯延伸到他们在代码评审期间与其他工程师的互动中，因为高级软件工程师试图提升他的同事，帮助他们编写更干净的代码。

# 3.情商和成熟度

一个高级软件工程师需要明白他们不是他们的代码。在任何争论或讨论中，你都必须放下自我。理解对代码的批评不是对人的批评。

高级软件工程师需要能够与其他人进行艰难的对话，无论是关于代码质量、工作表现，还是关于项目时间表的管理期望。

同时，高级软件工程师需要了解他们的言行如何影响其他人。情商不仅包括自我意识，还包括对周围人的意识。高级软件工程师应该知道如何建设性地提供反馈，以提升和激励，而不是降低和打击士气。

# 4.谦逊

高级软件工程师并不是什么都知道。他们也不应该！工程领域太大了，任何一个人都无法完全掌握。

一个优秀的高级软件工程师明白这个事实，并能坦然接受。你不需要知道所有的事情，但是你应该能够认识到什么时候你不知道什么时候应该寻求帮助，以及如何自己找到问题的答案。

这种理解导致谦逊。在软件工程的世界里，没有自我的空间。你永远不要觉得你是不可或缺的，或者你是唯一一个能充分履行工作职责的人。如果你离开了，这个团队和公司会继续运作。

# 5.卓越工程的高标准

高级软件工程师为他们自己和他们的同事设立了一个很高的标准。他们使用有效的工具，如代码格式化程序和代码 linters 来执行标准和编码风格。他们理解测试的重要性，不允许未测试的代码被合并到主分支中。他们配置 CI/CD 管道，以自动化代码审查的繁琐部分，并确保主分支始终处于工作状态。

高级软件工程师将编程视为一门手艺，而不仅仅是一份工作。他们为自己的工作感到自豪。

# 6.他们领域的专家

高级软件工程师能进阶到这个层次是有原因的:他们擅长自己的工作。这意味着了解他们的技术堆栈或选择的语言，而不仅仅是表面的。他们的专长是 T 型的——在一些领域有深厚的知识，在许多其他领域有丰富的经验。

他们通过多年来解决难题、志愿参加艰难的项目、接受延伸任务以及向周围的人学习来获得这种专业知识。

高级软件工程师通过阅读经典编程书籍，如*干净的代码*、*重构*、*实用程序员*或*设计模式*，来利用他们可用的丰富经验。在这样做的过程中，他们从那些花了几十年时间来完善自己技术的领域专家那里获得了经验。

# 7.学习热情

高级软件工程师对他们的工作感到兴奋。他们对新事物感兴趣。他们总是通过阅读文章、观看视频、构建概念验证应用程序，甚至是写作和制作内容来探索新的想法。

简而言之，他们在职业生涯中没有停滞不前，这一点显而易见。

# 结论

硬技能很重要，高级软件工程师需要了解他们的东西。他们被期望成为各自领域的专家，或者至少是几个关键领域的专家。

然而，正如硬技能一样重要，我经常发现软技能最终也同样重要，如果不是更重要的话。

总之，下面是我在其他高级软件工程师身上寻找和看重的品质列表:

1.  能够与技术人员和非技术人员清楚地沟通
2.  注意细节
3.  情商和成熟度
4.  谦逊
5.  卓越工程的高标准
6.  他们领域的专家
7.  学习热情

我希望这有所帮助，并感谢阅读！
