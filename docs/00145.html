使用Docker Compose的Elasticsearch集群和Kibana

原文：https://betterprogramming.pub/elasticsearch-cluster-and-kibana-using-docker-compose-4f9c4d6c5470?source=collection_archive---------7-----------------------#2018-06-23

使用 Docker 为你的开发项目启动你的 Elasticsearch 实验。

我使用Docker Compose创建了三个节点静态弹性搜索7.5.1集群。Docker Compose还包括新的开源 Kibana 7.5.1，运行在 NGINX 之后。

发布说明

  • Elasticsearch 7.5.1发行说明

不建议在生产环境中使用该集群，但它为您提供了一个起点。

GitHub存储库中包含的服务:

  • 三节点弹性搜索版本7.5.1。
  • 基巴纳版本7.5.1。
  • 审计节拍版本7.5.1。
  • 公制节拍版本7.5.1。
  • 心跳版本7.5.1。
  • 包揍版本7.5.1。
  • 文件节拍版本7.5.1。
  • APM服务器版本7.5.1。
  • APP搜索7.5.1版本。
  • NGINX。

要求

  • 18.05号文件
  • docker-撰写1.21

在守护模式下启动堆栈

docker-compose up -d

检查Docker撰写群集的状态

docker-compose ps -a

群集节点信息

curl http://localhost:9200/_nodes?pretty=true

访问基巴纳

http://localhost:5601

通过NGINX访问Kibana

http://localhost:8080

访问弹性搜索

http://localhost:9200

访问应用程序搜索

http://localhost:3002
