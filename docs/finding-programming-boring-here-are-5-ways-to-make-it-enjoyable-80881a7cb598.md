面对现实吧:虽然编程通常很有趣，但有时也会很无聊。有时候你只是碰壁，忘记了潜在的目的，最终筋疲力尽。

你可能会几天或几周不编程——除了坐在电脑前什么都不做。如果编程是你的主要收入来源，那么这些休息日会变得不堪重负，最终导致你放弃编程。

所以问题是:如何让自己在编程中保持高效和活跃？当你感到困难的时候，你是如何让旅途变得愉快的？

我有过想放弃的时刻，怀疑这是否是正确的道路，评估其他工作领域，等等。我是那种很难在电脑前坐上几个小时的人。

如果你曾经陷入无聊状态，这里有一些我的秘密成分可以让编程再次变得有趣。这不是一个普遍的法则——只是到目前为止对我有用的东西。然而，从这篇文章中你仍然可以学到一些东西。

## 以下是我们要解决的问题:

1.  音乐被低估了
2.  太多事情要做
3.  “我能行”的心态
4.  学习的艺术
5.  定期休息

# 音乐被低估了

编程是一项心理运动。即使你不慢跑或举重，解决问题的思考过程也会耗尽你的精力。

此外，你会经历许多情绪——当你设法解决某事时感觉良好的情绪，或者因为找不到解决方案而感到压力和愤怒的情绪。大多数程序员都面临这一切。

处理这些情绪的一个很好的方法是听好的音乐——当然不是任何类型的，而是那些让你兴奋并帮助你正确思考的音乐。对我来说，我喜欢听八九十年代的老派音乐。音乐是世界上为数不多的能在几秒钟内改变一个人情绪的东西之一。听一些能让你开心，勾起美好回忆的东西。

> 小贴士:听一些有助于你思考的音乐，一些节拍和声音可以帮助你更长时间地集中注意力。太多的噪音只会让你更快疲倦。

我也听 Spotify 上的几个播放列表:Anjundeeep New Releases、Night Rider、Synthawave from Space、Chill Tracks 和 Downtempo Beats。

# 有太多事情要做

你开始恐慌地思考你需要实现的所有事情。在你意识到之前，你什么也没做，只是想多了。

让我首先说这是完全正常的。建筑工人也有同样的问题——大多数工作都是如此。

当这些想法进入你的脑海时，不要跟着情绪走。相反，使用一个简单的技巧:专注于你目前正在做的事情(如果它变得复杂，画一些图)。有时候问题是我们脑子里有太多的想法。当这种情况发生时，一个有效的方法是把事情写下来，要么创建一个项目列表，要么画一些图。这样做有助于正确看待事物，理清思路。如果这不起作用，试着休息 15 分钟/小睡一下。睡眠被证明是减轻压力的有效工具。

> 提示:如果你有太多的事情要做，开始区分哪些事情重要，哪些事情不重要。问问你自己:我真的需要知道这些吗？如果是，那么它提供了什么价值？

# “我能行”的心态

这种心态“在承担解决问题的责任方面是一件好事。

然而，你必须意识到你永远不可能完全学到东西——总有改进的空间。提问是获得洞察力和想法的好方法，尤其是如果你在一个组织中工作。我所见过的在公司里做得很好的人是那些提问的人——那些精心设计的问题。

你从阅读文章或观看视频中获得的是巨大的，但你从询问中获得的是基于经验的独特知识。编程是一份工作，它是关于分享和互相帮助的。我们一起更快地实现目标。

> **提示:**问问题时，确保你理解了问题。在现有知识的基础上积累知识会让学习变得更容易。但是不要让这阻止你去问——有时候我们什么都不知道，这很好。

# 学习的艺术

有句话说:当你真正学到东西时，你会意识到你有多少不知道。

学习是一个依赖于时间的过程，通过持续的训练。我们生活在一个时代，我们认为仅仅因为我们看一些在线视频，阅读文章，或听演讲，我们就变得更聪明。

但遗憾的是，事实并非如此。知识是建立在使用你所学的知识并付诸实践的基础上的。想学习如何使用 React、Angular 或其他框架吗？建造一些简单的东西。

> **提示:**如果你在学习方面有困难，[这里有一篇我写的文章](https://medium.com/free-code-camp/a-chaotic-mind-leads-to-chaotic-code-e7d6962777c0)。

# 定期休息

在我编程的早期，我认为编程就是把自己锁在一个黑暗的房间里，然后整天编码。类似于我们现在在电影上看到的。然而，我逐渐意识到编程需要休息，就像运动员在拳击、足球、篮球等大多数体育活动中所做的那样。

把大脑想象成汽车引擎。引擎需要什么？它需要石油和汽油来运转，对吗？对你来说，这意味着你需要吃健康的食物，喝水，休息来完成一份好的工作。休息大脑会带来创造性思维，而创造性思维会带来更好的解决问题的方法。

> 小贴士:确保充足的睡眠。这是大多数程序员忽略的一个关键因素。

# 摘要

总的来说，编程是一项有时会令人厌烦的工作或爱好。我不会撒谎说这很有趣。它会耗尽你对生活中很多事情的精力和动力。

但不一定要这样。在大多数情况下，这不是关于代码或挑战性的问题，而是对它的心态。

所以听听你想要的——有时候我们只是需要短暂的休息，或者有时候我们只是需要某人的帮助。不要犹豫寻求帮助！
